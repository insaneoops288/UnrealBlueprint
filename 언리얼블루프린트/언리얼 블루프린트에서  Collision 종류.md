# **언리얼 엔진 블루프린트 콜리전(Collision) 가이드**

언리얼 엔진의 콜리전 시스템은 크게 **반응(Responses)**, **채널(Channels)**, 그리고 \*\*프리셋(Presets)\*\*으로 나뉩니다.

## **1\. 콜리전 반응 (Collision Responses)**

모든 콜리전 설정의 기초가 되는 세 가지 반응 모드입니다. 두 물체가 만났을 때 각각의 설정값 중 '최소 공통분모'에 해당하는 반응이 일어납니다.

| 반응 종류 | 설명 | 주요 특징 |
| :---- | :---- | :---- |
| **Ignore (무시)** | 아무런 반응을 하지 않습니다. | 물체가 그대로 통과하며, 이벤트도 발생하지 않습니다. |
| **Overlap (겹침)** | 물체가 통과되지만, 겹쳤음을 감지합니다. | BeginOverlap, EndOverlap 이벤트가 발생합니다. (트리거 제작용) |
| **Block (블록)** | 물체가 통과하지 못하게 막습니다. | 물리적 충돌이 발생하며, Hit 이벤트가 발생합니다. |

## **2\. 오브젝트 타입과 트레이스 채널 (Object Types & Trace Channels)**

콜리전이 누구인지를 정의하고, 레이캐스트(Line Trace) 등에 어떻게 반응할지 결정합니다.

### **오브젝트 타입 (Object Types)**

* **WorldStatic:** 움직이지 않는 배경, 벽, 바닥 등.  
* **WorldDynamic:** 움직이는 액터, 엘리베이터, 발판 등.  
* **Pawn:** 플레이어 또는 AI 캐릭터.  
* **PhysicsBody:** 물리 시뮬레이션(Ragdoll 등)이 적용된 물체.  
* **Vehicle / Destructible:** 차량이나 파괴 가능한 물체 전용.

### **트레이스 채널 (Trace Channels)**

* **Visibility:** 시각적으로 보이는지 확인 (마우스 클릭, 조준 등).  
* **Camera:** 카메라 사야를 가리는지 확인 (카메라 줌 방지 등).

## **3\. 콜리전 프리셋 (Collision Presets)**

자주 사용하는 설정을 묶어놓은 템플릿입니다.

* **NoCollision:** 모든 콜리전 무시.  
* **BlockAll:** 모든 물체를 막음.  
* **OverlapAll:** 모든 물체와 겹침 이벤트 발생.  
* **Pawn:** 캐릭터 모델에 최적화된 설정.  
* **Trigger:** 플레이어가 들어왔을 때 이벤트를 발생시키기 위한 전용 설정.

## **4\. 단순 콜리전 vs 복잡 콜리전 (Simple vs Complex)**

성능 최적화와 정확도 사이의 선택입니다.

1. **Simple Collision (단순 콜리전):**  
   * 박스, 구체, 캡슐 등 단순한 도형으로 구성.  
   * 연산 속도가 매우 빠름.  
   * 캐릭터 이동이나 물리 시뮬레이션에 사용.  
2. **Complex Collision (복잡 콜리전):**  
   * 메시의 실제 폴리곤(Per-poly) 형태를 그대로 사용.  
   * 매우 정확하지만 연산 비용이 높음.  
   * 총알이 정밀하게 박혀야 하는 경우 등에 사용 (물리 시뮬레이션 불가).

## **5\. 블루프린트 주요 이벤트**

블루프린트 노드에서 콜리전 결과를 활용할 때 가장 많이 사용되는 노드들입니다.

* **On Component Begin Overlap:** 트리거 구역에 들어왔을 때 실행.  
* **On Component End Overlap:** 구역에서 나갔을 때 실행.  
* **On Component Hit:** 물체가 무언가에 부딪혔을 때(Block 설정 시) 실행.  
* **Line Trace By Channel:** 특정 방향으로 가상의 선을 쏴서 충돌 여부 확인.

## **💡 팁: 콜리전이 작동하지 않을 때 체크리스트**

1. 두 물체 모두 **Generate Overlap Events**가 체크되어 있는가?  
2. 서로의 채널 설정이 **Ignore**로 되어 있지는 않은가?  
3. 물리 시뮬레이션이 필요하다면 **Simulate Physics**가 켜져 있는가?  
4. 적절한 **Collision Enabled** 설정(Query and Physics 등)이 되어 있는가?
