# **템플릿 애니메이션 블루프린트 (Template Animation Blueprint) 역할 및 이해**

언리얼 엔진(특히 UE5 이상)에서 **템플릿 애니메이션 블루프린트**는 애니메이션 그래프 로직과 데이터(애셋)를 분리하여, 하나의 로직을 여러 캐릭터나 스켈레톤에 공유할 수 있게 해주는 핵심 기능입니다.

## **1\. 주요 역할 (Core Roles)**

### **① 스켈레톤 독립성 (Skeleton Independence)**

가장 큰 역할은 **특정 스켈레톤에 묶이지 않는 애니메이션 로직**을 만드는 것입니다.

* 일반 AnimBP: 생성 시 반드시 특정 'Skeleton' 애셋을 선택해야 하며, 해당 스켈레톤 전용으로 작동합니다.  
* 템플릿 AnimBP: 특정 스켈레톤 없이 로직(스테이트 머신, 블렌드 스페이스 구성 등)만 정의합니다.

### **② 로직의 추상화 및 재사용**

복잡한 이동 로직(Locomotion), 전투 시스템, 레이어드 블렌딩 등의 '구조'만 템플릿으로 짭니다. 이후 이 구조를 사용하는 각 캐릭터는 자신의 스켈레톤에 맞는 애니메이션 애셋만 할당하면 됩니다.

### **③ 런타임 인스턴스 생성**

Create Template Animation Blueprint와 관련된 노드나 설정을 통해, 게임 실행 중에 특정 스켈레톤 메시 컴포넌트에 대해 템플릿 기반의 애니메이션 인스턴스를 동적으로 할당하고 실행할 수 있습니다.

## **2\. 기존 방식과의 차이점**

| 특징 | 일반 애니메이션 블루프린트 | 템플릿 애니메이션 블루프린트 |
| :---- | :---- | :---- |
| **스켈레톤 지정** | 생성 시 필수 선택 | 생성 시 선택하지 않음 (Template 모드) |
| **애셋 바인딩** | 특정 시퀀스/블렌드 스페이스 고정 | 변수나 바인딩을 통해 나중에 결정 가능 |
| **유지보수** | 캐릭터마다 AnimBP를 복사해야 함 | 템플릿 하나만 수정하면 모든 캐릭터에 반영 |
| **범용성** | 낮음 (동일 스켈레톤 내에서만 가능) | 높음 (본 구조가 유사하면 다른 스켈레톤도 가능) |

## **3\. 작동 원리 및 워크플로우**

1. **템플릿 생성**: AnimBP를 생성할 때 'Template' 옵션을 선택하거나 기존 AnimBP를 템플릿으로 전환합니다.  
2. **데이터 바인딩**: 내부의 애니메이션 노드(예: Play Sequence)에 구체적인 애셋 대신 \*\*'Property Binding'\*\*을 사용하여, 외부에서 애니메이션 애셋을 주입받을 수 있도록 설계합니다.  
3. **자식 클래스 또는 인스턴스화**:  
   * **AnimBP 자식**: 템플릿을 상속받아 각 캐릭터(인간, 괴물 등)에 맞는 실제 애니메이션 파일만 '에셋 오버라이드' 탭에서 교체합니다.  
   * **런타임 할당**: 메시 컴포넌트의 Set Anim Instance Class 등을 통해 템플릿 기반 클래스를 지정합니다.

## **4\. 활용 시 이점**

* **생산성 향상**: 수십 명의 NPC가 있을 때 각자 AnimBP를 만들 필요 없이, 하나의 'NPC\_Template'만 관리하면 됩니다.  
* **메모리 최적화**: 공통 로직을 공유하므로 엔진 내부적으로 최적화된 처리가 가능합니다.  
* **모듈화**: 기능별로 템플릿을 나누어(예: 수영 템플릿, 비행 템플릿) 필요할 때만 레이어로 섞어 쓸 수 있습니다.

## **5\. 요약**

Create Template Animation Blueprint 관련 기능은 \*\*"애니메이션 로직은 하나로 만들고, 실제 움직이는 데이터(FBX/애셋)만 갈아 끼우겠다"\*\*는 의도를 구현하는 도구입니다. 이는 특히 규모가 큰 프로젝트나 다양한 체형의 캐릭터가 등장하는 게임에서 필수적인 워크플로우입니다.
