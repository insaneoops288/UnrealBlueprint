# **AbilitySystemComponent (ASC)의 역할과 기능**

AbilitySystemComponent는 언리얼 엔진 GAS(Gameplay Ability System)를 사용하는 모든 액터의 핵심입니다. 이 컴포넌트는 능력, 속성, 상태를 통합 관리하며 서버와 클라이언트 간의 동기화를 담당합니다.

## **1\. 주요 역할 (Core Responsibilities)**

### **① Gameplay Abilities (능력) 관리**

액터가 실행할 수 있는 모든 행동(스킬, 점프, 공격 등)을 소유하고 실행합니다.

* **부여 (Granting):** 특정 능력을 액터에게 할당합니다 (GiveAbility).  
* **활성화 (Activation):** 입력이나 이벤트를 통해 능력을 실행합니다 (TryActivateAbility).  
* **중단 (Termination):** 실행 중인 능력을 강제로 종료하거나 완료 처리합니다.

### **② Gameplay Attributes (속성) 보관 및 변경**

체력(HP), 마나(MP), 공격력 등 수치 데이터를 포함하는 AttributeSet과 상호작용합니다.

* 속성값의 현재 수치(Current Value)와 기본 수치(Base Value)를 유지합니다.  
* 데미지 계산이나 버프에 의한 수치 변동을 처리합니다.

### **③ Gameplay Tags (태그) 처리**

액터의 상태를 나타내는 문자열 기반의 레이블인 '태그'를 관리합니다.

* **상태 표현:** State.Dead, State.Stunned 같은 태그를 통해 현재 액터의 상태를 정의합니다.  
* **조건 검사:** 특정 능력이 실행될 때 "스턴 상태인가?" 등을 태그를 통해 즉시 확인합니다.

### **④ Gameplay Effects (이펙트) 적용**

능력에 의해 발생하는 변화(지속 데미지, 스탯 상승, 상태 이상 등)를 처리합니다.

* GameplayEffect를 통해 속성을 수정하거나 태그를 추가/제거합니다.  
* 이펙트의 지속 시간(Duration)과 중첩(Stacking) 로직을 관리합니다.

## **2\. 블루프린트에서의 주요 노드 및 활용**

블루프린트에서 ASC를 다룰 때 가장 자주 사용하게 되는 기능들은 다음과 같습니다.

| 기능 | 블루프린트 노드 이름 (예시) | 설명 |
| :---- | :---- | :---- |
| **능력 활성화** | Try Activate Ability by Class | 클래스 지정을 통해 스킬을 실행 시도합니다. |
| **이벤트 전달** | Send Gameplay Event to Actor | 특정 태그를 가진 이벤트를 보내 능력을 트리거합니다. |
| **이펙트 적용** | Apply Gameplay Effect to Self | 자신에게 버프나 디버프를 적용합니다. |
| **태그 확인** | Has Matching Gameplay Tag | 액터가 특정 상태(태그)를 가지고 있는지 확인합니다. |
| **속성 변경 감지** | Get Gameplay Attribute Value Change Delegate | 체력 등 수치가 변할 때 UI를 업데이트하는 등에 사용합니다. |

## **3\. 네트워크 복제 (Replication)**

ASC는 멀티플레이어 환경에서 매우 강력합니다.

* **동기화:** 서버에서 발생한 능력 실행, 이펙트 적용, 속성 변화를 클라이언트에 자동으로 복제합니다.  
* **예측 (Prediction):** 네트워크 지연(Lag)을 숨기기 위해 클라이언트에서 즉각적인 반응을 보여주고, 나중에 서버의 결과와 대조하여 보정합니다.

## **4\. 배치 위치 전략 (Architecture)**

ASC를 액터의 어디에 추가하느냐에 따라 성격이 달라집니다.

1. **Pawn/Character에 추가:**  
   * 주로 AI 캐릭터나 단순한 몬스터에게 적합합니다.  
   * 캐릭터가 파괴되면 ASC 정보도 사라집니다.  
2. **PlayerState에 추가:**  
   * 플레이어 캐릭터에게 권장되는 방식입니다.  
   * 캐릭터가 리스폰(재소환)되어도 레벨, 스킬 쿨타임, 능력치 정보를 그대로 유지할 수 있습니다.

## **요약**

**AbilitySystemComponent**는 게임의 \*\*'규칙 엔진'\*\*입니다. 캐릭터가 어떤 행동을 할 때 그것이 가능한지 체크하고, 결과적으로 어떤 변화를 일으키며, 이 모든 것이 멀티플레이 환경에서 공정하게 돌아가도록 관리하는 컨트롤 타워입니다.
