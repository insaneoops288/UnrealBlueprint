# **언리얼 엔진 블루프린트: Acos 노드 상세 가이드**

Acos(Arccosine) 노드는 수학적 삼각함수의 역함수로, 코사인 비율을 각도로 변환하는 강력한 도구입니다.

## **1\. 노드 기본 사양**

| 항목 | 설명 |
| :---- | :---- |
| **입력 (Value)** | \-1.0 \~ 1.0 사이의 Float 값 (코사인 값) |
| **출력 (Return)** | 0 \~ 180 (Degrees 기준) 또는 0 \~ ![][image1] (Radians 기준) |
| **변종** | Acos (Degrees), Acos (Radians) |

## **2\. 왜 사용하는가? (핵심 개념)**

코사인 함수(Cos)가 **각도 → 비율**을 만든다면, 역코사인(Acos)은 **비율 → 각도**를 만듭니다.

* **Cos(60°)** \= 0.5  
* **Acos(0.5)** \= 60°

게임에서는 순수 각도 데이터를 직접 다루기보다 벡터의 방향(내적값)을 먼저 얻는 경우가 많기 때문에, 이를 다시 우리가 이해하기 쉬운 '도(Degree)' 단위로 바꿀 때 Acos가 필요합니다.

## **3\. 실전 활용 사례: "적의 시야 판별"**

AI가 플레이어를 감지할 때, 단순히 거리만 보는 게 아니라 **전방 45도 이내**에 있는지를 판별하는 로직입니다.

### **로직 단계**

1. **AI Forward Vector**: AI가 바라보는 방향 벡터.  
2. **Direction to Player**: 플레이어 위치에서 AI 위치를 뺀 후 정규화(Normalize)한 벡터.  
3. **Dot Product**: 두 벡터를 내적합니다. (결과는 \-1 \~ 1 사이의 값)  
4. **Acos (Degrees)**: 내적 결과값을 넣습니다.  
5. **Comparison**: 결과값이 45보다 작으면 AI의 시야 안에 플레이어가 있는 것입니다.

## **4\. 주의사항**

1. **입력 값 범위 제어 (Clamping):**  
   * 수학적으로 Acos의 입력은 반드시 \-1에서 1 사이여야 합니다.  
   * 부동 소수점 오차로 인해 내적값이 1.000001 같은 값이 들어오면 에러(NaN, Not a Number)가 발생할 수 있습니다.  
   * 안전을 위해 Clamp (Float) 노드를 사용하여 입력값을 \-1\~1 사이로 제한하는 것이 좋습니다.  
2. **단위 벡터 필수:**  
   * 내적을 통해 각도를 구할 때는 반드시 두 벡터가 **Normalize** 되어 있어야 합니다. 그렇지 않으면 Acos에 1보다 크거나 \-1보다 작은 값이 들어가 올바른 각도가 나오지 않습니다.  
3. **성능:**  
   * Acos는 일반적인 산술 연산보다 비용이 약간 더 듭니다. 만약 정확한 각도가 필요한 게 아니라 단순히 "범위 안에 있는가"만 체크한다면, Acos를 생략하고 내적값(Dot Product)끼리 직접 비교하는 것이 더 효율적입니다.  
   * 예: Acos(Dot) \< 45 대신 Dot \> Cos(45)로 비교.

## **5\. 요약**

Acos 노드는 **"방향(벡터) 데이터를 우리가 읽을 수 있는 각도(0\~180도) 데이터로 변환할 때"** 사용하는 가장 대표적인 노드입니다.
