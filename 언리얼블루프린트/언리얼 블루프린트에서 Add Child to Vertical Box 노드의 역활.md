# **Add Child to Vertical Box 노드 상세 설명**

언리얼 엔진 블루프린트에서 Add Child to Vertical Box 노드는 UI 레이아웃을 동적으로 구성할 때 필수적인 역할을 합니다.

## **1\. 핵심 역할**

이 노드는 이름 그대로 **Vertical Box 위젯 안에 새로운 자식 위젯을 삽입**하는 기능을 합니다. 버티컬 박스는 자식들을 위에서 아래로 수직으로 정렬하므로, 이 노드를 호출할 때마다 새로운 위젯이 리스트의 가장 아래쪽에 추가됩니다.

## **2\. 주요 입력 파라미터**

* **Target (Vertical Box):** 위젯을 추가할 대상이 되는 버티컬 박스 컴포넌트입니다.  
* **Content (Widget):** 버티컬 박스 안에 넣고 싶은 실제 위젯 객체(Object Reference)입니다. 보통 Create Widget 노드를 통해 생성된 결과물을 연결합니다.

## **3\. 반환값: Vertical Box Slot**

이 노드의 가장 중요한 특징 중 하나는 **Vertical Box Slot** 객체를 반환한다는 점입니다.

* 위젯을 단순히 추가하는 데 그치지 않고, 추가된 직후의 **레이아웃 설정**을 변경할 수 있게 해줍니다.  
* **슬롯을 통해 설정 가능한 항목:**  
  * **Padding:** 위젯 주변의 여백 설정  
  * **Size:** 'Auto'(콘텐츠 크기에 맞춤) 또는 'Fill'(남은 공간 채우기) 설정  
  * **Horizontal/Vertical Alignment:** 슬롯 내에서의 수평/수직 정렬 방식

## **4\. 실제 활용 사례**

이 노드는 정적인 UI보다는 상황에 따라 변하는 UI를 만들 때 주로 사용됩니다.

* **인벤토리 시스템:** 아이템을 획득할 때마다 아이템 슬롯 위젯을 리스트에 추가.  
* **채팅창:** 새로운 메시지가 올 때마다 텍스트 위젯을 아래에 추가.  
* **설정 메뉴:** 옵션 리스트를 데이터 테이블에서 불러와 동적으로 생성.

## **5\. 블루프린트 작업 흐름 (Workflow)**

일반적으로 다음과 같은 순서로 노드를 구성합니다.

1. Create Widget: 추가할 자식 위젯(예: 아이템 슬롯)을 생성합니다.  
2. Add Child to Vertical Box: 생성된 위젯을 버티컬 박스에 연결합니다.  
3. (선택 사항) 반환된 Slot 결과값에서 Set Padding이나 Set Horizontal Alignment 등을 호출하여 디자인을 미세 조정합니다.

## **6\. 주의사항**

* **계층 구조:** 이미 다른 부모 위젯에 속해 있는 위젯을 추가하려고 하면, 기존 부모에서 자동으로 제거된 후 새로운 버티컬 박스로 이동합니다.  
* **성능:** 너무 많은 자식 위젯(수백 개 이상)을 동적으로 추가하면 성능 저하가 발생할 수 있습니다. 이 경우 ListView나 TileView 사용을 고려해야 합니다.
