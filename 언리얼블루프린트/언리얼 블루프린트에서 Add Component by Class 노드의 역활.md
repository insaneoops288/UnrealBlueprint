# **Add Component by Class 노드 상세 가이드**

언리얼 엔진 블루프린트에서 Add Component by Class 노드는 미리 정의된 컴포넌트 목록에 없는 기능을 게임 도중에 필요에 따라 액터에 이식하는 역할을 합니다.

## **1\. 주요 역할**

기본적으로 액터는 에디터의 '컴포넌트(Components)' 패널에서 미리 컴포넌트를 추가해 두지만, 다음과 같은 상황에서는 이 노드가 필수적입니다.

* **동적 장착:** 캐릭터가 아이템을 주웠을 때 특정 스태틱 메시나 효과 컴포넌트를 추가해야 할 때.  
* **절차적 생성:** 게임 환경에 따라 액터의 외형이나 기능을 실시간으로 구성해야 할 때.  
* **성능 최적화:** 항상 필요하지 않은 복잡한 컴포넌트를 특정 조건에서만 생성하여 메모리를 관리할 때.

## **2\. 주요 입력 및 출력 항목**

| 항목 | 유형 | 설명 |
| :---- | :---- | :---- |
| **Target** | Actor Object Reference | 컴포넌트를 추가할 대상 액터입니다. (기본값은 Self) |
| **Class** | Component Class Reference | 추가할 컴포넌트의 클래스를 선택합니다. (예: Static Mesh, Particle System, Point Light 등) |
| **Relative Transform** | Transform | 추가될 컴포넌트의 상대적인 위치, 회전, 스케일을 결정합니다. |
| **Manual Attachment** | Boolean | True인 경우 부모 컴포넌트에 자동으로 붙지 않으며, 이후 AttachToComponent 노드로 직접 연결해야 합니다. |
| **Return Value** | Component Object Reference | 생성된 컴포넌트의 참조값을 반환합니다. 이를 통해 변수에 저장하거나 설정을 변경할 수 있습니다. |

## **3\. 사용 예시 (Scenario)**

### **A. 무기 장착 시스템**

플레이어가 칼을 주웠을 때, 캐릭터 액터에 Static Mesh Component를 추가하고 칼 메쉬를 할당한 뒤 손 소켓에 부착합니다.

### **B. 상태 이상 효과**

캐릭터가 불에 닿았을 때 Niagara Particle System 컴포넌트를 동적으로 추가하여 불타는 효과를 내고, 일정 시간이 지나면 Destroy Component로 제거합니다.

### **C. 인벤토리 미리보기**

UI에서 아이템을 클릭했을 때, 빈 액터에 해당 아이템의 클래스에 맞는 컴포넌트들을 순차적으로 추가하여 화면에 보여줍니다.

## **4\. 주의사항 및 팁**

1. **반환 값 활용:** 생성된 컴포넌트는 'Return Value'를 통해 즉시 수정이 가능합니다. 예를 들어 클래스를 Static Mesh Component로 설정했다면, 반환 값을 Set Static Mesh 노드에 연결하여 실제 모양을 입혀야 합니다.  
2. **트랜스포름 설정:** 추가되는 순간의 위치가 중요합니다. 부모 액터의 원점(Root)을 기준으로 상대적인 위치를 입력하세요.  
3. **메모리 관리:** 런타임에 생성된 컴포넌트는 더 이상 필요 없을 때 반드시 Destroy Component를 호출하여 제거해 주는 것이 성능상 좋습니다.  
4. **Cast(형변환):** Return Value는 일반적인 'Actor Component' 타입으로 나오므로, 특정 기능(예: Light의 색상 변경)을 쓰려면 해당 클래스로 형변환(Cast)이 필요할 수 있습니다.

**요약:** 이 노드는 액터의 구성을 \*\*정적(Static)\*\*인 상태에서 \*\*동적(Dynamic)\*\*인 상태로 바꿔주는 강력한 도구입니다.
