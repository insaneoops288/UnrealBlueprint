# **Unreal Engine: Add Local Offset 노드 완벽 가이드**

Add Local Offset 노드는 언리얼 엔진 블루프린트에서 액터나 컴포넌트의 위치를 변경할 때 가장 자주 사용되는 노드 중 하나입니다. 이 노드의 핵심은 '상대적 이동'과 '로컬 좌표계'에 있습니다.

## **1\. 핵심 역할**

이 노드는 타겟(Target)이 되는 컴포넌트의 **현재 위치에 특정 벡터값(Delta Location)을 더해줍니다.** 이때 기준이 되는 좌표계는 월드(World)가 아닌 \*\*로컬(Local)\*\*입니다.

* **로컬 좌표계 기준:** 액터가 회전되어 있다면, 그 회전된 방향을 기준으로 X, Y, Z축이 결정됩니다.  
* **예시:** 캐릭터가 북쪽을 보든 동쪽을 보든, 로컬 X축으로 \+10만큼 이동시키면 항상 캐릭터의 '앞'으로 10만큼 이동합니다.

## **2\. 주요 입력 핀 (Input Pins)**

| 핀 이름 | 역할 |
| :---- | :---- |
| **Target** | 이동시킬 대상(Scene Component)입니다. 보통 'Self'나 특정 'Static Mesh' 등이 연결됩니다. |
| **Delta Location** | 현재 위치에서 얼마나 더 이동할 것인지 나타내는 벡터값입니다. |
| **Sweep** | 체크 시, 이동 경로에 장애물이 있는지 확인합니다. 장애물이 있으면 멈추거나 부딪힙니다. |
| **Teleport** | 체크 시, 물리 시뮬레이션(Physics) 상태를 유지하면서 순식간에 이동합니다. (속도/가속도 초기화 방지) |
| **Hit Result** | Sweep이 켜져 있을 때, 무엇에 부딪혔는지에 대한 정보를 출력합니다. |

## **3\. Add World Offset vs Add Local Offset**

가장 혼동하기 쉬운 두 노드의 차이점은 다음과 같습니다.

### **Add World Offset (월드 오프셋)**

* 게임 세상의 고정된 동서남북 방향을 기준으로 이동합니다.  
* 액터가 어디를 보든 상관없이 X: 100을 주면 항상 월드의 X축 방향으로 이동합니다.

### **Add Local Offset (로컬 오프셋)**

* 액터가 바라보는 방향을 기준으로 이동합니다.  
* 액터가 90도 회전했다면, 로컬 X축은 월드의 Y축 방향이 될 수 있습니다. 즉, \*\*"내 기준 앞/뒤/좌/우"\*\*를 계산할 때 필수적입니다.

## **4\. 활용 사례**

1. **캐릭터 이동:** 입력 값에 따라 캐릭터를 전진(Local X)시키거나 좌우(Local Y)로 게걸음질 치게 할 때 사용합니다.  
2. **발사체(Projectile):** 총알이 발사된 후 매 프레임마다 자신의 '앞' 방향으로 계속 이동하게 할 때 사용합니다.  
3. **문 열기 애니메이션:** 문(Door) 컴포넌트를 문틀 기준의 로컬 좌표로 살짝 밀어낼 때 유용합니다.  
4. **카메라 흔들림:** 카메라의 로컬 좌표를 무작위로 변경하여 흔들리는 효과를 줄 수 있습니다.

## **5\. 주의사항**

* **부모-자식 관계:** 자식 컴포넌트에 Add Local Offset을 사용하면 부모 컴포넌트의 좌표계를 기준으로 이동합니다.  
* **Sweep 성능:** Sweep을 켜면 이동할 때마다 충돌 계산을 수행하므로, 수천 개의 오브젝트에 동시에 적용할 때는 성능 최적화를 고려해야 합니다.  
* **좌표계 확인:** 언리얼에서 전방(Forward)은 보통 **X축**, 우측(Right)은 **Y축**, 상단(Up)은 **Z축**임을 기억하세요.

### **요약**

Add Local Offset은 **"대상이 바라보는 방향을 기준으로 위치를 변화시키고 싶을 때"** 사용하는 노드입니다. 월드 기준이 아닌 자기 자신 기준의 좌표계를 사용한다는 점이 핵심입니다.
