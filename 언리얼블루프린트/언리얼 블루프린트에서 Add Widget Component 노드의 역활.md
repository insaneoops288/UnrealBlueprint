# **Add Widget Component 노드 상세 가이드**

언리얼 엔진 블루프린트에서 **Add Widget Component** 노드는 일반적인 2D UMG 위젯을 3D 월드 공간 내에 물리적으로 존재하는 컴포넌트로 변환하여 액터에 부착할 때 사용합니다.

## **1\. 주요 역할 (Role)**

이 노드의 핵심은 \*\*"2D UI를 3D 세계의 일부로 만드는 것"\*\*입니다.

* **동적 생성:** 미리 컴포넌트 패널에 추가해두지 않고, 특정 이벤트(예: 아이템 근처에 다가갔을 때)가 발생했을 때만 UI를 생성할 수 있습니다.  
* **3D 공간 배치:** 뷰포트에 직접 그리는 Create Widget \+ Add to Viewport 방식과 달리, 캐릭터의 머리 위나 키오스크 화면 같은 3D 좌표에 UI를 위치시킵니다.

## **2\. 주요 설정 항목 (Key Pins)**

노드를 선택하면 디테일 패널이나 입력 핀에서 다음 설정을 조절할 수 있습니다.

| 설정 항목 | 설명 |
| :---- | :---- |
| **Widget Class** | 표시할 UMG 위젯 블루프린트 클래스를 선택합니다. |
| **Widget Space** | **World:** 다른 3D 오브젝트처럼 거리에 따라 작아지고 가려짐. **Screen:** 3D 위치에 고정되되, 항상 화면 정면을 바라보는 2D 형태로 표시. |
| **Draw Size** | 위젯이 렌더링될 가상 캔버스의 해상도(크기)를 결정합니다. |
| **Pivot** | 위젯의 중심점 좌표를 설정합니다. |

## **3\. 사용 시나리오**

* **상호작용 UI:** 플레이어가 오브젝트에 접근했을 때 나타나는 "E 키를 눌러 조사" 같은 플로팅 메시지.  
* **체력바:** 적 캐릭터의 머리 위에 실시간으로 따라다니는 HP 바.  
* **인게임 스크린:** 게임 내 컴퓨터 모니터나 뉴스 전광판 같은 연출.  
* **VR UI:** 플레이어의 손목이나 허공에 떠 있는 메뉴 시스템.

## **4\. 작동 흐름 (Workflow Example)**

1. **Add Widget Component** 노드 실행.  
2. **Set Widget Class**를 통해 보여줄 UI 할당.  
3. (필요 시) **Set Relative Transform**으로 액터 대비 위치 조정.  
4. 생성된 컴포넌트 리턴값에서 **Get User Widget Object**를 호출하여 위젯 내부의 함수나 변수(예: 텍스트 변경)에 접근.

## **5\. 주의사항**

* **성능:** 너무 많은 위젯 컴포넌트를 World Space로 실시간 생성하면 렌더링 비용이 증가할 수 있습니다.  
* **입력 처리:** 3D 위젯과 마우스 클릭 등의 상호작용을 하려면 Widget Interaction Component가 추가로 필요합니다.  
* **초기화 시점:** 노드가 실행된 직후에는 위젯 인스턴스가 즉시 생성되지 않을 수 있으므로, 생성 직후 위젯 내부 값을 수정할 때는 약간의 지연이나 유효성 검사(Is Valid)가 필요할 수 있습니다.
