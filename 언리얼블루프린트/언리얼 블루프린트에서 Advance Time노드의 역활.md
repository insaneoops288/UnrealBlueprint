# **Unreal Engine: Advance Time 노드의 역할과 활용**

언리얼 엔진 블루프린트에서 Advance Time 노드는 기본적으로 \*\*"현재 흐르는 시간과는 별개로, 특정 대상의 내부 시계를 강제로 진행시키는 역할"\*\*을 합니다.

주로 다음과 같은 세 가지 주요 컨텍스트에서 사용됩니다.

## **1\. 미디어 플레이어 (Media Player)에서의 역할**

비디오나 오디오 파일을 재생하는 **Media Player** 컴포넌트에서 가장 흔하게 볼 수 있습니다.

* **역할:** 미디어의 재생 위치를 특정 초(Seconds)만큼 앞으로 건너뛰게 하거나, 매 프레임 수동으로 시간을 업데이트할 때 사용합니다.  
* **활용:** \- 영상의 특정 구간으로 빠르게 넘기기(Fast Forward).  
  * 게임의 일시정지 상태에서도 영상만 따로 흐르게 하고 싶을 때 수동으로 호출.

## **2\. 카오스 캐시 / 피지컬 시뮬레이션 (Chaos Cache)**

물리 시뮬레이션 결과를 저장한 '캐시' 데이터를 다룰 때 사용됩니다.

* **역할:** 이미 구워진(Baked) 물리 시뮬레이션의 진행 상태를 제어합니다.  
* **활용:**  
  * 시네마틱 연출 시, 플레이어의 위치나 특정 이벤트에 맞춰 물리 파괴 효과가 서서히 진행되도록 설계할 때.  
  * Advance Time에 넣는 값을 조절하여 물리 파괴 속도를 느리게(Slow Motion) 혹은 빠르게 제어 가능.

## **3\. 나이아가라(Niagara) 수동 틱 (Manual Tick)**

나이아가라 파티클 시스템에서 '수동 틱' 옵션을 켰을 때 사용됩니다.

* **역할:** 파티클 시뮬레이션을 엔진의 기본 틱에 맡기지 않고, 개발자가 원하는 타이밍에 원하는 만큼만 업데이트합니다.  
* **활용:** \- 성능 최적화를 위해 특정 조건에서만 파티클을 계산하고 싶을 때.  
  * 타임스톱(시간 정지) 기능 구현 중 특정 파티클만 움직이게 하고 싶을 때.

## **💡 주요 입력 핀(Input Pins) 설명**

| 핀 이름 | 설명 |
| :---- | :---- |
| **Target** | 시간을 진행시킬 대상 (Media Player, Chaos Cache Player 등). |
| **Duration / Delta Time** | 진행시키고자 하는 시간의 양(초 단위). |

## **⚠️ 주의사항**

1. **자동 재생과의 충돌:** 만약 대상이 이미 '자동 재생(Auto Play)' 중인데 Advance Time을 지속적으로 호출하면, 시간이 이중으로 빠르게 흐를 수 있습니다. 수동 제어를 원한다면 대상을 일시정지(Pause)하거나 수동 모드로 설정해야 합니다.  
2. **프레임 종속성:** Tick 이벤트에서 Delta Seconds를 Advance Time에 그대로 넣으면 일반적인 실시간 재생과 동일하게 작동합니다. 하지만 고정된 값을 넣으면 프레임 속도에 따라 체감 속도가 달라질 수 있으므로 주의가 필요합니다.

## **요약**

Advance Time은 \*\*"대상의 내부 시계바늘을 강제로 돌리는 노드"\*\*입니다. 주로 **영상 제어, 물리 시뮬레이션 연출, 특수 파티클 제어**에 필수적인 도구입니다.
