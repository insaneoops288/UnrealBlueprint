# **언리얼 엔진 AnimNotify(애니메이션 노티파이)의 역할**

**AnimNotify**는 애니메이션 시퀀스(Animation Sequence)나 애니메이션 몽타주(Animation Montage) 내의 타임라인에서 특정 프레임에 도달했을 때 이벤트를 호출하는 메커니즘입니다. 애니메이션의 시각적 움직임과 게임의 논리적 기능을 동기화하는 데 핵심적인 역할을 합니다.

## **1\. 주요 역할**

AnimNotify는 주로 다음과 같은 상황에서 사용됩니다:

1. **사운드 및 파이클 효과 실행**:  
   * 발이 땅에 닿는 순간 '발소리' 사운드를 재생하거나 '먼지' 파티클을 스폰합니다.  
2. **게임 로직 트리거**:  
   * 공격 애니메이션 도중 실제 대미지가 들어가는 판정(Collision On/Off)을 활성화합니다.  
   * 무기를 손에서 놓거나 다시 잡는 타이밍을 설정합니다.  
3. **상태 변경**:  
   * 특정 동작 중에는 캐릭터의 이동 속도를 제한하거나, 다른 애니메이션으로의 캔슬 가능 여부를 결정합니다.  
4. **UI 업데이트**:  
   * 특정 동작이 끝나는 시점에 UI 메시지를 띄우거나 연출을 시작합니다.

## **2\. 주요 종류**

언리얼 엔진은 상황에 맞게 세 가지 방식의 노티파이를 제공합니다.

| 종류 | 특징 | 주요 용도 |
| :---- | :---- | :---- |
| **AnimNotify** | 특정 '지점'에서 단발성 이벤트를 발생시킵니다. | 사운드 재생, 이펙트 생성, 변수 변경 |
| **AnimNotifyState** | 시작(Begin)과 끝(End)이 존재하며, 그 '구간' 동안 이벤트를 유지합니다. | 무기 트레이스 활성화, 무적 상태 유지, 이동 속도 보정 |
| **Custom Notify** | 개발자가 C++나 블루프린트 클래스로 직접 로직을 설계한 노티파이입니다. | 복잡한 게임 로직과의 연동 |

## **3\. 블루프린트에서의 활용 방법**

### **애니메이션 블루프린트(AnimBP)에서 수신**

애니메이션 에디터에서 생성한 노티파이는 애니메이션 블루프린트의 \*\*이벤트 그래프(Event Graph)\*\*에서 노드로 호출할 수 있습니다.

1. 애니메이션 타임라인에서 우클릭 \-\> **Add Notify...** \-\> **New Notify** 선택 및 이름 지정.  
2. 애니메이션 블루프린트로 이동.  
3. 그래프에서 AnimNotify\_이름을 검색하여 이벤트 노드를 배치.  
4. 해당 노드 뒤에 실행하고 싶은 로직(예: 발소리 재생) 연결.

### **전용 클래스 활용 (추천 방식)**

재사용성을 높이려면 AnimNotify 또는 AnimNotifyState 클래스를 상속받는 블루프린트 클래스를 별도로 만드는 것이 좋습니다. 이렇게 하면 모든 애니메이션에서 해당 노티파이를 공통으로 사용할 수 있습니다.

* Received\_Notify: 노티파이가 실행될 때 호출되는 함수입니다.  
* MeshComp: 애니메이션을 재생 중인 스켈레탈 메시 컴포넌트 정보를 전달받아 캐릭터 정보에 접근할 수 있습니다.

## **4\. 실전 예시: 발소리 시스템**

1. **시퀀스 에디터**: 걷기 애니메이션에서 왼쪽 발이 닿는 프레임에 Notify\_Footstep\_L 추가.  
2. **AnimBP**: Event AnimNotify\_Footstep\_L 이벤트 배치.  
3. **로직**: 캐릭터의 현재 바닥 재질(Physical Material)을 체크하여 나무 소리 혹은 흙 소리를 재생하도록 연결.

## **5\. 주의사항**

* **성능**: 너무 많은 노티파이가 매 프레임 실행되도록 설정하면 성능에 영향을 줄 수 있습니다.  
* **틱(Tick)**: 애니메이션 최적화(LOD 등)에 의해 멀리 있는 캐릭터의 애니메이션 틱이 생략될 경우, 노티파이가 호출되지 않을 수도 있으므로 중요한 게임 로직은 서버나 더 안정적인 곳에서 관리해야 합니다.
