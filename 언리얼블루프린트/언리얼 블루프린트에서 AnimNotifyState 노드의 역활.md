# **AnimNotifyState의 역할과 활용 방법**

언리얼 엔진(Unreal Engine)의 **AnimNotifyState**는 애니메이션 시퀀스나 몽타주 내에서 \*\*'시작(Begin) \- 진행(Tick) \- 종료(End)'\*\*의 흐름을 가지는 상태 기반의 노티파이입니다.

## **1\. 핵심 역할: 구간 관리**

일반적인 AnimNotify는 애니메이션 타임라인의 특정 프레임에서 한 번만 호출되지만, AnimNotifyState는 타임라인상에 **막대(Bar)** 형태로 배치되어 해당 영역이 실행되는 동안 지속적인 로직을 수행합니다.

### **주요 3단계 함수 (Override)**

블루프린트에서 AnimNotifyState를 상속받아 생성하면 다음 세 가지 핵심 함수를 오버라이드할 수 있습니다:

1. **Received\_NotifyBegin**: 노티파이 구간이 시작될 때 **단 한 번** 호출됩니다. (예: 칼을 휘두르기 시작할 때 트레이스 시작)  
2. **Received\_NotifyTick**: 노티파이 구간 내에서 **매 프레임** 호출됩니다. (예: 구간 내내 캐릭터의 위치나 상태를 체크)  
3. **Received\_NotifyEnd**: 노티파이 구간이 끝날 때 **단 한 번** 호출됩니다. (예: 공격 판정 종료, 효과 제거)

## **2\. 주요 사용 사례**

### **① 공격 판정 (Hitbox / Trace)**

가장 대표적인 사용 사례입니다. 캐릭터가 칼을 휘두를 때, 휘두르는 '동작'이 일어나는 특정 구간 동안만 충돌 판정(Line Trace 또는 Sphere Trace)을 활성화하고 싶을 때 사용합니다.

* **Begin**: 충돌 검사 시작 변수 ON  
* **Tick**: 현재 무기 위치에서 트레이스 실행  
* **End**: 충돌 검사 종료 변수 OFF

### **② 파티클 및 트레일 효과 (FX / Trail)**

칼의 잔상(Trail) 효과나 캐릭터 몸에서 뿜어져 나오는 오라(Aura) 같은 효과를 애니메이션 특정 구간에만 연출할 때 사용합니다.

* **Begin**: 이펙트 생성 (Spawn Emitter)  
* **End**: 이펙트 제거 (Destroy/Deactivate)

### **③ 캐릭터 상태 제어**

애니메이션이 진행되는 동안 캐릭터의 능력을 제한하거나 변경해야 할 때 사용합니다.

* **예시**: "회피" 애니메이션 도중에는 무적 상태로 만들기.  
* **예시**: "스킬 사용" 중에는 이동 속도를 0으로 고정하기.

## **3\. AnimNotify vs AnimNotifyState 차이점**

| 특징 | AnimNotify | AnimNotifyState |
| :---- | :---- | :---- |
| **형태** | 특정 프레임(Point) | 특정 구간(Duration/Bar) |
| **호출 횟수** | 1회 | 시작(1회), 틱(매 프레임), 종료(1회) |
| **주요 용도** | 사운드 재생, 단발성 이펙트 생성 | 공격 판정 유지, 지속 효과, 상태 변경 |
| **정밀도** | 순간적인 이벤트에 적합 | 시간의 흐름이 필요한 로직에 적합 |

## **4\. 구현 시 주의사항**

1. **Mesh 및 Animation 참조**: 각 함수(Begin, Tick, End)는 MeshComp 인수를 전달받으므로, 이를 통해 캐릭터(Owner)에 접근하여 변수를 수정하거나 함수를 호출할 수 있습니다.  
2. **퍼포먼스**: Received\_NotifyTick은 매 프레임 실행되므로 너무 무거운 연산을 넣지 않도록 주의해야 합니다.  
3. **중단 처리**: 애니메이션이 도중에 캔슬되더라도 Received\_NotifyEnd는 호출되므로, 상태를 원복하는 로직(예: 변수 초기화)은 반드시 End에서 처리하는 것이 안전합니다.
