# **Attach Actor To Actor 노드 상세 설명**

언리얼 엔진의 Attach Actor To Actor 노드는 한 액터(Child)를 다른 액터(Parent)에게 물리적/계층적으로 부착하는 데 사용됩니다. 이 작업을 통해 자식 액터는 부모 액터의 트랜스폼(위치, 회전, 스케일) 변화를 그대로 상속받게 됩니다.

## **1\. 주요 역할**

* **계층 구조 생성**: 런타임 중에 액터 간의 부모-자식 관계를 형성합니다.  
* **트랜스폼 상속**: 부모 액터가 움직이거나 회전할 때 자식 액터가 마치 한 몸처럼 함께 움직입니다.  
* **소켓 시스템 활용**: 부모 액터의 특정 \*\*소켓(Socket)\*\*이나 **본(Bone)** 이름에 정밀하게 부착할 수 있습니다.

## **2\. 주요 파라미터 (Inputs)**

| 파라미터 | 설명 |
| :---- | :---- |
| **Target** | 부착될 자식 액터(Child Actor)입니다. |
| **Parent Actor** | 부모가 될 액터(Parent Actor)입니다. |
| **Socket Name** | 부모 액터의 메시(Mesh)에 설정된 소켓이나 본의 이름입니다. 비워두면 부모의 루트 컴포넌트에 부착됩니다. |
| **Location Rule** | 부착 시 위치를 어떻게 처리할지 결정합니다. (아래 상세 설명 참조) |
| **Rotation Rule** | 부착 시 회전값을 어떻게 처리할지 결정합니다. |
| **Scale Rule** | 부착 시 스케일값을 어떻게 처리할지 결정합니다. |
| **Weld Simulated Bodies** | 물리 시뮬레이션 중인 바디들을 하나로 합칠지 여부입니다. |

## **3\. Attachment Rules (위치/회전/스케일 규칙)**

부착 시 가장 중요한 설정값들입니다:

1. **Keep Relative (상대적 유지)**:  
   * 자식 액터가 현재 가지고 있는 상대적인 오프셋 값을 유지합니다.  
   * 예: 부모로부터 100 유닛 떨어져 있었다면 부착 후에도 그 간격을 유지합니다.  
2. **Keep World (월드 좌표 유지)**:  
   * 자식 액터가 월드 상에 있던 현재 위치를 그대로 고수합니다. 부모와의 상대적 거리는 자동으로 계산됩니다.  
   * 부착 시 비주얼적인 튀는 현상(Snap)을 방지할 때 유용합니다.  
3. **Snap to Target (타겟에 맞춤)**:  
   * 자식 액터를 부모(또는 소켓)의 위치로 즉시 이동시킵니다.  
   * 자식의 상대 좌표가 (0, 0, 0)이 됩니다.

## **4\. 대표적인 활용 사례**

* **무기 장착**: 캐릭터의 손 소켓(hand\_r\_socket)에 칼이나 총 액터를 부착할 때 사용합니다.  
* **탈것 탑승**: 캐릭터를 자동차나 말 액터의 특정 위치에 부착하여 함께 이동하게 합니다.  
* **아이템 획득**: 플레이어가 상자를 집어 들었을 때, 상자 액터를 플레이어 캐릭터 앞에 부착합니다.  
* **특수 효과**: 이동하는 발판 위에 파티클 효과 액터를 부착하여 발판을 따라다니게 합니다.

## **5\. 주의사항**

* **충돌(Collision)**: 부착된 두 액터가 서로 충돌 설정을 가지고 있으면 물리적 오류(떨림 현상 등)가 발생할 수 있습니다. 부착 시에는 자식 액터의 충돌을 끄거나 무시하도록 설정하는 것이 좋습니다.  
* **Detach**: 부착을 해제하고 싶을 때는 Detach From Actor 노드를 사용합니다.  
* **루트 컴포넌트**: 이 노드는 내부적으로 자식 액터의 Root Component를 부모 액터의 Root Component(또는 지정된 소켓)에 부착하는 방식으로 작동합니다.
