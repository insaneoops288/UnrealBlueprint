# **Attach Actor To Component 노드 이해하기**

언리얼 엔진에서 **Attach Actor To Component** 노드는 한 액터를 다른 액터의 특정 부위에 '자식(Child)'으로 귀속시키는 역할을 합니다. 부착된 액터는 부모 컴포넌트의 위치, 회전, 크기 변화를 그대로 따르게 됩니다.

## **1\. 주요 파라미터 (Input Pins)**

| 파라미터 | 설명 |
| :---- | :---- |
| **Target** | 부모에게 달라붙을 **자식 액터**입니다. |
| **Parent** | 자식 액터가 붙게 될 **대상 컴포넌트**입니다. (예: 캐릭터의 Mesh) |
| **Socket Name** | 부모 컴포넌트에 설정된 특정 \*\*소켓(Socket)\*\*이나 \*\*본(Bone)\*\*의 이름입니다. 비워두면 컴포넌트의 루트에 붙습니다. |
| **Location Rule** | 위치 값을 어떻게 처리할지 결정합니다. |
| **Rotation Rule** | 회전 값을 어떻게 처리할지 결정합니다. |
| **Scale Rule** | 크기 값을 어떻게 처리할지 결정합니다. |
| **Weld Simulated Bodies** | 물리 시뮬레이션 중인 바디들을 하나로 합칠지 여부입니다. |

## **2\. 부착 규칙 (Attachment Rules)**

가장 중요한 설정은 위치, 회전, 크기 규칙입니다. 보통 다음 세 가지 옵션 중 하나를 선택합니다.

1. **Keep Relative (상대값 유지):** 부착 직전의 상대적 거리를 유지합니다. (이미 적절한 위치에 있을 때 사용)  
2. **Keep World (월드값 유지):** 부모에게 붙더라도 현재 월드 공간에서의 시각적 위치를 그대로 유지하도록 상대 좌표를 계산합니다.  
3. **Snap to Target (대상에 스냅):** 자식 액터의 위치를 부모 소켓/컴포넌트의 위치와 **강제로 일치**시킵니다. (무기를 손에 쥐어줄 때 가장 많이 사용)

## **3\. 주요 활용 사례**

* **아이템 장착:** 캐릭터가 바닥에 떨어진 무기를 주웠을 때, 무기 액터를 캐릭터 메시의 Hand\_Socket에 부착합니다.  
* **탑승 시스템:** 캐릭터가 차량에 탔을 때, 캐릭터 액터를 차량 메시의 특정 좌석 소켓에 부착합니다.  
* **장비 및 코스튬:** 모자, 가방 등의 액세서리를 캐릭터 모델의 특정 부위에 고정할 때 사용합니다.

## **4\. 관련 노드**

* **Detach from Actor:** 부착된 관계를 끊고 다시 독립적인 액터로 만듭니다.  
* **Attach Actor To Actor:** 컴포넌트 단위가 아닌 액터의 루트 대 루트로 부착합니다.  
* **Get Socket Transform:** 소켓의 현재 위치와 회전값을 가져올 때 함께 자주 사용됩니다.

## **5\. 주의사항**

1. **무브먼트 컴포넌트:** 캐릭터가 부착된 상태에서 물리 엔진이 충돌을 일으키지 않도록, 부착된 자식 액터의 **Collision** 설정을 적절히 변경(예: NoCollision 또는 Ignore)해야 하는 경우가 많습니다.  
2. **계층 구조:** 부모가 파괴(Destroy)되면 자식 액터도 함께 영향을 받을 수 있으므로 생명 주기를 고려해야 합니다.
