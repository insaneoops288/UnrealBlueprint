# **Blend Poses (Enum) 노드의 역할과 활용**

언리얼 엔진(Unreal Engine)의 애니메이션 블루프린트 애님 그래프(AnimGraph)에서 **Blend Poses (Enum)** 노드는 특정 **열거형(Enumeration)** 변수의 값에 따라 출력할 포즈를 결정하는 "스위치(Switch)" 역할을 합니다.

## **1\. 핵심 역할**

이 노드는 캐릭터의 상태가 바뀔 때(예: 대기 \-\> 걷기 \-\> 달리기) 해당 상태에 맞는 애니메이션 포즈를 선택하고, 상태가 변할 때 포즈 간의 \*\*블렌딩(Blending, 보간)\*\*을 처리하여 끊김 없는 움직임을 만들어냅니다.

## **2\. 주요 특징**

### **1\) 열거형(Enum) 기반 동적 핀 생성**

* 사용자가 정의한 Enum 타입을 선택하면, 해당 Enum에 포함된 모든 항목(Enumerator)이 노드의 입력 핀으로 자동 생성됩니다.  
* 예를 들어, E\_MovementState라는 Enum에 Idle, Walking, Running이 있다면 노드에 이 세 가지 입력을 위한 포즈 핀이 생깁니다.

### **2\) 상태 기반 포즈 전환**

* Active Enum Value 핀에 현재 캐릭터의 상태 변수를 연결하면, 그 값에 일치하는 포즈 핀의 애니메이션이 최종 출력으로 나갑니다.

### **3\) 개별 블렌드 설정 (Blend Time)**

* 각 상태로 전환될 때 걸리는 시간(Blend Time)을 개별적으로 설정할 수 있습니다.  
* 예를 들어, 'Idle'에서 'Running'으로 갈 때는 0.2초, 'Running'에서 'Idle'로 올 때는 0.5초로 다르게 설정하여 자연스러운 연출이 가능합니다.

## **3\. 주요 설정 항목 (Details 패널)**

* **Blend Time**: 포즈가 전환될 때 소요되는 시간(초)입니다.  
* **Transition Type**: 블렌딩 시 사용할 보간 방식(Linear, Cubic, Hermite 등)을 선택합니다.  
* **Blend Curve**: 커브 자산을 사용하여 더 정교한 가감속을 표현할 수 있습니다.  
* **Reset Child on Activation**: 해당 포즈가 활성화될 때 애니메이션의 시간(Time)을 0으로 리셋할지 여부를 결정합니다.

## **4\. 사용 예시 (Scenario)**

가장 흔한 사례는 **캐릭터의 전투 상태 관리**입니다.

1. E\_CombatStance라는 Enum을 만듭니다 (Unarmed, Sword, Bow).  
2. 애님 그래프에서 Blend Poses (E\_CombatStance) 노드를 배치합니다.  
3. 각 핀에 해당하는 로코모션(Locomotion) 캐시 포즈나 스테이트 머신을 연결합니다.  
4. 변수 값이 바뀌면 노드가 알아서 맨손 애니메이션에서 검 공격 자세로 부드럽게 섞어줍니다.

## **5\. 장점**

* **가독성**: 여러 개의 Blend Poses by bool을 중첩해서 사용하는 것보다 훨씬 깔끔하게 로직을 짤 수 있습니다.  
* **효율성**: 현재 활성화된 포즈만 계산(Evaluation)하므로 성능 면에서 유리합니다.  
* **확장성**: Enum에 새로운 상태를 추가하기만 하면 노드에 즉시 반영되어 확장이 쉽습니다.

**팁**: 노드를 우클릭하여 "Add Pose for Enum" 메뉴를 통해 필요한 특정 상태만 핀으로 노출시켜 사용할 수도 있습니다.
