언리얼 엔진에서 \*\*Blueprint Function Library(블루프린트 함수 라이브러리)\*\*는 프로젝트 어디에서나 공통적으로 사용할 수 있는 **'공용 도구 상자'** 같은 역할을 합니다.

특정 액터나 캐릭터에 종속되지 않는 독립적인 함수들을 모아두는 곳으로, 코드의 재사용성과 관리 효율을 극대화하는 데 필수적인 요소입니다.

## ---

**1\. 핵심 개념: "언제 어디서든 호출 가능"**

일반적인 블루프린트 함수는 해당 블루프린트 클래스의 인스턴스가 있어야 호출할 수 있지만, 함수 라이브러리에 선언된 함수는 **스태틱(Static)** 성격을 가집니다.

* **전역 접근성:** 레벨 블루프린트, 캐릭터, UI(UMG), 게임 모드 등 어떤 블루프린트에서도 노드를 바로 검색해서 사용할 수 있습니다.  
* **인스턴스 불필요:** 별도의 '형변환(Cast To)'이나 변수 참조 없이 함수 이름만으로 실행됩니다.

## ---

**2\. 주요 역할과 장점**

### **① 중복 코드 방지 (DRY: Don't Repeat Yourself)**

여러 클래스에서 똑같이 반복되는 로직이 있다면 라이브러리로 옮기는 것이 좋습니다.

* *예: "체력 퍼센트 계산", "아이템 이름 포맷팅", "특정 거리 내의 적 찾기"* 등.

### **② 프로젝트 구조의 체계화**

기능별로 라이브러리를 나누어 관리하면 프로젝트가 커져도 길을 잃지 않습니다.

* Math\_Library: 복잡한 수학 계산식 모음  
* UI\_Utility\_Library: 위젯 애니메이션이나 텍스트 변환 모음  
* GameHelpers: 게임 규칙 관련 공통 로직

### **③ 유지보수의 용이성**

로직을 수정해야 할 때, 수십 개의 블루프린트를 찾아다닐 필요 없이 **라이브러리 안의 함수 하나만 수정**하면 프로젝트 전체에 적용됩니다.

## ---

**3\. 실무 활용 예시**

함수 라이브러리는 주로 다음과 같은 **유틸리티(Utility)** 성격의 기능을 만들 때 가장 빛을 발합니다.

| 분류 | 활용 예시 |
| :---- | :---- |
| **수학 연산** | 두 점 사이의 특정 각도 계산, 확률 판정(Random Chance) 등 |
| **데이터 변환** | 초(Second) 단위를 "00:00" 형식의 문자열로 변환, 열거형(Enum)을 텍스트로 변환 |
| **게임 로직** | 특정 태그를 가진 모든 액터 중 가장 가까운 대상 찾기 |
| **시스템** | 현재 플랫폼 확인(모바일/PC), 세이브 파일 경로 생성 |

## ---

**4\. 주의사항 및 팁**

* **상태 저장 불가:** 함수 라이브러리는 '함수'만 가질 수 있으며, **변수를 가질 수 없습니다.** (상태를 기억해야 한다면 게임 인스턴스나 게임 스테이트를 사용해야 합니다.)  
* **월드 컨텍스트(World Context):** Get Player Character나 Spawn Actor From Class처럼 월드 정보가 필요한 노드를 라이브러리에서 쓸 때는, 함수의 입력 핀에 'World Context Object'가 자동으로 생기거나 명시해줘야 할 때가 있습니다.  
* **C++ 라이브러리와의 조화:** 성능이 중요한 복잡한 계산은 C++에서 UBlueprintFunctionLibrary를 상속받아 만들고, 블루프린트에서 호출하는 방식이 가장 이상적입니다.

**결론**

"이 로직을 다른 곳에서도 또 쓸 것 같은데?"라는 생각이 드는 순간이 **Blueprint Function Library**를 만들 때입니다. 프로젝트 초기에 유틸리티 라이브러리를 잘 구축해두면 개발 속도가 비약적으로 빨라집니다.

혹시 지금 만들고 계신 기능 중에 라이브러리로 옮길지 말지 고민되는 특정 로직이 있으신가요? 상황을 말씀해 주시면 구조 설계에 도움을 드릴 수 있습니다.
