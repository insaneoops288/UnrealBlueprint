# **Break Hit Result 노드 가이드**

언리얼 엔진에서 Hit Result 구조체는 물리적 충돌이나 레이캐스트(Line Trace)가 발생했을 때 얻을 수 있는 방대한 정보를 담고 있습니다. **Break Hit Result** 노드는 이 뭉쳐진 데이터를 우리가 직접 사용할 수 있는 개별 변수(위치, 대상, 노멀 등)로 쪼개주는 역할을 합니다.

## **1\. 주요 출력 핀(Output Pins) 상세 설명**

가장 자주 사용되는 핵심 정보들은 다음과 같습니다.

| 핀 이름 | 데이터 타입 | 설명 |
| :---- | :---- | :---- |
| **Blocking Hit** | Boolean | 무언가에 실제로 부딪혔는지 여부를 반환합니다. (True/False) |
| **Initial Overlap** | Boolean | 트레이스가 시작될 때 이미 물체 내부에 겹쳐져 있었는지 여부입니다. |
| **Time** | Float | 트레이스 시작점(0.0)에서 끝점(1.0) 사이 중 어느 지점에서 충돌했는지의 비율입니다. |
| **Location** | Vector | 충돌이 일어난 실제 좌표입니다. (주로 부딪힌 위치에 이펙트를 소환할 때 사용) |
| **Impact Point** | Vector | 실제 표면에 닿은 정확한 지점입니다. (Location과 유사하지만 물리 계산 시 더 정밀함) |
| **Normal** | Vector | 충돌한 표면이 바라보는 방향(법선 벡터)입니다. |
| **Impact Normal** | Vector | 충돌 시 충격을 받은 표면의 방향입니다. |
| **Phys Mat** | Phys Material | 부딪힌 물체의 **물리 재질**입니다. (발소리 사운드나 바닥 재질에 따른 이펙트 구분 시 필수) |
| **Hit Actor** | Actor | **가장 중요한 핀** 중 하나로, 어떤 액터와 부딪혔는지를 알려줍니다. |
| **Hit Component** | Component | 액터 내의 구체적으로 어떤 컴포넌트(스태틱 메시, 콜리전 박스 등)와 부딪혔는지 알려줍니다. |
| **Hit Bone Name** | Name | 스켈레탈 메시일 경우, 정확히 어떤 \*\*뼈(Bone)\*\*를 맞췄는지 알려줍니다. (헤드샷 판정 등) |

## **2\. 사용 예시 (Use Cases)**

### **① 라인 트레이스(Line Trace) 처리**

사용자가 총을 쐈을 때 Line Trace By Channel 노드를 사용하면 결과값으로 Out Hit이 나옵니다. 이때 **Break Hit Result**를 연결하여:

* **Hit Actor**를 가져와 데미지를 입힘 (Apply Damage).  
* **Impact Point** 위치에 총알 자국(Decal)이나 불꽃 파티클을 소환.  
* **Hit Bone Name**이 'head'인지 확인하여 헤드샷 판정.

### **② 물리 재질에 따른 사운드 변경**

캐릭터가 걷는 바닥을 체크할 때:

* **Phys Mat** 값을 가져와서, 재질이 'Wood'면 나무 걷는 소리를, 'Water'면 물소리를 재생하도록 분기 처리합니다.

### **③ 표면 각도에 따른 도탄(Ricochet) 계산**

* **Impact Normal** 값을 활용하여 총알이 반사되어 나갈 각도를 계산할 수 있습니다.

## **3\. 팁 (Tips)**

* **노드 확장**: 기본적으로 노드는 작게 보이지만, 노드 하단의 화살표(숨겨진 핀 표시)를 누르면 위에 나열된 모든 세부 정보들을 확인할 수 있습니다.  
* **성능**: 단순히 충돌 여부만 궁금하다면 Blocking Hit 핀만 확인하면 되므로, 복잡한 로직을 수행하기 전 항상 Blocking Hit이 True인지 먼저 체크하는 습관이 중요합니다.
