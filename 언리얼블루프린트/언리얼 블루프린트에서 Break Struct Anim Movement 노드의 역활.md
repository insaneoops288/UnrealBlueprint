# **언리얼 엔진: Break Struct Anim Movement 노드 이해하기**

언리얼 엔진에서 \*\*구조체(Structure)\*\*는 서로 연관된 여러 변수들을 하나의 단위로 묶은 데이터 타입입니다. Break 노드는 이 묶음을 해제하여 내부의 구체적인 값들을 추출할 때 사용합니다.

## **1\. 핵심 역할: 데이터의 분해 (Deconstruction)**

Break Struct Anim Movement 노드의 가장 주요한 역할은 \*\*하나의 핀(Struct)\*\*으로 들어온 복합적인 데이터를 \*\*여러 개의 핀(Individual Variables)\*\*으로 나누는 것입니다.

* **입력:** Anim Movement 구조체 변수 (보통 이동 속도, 방향, 지면 접지 여부, 기울기 등의 정보를 포함).  
* **출력:** 구조체 내부에 정의된 개별 변수들 (예: float Speed, bool IsFalling, vector Velocity 등).

## **2\. 왜 사용하는가?**

### **① 노드 그래프의 정문화 (Clean Code)**

이동 관련 데이터를 개별 변수로 하나하나 전송하면 블루프린트 선(Wire)이 너무 복잡해집니다. 캐릭터에서 애니메이션 블루프린트로 데이터를 보낼 때, 10개의 변수를 따로 보내는 대신 1개의 구조체로 보낸 뒤, 필요한 곳에서만 Break를 통해 꺼내 쓰는 것이 훨씬 깔끔합니다.

### **② 데이터 일관성**

이동에 필요한 데이터 세트를 하나의 구조체로 관리하면, 함수나 인터페이스를 통해 데이터를 전달할 때 누락되는 값 없이 통째로 전달할 수 있어 관리가 용이합니다.

## **3\. 실무 활용 예시 (애니메이션 블루프린트)**

일반적으로 애니메이션 블루프린트의 \*\*이벤트 그래프(Event Graph)\*\*에서 다음과 같이 사용됩니다.

1. **Update Animation 이벤트:** 매 프레임마다 캐릭터의 상태를 확인합니다.  
2. **구조체 획득:** 캐릭터의 무브먼트 컴포넌트나 변수에서 MovementData 구조체를 가져옵니다.  
3. **Break Struct 실행:** Break Struct Anim Movement 노드를 배치합니다.  
4. **로직 연결:**  
   * Speed 출력 핀을 블렌드 스페이스(BlendSpace)의 입력 값으로 연결.  
   * IsFalling 출력 핀을 사용하여 점프 애니메이션 전환(Transition) 조건으로 연결.

## **4\. 팁: 필요한 핀만 보이기**

Break Struct 노드는 구조체 내부 변수가 많을 경우 노드 크기가 매우 커질 수 있습니다.

* **사용 방법:** 노드를 선택한 후 **디테일(Details) 패널**을 보면 구조체 내부의 변수 목록이 체크박스로 나타납니다.  
* **최적화:** 현재 로직에 사용하지 않는 변수들의 체크를 해제하면, 해당 출력 핀이 숨겨져 그래프를 더 간결하게 유지할 수 있습니다.

## **5\. 요약**

Break Struct Anim Movement 노드는 \*\*"이동 상태라는 종합 선물 세트를 열어서, 지금 당장 필요한 '속도'나 '상태'만 꺼내 쓰기 위한 도구"\*\*라고 이해하시면 됩니다.
