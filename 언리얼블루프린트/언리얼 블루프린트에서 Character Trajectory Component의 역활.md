# **Character Trajectory Component의 역할과 기능**

언리얼 엔진(특히 UE 5.1 이상)에서 **Character Trajectory Component**는 차세대 애니메이션 시스템인 \*\*모션 매칭(Motion Matching)\*\*을 구동하기 위한 핵심 데이터를 제공합니다. 이 컴포넌트는 캐릭터의 이동 상태를 분석하여 '궤적(Trajectory)' 정보를 생성합니다.

## **1\. 핵심 역할: 데이터의 가교**

이 컴포넌트의 가장 큰 역할은 \*\*캐릭터 이동 컴포넌트(Character Movement Component)\*\*의 물리적 데이터(속도, 가속도, 위치)를 애니메이션 블루프린트가 이해할 수 있는 **궤적 데이터**로 변환하는 것입니다.

### **주요 기능**

* **과거 기록(History Tracking):** 캐릭터가 직전 몇 초 동안 어디에 있었고 어떤 방향을 보고 있었는지 기록합니다.  
* **미래 예측(Future Prediction):** 현재의 입력값(Input)과 속도를 바탕으로, 캐릭터가 앞으로 몇 초 뒤에 도달할 위치와 회전값을 계산합니다.  
* **데이터 샘플링:** 특정 시간 간격(예: 0.1초, 0.5초 등)으로 위치와 방향 데이터를 샘플링하여 궤적을 선형 또는 곡선 형태로 구성합니다.

## **2\. 왜 필요한가? (모션 매칭과의 관계)**

기존의 스테이트 머신(State Machine) 방식은 "현재 속도가 얼마인가?"를 보고 애니메이션을 바꿨지만, 모션 매칭은 다음과 같은 질문을 던집니다.

**"지금 캐릭터가 가려는 경로(미래 궤적)와 가장 잘 일치하는 애니메이션 포즈는 무엇인가?"**

Character Trajectory Component가 생성한 미래 예측 선과 \*\*포즈 서치 데이터베이스(Pose Search Database)\*\*에 저장된 애니메이션의 궤적을 비교하여, 가장 오차가 적은 프레임을 즉시 재생합니다. 이를 통해 매우 자연스럽고 끊김 없는 방향 전환과 가속/감속 표현이 가능해집니다.

## **3\. 주요 속성 및 구조**

컴포넌트 설정에서 조절할 수 있는 주요 파라미터는 다음과 같습니다.

* **History Settings:** 얼마나 오래전까지의 궤적을 저장할지 결정합니다. (보통 발자국 소리나 관성 효과 처리에 사용)  
* **Prediction Settings:** 미래를 얼마나 멀리 예측할지(Seconds), 얼마나 세밀하게 샘플링할지 설정합니다.  
* **Sample Rate:** 초당 몇 개의 샘플을 생성할지 결정하며, 수치가 높을수록 궤적이 정교해지지만 연산 비용이 늘어납니다.

## **4\. 시각적 확인 (Visualizer)**

에디터에서 캐릭터를 선택하고 컴포넌트를 활성화하면 캐릭터 발아래에 **색상이 있는 점들과 선**이 나타납니다.

* **과거 경로:** 보통 파란색이나 어두운 계열로 표시됩니다.  
* **미래 경로:** 보통 오렌지색이나 밝은 계열로 표시되며, 조이스틱이나 키보드 입력에 따라 실시간으로 구부러지며 변화합니다.

## **5\. 요약**

| 구분 | 역할 설명 |
| :---- | :---- |
| **입력 분석** | 사용자의 이동 입력을 기반으로 가속/감속 경로 계산 |
| **애니메이션 선택** | 모션 매칭 엔진에 '목표 경로' 데이터 전달 |
| **자연스러운 전환** | 급격한 방향 전환 시 관성을 계산하여 발 미끄러짐(Foot Sliding) 방지 |
| **디버깅** | 런타임에서 캐릭터의 이동 의도를 시각적으로 파악 가능 |

Character Trajectory Component는 단순한 위치 기록 장치를 넘어, **캐릭터의 '이동 의도'를 데이터화**하여 애니메이션의 질을 한 단계 높여주는 필수 도구입니다.
