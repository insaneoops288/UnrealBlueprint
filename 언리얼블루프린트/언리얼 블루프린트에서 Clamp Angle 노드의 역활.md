# **언리얼 엔진 블루프린트: Clamp Angle 노드 이해하기**

Clamp Angle 노드는 입력된 각도(단위: 도/Degrees)가 지정된 최소(Min) 및 최대(Max) 범위를 벗어나지 않도록 고정하는 역할을 합니다. 주로 카메라 회전 제한, 캐릭터의 고개 돌리기 제한, 포탑의 회전 반경 설정 등에 사용됩니다.

## **1\. 노드의 주요 기능**

일반적인 Clamp (Float) 노드는 단순히 숫자의 크기만 비교하지만, **Clamp Angle**은 다음과 같은 특징을 가집니다.

* **순환성 처리**: 각도는 360도가 되면 다시 0도가 됩니다. 이 노드는 내부적으로 각도를 정규화(Normalize)하여 계산하므로, \-180\~180이나 0\~360 사이의 값들을 논리적으로 처리합니다.  
* **최단 경로 유지**: 회전 값이 범위를 넘어갈 때, 단순히 수치를 깎는 것이 아니라 각도 체계 안에서 가장 가까운 제한값으로 고정합니다.

## **2\. 노드 핀(Pin) 설명**

| 핀 이름 | 타입 | 설명 |
| :---- | :---- | :---- |
| **Angle** | Float | 제한하고 싶은 현재 각도 값입니다. |
| **Min Angle Degrees** | Float | 허용되는 최소 각도입니다. (예: \-90) |
| **Max Angle Degrees** | Float | 허용되는 최대 각도입니다. (예: 90\) |
| **Return Value** | Float | 제한 범위 내로 조정된 결과 각도 값입니다. |

## **3\. 왜 일반 Clamp 대신 사용하나요?**

일반 Clamp를 각도에 사용하면 문제가 발생할 수 있습니다.

예를 들어, 현재 각도가 350도이고 제한 범위가 0\~20도라고 가정해 봅시다.

* **일반 Clamp (Float)**: 350은 20보다 크므로 결과값을 20으로 강제 고정합니다.  
* **Clamp Angle**: 350도는 실제로는 \-10도와 같습니다. 따라서 범위(0\~20)에 맞추기 위해 가장 가까운 0으로 부드럽게 처리하거나 논리적인 회전 방향을 고려합니다.

## **4\. 실전 활용 사례**

### **① 1인칭/3인칭 카메라 상하 회전(Pitch) 제한**

플레이어가 화면을 너무 위로 보거나 아래로 봐서 카메라가 뒤집히는 것을 방지합니다.

* **Min**: \-80.0  
* **Max**: 80.0  
* 이렇게 설정하면 캐릭터가 하늘이나 바닥을 수직으로 뚫고 지나가는 현상을 막을 수 있습니다.

### **② 캐릭터 고개 돌리기 (Head Look-at)**

캐릭터가 특정 대상을 바라볼 때 목이 360도 돌아가지 않도록 제한합니다.

* **Min**: \-45.0  
* **Max**: 45.0  
* 좌우 회전(Yaw) 값을 이 노드에 통과시키면 자연스러운 시선 처리가 가능합니다.

## **5\. 함께 사용하면 좋은 노드**

* **Normalize Axis**: 각도를 \-180 \~ 180 사이의 표준 값으로 변환해 줍니다. Clamp Angle 사용 전후에 데이터의 일관성을 위해 자주 함께 사용됩니다.  
* **Combine Rotators**: 두 개의 회전값을 합친 후 특정 축만 Clamp 할 때 유용합니다.

## **💡 팁**

* Clamp Angle은 내부적으로 각도를 \-180에서 180 사이로 변환하여 계산하는 경우가 많습니다. 만약 결과값이 예상과 다르다면 입력값이 너무 큰 값(예: 720도 등)은 아닌지 확인해 보세요.
