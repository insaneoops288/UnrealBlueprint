# **언리얼 엔진 애니메이션: Component Space 노드 이해하기**

언리얼 엔진의 애니메이션 블루프린트(AnimGraph)에서 포즈 데이터를 처리할 때 가장 중요한 개념 중 하나가 \*\*공간(Space)\*\*의 변환입니다.

## **1\. 공간의 두 가지 종류**

### **로컬 공간 (Local Space)**

* **특징**: 각 뼈의 트랜스폼(위치, 회전, 스케일)이 \*\*부모 뼈(Parent Bone)\*\*를 기준으로 계산됩니다.  
* **색상**: AnimGraph에서 노드 연결선이 **연한 파란색**으로 표시됩니다.  
* **장점**: 관절의 굽힘이나 회전 등 자연스러운 계층 구조 움직임을 표현하기에 적합합니다.

### **컴포넌트 공간 (Component Space)**

* **특징**: 각 뼈의 트랜스폼이 부모와 상관없이 \*\*스켈레탈 메쉬 컴포넌트의 루트(Origin)\*\*를 기준으로 계산됩니다.  
* **색상**: AnimGraph에서 노드 연결선이 **흰색**으로 표시됩니다.  
* **장점**: 특정 뼈를 월드 좌표나 캐릭터 정면 방향 등 절대적인 기준에 맞춰 움직여야 할 때(예: IK, LookAt) 사용됩니다.

## **2\. 주요 변환 노드**

AnimGraph에서 이 두 공간을 서로 전환하기 위해 사용하는 핵심 노드는 다음과 같습니다.

### **\[Local to Component\] 노드**

* **역할**: 로컬 공간 포즈(파란색 선)를 컴포넌트 공간 포즈(흰색 선)로 변환합니다.  
* **사용 시점**: 로컬로 계산된 애니메이션 데이터를 기반으로 전체적인 좌표 수정이 필요할 때 입구 역할을 합니다.

### **\[Component to Local\] 노드**

* **역할**: 컴포넌트 공간 포즈(흰색 선)를 다시 로컬 공간 포즈(파란색 선)로 되돌립니다.  
* **사용 시점**: 컴포넌트 공간에서 IK나 회전 수정을 마친 후, 최종적으로 엔진이 이해하는 표준 로컬 포즈로 반환하여 출력 포즈(Output Pose)에 전달하기 전에 사용합니다.

## **3\. 왜 Component Space가 필요한가요? (주요 활용 사례)**

단순히 걷기, 뛰기 애니메이션을 재생할 때는 로컬 공간만으로 충분하지만, 다음과 같은 복잡한 계산에는 컴포넌트 공간이 필수적입니다.

1. **IK (Inverse Kinematics)**:  
   * 발이 지면의 경사에 맞춰 닿아야 할 때, 발 뼈의 위치는 부모인 종아리 뼈가 아니라 캐릭터의 전체 높이(컴포넌트 기준)에서 계산되어야 정확합니다.  
2. **LookAt / 머리 회전**:  
   * 캐릭터가 특정 타겟을 쳐다보게 만들 때, 머리 뼈를 "캐릭터 정면 기준"으로 회전시켜야 합니다. 이때 컴포넌트 공간에서 작업하는 것이 훨씬 직관적입니다.  
3. **특정 뼈의 직접적인 제어 (Modify Bone)**:  
   * 특정 뼈를 월드 좌표나 메쉬 좌표 시스템에 맞춰 고정하거나 강제로 회전시키고 싶을 때 사용합니다.

## **4\. AnimGraph에서의 일반적인 흐름**

보통 다음과 같은 파이프라인 구조를 가집니다.

1. **State Machine / Blend Space**: 로컬 공간에서 기본 애니메이션 생성 (파란색 선)  
2. **Local to Component**: 공간 변환 (흰색 선으로 변경)  
3. **Skeletal Control Nodes**: IK, LookAt, LegIK 등 컴포넌트 공간 노드 적용 (흰색 선 구간)  
4. **Component to Local**: 다시 로컬 공간으로 변환 (파란색 선으로 복구)  
5. **Output Pose**: 최종 결과 출력

## **5\. 주의사항 및 팁**

* **성능**: 공간 변환 노드(Local to Component 등)를 너무 자주 남발하면 연산 비용이 발생합니다. 가능한 한 로컬 공간에서 처리할 수 있는 것은 로컬에서 하고, 컴포넌트 공간이 필요한 노드들을 한데 모아서 한 번에 변환하는 것이 효율적입니다.  
* **데이터 유실 없음**: 공간을 변환한다고 해서 애니메이션 데이터가 변하는 것이 아니라, 데이터를 바라보는 \*\*'기준점'\*\*만 바뀌는 것입니다.
