# **언리얼 엔진 Control Rig 노드의 역할과 활용**

언리얼 엔진의 애니메이션 블루프린트(Animation Blueprint) 내에 존재하는 **Control Rig** 노드는 캐릭터의 본(Bone) 구조를 실시간으로 제어하고, 절차적(Procedural)인 움직임을 구현하는 핵심적인 연결 고리입니다.

## **1\. 핵심 개념**

Control Rig는 엔진 내부에서 리깅(Rigging) 시스템을 구축할 수 있게 해주는 프레임워크입니다. 애니메이션 블루프린트에서 이 노드를 호출한다는 것은, \*\*"미리 정의된 리깅 로직을 현재 애니메이션 포즈 위에 덮어씌우거나 수정하겠다"\*\*는 의미입니다.

## **2\. 주요 역할**

### **① 실시간 역운동학 (IK \- Inverse Kinematics) 구현**

가장 대중적인 역할입니다. 발이 지면의 경사에 맞춰 정확히 놓이게 하거나(Foot IK), 캐릭터가 특정 물체를 집으려고 손을 뻗는 동작 등을 계산할 때 사용됩니다.

* **기존 방식:** 애니메이션 블루프린트의 기본 IK 노드 사용.  
* **Control Rig 방식:** 복잡한 다관절 IK나 커스텀 로직을 직접 설계하여 훨씬 정교한 움직임 구현 가능.

### **② 절차적 애니메이션 (Procedural Animation)**

미리 제작된 FBX 애니메이션 파일 없이, 수학적 계산이나 입력 값을 통해 실시간으로 움직임을 생성합니다.

* 예: 촉수의 구부러짐, 캐릭터의 머리가 타겟을 바라보는 Look-at 기능, 물리 시뮬레이션과 결합된 본의 움직임.

### **③ 리타겟팅 및 포스트 프로세스 (Post-Process)**

애니메이션이 재생된 후, 마지막 단계에서 본의 위치를 교정합니다. 예를 들어, 무기가 손바닥에서 어긋나지 않도록 강제로 위치를 고정하거나, 특정 본이 회전할 때 주변 근육 본이 함께 움직이도록 하는 로직을 처리합니다.

### **④ 변수 공유 및 데이터 전달**

블루프린트에서 계산된 변수(예: 타겟 좌표, 속도, 알파 값 등)를 Control Rig 내부의 파라미터로 전달하여, 게임 플레이 상황에 유동적으로 반응하는 애니메이션을 만듭니다.

## **3\. 작동 워크플로우**

1. **Control Rig 에셋 생성:** Control Rig 에셋 내에서 본들을 어떻게 제어할지 그래프(Graph)를 작성합니다.  
2. **애니메이션 블루프린트 구성:** 애님 그래프(AnimGraph)에서 Control Rig 노드를 추가합니다.  
3. **에셋 할당:** 노드의 상세 디테일 패널에서 위에서 만든 에셋을 선택합니다.  
4. **입력/출력 매핑:** 블루프린트의 변수(예: TargetLocation)를 Control Rig의 변수로 노출시켜 연결합니다.

## **4\. 왜 Control Rig 노드를 사용해야 하는가?**

* **유연성:** 애니메이션 소스 파일 자체를 수정할 필요 없이 엔진 내에서 모든 보정이 가능합니다.  
* **재사용성:** 한 번 만든 리깅 로직을 동일한 스켈레톤을 사용하는 여러 캐릭터에게 쉽게 적용할 수 있습니다.  
* **퍼포먼스:** 최적화된 하드웨어 가속 기능을 활용하여 복잡한 리깅 연산도 실시간으로 처리할 수 있도록 설계되었습니다.

## **5\. 대표적인 활용 사례**

* **가상 유튜버/모션 캡쳐:** 실시간으로 들어오는 센서 데이터를 본에 직접 매핑하여 부드러운 움직임 보정.  
* **복잡한 지형 탐색:** 계단, 경사로, 장애물 등에서 캐릭터의 다리와 몸의 수평을 유지.  
* **무기 반동 시스템:** 사격 시 어깨와 팔의 본을 절차적으로 흔들어 생동감 부여.
