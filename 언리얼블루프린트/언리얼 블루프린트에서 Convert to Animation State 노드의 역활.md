# **언리얼 엔진: Convert to Animation State 노드의 역할**

언리얼 엔진의 애니메이션 블루프린트 내에서 **Convert to Animation State** 노드는 특정 애니메이션 에셋(Sequence, Blendspace 등)이나 포즈 로직을 **상태 머신(State Machine)이 이해할 수 있는 '상태(State)' 객체로 변환**하는 역할을 합니다.

## **1\. 핵심 개념**

일반적으로 스테이트 머신 내부의 각 노드는 독립적인 '상태'를 나타냅니다. 하지만 때로는 그래프 내부의 특정 애니메이션 로직을 동적으로 처리하거나, 외부에서 전달된 애니메이션 데이터를 현재 상태 머신의 흐름에 통합해야 할 때가 있습니다. 이때 이 노드가 가교 역할을 합니다.

## **2\. 주요 역할 및 기능**

### **① 애니메이션 데이터의 상태화**

* 단순한 애니메이션 시퀀스나 블렌드 스페이스 출력을 상태 머신의 **트랜지션(Transition)** 조건이나 **상태 노드**처럼 취급할 수 있게 합니다.  
* 애님 그래프(AnimGraph) 내의 데이터를 상태 머신의 구조에 맞게 래핑(Wrapping)합니다.

### **② 상태 머신의 가독성 및 재사용성 향상**

* 복잡한 로직을 하나의 '상태'로 묶어 처리함으로써 스테이트 머신 그래프를 깔끔하게 유지할 수 있습니다.  
* 특히 **Linked Anim Graph**나 **Animation Layers**를 사용할 때, 외부 계층에서 정의된 포즈를 현재 상태 머신의 한 상태로 삽입할 때 필수적입니다.

### **③ 최적화 및 블렌딩 제어**

* 이 노드를 통해 변환된 상태는 상태 머신의 표준 블렌딩 규칙(내부 보간, 트랜지션 시간 등)을 그대로 적용받습니다.  
* 현재 재생 중인 애니메이션의 남은 시간(Time Remaining)이나 재생 비율 등을 상태 머신이 더 정확하게 추적할 수 있도록 돕습니다.

## **3\. 주요 사용 시나리오**

1. **애니메이션 레이어 인터페이스(Animation Layer Interface):**  
   * 부모 블루프린트에서 정의한 상태 머신에 자식 블루프린트가 특정 애니메이션 포즈를 제공해야 할 때, 해당 포즈를 '상태'로 변환하여 연결합니다.  
2. **동적 상태 전환:**  
   * 특정 조건에 따라 여러 애니메이션 시퀀스 중 하나를 선택하여 상태 머신의 현재 상태로 밀어넣어야 할 때 사용합니다.  
3. **스테이트 머신 확장:**  
   * 기존의 고정된 스테이트 머신 구조를 유지하면서도, 내부의 구체적인 애니메이션 구현부만 별도의 로직으로 분리하고 싶을 때 유용합니다.

## **4\. 관련 노드와의 차이점**

* **Play Animation Sequence:** 단순히 애니메이션을 재생하고 포즈를 출력합니다.  
* **Convert to Animation State:** 재생되는 애니메이션을 포함한 '논리적인 상태'를 생성하여 상태 머신의 흐름(상태 전환 로직)에 편입시킵니다.

## **5\. 요약**

**Convert to Animation State** 노드는 애니메이션 블루프린트의 유연성을 극대화하는 도구입니다. 단순히 "애니메이션을 보여주는 것"을 넘어, \*\*"이 애니메이션 로직이 현재 시스템의 공식적인 한 상태(State)이다"\*\*라고 선언함으로써 상태 머신의 강력한 트랜지션 기능을 십분 활용할 수 있게 해줍니다.
