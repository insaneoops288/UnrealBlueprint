# **Create Child Animation Blueprint의 역할과 활용**

언리얼 엔진에서 \*\*Child Animation Blueprint (자식 애니메이션 블루프린트)\*\*는 기존에 작성된 애니메이션 블루프린트(부모)의 모든 로직과 구조를 그대로 상속받으면서, 특정 애니메이션 에셋이나 변수값만 교체하여 사용할 수 있게 해주는 기능입니다.

## **1\. 핵심 역할: "로직의 재사용과 데이터의 분리"**

부모 애니메이션 블루프린트가 \*\*'어떻게 움직이는가(How to move)'\*\*에 대한 복잡한 논리 구조(스테이트 머신, 블렌드 스페이스 계산, 레이어드 본 등)를 정의한다면, 자식 애니메이션 블루프린트는 그 구조 안에서 \*\*'어떤 애니메이션을 보여줄 것인가(What to play)'\*\*를 결정합니다.

### **주요 기능**

* **애니메이션 에셋 오버라이드 (Asset Override):** 부모에서 사용한 애니메이션 시퀀스, 블렌드 스페이스 등을 자식에서 다른 파일로 바꿀 수 있습니다.  
* **로직 상속:** 부모의 이벤트 그래프(Event Graph)와 애님 그래프(AnimGraph)를 다시 만들 필요 없이 그대로 사용합니다.  
* **변수 오버라이드:** 부모에 정의된 변수들의 기본값(Default Value)을 자식마다 다르게 설정할 수 있습니다.

## **2\. 왜 사용하는가? (장점)**

### **① 작업 효율성 증대**

모든 캐릭터마다 복잡한 스테이트 머신(Idle \-\> Walk \-\> Run \-\> Jump)을 새로 만들 필요가 없습니다. 공통적인 인간형 캐릭터라면 부모 하나만 잘 만들어두고, 자식에서 애니메이션 클립만 갈아 끼우면 됩니다.

### **② 유지보수의 용이성**

애니메이션 전환 로직에 버그가 있거나 기능을 추가해야 할 때, **부모 블루프린트 하나만 수정**하면 이를 상속받는 모든 자식 캐릭터들에게 즉시 적용됩니다.

### **③ 성능 최적화**

로직이 동일한 여러 캐릭터를 다룰 때 엔진 차원에서 구조적 중복을 줄여주며, 개발자가 관리해야 할 블루프린트 파일 숫자가 체계적으로 정리됩니다.

## **3\. 실무 활용 예시**

### **예시: 인간형 적 캐릭터 시스템**

1. **부모 (Base\_AnimBP):** 걷기, 뛰기, 피격, 사망 로직을 포함한 복잡한 스테이트 머신 구성.  
2. **자식 A (Warrior\_AnimBP):** 부모의 로직을 상속받되, '걷기' 애니메이션을 '검을 든 걷기'로 교체.  
3. **자식 B (Archer\_AnimBP):** 부모의 로직을 상속받되, '걷기' 애니메이션을 '활을 든 걷기'로 교체.  
4. **자식 C (Zombie\_AnimBP):** 부모의 로직을 상속받되, '걷기'를 '비틀거리는 걷기'로 교체.

## **4\. 사용 방법**

1. 콘텐츠 브라우저에서 부모가 될 **Animation Blueprint**를 우클릭합니다.  
2. **Create Child Animation Blueprint**를 선택합니다.  
3. 생성된 자식 블루프린트를 엽니다.  
4. 하단의 **Asset Override Editor (에셋 오버라이드 에디터)** 탭에서 부모가 사용 중인 애니메이션 에셋 목록을 확인하고, 원하는 에셋으로 교체(Override)합니다.

## **5\. 주의사항**

* 자식 블루프린트에서는 **AnimGraph(애님 그래프)를 직접 수정할 수 없습니다.** 그래프 구조 자체를 바꾸고 싶다면 부모를 수정하거나, 별도의 애니메이션 블루프린트를 만들어야 합니다.  
* 부모에서 사용된 변수들은 자식의 '디테일' 패널에서 기본값을 수정하여 차별화를 둘 수 있습니다.

결론적으로, **Create Child Animation Blueprint**는 동일한 해골(Skeleton)을 공유하거나 유사한 행동 양식을 가진 캐릭터 집합을 만들 때 없어서는 안 될 필수적인 도구입니다.
