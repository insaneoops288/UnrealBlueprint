# **Create Dynamic Material Instance 노드 이해하기**

언리얼 엔진에서 일반적인 머티리얼 에셋은 '정적'입니다. 즉, 게임이 실행되는 동안 블루프린트를 통해 머티리얼 내부의 값을 직접 바꿀 수 없습니다. 이를 가능하게 해주는 것이 바로 \*\*Dynamic Material Instance (MID)\*\*입니다.

## **1\. 핵심 역할**

이 노드는 지정된 \*\*부모 머티리얼(Parent Material)\*\*을 바탕으로, 메모리 상에 고유한 머티리얼 인스턴스를 생성합니다.

* **실시간 수정 가능:** Set Scalar Parameter Value, Set Vector Parameter Value 등의 노드를 사용하여 파라미터를 즉시 변경할 수 있습니다.  
* **개별성 유지:** 같은 머티리얼을 사용하는 여러 액터가 있더라도, 각 액터가 자신만의 '다이내믹 인스턴스'를 가지면 서로에게 영향을 주지 않고 개별적으로 색상이나 효과를 바꿀 수 있습니다.

## **2\. 주요 입력 핀 (Inputs)**

* **Parent:** 복사본의 바탕이 될 원본 머티리얼이나 머티리얼 인스턴스 상수를 지정합니다.  
* **Optional Name:** 인스턴스의 이름을 지정할 수 있습니다 (디버깅 용도).  
* **Source Material:** (선택 사항) 특정 소스를 기반으로 할 때 사용합니다.

## **3\. 일반적인 사용 흐름**

보통 BeginPlay 이벤트에서 이 노드를 호출하여 변수에 저장해두고 사용합니다.

1. **인스턴스 생성:** Create Dynamic Material Instance 호출.  
2. **메시 적용:** 생성된 리턴 값을 Set Material 노드를 통해 스태틱/스켈레탈 메시에 할당.  
3. **변수 저장:** 리턴 값을 'Material Instance Dynamic' 타입 변수에 저장 (나중에 참조하기 위함).  
4. **값 변경:** 게임 플레이 중 이벤트(예: 대미지 입음, 버튼 클릭) 발생 시 저장된 변수를 통해 파라미터 값 수정.

## **4\. 활용 예시**

* **캐릭터 대미지 효과:** 적에게 맞았을 때 캐릭터의 몸을 순간적으로 붉게 만듭니다.  
* **디졸브(Dissolve) 효과:** 적이 죽을 때 서서히 타들어 가며 사라지는 연출을 구현합니다.  
* **아이템 등급 표현:** 아이템의 등급에 따라 테두리 빛의 색상을 동적으로 변경합니다.  
* **진행률 표시:** 머티리얼 내부의 Glow 값을 조절하여 게이지가 차오르는 듯한 효과를 줍니다.

## **5\. 주의사항 및 팁**

* **성능:** 매 프레임(Tick)마다 Create Dynamic Material Instance를 호출하지 마세요. 생성 비용이 발생하므로 BeginPlay에서 한 번만 만들고 변수에 담아 재사용하는 것이 원칙입니다.  
* **파라미터 이름:** 블루프린트에서 값을 바꿀 때 입력하는 'Parameter Name'은 머티리얼 에디터 내부에 설정된 파라미터 노드의 이름과 대소문자까지 일치해야 합니다.  
* **Construction Script:** 가끔 컨스트럭션 스크립트에서 사용하기도 하지만, 런타임 변화가 목적이라면 이벤트 그래프의 BeginPlay가 더 적합합니다.

### **요약**

이 노드는 \*\*"읽기 전용인 머티리얼을 읽기/쓰기가 가능한 상태로 복사하여 특정 액터 전용으로 할당하는 도구"\*\*라고 이해하시면 됩니다.
