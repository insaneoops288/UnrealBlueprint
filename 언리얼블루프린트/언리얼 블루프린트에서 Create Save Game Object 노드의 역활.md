# **Create Save Game Object 노드 완벽 이해하기**

언리얼 엔진에서 게임의 진행 상황(레벨, 점수, 아이템 등)을 하드 드라이브에 저장하려면 SaveGame 시스템을 사용해야 합니다. 그 과정에서 가장 핵심적인 첫 번째 단계가 바로 Create Save Game Object 노드입니다.

## **1\. 노드의 핵심 역할**

이 노드는 사용자가 미리 정의한 **SaveGame 블루프린트 클래스의 인스턴스(객체)를 생성**합니다.

* **메모리 할당:** 하드 드라이브에 파일을 쓰기 전, 데이터를 임시로 담아둘 공간을 메모리(RAM)에 만듭니다.  
* **데이터 준비:** 저장하고 싶은 변수들(예: HP, 위치, 인벤토리 리스트)을 담을 수 있는 '빈 틀'을 준비하는 과정입니다.  
* **주의사항:** 이 노드 자체는 **파일을 생성하거나 저장하지 않습니다.** 오직 데이터를 담을 '객체'만 만듭니다.

## **2\. 세이브 시스템의 전체 흐름**

보통 다음과 같은 4단계 과정을 거쳐 데이터가 저장됩니다.

1. **SaveGame 클래스 생성:** SaveGame을 부모 클래스로 하는 블루프린트를 만들고 저장할 변수들을 추가합니다.  
2. **Create Save Game Object (현재 노드):** 위에서 만든 클래스를 선택해 객체를 생성합니다.  
3. **데이터 세팅:** 생성된 객체에 현재 게임의 실제 값(현재 점수 등)을 대입합니다.  
4. **Save Game to Slot:** 준비된 객체를 실제 파일(.sav)로 저장합니다.

## **3\. 노드 구성 요소**

### **입력 (Inputs)**

* **Save Game Class:** 미리 만들어둔 SaveGame 기반의 블루프린트 클래스를 선택합니다. 어떤 형태의 데이터를 저장할지 결정하는 부분입니다.

### **출력 (Outputs)**

* **Return Value:** 생성된 객체의 참조(Reference)를 반환합니다.  
  * *팁:* 이 값을 사용하기 위해서는 대개 **'Cast To \[사용자 세이브 클래스 명\]'** 노드를 통해 형변환을 해줘야 내부 변수에 접근할 수 있습니다.

## **4\. 실전 사용 예시 (블루프린트 로직)**

\[Create Save Game Object\] (Class: MySaveGame\_C)  
          |  
\[Cast To MySaveGame\] (Object: Return Value)  
          |  
\[Set CurrentLevel\] (값: 5\) \--- 세이브 객체 내부의 변수 설정  
          |  
\[Save Game to Slot\] (Slot Name: "SaveSlot1") \--- 실제 파일 저장

## **5\. 요약 및 팁**

| 구분 | 내용 |
| :---- | :---- |
| **언제 사용하나?** | 새로운 세이브 파일을 만들거나, 저장하기 전 데이터를 임시로 담을 때 |
| **저장 파일 형식** | 결과적으로 저장하면 /Saved/SaveGames/ 폴더에 .sav 파일이 생김 |
| **보안** | 이 방식은 바이너리 형태지만 보안이 완벽하지 않으므로 중요한 데이터는 암호화 로직을 추가하기도 함 |

**결론:** Create Save Game Object는 실제 저장 버튼을 누르기 전, \*\*"어떤 데이터를 저장할지 결정하고 그 데이터를 담을 빈 상자를 가져오는 작업"\*\*이라고 이해하시면 가장 정확합니다.
