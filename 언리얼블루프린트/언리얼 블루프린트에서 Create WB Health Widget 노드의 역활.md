# **Create WB Health Widget 노드 이해하기**

언리얼 엔진에서 Create WB Health Widget 노드는 UI 요소를 게임 화면에 표시하기 위한 **첫 번째 단계**입니다. 여기서 'WB'는 일반적으로 'Widget Blueprint'의 약자로 사용됩니다.

## **1\. 핵심 역할: 인스턴스화 (Instantiation)**

블루프린트 에디터에서 만든 위젯은 일종의 '설계도'입니다. 이 노드는 그 설계도를 바탕으로 실제 게임 세계(메모리)에 존재하게 하는 역할을 합니다.

* **클래스 선택:** WB\_Health라는 위젯 클래스를 기반으로 객체를 생성합니다.  
* **메모리 할당:** 이 노드가 실행되는 순간, 컴퓨터 메모리에 해당 체력바 위젯의 공간이 생깁니다.

## **2\. 노드의 구성 요소**

이 노드를 호출할 때 주로 보이는 핀들의 역할은 다음과 같습니다.

| 핀 이름 | 역할 |
| :---- | :---- |
| **Class** | 생성할 위젯의 종류를 결정합니다. (여기서는 WB\_Health) |
| **Owning Player** | 이 위젯을 소유할 플레이어 컨트롤러를 지정합니다. (보통 Get Player Controller 연결) |
| **Return Value** | 생성된 위젯의 \*\*참조(Reference)\*\*를 반환합니다. 이를 통해 나중에 체력 수치를 업데이트하거나 위젯을 제거할 수 있습니다. |

## **3\. 실행 프로세스 (중요)**

이 노드 하나만으로는 화면에 UI가 나타나지 않습니다. 반드시 다음과 같은 흐름을 거쳐야 합니다.

1. **Create WB Health Widget**: 위젯을 메모리에 생성합니다.  
2. **변수 저장 (선택 사항)**: 나중에 체력을 깎거나 색상을 바꿀 때 접근하기 위해 Return Value를 변수로 승격(Promote to Variable)시켜 저장해둡니다.  
3. **Add to Viewport**: 생성된 위젯 참조를 입력받아 실제 플레이어의 **화면에 렌더링**합니다.

## **4\. 활용 예시: Expose on Spawn**

만약 WB\_Health 위젯 안에 CurrentHealth라는 변수가 있고, 이 변수에 **Expose on Spawn** 설정이 되어 있다면, 노드 자체에서 위젯이 생성되자마자 초기 체력 값을 입력할 수 있는 핀이 나타납니다.

## **5\. 요약**

* **역할**: 설계도(Class)를 실체(Object)로 만드는 것.  
* **주의점**: 생성(Create)과 표시(Add to Viewport)는 별개의 단계이다.  
* **팁**: 생성된 결과값(Return Value)은 반드시 변수로 저장해두는 것이 유지보수에 유리하다.
