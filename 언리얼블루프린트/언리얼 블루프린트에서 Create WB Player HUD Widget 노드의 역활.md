# **Unreal Engine: Create Widget 노드 이해하기**

언리얼 엔진 블루프린트에서 Create WB\_PlayerHUD\_Widget (또는 Create Widget) 노드는 단순히 화면에 UI를 그리는 것이 아니라, **User Widget 클래스를 기반으로 실제 동작하는 '객체(Object)'를 만드는 과정**입니다.

## **1\. 주요 역할**

### **① 메모리 상의 인스턴스화 (Instantiation)**

콘텐츠 브라우저에 있는 'WB\_PlayerHUD'는 일종의 설계도(클래스)입니다. 이 노드는 이 설계도를 바탕으로 실제 게임 안에서 살아 움직이는 \*\*복사본(인스턴스)\*\*을 메모리에 생성합니다.

### **② 데이터 연결 및 초기화**

위젯이 생성될 때 해당 위젯 내부의 변수나 로직이 작동할 수 있도록 준비합니다. 'Expose on Spawn'으로 설정된 변수가 있다면 이 노드에서 직접 데이터를 전달할 수도 있습니다.

### **③ 소유권 설정 (Owning Player)**

이 위젯을 어떤 플레이어가 소유할지 결정합니다. 주로 로컬 플레이어 컨트롤러(Get Player Controller)를 연결하며, 이는 위젯 내에서 플레이어의 정보를 참조하거나 입력을 처리할 때 기준이 됩니다.

## **2\. 노드의 주요 핀 (Pins) 설명**

| 핀 이름 | 역할 |
| :---- | :---- |
| **Class** | 생성할 위젯 블루프린트를 선택합니다. (예: WB\_PlayerHUD) |
| **Owning Player** | 위젯을 소유할 플레이어 컨트롤러를 입력합니다. 비워두면 기본 로컬 플레이어가 할당되지만, 명시적으로 연결하는 것이 좋습니다. |
| **Return Value** | 생성된 위젯의 \*\*참조(Reference)\*\*를 반환합니다. 이 값을 변수로 저장해두어야 나중에 UI를 끄거나 내용을 수정할 수 있습니다. |

## **3\. 가장 중요한 주의사항: "생성 \!= 출력"**

많은 초보 개발자가 실수하는 부분입니다. Create Widget 노드만 실행해서는 화면에 아무것도 나타나지 않습니다.

1. **Create Widget**: 메모리에 위젯 객체를 만듦 (보이지 않음).  
2. **Add to Viewport**: 생성된 위젯을 실제 플레이어의 화면(뷰포트)에 부착함 (이때 비로소 보임).

## **4\. 권장되는 워크플로우 (Best Practice)**

보통 다음과 같은 순서로 노드를 구성합니다.

1. **Create Widget**: 위젯 생성.  
2. **Promote to Variable**: 반환값을 변수로 저장 (나중에 접근하기 위함).  
3. **Add to Viewport**: 화면에 표시.

## **5\. 예시 시나리오**

게임이 시작될 때(BeginPlay) 플레이어의 체력바와 미니맵이 포함된 WB\_PlayerHUD를 띄우고 싶다면:

* Event BeginPlay \-\> Create WB\_PlayerHUD\_Widget \-\> SET MyHUD\_Ref \-\> Add to Viewport 순으로 연결합니다.

**요약하자면:** Create WB\_PlayerHUD\_Widget 노드는 UI라는 '붕어빵'을 굽기 위해 **틀에 반죽을 부어 붕어빵(객체)을 완성하는 단계**이며, 이를 \*\*접시(화면)에 내놓는 단계(Add to Viewport)\*\*가 뒤따라야 합니다.
