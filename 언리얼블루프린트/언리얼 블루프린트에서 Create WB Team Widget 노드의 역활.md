# **Create WB Team Widget 노드 역할 및 분석**

Create WB Team Widget 노드는 언리얼 엔진(Unreal Engine)의 UI 시스템인 **UMG(Unreal Motion Graphics)** 환경에서 특정 팀 정보를 표시하기 위한 위젯을 동적으로 생성할 때 사용되는 노드입니다. 이는 엔진 기본 노드인 Create Widget을 특정 목적에 맞게 래핑(Wrapping)한 커스텀 노드입니다.

## **1\. 주요 역할 (Core Responsibility)**

이 노드의 핵심 목적은 **메모리상에 팀 UI 객체를 인스턴스화**하는 것입니다.

1. **위젯 인스턴스 생성**: WB\_Team (Widget Blueprint Team) 클래스를 기반으로 실제 게임 화면에 띄울 수 있는 객체를 생성합니다.  
2. **소유자 설정 (Owning Player)**: 해당 위젯을 제어할 플레이어 컨트롤러를 연결하여 입력 처리나 타겟팅을 설정합니다.  
3. **초기 데이터 전달**: 생성 시점에 팀 이름, 팀 점수, 팀 컬러 등의 데이터를 변수로 넘겨주어 위젯이 생성되자마자 올바른 정보를 표시하게 합니다.

## **2\. 노드의 내부 구조 (추정)**

보통 이 노드 내부(Function Graph)는 다음과 같은 표준 노드들의 조합으로 구성됩니다.

* **Create Widget**: 클래스 선택 항목에서 WB\_Team이 미리 선택되어 있는 상태의 노드입니다.  
* **Expose on Spawn 변수들**: 위젯 내부에 '인스턴스 편집 가능(Instance Editable)' 및 '생성 시 노출(Expose on Spawn)'로 설정된 변수들이 있다면, 이 노드의 입력 핀(Pin)으로 나타납니다.  
* **Return Value**: 생성된 위젯의 레퍼런스를 반환하여 이후 Add to Viewport나 다른 부모 위젯의 Add Child 노드에서 사용할 수 있게 합니다.

## **3\. 일반적인 사용 흐름 (Workflow)**

이 노드는 단독으로 사용된다고 해서 화면에 바로 보이지 않습니다. 일반적으로 다음과 같은 단계를 거칩니다.

1. **Event BeginPlay / Update**: 게임 시작 시나 팀 목록이 갱신될 때 호출됩니다.  
2. **Create WB Team Widget**: 노드를 실행하여 위젯 객체를 생성합니다.  
3. **Set Variables (Optional)**: 노드에 핀이 없다면 생성된 참조를 통해 추가 데이터를 설정합니다.  
4. **Add to Viewport / Add to Slot**:  
   * 전체 화면에 띄우려면 Add to Viewport를 연결합니다.  
   * 팀 리스트(Scroll Box 등) 내부에 넣으려면 해당 컨테이너의 자식으로 추가합니다.

## **4\. 왜 'Create Widget' 대신 이 노드를 쓰는가?**

* **재사용성**: 여러 곳에서 팀 위젯을 생성해야 할 때 설정 과정을 매번 반복하지 않기 위함입니다.  
* **가독성**: 단순히 Create Widget이라고 적힌 것보다 Create WB Team Widget이라고 명시되어 있으면 블루프린트 전체의 흐름을 파악하기 훨씬 쉽습니다.  
* **자동화**: 위젯 생성 직후 특정 애니메이션을 재생하거나, 특정 사운드를 출력하는 등의 로직을 함수 내부에 미리 포함시켜 둘 수 있습니다.

## **5\. 주의사항**

* **유효성 검사**: 이 노드가 반환하는 Return Value가 유효한지 항상 Is Valid 노드로 확인하는 것이 좋습니다.  
* **중복 생성**: 루프(For Each Loop) 내에서 이 노드를 사용할 경우, 기존에 생성된 위젯을 제거(Remove from Parent)하지 않으면 화면에 위젯이 겹쳐서 보일 수 있습니다.

**요약하자면:** 이 노드는 \*\*"팀 정보를 보여주는 UI 조각을 만들어서 사용할 준비를 하는 노드"\*\*라고 이해하시면 됩니다.
