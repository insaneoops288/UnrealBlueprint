# **언리얼 엔진 블루프린트: Dead Zone 노드의 역할과 활용**

## **1\. 데드 존(Dead Zone)이란?**

데드 존은 입력 장치(주로 게임패드의 아날로그 스틱)의 **입력값 중 무시되는 범위**를 의미합니다. 하드웨어의 노후화나 정밀도 한계로 인해 사용자가 스틱을 만지지 않아도 아주 미세한 입력값(예: 0.02)이 전달될 수 있는데, 이를 방지하기 위해 설정합니다.

## **2\. 주요 역할**

* **스틱 드리프트(Stick Drift) 방지**: 스틱이 중앙으로 완벽하게 돌아오지 않아 캐릭터가 저절로 움직이는 현상을 막습니다.  
* **입력 정밀도 향상**: 의도치 않은 미세한 움직임을 필터링하여 플레이어에게 더 깔끔한 조작감을 제공합니다.  
* **최대치 보정**: 상단 임계값(Upper Threshold)을 설정하여 스틱을 끝까지 밀지 않아도 최대 입력값(1.0)에 도달하게 할 수 있습니다.

## **3\. 향상된 입력(Enhanced Input)에서의 Dead Zone**

언리얼 엔진 5의 향상된 입력 시스템에서는 \*\*Input Modifier(입력 모디파이어)\*\*로 주로 사용됩니다.

### **설정 항목**

* **Lower Threshold (하단 임계값)**: 이 값보다 작은 입력은 0으로 처리됩니다. (기본값: 보통 0.2 내외)  
* **Upper Threshold (상단 임계값)**: 이 값보다 큰 입력은 1로 처리됩니다. (기본값: 보통 1.0)  
* **Type (데드존 타입)**:  
  * **Radial (방사형)**: 2축(X, Y) 입력 시 스틱의 중앙으로부터의 거리를 기준으로 원형 데드존을 형성합니다. (가장 일반적임)  
  * **Axial (축별)**: X축과 Y축 각각에 독립적으로 데드존을 적용합니다.

## **4\. 블루프린트에서의 작동 원리**

데드 존 처리는 단순히 값을 자르는 것이 아니라, 남은 범위를 \*\*재매핑(Remapping)\*\*합니다.

예를 들어, Lower Threshold가 0.2이고 Upper Threshold가 1.0인 경우:

* 실제 입력 0.1 \-\> 결과 0.0  
* 실제 입력 0.2 \-\> 결과 0.0  
* 실제 입력 0.6 \-\> 결과 약 0.5 (0.2\~1.0 사이의 구간을 0.0\~1.0으로 다시 계산)  
* 실제 입력 1.0 \-\> 결과 1.0

## **5\. 직접 구현 시의 로직 (Math)**

만약 시스템에서 제공하는 모디파이어 대신 직접 노드를 짠다면 아래와 같은 수식이 적용됩니다.

1. 입력값의 길이를 계산 (Vector Length)  
2. Map Range Clamped 노드를 사용하여:  
   * In Range A: 데드존 하한값 (0.2)  
   * In Range B: 데드존 상한값 (1.0)  
   * Out Range A: 0.0  
   * Out Range B: 1.0  
3. 결과값에 방향 벡터를 곱해 최종 입력값 산출
