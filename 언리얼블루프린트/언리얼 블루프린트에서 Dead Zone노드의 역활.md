언리얼 블루프린트에서 Dead Zone노드의 역활?

언리얼 엔진(특히 Enhanced Input 시스템)에서 Dead Zone(데드존) 노드(정확히는 모디파이어)는 조이스틱이나 트리거 같은 아날로그 입력 장치의 '미세한 오차'를 무시하기 위해 사용됩니다.

쉽게 말해, **"살짝 건드린 건 입력으로 치지 마!"**라고 선을 긋는 역할을 합니다.

1. 데드존이 왜 필요한가요? (Stick Drift 문제)
대부분의 게임 패드 조이스틱은 오래 사용하다 보면 중심축이 미세하게 틀어집니다. 이를 **스틱 드리프트(Stick Drift)**라고 하는데요.

데드존이 없다면: 손가락을 떼고 있어도 캐릭터가 혼자서 아주 천천히 옆으로 걷거나 카메라가 스르르 돌아가는 짜증 나는 상황이 발생합니다.

데드존이 있다면: 중심 근처의 아주 작은 값(예: 0.1 이하)은 0으로 취급하여 이런 오작동을 막아줍니다.

2. 주요 설정 항목 (Enhanced Input 기준)
데드존 모디파이어를 추가하면 보통 다음과 같은 수치를 조절합니다.

Lower Threshold (최하 임계값): 이 값보다 낮은 입력은 무시하고 0으로 만듭니다. (보통 0.1 ~ 0.2 정도를 설정합니다.)

Upper Threshold (최상 임계값): 이 값보다 높은 입력은 무조건 최대치(1.0)로 취급합니다. (예를 들어 0.9로 설정하면 끝까지 밀지 않아도 100% 입력이 됩니다.)

Type (데드존 타입):

Axial (축별): X축과 Y축을 각각 독립적으로 계산합니다. (십자 모양 데드존)

Radial (방사형): 중심점에서부터의 거리를 계산합니다. (원형 데드존, 더 자연스러운 대각선 이동을 위해 주로 사용됨)

3. 수학적인 처리 방식
단순히 "0.1 미만은 0"이라고만 하면, 0.1을 넘는 순간 입력값이 갑자기 0에서 0.11로 툭 튀어 오르게 됩니다. 이를 부드럽게 만들기 위해 언리얼은 내부적으로 **보간(Remapping)**을 수행합니다.

Output= 
Upper−Lower
Input−Lower
​
 
이 과정을 통해 사용자는 데드존을 벗어나는 지점부터 0에서 1까지 매끄러운 가속을 느낄 수 있습니다.

요약하자면
"데드존은 하드웨어의 물리적인 결함이나 손가락의 미세한 떨림을 무시해서, 플레이어가 의도한 확실한 움직임만 입력으로 받아들이게 만드는 필터다!"

라고 이해하시면 됩니다. 게임 개발 시 조이스틱 입력을 구현한다면 선택이 아닌 필수로 설정해줘야 하는 옵션입니다.
