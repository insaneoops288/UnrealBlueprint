# **Draw Debug Vector 노드 완벽 가이드**

언리얼 엔진에서 **Draw Debug Vector** 노드는 물리 계산, AI 경로, 캐릭터의 시선 방향 등 수치로만 존재하는 벡터 데이터를 3D 뷰포트 상에 **화살표**로 그려주는 역할을 합니다.

## **1\. 주요 역할 (Why use it?)**

1. **방향 시각화**: 캐릭터의 Forward Vector, Right Vector 등이 의도한 방향을 가리키고 있는지 즉각 확인 가능합니다.  
2. **크기(힘) 확인**: 화살표의 길이를 통해 속도(Velocity)나 가해지는 힘(Force)의 세기를 직관적으로 파악할 수 있습니다.  
3. **레이캐스트(Line Trace) 보조**: 레이가 어디서 시작해서 어디로 향하는지 시각적 가이드라인을 제공합니다.  
4. **좌표계 확인**: 월드 좌표 대비 로컬 좌표가 어떻게 회전되어 있는지 디버깅할 때 유용합니다.

## **2\. 주요 입력 파라미터 (Inputs)**

| 파라미터 | 설명 |
| :---- | :---- |
| **Line Start** | 화살표가 시작될 위치 (World Location) 입니다. |
| **Line End** | 화살표의 끝점입니다. (시작점 \+ 벡터값으로 주로 계산합니다) |
| **Line Color** | 화살표의 색상입니다. 상황별로 색을 다르게 지정하면 구분이 쉽습니다. |
| **Duration** | 화살표가 화면에 유지되는 시간(초)입니다. 0이면 한 프레임만 표시됩니다. |
| **Thickness** | 화살표 선의 두께입니다. |

## **3\. 실전 활용 사례**

### **A. 캐릭터 이동 속도 시각화**

캐릭터의 Get Velocity를 Line End에 더해 현재 캐릭터가 어느 방향으로 얼마나 빠르게 이동 중인지 확인할 수 있습니다.

* **Line Start**: Get Actor Location  
* **Line End**: Get Actor Location \+ Get Velocity

### **B. 발사체 경로 예측**

발사체가 날아갈 방향 벡터를 미리 그려보아 총구나 발사 위치가 정확한지 테스트할 때 사용합니다.

### **C. AI 감지 범위 및 시선**

AI가 플레이어를 바라보는 '방향'을 화살표로 그려서 AI의 시야 로직이 정상인지 체크합니다.

## **4\. 주의 사항 및 팁**

1. **좌표 계산**: Line End는 절대 좌표를 입력받습니다. 단순히 '벡터값'만 넣으면 월드 원점(0,0,0)에서 시작하는 화살표가 그려지므로, 반드시 \*\*\[시작 위치 \+ 벡터값\]\*\*을 해주어야 해당 위치에서 뻗어 나가는 화살표가 됩니다.  
2. **성능**: 디버그 노드는 런타임 성능에 영향을 줄 수 있습니다. 개발 단계에서만 사용하고, 최종 빌드(Shipping Build)에서는 자동으로 제외되거나 실행되지 않도록 관리해야 합니다.  
3. **Print String과의 차이**: 숫자로 좌표를 보는 것보다 3D 공간에서 직접 화살표를 보는 것이 공간적인 로직 오류(예: 좌우 반전, 회전 값 오류)를 잡는 데 훨씬 효율적입니다.

## **5\. 요약**

Draw Debug Vector는 \*\*"내 수학적 계산(벡터)이 실제 월드에서 맞게 작동하는가?"\*\*를 눈으로 확인시켜 주는 가장 빠른 방법입니다.
