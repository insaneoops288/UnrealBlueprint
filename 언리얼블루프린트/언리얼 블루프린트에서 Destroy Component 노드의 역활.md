# **언리얼 엔진: Destroy Component 노드 완벽 이해**

**Destroy Component** 노드는 언리얼 블루프린트에서 매우 자주 사용되는 노드로, 액터 자체는 유지하면서 그 안에 포함된 특정 기능이나 외형(컴포넌트)만을 선택적으로 제거할 때 사용됩니다.

## **1\. 주요 역할 (Core Roles)**

### **① 컴포넌트 즉각 제거**

지정된 컴포넌트를 게임 세션에서 즉시 삭제합니다. 호출되는 즉시 해당 컴포넌트는 다음과 같은 상태가 됩니다.

* **렌더링 중지:** 화면에서 보이지 않게 됩니다.  
* **물리 처리 중지:** 충돌(Collision)이나 물리 시뮬레이션이 비활성화됩니다.  
* **틱(Tick) 중지:** 해당 컴포넌트가 가지고 있던 주기적인 로직 처리가 멈춥니다.

### **② 메모리 최적화**

단순히 숨기는(Set Hidden in Game) 것이 아니라, 엔진의 가비지 컬렉션(Garbage Collection) 시스템이 해당 컴포넌트를 메모리에서 완전히 수거할 수 있도록 준비시킵니다. 불필요한 컴포넌트를 제거함으로써 런타임 성능을 향상시킬 수 있습니다.

## **2\. Destroy Actor vs. Destroy Component**

가장 흔히 혼동하는 두 노드의 차이점은 '범위'에 있습니다.

| 구분 | Destroy Actor | Destroy Component |
| :---- | :---- | :---- |
| **제거 범위** | 액터 전체 (포함된 모든 컴포넌트 일괄 삭제) | 특정 개별 컴포넌트 하나만 삭제 |
| **활용 예시** | 몬스터가 죽었을 때 시체 제거 | 캐릭터가 장착한 방어구가 파괴되었을 때 |
| **결과** | 월드에서 해당 객체가 완전히 사라짐 | 액터는 남아있으나 특정 기능/외형만 사라짐 |

## **3\. 주요 사용 사례 (Use Cases)**

1. **파괴 가능한 장비:** 캐릭터의 방어구(Static Mesh Component)가 내구도를 다해 부서질 때 해당 컴포넌트만 제거.  
2. **동적 효과 관리:** Add Component로 생성한 파티클 시스템이나 오디오 컴포넌트가 재생을 마친 후 리소스를 아끼기 위해 제거.  
3. **게임플레이 상태 변화:** 캐릭터의 '비행' 기능을 담당하는 무브먼트 컴포넌트나 충돌 판정용 박스 컴포넌트를 특정 상황에서 영구적으로 없애야 할 때.

## **4\. 사용 시 주의사항**

* **유효성 검사 (Is Valid):** 이미 제거된 컴포넌트를 참조하려고 하면 'Accessed None' 에러가 발생합니다. Destroy Component를 호출한 직후나 이후에는 반드시 Is Valid 노드를 통해 해당 컴포넌트가 존재하는지 확인하는 습관이 중요합니다.  
* **루트 컴포넌트 삭제:** 액터의 루트(Root) 컴포넌트를 삭제하면 해당 액터의 위치 정보 등이 사라지므로 주의해야 합니다. 보통은 자식 컴포넌트들을 제거할 때 주로 사용합니다.  
* **복제(Replication):** 네트워크 게임의 경우, 서버에서 컴포넌트를 제거하면 클라이언트에게도 해당 변화가 전파되도록 설정되어 있어야 합니다.

## **5\. 블루프린트 구성 예시**

\[Event Any Damage\] \-\> \[Branch (HP \<= 0)\] \-\> \[Target: MyShieldComponent\] \-\> \[Destroy Component\]

*설명: 대미지를 입었을 때 체력이 0 이하라면, 캐릭터는 그대로 두고 '방패(Shield)' 컴포넌트만 파괴합니다.*
