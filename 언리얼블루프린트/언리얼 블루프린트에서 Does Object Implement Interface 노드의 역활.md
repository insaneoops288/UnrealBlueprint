# **Does Object Implement Interface 노드 완벽 가이드**

언리얼 엔진 블루프린트 프로그래밍에서 Does Object Implement Interface 노드는 객체 지향 설계의 핵심인 \*\*인터페이스(Interface)\*\*를 안전하고 효율적으로 다루기 위해 사용됩니다.

## **1\. 노드의 기본 역할**

이 노드는 입력으로 받은 \*\*객체(Object)\*\*가 특정 \*\*인터페이스(Interface Class)\*\*를 구현(Implement)하고 있는지 여부를 확인하여 **Boolean(True/False)** 값으로 반환합니다.

* **Input (Test Object):** 확인하려는 액터, 컴포넌트 또는 일반 오브젝트 레퍼런스입니다.  
* **Input (Interface):** 객체가 가지고 있는지 확인하려는 블루프린트 인터페이스 클래스입니다.  
* **Output (Return Value):** 해당 객체가 인터페이스를 가지고 있다면 True, 그렇지 않다면 False를 반환합니다.

## **2\. 왜 이 노드를 사용하는가? (주요 역할)**

### **① 안전한 함수 호출 (Safety Check)**

인터페이스 메시지를 보낼 때, 대상 객체가 해당 인터페이스를 구현하지 않은 상태에서 함수를 호출하면 아무 일도 일어나지 않지만, 로직 흐름상 반드시 구현된 객체에만 특정 액션을 취해야 할 때가 있습니다. 이때 이 노드로 미리 체크하면 런타임 오류를 방지하고 논리적인 흐름을 제어할 수 있습니다.

### **② 의존성 제거 및 결합도 감소 (Decoupling)**

Cast To \[Specific Blueprint\] 노드를 사용하면 해당 블루프린트와 강한 의존성(Hard Reference)이 생깁니다. 하지만 Does Object Implement Interface를 사용하면 상대방이 정확히 어떤 클래스인지 몰라도 \*\*"특정 기능을 할 줄 아는가?"\*\*만 확인하면 되므로 코드가 훨씬 유연해집니다.

### **③ 불필요한 캐스팅 방지**

많은 수의 액터와 상호작용할 때, 일일이 클래스별로 캐스팅하는 것은 성능상 좋지 않고 노드가 복잡해집니다. 인터페이스 구현 여부만 확인하면 수많은 서로 다른 클래스들을 하나의 인터페이스 기준으로 묶어서 처리할 수 있습니다.

## **3\. 실제 활용 예시 (상호작용 시스템)**

플레이어가 앞에 있는 물체와 상호작용(E 키)을 하려고 할 때의 로직입니다.

1. **Line Trace**를 통해 앞에 있는 액터를 감지합니다.  
2. 감지된 액터를 Does Object Implement Interface 노드의 **Test Object**에 연결합니다.  
3. **Interface** 핀에 미리 만들어둔 BPI\_Interactable을 선택합니다.  
4. **Branch** 노드를 통해 결과가 True인 경우에만 Interact 인터페이스 함수를 호출합니다.

이렇게 하면 문, 상자, NPC 등 서로 다른 클래스라도 BPI\_Interactable만 구현하고 있다면 동일한 로직으로 상호작용이 가능해집니다.

## **4\. Cast To 노드와의 차이점**

| 특징 | Cast To \[Class\] | Does Object Implement Interface |
| :---- | :---- | :---- |
| **확인 대상** | 특정 클래스 또는 그 자식 클래스인가? | 특정 인터페이스를 구현했는가? |
| **참조 방식** | 하드 레퍼런스 (메모리 로드 발생 가능) | 소프트/인터페이스 참조 (가벼움) |
| **유연성** | 낮음 (정해진 클래스만 가능) | 높음 (전혀 다른 클래스라도 인터페이스만 있으면 OK) |
| **용도** | 해당 클래스의 변수/함수에 접근할 때 | 인터페이스를 통한 공통 기능 실행 시 |

## **5\. 사용 팁**

* **Is Valid 확인:** Test Object가 유효한지(Not Null) 먼저 확인하는 습관을 들이는 것이 좋습니다.  
* **순수 함수:** 이 노드는 실행 핀이 없는 'Pure' 노드이므로, 값이 필요할 때만 계산되어 성능 최적화에 유리합니다.  
* **C++ 호환:** C++에서 만든 인터페이스도 블루프린트에서 이 노드를 통해 동일하게 확인할 수 있습니다.
