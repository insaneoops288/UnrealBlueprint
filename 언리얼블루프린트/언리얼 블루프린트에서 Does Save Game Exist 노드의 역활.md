# **Does Save Game Exist 노드 상세 가이드**

언리얼 엔진에서 세이브 시스템을 구축할 때 가장 먼저 사용하게 되는 노드 중 하나입니다. 게임 데이터를 불러오기 전에 해당 데이터가 있는지 확인하여 오류를 방지하는 '가드(Guard)' 역할을 수행합니다.

## **1\. 노드의 주요 역할**

* **데이터 존재 확인**: 지정한 Slot Name과 User Index를 가진 세이브 파일(.sav)이 저장 폴더(Saved/SaveGames)에 있는지 검색합니다.  
* **분기 처리**: 파일이 있다면 True, 없다면 False를 반환하여 이후 로직(데이터 로드 vs 초기 데이터 생성)을 결정하게 합니다.

## **2\. 입력 및 출력 핀 설명**

| 핀 이름 | 타입 | 설명 |
| :---- | :---- | :---- |
| **Slot Name** | String | 확인하려는 세이브 파일의 고유 이름 (예: "PlayerSave\_01") |
| **User Index** | Integer | 로컬 사용자 번호. 일반적으로 단일 플레이어 게임에서는 0을 사용합니다. |
| **Return Value** | Boolean | 파일이 존재하면 **True**, 존재하지 않으면 **False**를 반환합니다. |

## **3\. 주요 활용 사례**

### **① 메인 메뉴의 '이어하기' 버튼 활성화**

게임 실행 시 세이브 데이터가 있는지 체크하여, 데이터가 있을 때만 'Continue' 버튼을 활성화하고 없을 때는 버튼을 숨기거나 비활성화할 때 사용합니다.

### **② 데이터 로드 전 안전장치**

Load Game from Slot 노드를 호출하기 전에 반드시 이 노드로 먼저 확인해야 합니다. 만약 파일이 없는데 로드를 시도하면 None 값을 반환하여 게임이 크래시되거나 오류가 발생할 수 있습니다.

### **③ 초기 세이브 생성 로직**

게임을 처음 실행했을 때 세이브 파일이 없다면 새 세이브 객체를 생성(Create Save Game Object)하고 저장하는 로직으로 연결하는 분기점 역할을 합니다.

## **4\. 논리적 흐름 예시 (Pseudocode)**

1. **게임 시작/로드 버튼 클릭**  
2. **Does Save Game Exist** (SlotName: "MySave") 호출  
3. **Branch (분기)**  
   * **True (있음)**: Load Game from Slot \-\> 캐스팅 \-\> 변수 적용 \-\> 게임 시작  
   * **False (없음)**: Create Save Game Object \-\> 기본값 설정 \-\> Save Game to Slot \-\> 게임 시작

## **5\. 주의사항**

* **대소문자 구분**: Slot Name은 대소문자를 구분하므로 저장할 때와 확인할 때의 이름을 정확히 일치시켜야 합니다.  
* **경로**: 이 노드는 언리얼이 지정한 기본 세이브 경로만 확인합니다. 별도의 데이터베이스나 클라우드 저장소를 확인할 때는 다른 방식을 사용해야 합니다.
