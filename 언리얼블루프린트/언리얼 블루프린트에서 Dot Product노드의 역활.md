# **언리얼 엔진 블루프린트: 내적(Dot Product) 노드 가이드**

내적(Dot Product)은 두 개의 벡터를 입력받아 하나의 숫자(Float)를 결과로 내놓는 연산입니다. 기하학적으로는 \*\*"두 벡터가 얼마나 같은 방향을 향하고 있는가"\*\*를 측정하는 지표로 사용됩니다.

## **1\. 수학적 핵심 (정규화된 벡터 기준)**

보통 내적을 사용할 때는 두 벡터의 길이를 1로 만든 \*\*정규화(Normalize)\*\*된 벡터를 사용합니다. 이때 결과값은 항상 **\-1.0에서 1.0 사이**로 나옵니다.

| 내적 결과값 | 의미 (두 벡터 사이의 각도) | 시각적 설명 |
| :---- | :---- | :---- |
| **1.0** | 0도 (완전히 같은 방향) | 두 벡터가 평행하며 같은 곳을 봄 |
| **0.5** | 60도 | 비스듬히 같은 방향을 향함 |
| **0.0** | 90도 (직교) | 두 벡터가 서로 수직임 |
| **\-1.0** | 180도 (완전히 반대 방향) | 서로 마주 보거나 등지고 있음 |

## **2\. 주요 활용 사례 (실전 예시)**

### **① 적이 내 정면에 있는가? (시야 확인)**

가장 많이 쓰이는 방식입니다. 플레이어의 \*\*Forward Vector(앞방향 벡터)\*\*와 **플레이어에서 적을 향하는 벡터**를 내적합니다.

* 결과가 **0.8 이상**이면 "플레이어의 정면 시야 안에 적이 있다"고 판단할 수 있습니다.  
* 결과가 \*\*음수(-)\*\*라면 "적이 내 뒤에 있다"는 뜻입니다.

### **② 조명 및 그림자 계산**

표면의 방향(Normal Vector)과 빛이 들어오는 방향(Light Vector)을 내적합니다.

* 내적 값이 클수록 빛을 정면으로 받아 밝게 표현되고, 0에 가까워질수록 어두워집니다.

### **③ 경사도 판별**

캐릭터가 서 있는 바닥의 **Normal Vector**와 월드의 \*\*Up Vector(0, 0, 1)\*\*를 내적합니다.

* 결과가 1.0이면 완전한 평지입니다.  
* 결과가 낮아질수록 경사가 가파르다는 뜻이므로, 특정 수치 이하일 때 캐릭터가 미끄러지게 만들 수 있습니다.

## **3\. 블루프린트 사용 팁**

1. **Normalize 노드 활용:** 정확한 각도 관계를 알고 싶다면 입력 벡터들을 반드시 Normalize 노드에 먼저 통과시킨 후 Dot 노드에 연결하세요.  
2. **Acos(Degrees) 노드:** 내적 결과값으로부터 실제 각도(도 단위)를 얻고 싶다면, 내적 결과값을 Acos (Degrees) 노드에 넣으면 됩니다.  
3. **성능 최적화:** Distance 연산보다 Dot Product 연산이 수학적으로 훨씬 가볍습니다. 단순히 "내 앞에 있는가"를 체크할 때는 거리 계산보다 내적을 우선적으로 고려하세요.

## **4\. 요약**

* **내적 \> 0**: 두 벡터가 예각 (같은 방향 쪽)  
* **내적 \= 0**: 두 벡터가 수직  
* **내적 \< 0**: 두 벡터가 둔각 (반대 방향 쪽)
