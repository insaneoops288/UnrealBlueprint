# **Draw Debug Capsule 노드 상세 가이드**

Draw Debug Capsule 노드는 개발 단계에서 보이지 않는 물리 엔진의 데이터나 논리적 범위를 시각화하여 문제를 해결(디버깅)하는 데 필수적인 도구입니다.

## **1\. 주요 역할**

* **충돌 영역 시각화**: 캐릭터의 캡슐 컴포넌트나 트리거 볼륨이 의도한 위치와 크기에 있는지 확인합니다.  
* **범위 체크**: AI의 감지 범위, 폭발 반경, 스킬 영향 범위 등을 시각적으로 표시합니다.  
* **위치 추적**: 특정 프레임에서 액터나 소켓의 정확한 위치와 회전값을 확인합니다.

## **2\. 주요 파라미터 (입력 항목)**

| 파라미터 | 설명 |
| :---- | :---- |
| **Center** | 월드 공간에서 캡슐의 중심점 좌표(Vector)입니다. |
| **Half Height** | 캡슐의 **전체 높이의 절반**입니다. (구 형태의 끝부분 포함) |
| **Radius** | 캡슐의 반지름입니다. |
| **Rotation** | 캡슐의 회전값(Rotator)입니다. 캡슐이 기울어진 정도를 결정합니다. |
| **Line Color** | 캡슐 선의 색상을 지정합니다. |
| **Duration** | 화면에 표시될 시간(초 단위)입니다. 0이면 한 프레임만 표시됩니다. |
| **Thickness** | 캡슐을 구성하는 선의 두께입니다. |

## **3\. 사용 시 주의사항**

### **Half Height의 개념**

언리얼에서 캡슐의 높이는 보통 'Half Height'로 계산됩니다.

* 만약 실제 높이가 200cm인 캐릭터를 그리려면 Half Height에 **100**을 입력해야 합니다.  
* Half Height는 반드시 Radius보다 크거나 같아야 합니다. (반지름보다 작으면 형태가 깨질 수 있습니다.)

### **성능 영향**

이 노드는 **디버깅용**입니다.

* 수백 개의 캡슐을 매 프레임(Event Tick)마다 그리면 성능이 저하될 수 있습니다.  
* 실제 게임 출시(Shipping) 빌드에서는 자동으로 제거되거나 작동하지 않도록 설정되는 경우가 일반적입니다.

## **4\. 실전 활용 예시 (블루프린트 로직)**

1. **라인 트레이스 확인**: Line Trace By Channel이 실패했을 때, 충돌 판정 범위를 확인하기 위해 해당 위치에 캡슐을 그려봅니다.  
2. **캐릭터 피격 판정**: 캐릭터가 공격을 받았을 때, 피격된 부위 주위에 잠깐 붉은색 캡슐을 그려서 판정이 정확한지 확인합니다.  
3. **AI 경로 확인**: AI가 이동하는 경로의 각 노드에 캡슐을 배치하여 경로가 끊기지 않았는지 시각적으로 검토합니다.

## **5\. 유사한 노드들**

* Draw Debug Sphere: 구체 형태 시각화  
* Draw Debug Box: 박스 형태 시각화  
* Draw Debug Line: 선 시각화  
* Draw Debug Coordinate System: 좌표축(X, Y, Z) 시각화
