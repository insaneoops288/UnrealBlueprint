# **언리얼 엔진 향상된 입력 액션(Enhanced Input Action) 노드 이해하기**

언리얼 엔진 5에서 **향상된 입력 액션(Enhanced Input Action)** 노드는 특정 입력(키보드, 마우스, 게임패드 등)이 발생했을 때 게임 로직을 실행하는 핵심 관문입니다.

## **1\. 노드의 주요 역할**

이 노드는 미리 정의된 **입력 액션(Input Action, IA)** 에셋을 블루프린트에서 호출하여 사용하며, 입력의 '상태'와 '값'을 실시간으로 전달합니다.

* **이벤트 트리거**: 키를 눌렀을 때, 떼었을 때, 누르고 있을 때 등 다양한 상황에 맞춰 실행 핀을 제공합니다.  
* **데이터 전달**: 단순한 On/Off(Boolean)뿐만 아니라 마우스 이동량, 스틱의 기울기 등 복합적인 데이터를 전달합니다.

## **2\. 노드 구성 요소 (실행 핀)**

향상된 입력 액션 노드는 입력의 진행 단계에 따라 5가지 실행 핀을 제공합니다.

| 핀 이름 | 설명 |
| :---- | :---- |
| **Triggered** | 액션이 트리거되는 동안 매 프레임 실행됩니다. (예: 이동 중, 버튼 유지 중) |
| **Started** | 입력이 처음 시작된 순간(첫 번째 프레임)에 한 번 실행됩니다. |
| **Ongoing** | 입력이 활성화되어 있지만, 아직 '트리거(Trigger)' 조건이 충족되지 않은 상태에서 실행됩니다. |
| **Canceled** | 입력이 완료되기 전에 중단되었을 때 실행됩니다. (예: 버튼을 끝까지 안 누르고 뗌) |
| **Completed** | 입력이 완전히 끝났을 때(버튼을 떼었을 때 등) 실행됩니다. |

## **3\. 출력 데이터 (Action Value)**

노드 우측의 **Action Value**는 입력 액션 에셋에서 설정한 'Value Type'에 따라 다른 형태의 데이터를 출력합니다.

1. **Digital (bool)**: 참/거짓 (예: 점프 버튼)  
2. **Axis1D (float)**: 단일 축 값 (예: 가속 페달)  
3. **Axis2D (Vector2D)**: 두 축 값 (예: 캐릭터 이동, 마우스 회전)  
4. **Axis3D (Vector)**: 세 축 값 (주로 특수한 컨트롤러용)

## **4\. 향상된 입력의 핵심 장점**

1. **입력 매핑 컨텍스트(IMC)**: 상황에 따라 입력 체계를 통째로 갈아끼울 수 있습니다. (예: 도보 모드 ↔ 탑승 모드)  
2. **수정자(Modifiers)**: 입력값에 계산 로직을 넣지 않고도 값을 반전(Negative)하거나 감도를 조절(Scalar)할 수 있습니다.  
3. **트리거(Triggers)**: '두 번 누르기', '길게 누르기' 등의 복잡한 로직을 노드 하나로 처리할 수 있습니다.

## **5\. 사용 워크플로우**

1. **Input Action (IA) 에셋 생성**: 어떤 데이터 타입(bool, float 등)을 쓸지 결정합니다.  
2. **Input Mapping Context (IMC) 생성**: IA 에셋에 실제 키(W, S, A, D 등)를 할당합니다.  
3. **캐릭터 블루프린트 설정**: Add Input Mapping Context 노드를 통해 플레이어 컨트롤러에 IMC를 추가합니다.  
4. **이벤트 호출**: 그래프에서 해당 IA 이름을 검색하여 Enhanced Action Events 노드를 배치합니다.

## **6\. 요약**

**Enhanced Input Action 노드**는 사용자가 컨트롤러를 조작하는 방식(Triggers)과 그 강도(Action Value)를 블루프린트로 가져오는 통로입니다. Started는 입력의 시작을, Triggered는 지속적인 동작을 처리할 때 주로 사용합니다.
