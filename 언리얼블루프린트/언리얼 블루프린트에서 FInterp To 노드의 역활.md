# **Unreal Engine: FInterp To 노드 완벽 이해하기**

FInterp To (Float Interpolate To)는 두 실수(Float) 사이의 값을 **프레임 속도에 독립적으로(Frame-rate Independent)**, 그리고 **부드러운 감속(Ease-out) 효과**를 주며 보간(Interpolation)하는 함수입니다.

## **1\. 노드 구성 요소 (Inputs)**

| 입력 항목 | 설명 |
| :---- | :---- |
| **Current** | 현재 수치입니다. 매 프레임 업데이트된 결과값이 다시 이 자리에 들어와야 합니다. |
| **Target** | 도달하고자 하는 최종 목표 수치입니다. |
| **Delta Time** | 이전 프레임으로부터 경과된 시간입니다. 보통 Get World Delta Seconds나 Event Tick의 Delta Seconds를 연결합니다. |
| **Interp Speed** | 보간 속도입니다. 값이 클수록 목표치에 더 빠르게 도달하며, 0이면 수치가 변하지 않습니다. |

## **2\. 작동 원리: 왜 Lerp와 다른가?**

많은 초보자가 Lerp(Linear Interpolation)와 헷갈려합니다.

* **Lerp (선형 보간):** 0에서 1 사이의 '알파(Alpha)' 값을 기준으로 두 지점 사이의 특정 위치를 계산합니다. 일정한 속도로 움직입니다.  
* **FInterp To (지수 보간):** 목표치에 가까워질수록 속도가 줄어드는 **Ease-out** 효과가 자동으로 적용됩니다. 물리적으로는 "현재 위치에서 목표 방향으로 매 초마다 일정 비율만큼 이동"하는 방식입니다.

## **3\. 구현 패턴 (블루프린트 구성 방법)**

FInterp To를 제대로 사용하려면 반드시 **매 프레임 호출**되어야 하며, 결과값이 다시 Current로 피드백되는 구조여야 합니다.

### **올바른 흐름 예시:**

1. **Event Tick** 발생  
2. **Current Value** 변수 가져오기 \-\> FInterp To의 **Current**에 연결  
3. 원하는 목표치를 **Target**에 입력  
4. **Interp Speed** 설정 (예: 5.0)  
5. FInterp To의 결과값(Return Value)을 다시 **Current Value** 변수에 저장(Set)  
6. 저장된 변수를 실제 로직(예: 카메라 FOV, 불투명도 등)에 적용

## **4\. 주요 활용 사례**

1. **카메라 줌 (FOV 변화):**  
   * 마우스 휠을 굴릴 때 FOV 값을 즉시 바꾸지 않고 FInterp To를 거치면 부드러운 줌 효과가 납니다.  
2. **UI 프로그레스 바 (체력바):**  
   * 데미지를 입었을 때 체력바 수치가 툭 끊기지 않고 스르륵 줄어들게 할 때 사용합니다.  
3. **부드러운 조명 조절:**  
   * 불을 켤 때 조명 강도(Intensity)를 부드럽게 올릴 때 유용합니다.  
4. **캐릭터 회전:**  
   * 캐릭터가 갑자기 뒤를 돌아볼 때 RInterp To(회전용)와 함께 사용하여 자연스러운 회전을 만듭니다.

## **5\. 꿀팁: FInterp To Constant**

만약 "가까워질수록 느려지는 효과"가 아니라, **정해진 속도로 일정하게(Linear)** 목표값에 도달하고 싶다면 FInterp To Constant 노드를 사용하세요. 이 노드는 감속 없이 일정한 속도로 이동하며 목표치에 도달하면 멈춥니다.

## **요약**

* **부드러운 감속**이 필요할 때 사용한다.  
* **Event Tick**과 함께 사용하여 매 프레임 값을 업데이트해야 한다.  
* **Interp Speed**를 조절하여 반응 속도를 결정한다.
