# **FInterp To 노드의 역할과 이해**

언리얼 엔진 블루프린트에서 FInterp To (Float Interpolation To) 노드는 현재의 숫자(Float) 값을 목표하는 숫자까지 **프레임 단위로 부드럽게 이동**시키는 역할을 합니다. 단순히 값이 변하는 것이 아니라, 매 프레임마다 자연스러운 감속(Ease-out) 효과를 주며 변화하는 것이 특징입니다.

## **1\. 노드의 입력 핀(Inputs) 설명**

| 핀 이름 | 타입 | 역할 |
| :---- | :---- | :---- |
| **Current** | Float | 현재 프레임에서의 값입니다. 보통 변수 자체를 연결합니다. |
| **Target** | Float | 최종적으로 도달하고자 하는 목표 값입니다. |
| **Delta Time** | Float | 프레임 간의 시간 간격입니다. 보통 Get World Delta Seconds를 연결하여 프레임 속도와 상관없이 일정한 속도를 유지하게 합니다. |
| **Interp Speed** | Float | 보간 속도입니다. 값이 클수록 목표치에 더 빠르게 도달합니다. |

## **2\. 주요 특징**

### **지수적 감속 (Exponential Smoothing)**

FInterp To는 목표값에 가까워질수록 속도가 줄어듭니다.

* 처음에는 빠르게 변하다가 목표치에 근접할수록 아주 미세하게 움직이며 멈춥니다.  
* 이 방식은 UI 애니메이션이나 카메라 이동 시 매우 자연스러운 느낌을 줍니다.

### **프레임 독립성**

Delta Time을 입력받기 때문에, 게임의 FPS(초당 프레임 수)가 30이든 120이든 상관없이 실제 시간 기준으로 일정한 속도로 값이 변합니다.

## **3\. 작동 원리 (수식)**

내부적으로는 대략 다음과 같은 로직으로 계산됩니다:

NewValue \= Current \+ (Target \- Current) \* DeltaTime \* InterpSpeed

*(실제 엔진 내부 코드는 미세하게 다르지만, 개념적으로 타겟과의 거리에 비례하여 움직입니다.)*

## **4\. 자주 사용되는 사례**

1. **카메라 줌 (Camera Zoom):** 휠을 돌릴 때 시야각(FOV)을 즉시 바꾸지 않고 부드럽게 좁히거나 넓힐 때 사용합니다.  
2. **부드러운 UI 게이지:** 체력바가 깎일 때 툭 끊기지 않고 스르륵 줄어들게 할 때 유용합니다.  
3. **오브젝트 회전/이동 제어:** 캐릭터가 특정 방향을 부드럽게 쳐다보게 하거나, 문이 부드럽게 열리는 로직 등에 활용됩니다. (회전은 RInterp To, 벡터는 VInterp To를 사용)

## **5\. 주의사항 및 팁**

* **Tick 이벤트와 함께 사용:** FInterp To는 값을 한 번 계산하고 끝내는 것이 아니라, 목표치에 도달할 때까지 **매 프레임 호출**되어야 합니다. 따라서 Event Tick이나 Timeline의 Update 핀에 연결해야 합니다.  
* **값의 업데이트:** 노드의 결과값(Return Value)을 다시 Current에 사용된 변수에 **Set** 해주어야 값이 누적되어 변화합니다.  
* **정확한 정지:** 지수적 방식 특성상 수학적으로 타겟에 완벽히 도달하는 데 시간이 오래 걸릴 수 있습니다. 아주 정밀한 일치가 필요하다면 FInterp To Constant 노드를 고려해 보세요. (이 노드는 일정한 속도로 이동합니다.)

### **요약**

FInterp To는 \*\*"A에서 B까지 시간(DeltaTime)과 속도(Speed)를 고려하여 부드럽게(Ease-out) 수치를 변화시키는 노드"\*\*라고 이해하시면 됩니다.
