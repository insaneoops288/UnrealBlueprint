언리얼 블루프린트에서 **Float Spring Interp** 노드는 단순히 값을 부드럽게 연결하는 것을 넘어, **물리적인 '용수철(Spring)'의 움직임**을 시뮬레이션하여 값을 보간(Interpolation)해주는 노드입니다.

일반적인 FInterp To가 일정한 속도로 매끄럽게 움직인다면, Float Spring Interp는 **반동, 흔들림, 탄성** 같은 생동감 있는 움직임을 만들어낼 때 사용합니다.


**1\. 노드의 핵심 원리**

이 노드는 **질량-용수철-감쇠(Mass-Spring-Damper)** 모델을 기반으로 작동합니다. 목표치에 도달할 때 마치 실제 용수철에 매달린 물체처럼 목표를 살짝 지나쳤다가 돌아오는(Overshoot) 등의 물리적 반응을 계산합니다.


**2\. 주요 입력 파라미터 (중요\!)**

이 노드를 제대로 쓰려면 각 설정값이 의미하는 바를 알아야 합니다.

* **Current / Target:** 현재 값과 목표 값입니다.  
* **Spring State (구조체):** **가장 중요한 부분입니다.** 이 노드는 이전 프레임의 속도와 상태를 기억해야 하므로, 반드시 \*\*변수(Variable)\*\*를 생성해서 연결해야 합니다. (연결하지 않으면 매 프레임 초기화되어 작동하지 않습니다.)  
* **Stiffness (강성):** 용수철의 탄성 수치입니다. 값이 클수록 목표 지점으로 더 강하고 빠르게 끌어당깁니다.  
* **Critical Damping Factor (감쇠 계수):** 에너지를 얼마나 빨리 흡수할지 결정합니다.  
  * **1.0 미만:** 목표치에서 출렁거리는(Bouncy) 효과가 발생합니다.  
  * **1.0 이상:** 출렁거림 없이 부드럽게 멈춥니다.  
* **Mass (질량):** 물체의 무게입니다. 값이 클수록 움직임이 묵직해지고 멈추는 데 오래 걸립니다.


**3\. FInterp To vs Float Spring Interp 비교**

| 특징 | FInterp To | Float Spring Interp |
| :---- | :---- | :---- |
| **느낌** | 수학적, 인위적인 매끄러움 | 물리적, 생동감 있는 반동 |
| **오버슈트** | 없음 (목표에서 딱 멈춤) | 있음 (목표를 지나쳤다 돌아옴 가능) |
| **복잡도** | 단순함 | 상태 변수 관리가 필요함 |
| **주요 용도** | 일반적인 수치 변화 | UI 팝업, 무기 반동, 카메라 흔들림 |


**4\. 실무 활용 사례**

### **① 역동적인 UI 애니메이션**

버튼 위에 마우스를 올렸을 때 버튼 크기가 커지는 효과를 줄 때 사용합니다. Float Spring Interp를 쓰면 버튼이 살짝 커졌다가 띠용\~ 하고 제자리를 잡는 듯한 **쫀득한 손맛**을 줄 수 있습니다.

### **② 무기 반동 및 스웨이 (Weapon Sway)**

FPS 게임에서 총을 쏠 때 총구가 위로 튀었다가 다시 내려오는 움직임, 혹은 캐릭터가 걸을 때 총기가 위아래로 출렁이는 효과를 매우 사실적으로 구현할 수 있습니다.

### **③ 카메라 팔로잉 (Camera Lag)**

카메라가 캐릭터를 따라올 때, 단순히 따라오는 게 아니라 물리적인 텐션을 가지고 따라오게 하여 속도감을 더해줄 때 사용합니다.


**5\. 주의사항: Spring State 변수 만들기**

이 노드를 처음 쓰시는 분들이 가장 많이 하는 실수는 Spring State 핀에 변수를 연결하지 않는 것입니다.

1. Spring State 핀에서 드래그하여 \*\*\[변수로 승격(Promote to Variable)\]\*\*을 클릭합니다.  
2. 생성된 변수가 매 프레임 업데이트되도록 노드 구성을 확인하세요. 이 변수가 없으면 노드는 자신이 이전에 어디에 있었는지 잊어버려 제대로 된 물리 연산을 할 수 없습니다.

**요약**

**Float Spring Interp**는 "살아있는 듯한 움직임"을 만들고 싶을 때 쓰는 노드입니다. **탄성과 반동**이 필요한 모든 곳에 FInterp To 대신 이 노드를 고려해 보세요\!
