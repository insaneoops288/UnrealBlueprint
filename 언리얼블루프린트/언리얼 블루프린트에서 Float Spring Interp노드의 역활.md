언리얼 블루프린트에서 Float Spring Interp노드의 역활?

언리얼 엔진의 블루프린트에서 Float Spring Interp 노드는 물리적인 스프링-댐퍼(Spring-Damper) 시스템을 시뮬레이션하여 값을 부드럽게 변화시키는 역할을 합니다.

일반적인 FInterp To 노드가 단순히 현재 값에서 목표 값으로 일정 비율로 이동한다면, Float Spring Interp는 **탄성(Elasticity)**과 **관성(Inertia)**이 느껴지는 훨씬 유기적이고 생동감 있는 움직임을 만들어냅니다.

1. 주요 작동 원리
이 노드는 단순히 수치를 계산하는 것을 넘어, 내부적으로 **속도(Velocity)**를 기억합니다. 목표 지점에 도달했을 때 딱 멈추는 것이 아니라, 설정된 물리 값에 따라 목표치를 약간 지나쳤다가 되돌아오는(Overshoot) 등의 물리 현상을 재현할 수 있습니다.

2. 주요 파라미터 (Inputs)
파라미터	설명
Current	현재의 실수(Float) 값입니다.
Target	도달하고자 하는 목표 값입니다.
Spring State	가장 중요한 부분입니다. 스프링의 현재 속도와 상태를 저장하는 변수(Float Spring State 타입)를 연결해야 합니다. 이 변수가 있어야 프레임 간의 연속성이 유지됩니다.
Stiffness	강성. 스프링이 얼마나 뻣뻣한지를 결정합니다. 값이 클수록 목표치에 더 강하고 빠르게 돌아가려 합니다.
Critical Damping Ratio	
감쇠비. 진동을 얼마나 빨리 줄일 것인가를 결정합니다.


- 0: 영원히 진동함


- 1: 진동 없이 가장 빠르게 정지함 (임계 감쇠)


- 1 미만: 목표치에서 출렁거림(Overshoot) 발생

Mass	질량. 물체의 무게입니다. 값이 클수록 움직임이 묵직해지고 반응이 느려집니다.
Delta Time	프레임 간의 시간 간격입니다. 보통 Get World Delta Seconds를 연결합니다.
3. FInterp To vs Float Spring Interp 비교
FInterp To: 일정한 속도나 부드러운 가속으로 목표에 도달합니다. 깔끔하고 정적인 UI 애니메이션 등에 적합합니다.

Float Spring Interp: 목표 지점에서 튕기거나 흔들리는 물리적 효과가 있습니다. "살아있는" 느낌을 주어야 하는 게임 플레이 요소에 적합합니다.

4. 주요 활용 사례
카메라 흔들림 및 추적: 카메라가 캐릭터를 따라갈 때 약간의 지연과 탄성을 주어 더 부드럽게 연출할 때 사용합니다.

UI 애니메이션: 버튼 위에 마우스를 올렸을 때 버튼 크기가 커지면서 살짝 튕기는 효과(Pop-up)를 줄 때 유용합니다.

캐릭터 장비 움직임: 캐릭터가 움직일 때 등에 맨 가방이나 무기가 관성에 의해 출렁거리는 효과를 구현할 수 있습니다.

게이지 바(HP Bar): 체력이 깎일 때 게이지가 물리적으로 출렁이며 줄어드는 연출에 사용합니다.

[!TIP]
Spring State 변수를 반드시 생성해서 연결하세요! 단순히 노드만 배치하면 매 프레임 속도가 초기화되어 스프링 효과가 나타나지 않습니다. 변수 패널에서 Float Spring State 타입을 찾아 변수를 만들고 이를 ref로 연결해야 정상 작동합니다.
