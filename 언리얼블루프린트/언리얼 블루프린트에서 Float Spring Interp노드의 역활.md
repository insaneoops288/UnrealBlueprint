# **Float Spring Interp 노드 완벽 이해하기**

Float Spring Interp는 목적지(Target) 값으로 이동할 때, 물리적인 스프링의 탄성과 감쇠를 적용하여 **관성, 오버슛(목표치를 지나침), 진동** 효과를 만들어내는 노드입니다.

## **1\. 핵심 역할**

일반적인 FInterp To가 일정한 속도나 부드러운 감속으로 목표에 도달한다면, Float Spring Interp는 마치 고무줄에 매달린 물체처럼 목표 지점 근처에서 튕기거나 출렁거리는 **유기적이고 역동적인 움직임**을 만들어냅니다.

## **2\. 주요 입력 파라미터 (Inputs)**

| 파라미터 | 설명 |
| :---- | :---- |
| **Current** | 현재 값입니다. |
| **Target** | 도달하고자 하는 목표 값입니다. |
| **Spring State** | **가장 중요한 부분입니다.** 스프링의 이전 속도와 상태를 저장하는 구조체 변수입니다. 반드시 변수로 생성하여 연결해야 물리 효과가 유지됩니다. |
| **Stiffness (강성)** | 스프링의 세기입니다. 값이 클수록 목표 지점으로 더 빠르게 당겨집니다. |
| **Critical Damping Ratio** | 감쇠 비율입니다. \- **1.0**: 오버슛 없이 가장 빠르게 멈춤 (Critical Damping) \- **1.0 미만**: 목표치 근처에서 출렁거림 (Underdamped) \- **1.0 초과**: 매우 끈적하게 느릿느릿 도달 (Overdamped) |
| **Delta Time** | 프레임 간의 시간 간격입니다. 보통 Get World Delta Seconds를 연결합니다. |
| **Mass (질량)** | 물체의 무게입니다. 질량이 클수록 관성이 커져서 더 많이 출렁거리고 멈추는 데 오래 걸립니다. |

## **3\. FInterp To vs Float Spring Interp 비교**

| 특징 | FInterp To | Float Spring Interp |
| :---- | :---- | :---- |
| **방식** | 수학적 곡선 (Exponential) | 물리 시뮬레이션 (Spring-Damper) |
| **오버슛** | 절대 목표치를 넘지 않음 | 목표치를 지나쳤다 돌아올 수 있음 |
| **느낌** | 깔끔하고 기계적인 부드러움 | 찰지고 역동적인 느낌 |
| **용도** | 일반 UI, 부드러운 카메라 회전 | 카메라 흔들림, 캐릭터의 머리 움직임, 튀는 UI 애니메이션 |

## **4\. 사용 시 주의사항**

1. **Spring State 변수화**: Spring State 핀에 직접 값을 넣는 것이 아니라, 반드시 \*\*변수(Float Spring Interpolation State)\*\*를 생성해서 연결해야 합니다. 그렇지 않으면 매 프레임 속도가 초기화되어 스프링 효과가 나타나지 않습니다.  
2. **프레임 독립성**: Delta Time을 사용하므로 프레임 레이트가 달라도 결과가 일정하게 유지되지만, 물리 계산 특성상 프레임이 너무 낮으면 계산이 튈 수 있습니다.

## **5\. 활용 예시**

* **절차적 애니메이션**: 캐릭터가 걸을 때 머리가 위아래로 반동을 받는 효과.  
* **역동적인 UI**: 버튼 위에 마우스를 올렸을 때 버튼 크기가 "통" 하고 튀기듯 커지는 효과.  
* **카메라 랙(Camera Lag)**: 3인칭 카메라가 캐릭터를 따라올 때 물리적인 거리감을 유지하며 따라오는 효과.
