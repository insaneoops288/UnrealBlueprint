# **언리얼 엔진 Gameplay Ability System (GAS)의 역할과 이해**

\*\*Gameplay Ability System (GAS)\*\*은 RPG나 액션 게임에서 흔히 볼 수 있는 '능력(Ability)', '속성(Attribute)', '상태 효과(Effect)' 등을 구현하기 위한 고도로 모듈화된 프레임워크입니다. 블루프린트 사용자의 관점에서 GAS의 주요 역할과 구성 요소를 설명합니다.

## **1\. GAS의 핵심 역할**

전통적인 블루프린트 방식(함수와 변수 위주)으로 스킬을 만들면 코드가 스파게티처럼 꼬이기 쉽습니다. GAS는 이를 다음과 같은 체계로 정리합니다.

* **복잡성 관리:** 수백 개의 스킬과 아이템 효과가 서로 어떻게 상호작용하는지 규칙을 정합니다.  
* **네트워크 복제(Replication) 자동화:** 멀티플레이어 환경에서 클라이언트와 서버 간의 능력 실행, 애니메이션 동기화, 능력치 변화를 기본적으로 지원합니다.  
* **의존성 제거:** 캐릭터 클래스 안에 모든 스킬 로직을 넣는 대신, 각 '능력'을 독립적인 객체로 분리하여 관리합니다.

## **2\. 블루프린트에서의 주요 구성 요소**

블루프린트 개발자는 주로 다음 4가지 요소를 조합하여 게임을 만듭니다.

### **① Gameplay Ability (GA) \- "무엇을 할 것인가?"**

캐릭터가 사용할 수 있는 구체적인 행동입니다.

* **BP 역할:** 스킬의 시전 조건(쿨다운, 마나 소모), 애니메이션 재생, 발사체 소환 등의 로직을 'ActivateAbility' 노드 아래에 작성합니다.  
* **예시:** 파이어볼 발사, 점프, 은신, 물약 사용.

### **② Gameplay Effect (GE) \- "어떤 변화를 줄 것인가?"**

상태나 속성을 변경하는 데이터 기반의 객체입니다.

* **BP 역할:** 직접 로직을 짜기보다는 데이터 에셋 형태로 작성합니다. 능력치 증가/감소, 지속 데미지(DoT), 스턴 상태 부여 등을 정의합니다.  
* **예시:** 5초간 이동 속도 20% 감소, 초당 체력 10 회복.

### **③ Gameplay Tag \- "현재 상태는 무엇인가?"**

문자열 기반의 계층적 라벨 시스템으로, GAS의 '언어' 역할을 합니다.

* **BP 역할:** "현재 기절 상태임", "화염 속성 공격임" 같은 상태를 태그로 붙여 관리합니다. 특정 태그가 있을 때만 능력이 발동되게 하거나 차단할 수 있습니다.  
* **예시:** State.Debuff.Stun, Ability.FireBall.

### **④ Ability Task \- "비동기 작업 처리"**

능력이 실행되는 동안 특정 사건이 일어나기를 기다리는 노드들입니다.

* **BP 역할:** "애니메이션 몽타주가 끝날 때까지 대기", "입력이 들어올 때까지 대기" 같은 복잡한 비동기 로직을 블루프린트에서 매우 단순하게 구현하게 해줍니다.

## **3\. 일반적인 블루프린트 방식 vs GAS 비교**

| 비교 항목 | 일반적인 블루프린트 방식 | Gameplay Ability System (GAS) |
| :---- | :---- | :---- |
| **상태 관리** | 수많은 Boolean 변수 (IsJumping, IsStunned) | Gameplay Tag를 통한 통합 관리 |
| **네트워크** | RPC (Server, Multicast)를 직접 설계 | 프레임워크 수준에서 예측 및 복제 지원 |
| **쿨다운/비용** | 타이머와 변수로 직접 계산 로직 작성 | GE를 통해 선언적으로 처리 (데이터 입력) |
| **유지보수** | 스킬이 늘어날수록 캐릭터 클래스가 무거워짐 | 각 능력이 독립된 에셋으로 분리됨 |

## **4\. 블루프린트 사용 시 유의사항**

1. **C++ 기반의 초기 설정:** GAS는 블루프린트만으로는 100% 활용하기 어렵습니다. AttributeSet(체력, 마나 등 변수 선언)이나 기본적인 AbilitySystemComponent 등록은 C++에서 수행하는 것이 성능과 안정성 면에서 권장됩니다.  
2. **학습 곡선:** 처음에는 설정할 것이 많아 어렵게 느껴질 수 있지만, 프로젝트 규모가 커질수록 GAS가 주는 구조적 안정성이 매우 큽니다.  
3. **UI 연동:** GAS의 속성 변화를 UI(UMG)에 반영할 때는 델리게이트(Delegate)를 통해 값이 변할 때만 업데이트하도록 설계하는 것이 효율적입니다.

## **요약**

**GAS**는 블루프린트에서 \*\*"스킬의 로직(GA), 상태의 변화(GE), 논리적 조건(Tag)"\*\*을 완전히 분리하여 관리하게 해주는 시스템입니다. 특히 멀티플레이어 게임을 개발하거나 복잡한 스킬 상호작용이 필요한 RPG 게임을 만든다면 필수적인 도구입니다.
