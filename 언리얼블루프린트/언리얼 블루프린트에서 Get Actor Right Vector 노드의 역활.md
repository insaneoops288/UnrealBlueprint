# **Get Actor Right Vector 노드 이해하기**

언리얼 엔진에서 액터는 자신만의 \*\*로컬 좌표계(Local Space)\*\*를 가집니다. Get Actor Right Vector는 이 로컬 좌표 중 '오른쪽'을 나타내는 축을 \*\*월드 좌표계(World Space)\*\*의 벡터값으로 변환하여 반환합니다.

## **1\. 주요 역할**

* **오른쪽 방향 추출**: 액터가 현재 회전해 있는 상태를 기준으로, 정확히 오른쪽이 어디인지를 벡터(X, Y, Z) 형태로 제공합니다.  
* **단위 벡터(Normalized Vector)**: 이 노드가 반환하는 벡터의 길이는 항상 **1.0**입니다. 방향 정보만을 담고 있습니다.  
* **로컬 Y축**: 언리얼 엔진의 표준 좌표계에서 **Y축**은 오른쪽(Right)을 의미합니다. 즉, 액터의 로컬 Y축 방향이 월드에서 어떤 방향인지 계산해 줍니다.

## **2\. 왜 사용하는가? (활용 사례)**

### **① 캐릭터 좌우 이동 (Strafing)**

TPS나 FPS 게임에서 A, D 키를 눌러 옆으로 이동할 때 사용합니다.

* Get Actor Right Vector 값에 'Scale Value'(입력값)를 곱해 Add Movement Input 노드에 연결하면 캐릭터가 바라보는 방향 기준 오른쪽/왼쪽으로 이동합니다.

### **② 물체 스폰 위치 지정**

캐릭터의 오른손 위치나 오른쪽 어깨 너머에 마법 효과나 발사체를 생성하고 싶을 때 사용합니다.

* Get Actor Location \+ (Get Actor Right Vector \* 100.0)을 하면 캐릭터 중심에서 오른쪽으로 100유닛 떨어진 좌표를 구할 수 있습니다.

### **③ 왼쪽 방향 구하기 (Left Vector)**

언리얼에는 Get Actor Left Vector라는 노드는 따로 없습니다.

* Get Actor Right Vector 값에 **\-1**을 곱하면 바로 왼쪽 방향 벡터가 됩니다.

## **3\. 벡터의 특징 (수학적 배경)**

* **월드 좌표 기준**: 액터가 월드에서 (0, 90, 0)으로 회전해 있다면, 이 노드는 월드 좌표의 Forward(X) 방향을 가리키게 될 수도 있습니다. (액터가 회전하면 오른쪽 벡터값도 변합니다.)  
* **값의 범위**: X, Y, Z 각 성분은 \-1.0에서 1.0 사이의 값을 가집니다.

## **4\. 다른 노드와의 비교**

| 노드 이름 | 방향 (로컬 축) | 주로 쓰이는 곳 |
| :---- | :---- | :---- |
| **Get Actor Forward Vector** | 앞쪽 (X축) | 전진, 총알 발사 방향 |
| **Get Actor Right Vector** | 오른쪽 (Y축) | 옆걸음, 사이드 미러 위치 |
| **Get Actor Up Vector** | 위쪽 (Z축) | 점프 힘 방향, 머리 위 UI 위치 |

## **5\. 블루프린트 구성 예시 (좌우 이동)**

1. **Input Axis MoveRight** 이벤트 발생  
2. **Get Actor Right Vector** 호출 (Target은 Self)  
3. **Vector \* Float** 노드로 방향 벡터와 Axis Value를 곱함  
4. **Add Movement Input**의 'World Direction'에 연결

이 흐름을 통해 캐릭터가 어디를 보고 있든 상관없이 항상 옆으로 이동하는 로직을 구현할 수 있습니다.
