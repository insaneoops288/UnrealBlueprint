# **Get All Actors with Tag 노드 완벽 가이드**

언리얼 엔진(Unreal Engine) 블루프린트에서 **Get All Actors with Tag** 노드는 월드에 배치된 수많은 액터 중에서 특정 \*\*태그(Tag)\*\*를 가진 액터들만 필터링하여 배열(Array) 형태로 가져오는 강력한 기능을 수행합니다.

## **1\. 주요 역할 (Role)**

이 노드의 핵심 역할은 \*\*"이름표(Tag)가 붙은 모든 물체를 리스트로 뽑아줘"\*\*라고 요청하는 것입니다.

* **동적 참조:** 특정 액터를 변수로 직접 연결하지 않아도, 태그 이름만 알면 언제든지 해당 액터들에 접근할 수 있습니다.  
* **그룹 제어:** 동일한 태그를 가진 여러 액터(예: 모든 전등, 모든 적 캐릭터 등)를 한 번에 조작할 때 필수적입니다.

## **2\. 노드 구조 및 파라미터**

| 입력/출력 | 이름 | 설명 |
| :---- | :---- | :---- |
| **Input (입력)** | **Tag** | 찾고자 하는 태그의 이름을 입력합니다. (Name 타입) |
| **Output (출력)** | **Out Actors** | 태그가 일치하는 액터들이 담긴 \*\*배열(Array)\*\*입니다. |

## **3\. 작동 원리**

1. 노드가 실행되면 언리얼 엔진은 현재 월드에 존재하는 **모든 액터**를 전수 조사합니다.  
2. 각 액터의 Tags 배열 내에 사용자가 입력한 Tag와 일치하는 문자열이 있는지 확인합니다.  
3. 일치하는 액터들을 모아서 하나의 리스트(배열)로 반환합니다.

## **4\. 주요 활용 사례**

### **① 특정 그룹의 액터 일괄 파괴/수정**

예를 들어, 스테이지 클리어 시 "Enemy" 태그를 가진 모든 적 액터를 한꺼번에 제거하고 싶을 때 유용합니다.

* Get All Actors with Tag (Tag: Enemy) \-\> ForEachLoop \-\> Destroy Actor

### **② 특정 포인트 찾기**

월드 곳곳에 배치된 체크포인트나 아이템 스폰 위치에 "SpawnPoint" 태그를 넣고, 게임 시작 시 이 위치 정보를 모두 가져와 관리할 수 있습니다.

### **③ 상호작용 시스템**

스위치를 눌렀을 때 "Light\_RoomA" 태그를 가진 모든 전등 액터의 불을 끄거나 켜는 로직을 짤 때 사용합니다.

## **5\. Get All Actors of Class와 차이점**

| 비교 항목 | Get All Actors with Tag | Get All Actors of Class |
| :---- | :---- | :---- |
| **검색 기준** | 액터에 설정된 **Tag(이름표)** | 액터의 **Class(도면/타입)** |
| **유연성** | 서로 다른 클래스라도 같은 태그면 함께 가져옴 | 특정 클래스(또는 자식 클래스)만 가져옴 |
| **성능** | 클래스 필터링이 없으므로 비교적 더 무거울 수 있음 | 특정 클래스만 조사하므로 약간 더 효율적일 수 있음 |

## **6\. 성능 및 주의사항 (매우 중요)**

### **⚠️ Tick 노드에서 사용 금지**

이 노드는 실행될 때마다 월드의 모든 액터를 검색합니다. 액터가 수천 개라면 성능 부하가 매우 큽니다. 따라서 매 프레임 실행되는 Event Tick에 연결하는 것은 절대로 피해야 합니다.

### **⚠️ 가급적 캐싱(Caching) 사용**

게임 시작 시(BeginPlay) 한 번만 호출하여 변수(Array)에 저장해두고, 필요할 때마다 그 변수를 사용하는 것이 성능 최적화에 훨씬 유리합니다.

### **⚠️ 대소문자 구분**

태그 이름은 대소문자를 구분하지 않지만, 오타가 나면 빈 배열을 반환하므로 정확한 이름을 입력해야 합니다.

## **7\. 액터에 태그 설정하는 법**

1. 언리얼 에디터에서 액터를 선택합니다.  
2. **Details(디테일)** 패널에서 'Tag'를 검색합니다.  
3. **Actor \> Tags** 항목의 \+ 버튼을 눌러 원하는 이름을 추가합니다. (Component Tag와 헷갈리지 않도록 주의하세요\!)
