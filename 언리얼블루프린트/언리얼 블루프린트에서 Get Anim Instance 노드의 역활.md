# **Get Anim Instance 노드 이해하기**

언리얼 엔진에서 **Get Anim Instance** 노드는 스켈레탈 메시 컴포넌트(Skeletal Mesh Component)가 현재 사용 중인 \*\*애니메이션 인스턴스(Animation Instance)\*\*에 대한 참조를 가져오는 역할을 합니다.

## **1\. 핵심 역할**

이 노드의 가장 큰 목적은 \*\*"게임 로직(캐릭터 블루프린트 등)에서 애니메이션 블루프린트로 데이터를 전달하거나 함수를 호출하는 것"\*\*입니다.

* **브릿지 역할**: 캐릭터의 상태(이동 속도, 점프 여부, 공격 상태 등)를 애니메이션 블루프린트 내의 변수에 전달하기 위해 사용합니다.  
* **컴포넌트 연결**: 특정 스켈레탈 메시가 현재 어떤 애니메이션 로직에 의해 구동되고 있는지 확인합니다.

## **2\. 주요 사용 패턴 (The Casting Pattern)**

Get Anim Instance 노드는 기본적으로 Anim Instance 타입을 반환합니다. 하지만 우리가 만든 특정 애니메이션 블루프린트(예: ABP\_Player)의 변수에 접근하려면 반드시 **형 변환(Cast)** 과정이 필요합니다.

### **일반적인 실행 흐름**

1. **Mesh 컴포넌트**를 그래프로 드래그합니다.  
2. Mesh에서 선을 끌어 **Get Anim Instance** 노드를 생성합니다.  
3. 반환 값(Return Value)에서 선을 끌어 \*\*Cast To \[내 애니메이션 블루프린트 이름\]\*\*을 실행합니다.  
4. 형 변환에 성공하면, 해당 애니메이션 블루프린트 안에 선언된 변수(Speed, IsJumping 등)를 수정하거나 함수를 호출합니다.

## **3\. 왜 사용해야 하나요?**

애니메이션 블루프린트는 독립적인 공간입니다. 캐릭터 블루프린트에서 "공격 버튼을 눌렀다"는 사실을 애니메이션 블루프린트는 자동으로 알지 못합니다.

따라서 캐릭터 블루프린트의 Event Tick이나 특정 이벤트 발생 시점에 Get Anim Instance를 통해 애니메이션 블루프린트의 변수 값을 업데이트해 주어야 애니메이션이 그에 맞춰 변하게 됩니다.

## **4\. 활용 예시: 속도 동기화**

캐릭터가 움직일 때 애니메이션 블루프린트의 Speed 변수를 업데이트하는 예시입니다.

1. **Character Blueprint**의 Event Tick에서 호출.  
2. Get Velocity \-\> Vector Length로 현재 속도를 구함.  
3. Mesh \-\> **Get Anim Instance** \-\> Cast to ABP\_Player.  
4. ABP\_Player 내부의 Speed 변수를 찾아 구한 속도 값을 대입(Set).

## **5\. 주의사항**

* **유효성 검사 (Validity Check)**: 스켈레탈 메시에 애니메이션 블루프린트가 설정되어 있지 않거나, 게임이 시작되지 않은 시점에는 None을 반환할 수 있습니다. 형 변환 전후에 Is Valid 노드로 체크하는 것이 안전합니다.  
* **성능 고려**: 매 프레임(Tick)마다 Get Anim Instance와 Cast를 수행하는 것은 비용이 발생합니다. 보통 Begin Play에서 한 번만 가져와서 \*\*변수로 승격(Promote to Variable)\*\*하여 저장해두고 사용하는 방식이 권장됩니다.

### **요약**

**Get Anim Instance**는 "이 메시를 움직이고 있는 애니메이션 설계도(AnimBP)를 좀 가져와 봐\!"라고 명령하는 노드라고 이해하면 쉽습니다.
