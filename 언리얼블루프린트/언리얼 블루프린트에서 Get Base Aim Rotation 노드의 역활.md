# **Get Base Aim Rotation 노드 가이드**

언리얼 엔진의 Pawn 클래스에서 제공되는 Get Base Aim Rotation 노드는 캐릭터나 AI가 "바라보고 있는 방향"을 정의할 때 사용됩니다.

## **1\. 주요 역할**

이 노드는 기본적으로 폰의 \*\*조준 방향(Aiming Direction)\*\*을 계산하여 반환합니다. 단순히 몸체가 돌아간 방향이 아니라, 플레이어의 시선이나 컨트롤러의 입력을 반영한 실제 '조준' 수치를 가져옵니다.

## **2\. 작동 원리 (내부 로직)**

이 함수는 다음과 같은 우선순위에 따라 값을 결정합니다:

1. **컨트롤러가 있는 경우:** 폰에 Controller(PlayerController 또는 AIController)가 빙의(Possess)되어 있다면, 컨트롤러의 **Control Rotation** 값을 반환합니다.  
   * 플레이어가 마우스를 위아래로 움직여 카메라를 조절할 때, 캐릭터의 몸은 정면을 보더라도 이 노드는 카메라가 위를 보는 각도를 포함한 회전값을 반환합니다.  
2. **컨트롤러가 없는 경우:** 만약 컨트롤러가 할당되지 않은 상태라면, 해당 폰의 **Actor Rotation**(몸체가 바라보는 방향)을 기본값으로 반환합니다.

## **3\. Get Actor Rotation과의 차이점**

| 특징 | Get Actor Rotation | Get Base Aim Rotation |
| :---- | :---- | :---- |
| **반환값** | 폰(액터)의 물리적인 몸체 회전 | 컨트롤러의 조준/시선 회전 |
| **Pitch(상하) 값** | 보통 0 (캐릭터가 바닥에 서 있을 때) | 플레이어가 위/아래를 보는 각도 포함 |
| **주요 용도** | 캐릭터 이동 방향 계산 | 총기 발사, 라인 트레이스, 시선 처리 |

## **4\. 주요 사용 사례**

### **A. 투사체 발사 및 라인 트레이스 (Weapon Firing)**

총기를 발사할 때 총구가 아닌 **플레이어의 시선 정면**으로 총알이 나가게 하고 싶다면 이 노드를 사용합니다.

* Get Base Aim Rotation에서 Get Forward Vector를 추출하여 라인 트레이스의 방향으로 설정합니다.

### **B. 네트워크 멀티플레이어**

이 함수는 네트워크 환경에서 유용합니다. 서버와 클라이언트 간에 폰의 조준 방향을 동기화할 때, GetControlRotation은 클라이언트 측에서만 유효한 경우가 많지만, GetBaseAimRotation은 적절히 래핑되어 있어 조준 방향을 참조하기에 더 안정적입니다.

### **C. AI 조준**

AI 캐릭터가 플레이어를 추적하거나 특정 타겟을 조준할 때, AI 컨트롤러가 설정한 회전값을 가져와 애니메이션 블루프린트의 Aim Offset 등에 전달하는 용도로 사용됩니다.

## **5\. 블루프린트 구성 예시**

1. **Input Action (Fire)** 이벤트 발생.  
2. **Get Base Aim Rotation** 노드 호출.  
3. 반환된 Rotator를 **Get Forward Vector**에 연결.  
4. **Line Trace By Channel**의 Start 지점에 Get Actor Location을, End 지점에 Start \+ (Forward Vector \* 사거리)를 연결.

**요약:** 캐릭터의 몸 방향이 아닌, **카메라나 컨트롤러가 가리키는 실제 조준 방향**이 필요할 때 반드시 사용해야 하는 노드입니다.
