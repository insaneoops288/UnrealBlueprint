# **Get Base Rotation 노드 상세 설명**

언리얼 엔진 블루프린트에서 Get Base Rotation 함수(노드)는 Pawn 클래스의 멤버 함수로, 해당 폰의 \*\*지향 방향(Orientation)\*\*을 결정하는 가장 기본이 되는 회전값을 반환합니다.

## **1\. 핵심 동작 원리**

이 노드는 내부적으로 다음과 같은 우선순위에 따라 값을 선택하여 반환합니다:

1. **컨트롤러(Controller)가 있는 경우:** \* 현재 폰을 조종하고 있는 **Control Rotation** 값을 반환합니다.  
   * 이는 주로 플레이어가 마우스나 스틱으로 카메라를 회전시켰을 때의 회전값입니다.  
2. **컨트롤러가 없는 경우 (AI 또는 빈 폰):**  
   * 해당 폰(Actor) 자체의 **Actor Rotation** 값을 반환합니다.

## **2\. 왜 사용하는가? (Get Actor Rotation과의 차이점)**

가장 큰 차이점은 \*\*"몸체의 방향"\*\*과 **"시선의 방향"** 중 무엇을 기준으로 하느냐입니다.

* **Get Actor Rotation:** 캐릭터 모델(Mesh)이 현재 물리적으로 월드에서 회전되어 있는 방향을 가져옵니다. 캐릭터가 앞을 보고 달려가고 있다면 그 방향을 반환합니다.  
* **Get Base Rotation:** 캐릭터가 물리적으로는 옆을 보고 있더라도, 플레이어의 카메라(컨트롤러)가 앞을 조준하고 있다면 그 **조준 방향**을 가져옵니다.

## **3\. 주요 활용 사례**

### **A. 투사체 발사 (Projectile Spawning)**

총을 쏘거나 마법을 부릴 때, 캐릭터의 몸이 돌아가는 속도보다 카메라 회전이 더 빠를 수 있습니다. 이때 Get Base Rotation을 사용하면 캐릭터의 몸 방향이 아닌, 플레이어가 화면 중앙으로 조준하고 있는 정확한 방향으로 총알을 날릴 수 있습니다.

### **B. 애니메이션 블루프린트 (ABP)**

캐릭터의 상체만 조준 방향으로 돌리는 '에임 오프셋(Aim Offset)'을 구현할 때, 컨트롤러의 회전값과 캐릭터의 몸 회전값의 차이를 계산하기 위해 이 노드를 사용합니다.

### **C. 카메라 기반 이동**

캐릭터가 카메라가 바라보는 방향을 기준으로 이동해야 할 때(예: TPS 게임), 이동 벡터를 계산하는 기초 데이터로 활용됩니다.

## **4\. 요약**

| 특징 | 내용 |
| :---- | :---- |
| **대상 클래스** | Pawn (또는 Pawn을 상속받은 Character) |
| **반환 타입** | Rotator (Pitch, Yaw, Roll) |
| **우선순위** | 1\. Control Rotation (있을 때) / 2\. Actor Rotation (없을 때) |
| **한 줄 정의** | "이 캐릭터가 현재 조준하거나 의도하고 있는 기준 방향" |

**팁:** 만약 캐릭터의 발이 닿고 있는 바닥의 기울기 등에 맞춘 회전이 필요하다면 이 노드가 아닌 GetActorRotation이나 FindLookAtRotation 등을 조합해서 사용해야 합니다. Get Base Rotation은 철저히 **'입력/컨트롤'** 관점의 회전값입니다.
