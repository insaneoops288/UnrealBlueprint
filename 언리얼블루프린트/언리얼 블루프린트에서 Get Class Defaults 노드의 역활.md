# **언리얼 엔진 블루프린트: Get Class Defaults 노드 완벽 가이드**

Get Class Defaults 노드는 언리얼 엔진 블루프린트 프로그래밍에서 매우 중요한 역할을 합니다. 이 노드의 핵심은 \*\*"배치되거나 생성된 액터(인스턴스)가 없어도, 해당 클래스가 가진 기본 정보를 미리 읽어올 수 있다"\*\*는 점입니다.

## **1\. 주요 역할 (Role)**

보통 액터의 변수 값을 가져오려면 월드에 해당 액터가 배치되어 있어야 하고, 그 액터의 '참조(Reference)'가 필요합니다. 하지만 Get Class Defaults는 다릅니다.

* **인스턴스 미필요:** 액터를 Spawn Actor로 생성하거나 월드에 배치하지 않고도, 해당 클래스의 블루프린트 편집기 내 'Details' 창에서 설정한 기본값을 가져옵니다.  
* **데이터 접근:** 클래스 레퍼런스(Purple Pin)를 입력받아 해당 클래스에 정의된 모든 변수의 기본값에 접근할 수 있습니다.

## **2\. 주요 특징 및 작동 방식**

### **입력 (Input)**

* **Class:** 가져오고자 하는 클래스 타입을 지정합니다. (예: BP\_PlayerCharacter, BP\_EnemyBase 등)

### **출력 (Output)**

* 노드를 선택한 후 **Details(세부 정보) 패널**에서 출력하고 싶은 변수를 체크하여 핀으로 노출시킬 수 있습니다.  
* 기본적으로 모든 변수가 보이지 않으므로, 필요한 데이터만 핀으로 뽑아서 사용합니다.

### **특징**

* **Read-Only (읽기 전용):** 이 노드를 통해 값을 수정할 수는 없습니다. 오직 클래스 설정에 정의된 초기값만 읽어올 수 있습니다.  
* **정적 데이터 처리:** 게임 도중 특정 인스턴스의 변수가 변하더라도, 이 노드는 항상 '클래스 자체'의 기본값을 반환합니다.

## **3\. 언제 사용하나요? (Use Cases)**

1. **아이템/데이터 시트 조회:** \* 예를 들어, 아이템 클래스마다 가격, 이름, 아이콘 등이 기본값으로 설정되어 있다면, 상점 UI에서 아이템을 생성하기 전에 미리 정보를 보여줄 때 사용합니다.  
2. **스폰 조건 결정:** \* 액터를 생성하기 전에 해당 클래스의 SpawnCost(소환 비용) 등을 확인하여 생성 가능 여부를 판단할 때 유용합니다.  
3. **UI 바인딩:** \* 캐릭터 선택 창에서 각 캐릭터 클래스의 기본 능력치(공격력, 방어력 등)를 그래프로 표시할 때 사용합니다.

## **4\. 주의사항**

* **가변 데이터 불가:** 실시간으로 체력이 깎인 플레이어의 현재 체력을 가져오고 싶다면 이 노드가 아니라 실제 플레이어 액터의 참조를 사용해야 합니다.  
* **핀 노출 설정:** 노드를 배치했는데 아무런 출력 핀이 없다면, **노드를 클릭하고 오른쪽 Details 패널**에서 가져올 변수 이름 옆의 눈 모양 아이콘(Show Pin)을 체크했는지 확인하세요.

## **요약**

Get Class Defaults는 \*\*"붕어빵(인스턴스)을 굽기 전에, 붕어빵 틀(클래스)에 적힌 레시피(기본값)를 확인하는 노드"\*\*라고 이해하면 가장 쉽습니다.
