# **Get Class 노드 이해하기**

언리얼 엔진 블루프린트에서 **Get Class** 노드는 실행 시점에 특정 오브젝트 인스턴스의 '설계도(클래스)' 정보를 가져오는 데 사용됩니다.

## **1\. 기본 개념**

* **입력 (Target):** 확인하고자 하는 오브젝트의 레퍼런스 (예: Self, 타격한 액터, 오버랩된 컴포넌트 등).  
* **출력 (Return Value):** 해당 오브젝트가 속한 **클래스 레퍼런스 (Class Reference)**. 핀의 색상은 보통 **보라색**입니다.

## **2\. 주요 역할 및 활용 사례**

### **① 클래스 비교 (Equality Check)**

가장 흔한 용도는 "이 물체가 내가 찾는 그 종류인가?"를 확인하는 것입니다.

* **상황:** 총알이 무언가에 부딪혔을 때, 부딪힌 대상이 '적(Enemy)' 클래스인지 확인하고 싶을 때.  
* **방법:** Get Class \-\> Equal (==) 노드를 사용하여 특정 클래스와 비교합니다.

### **② 동일한 종류의 액터 생성 (Spawning)**

* **상황:** 어떤 액터를 파괴하면서 그와 똑같은 종류의 효과나 아이템을 그 자리에 생성하고 싶을 때.  
* **방법:** Get Class를 통해 현재 액터의 클래스를 알아낸 뒤, Spawn Actor from Class 노드의 클래스 핀에 연결합니다.

### **③ 클래스 이름 확인 및 디버깅**

* **상황:** 런타임 중에 어떤 오브젝트가 어떤 클래스인지 로그로 남기고 싶을 때.  
* **방법:** Get Class \-\> Get Display Name \-\> Print String 순으로 연결하여 화면에 클래스 이름을 출력합니다.

## **3\. 'Cast To' 노드와의 차이점**

| 구분 | Get Class (클래스 비교) | Cast To (형변환) |
| :---- | :---- | :---- |
| **목적** | 단순히 종류(타입)만 확인 | 대상의 내부 변수나 함수에 접근 |
| **정밀도** | **정확히** 일치해야 함 (자식 클래스는 False) | 부모/자식 관계라면 성공 (다형성 활용) |
| **비용** | 매우 가볍고 빠름 | 약간의 비용이 발생함 |

**팁:** 단순히 "이게 A 클래스야?"라고 묻고 싶다면 Equal (==)이나 Class Is Child Of를 쓰는 것이 성능상 유리하며, 그 클래스의 특정 함수를 실행해야 한다면 Cast To를 사용해야 합니다.

## **4\. 관련 노드**

* **Class Is Child Of:** Get Class에서 나온 결과가 특정 부모 클래스를 상속받았는지 확인할 때 유용합니다. (예: '적' 클래스뿐만 아니라 '적'을 상속받은 '보스' 클래스도 포함하고 싶을 때)  
* **Get Object Class:** Get Class와 동일한 기능을 수행합니다.
