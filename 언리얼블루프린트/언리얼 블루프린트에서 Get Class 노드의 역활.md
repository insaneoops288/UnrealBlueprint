# **언리얼 블루프린트: Get Class 노드 완벽 이해**

Get Class 노드는 언리얼 엔진 프로그래밍에서 매우 자주 사용되는 노드로, 런타임 중에 객체의 타입을 동적으로 파악할 때 핵심적인 역할을 합니다.

## **1\. 노드의 핵심 역할**

* **입력 (Target):** 객체 레퍼런스 (Object Reference). 예를 들어 'Self', 'Hit Actor', 'Overlapped Actor' 등이 들어갑니다.  
* **출력 (Return Value):** 클래스 레퍼런스 (Class Reference). 해당 객체의 원본 클래스 타입을 반환합니다.

## **2\. 주요 사용 시나리오**

### **① 객체 타입 비교 (Filtering)**

특정 액터와 충돌했을 때, 그 액터가 내가 찾는 특정 클래스인지 확인할 때 사용합니다.

* **예시:** Get Class \-\> Equal (==) \-\> Branch  
* **왜 쓰나요?** Cast To 노드를 사용할 수도 있지만, 단순히 타입이 일치하는지만 확인하고 싶을 때는 Get Class가 성능상 더 가볍고 직관적일 수 있습니다.

### **② 동일한 타입의 객체 생성 (Spawning)**

월드에 배치된 어떤 액터를 복제하거나, 같은 종류의 아이템을 스폰해야 할 때 유용합니다.

* **예시:** Get Class의 결과값을 Spawn Actor from Class 노드의 클래스 핀에 연결합니다.

### **③ 클래스 디폴트 값 접근 (Get Class Defaults)**

객체의 인스턴스 값이 아니라, 그 클래스 자체가 기본적으로 가지고 있는 설정값(CDO, Class Default Object)을 읽어오고 싶을 때 사용합니다.

* **예시:** Get Class \-\> Get Class Defaults 노드를 연결하여 해당 클래스의 기본 공격력, 이름 등을 참조합니다.

## **3\. Cast To 노드와의 차이점**

| 구분 | Get Class | Cast To (형변환) |
| :---- | :---- | :---- |
| **목적** | 단순 타입 확인 및 클래스 정보 추출 | 타입 확인 \+ 해당 클래스의 변수/함수 접근 |
| **결과** | 클래스 레퍼런스 반환 | 객체 레퍼런스 반환 (성공 시) |
| **성능** | 매우 가벼움 | 상대적으로 무거움 (종속성 발생) |

## **4\. 실전 팁**

* **자식 클래스 주의:** Get Class 노드는 정확히 그 객체의 클래스만 반환합니다. 만약 부모 클래스 타입을 포함하여 확인하고 싶다면 Is Child Of 노드와 함께 사용하는 것이 좋습니다.  
* **유효성 검사:** Get Class를 호출하기 전에 입력 객체가 유효한지(Is Valid) 먼저 확인하는 습관이 필요합니다. 객체가 None이면 노드가 제대로 작동하지 않습니다.

**요약하자면:** Get Class는 "이 물건의 설계도가 뭐야?"라고 묻는 노드입니다. 설계도(클래스)를 알면 같은 물건을 더 만들거나, 그 설계도에 적힌 기본 사양을 읽어올 수 있습니다.
