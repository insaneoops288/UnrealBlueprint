# **Get Component by Class 노드 가이드**

언리얼 엔진의 **Get Component by Class** 노드는 액터(Actor) 내에서 특정 종류의 부품(Component)을 동적으로 찾아야 할 때 사용하는 핵심적인 노드입니다.

## **1\. 기본 개념**

이 노드는 "이 액터에 '스태틱 메시'가 있어?", "이 액터에 '오디오 컴포넌트'가 있어?"와 같은 질문을 던지고, 존재한다면 해당 컴포넌트를 직접 제어할 수 있는 \*\*참조(Reference)\*\*를 반환합니다.

## **2\. 노드 파라미터 구성**

| 항목 | 이름 | 설명 |
| :---- | :---- | :---- |
| **Input** | **Target** | 컴포넌트를 검색할 대상 액터입니다. (기본값은 Self) |
| **Input** | **Component Class** | 찾으려는 컴포넌트의 클래스(예: StaticMeshComponent, CameraComponent 등)를 선택합니다. |
| **Output** | **Return Value** | 찾은 컴포넌트 객체입니다. 일치하는 것이 없으면 None(Null)을 반환합니다. |

## **3\. 주요 특징 및 장점**

### **이름에 의존하지 않는 검색**

컴포넌트의 개별 이름(예: "MySuperSwordMesh")을 몰라도, 그 컴포넌트가 \*\*어떤 종류(Class)\*\*인지만 알면 접근할 수 있습니다. 이는 다양한 액터와 상호작용해야 하는 범용 시스템을 만들 때 매우 유용합니다.

### **다형성 활용**

부모 클래스(예: SceneComponent)를 검색 대상으로 설정하면, 이를 상속받은 모든 자식 컴포넌트(StaticMesh, Billboard, Audio 등) 중 가장 먼저 발견되는 것을 가져옵니다.

## **4\. 권장 사용 패턴 (Best Practices)**

### **유효성 검사 (Is Valid)**

검색 결과가 없을 경우 None을 반환하므로, 반환된 값을 사용하기 전에 반드시 **Is Valid** 노드로 확인해야 런타임 에러(Crash)를 방지할 수 있습니다.

### **변수에 캐싱(Caching)**

이 노드는 내부적으로 액터의 컴포넌트 목록을 순회(Loop)하므로 성능 비용이 발생합니다.

* **나쁜 예:** Event Tick에서 매 프레임 호출하기.  
* **좋은 예:** BeginPlay에서 한 번만 호출하여 변수에 저장(Promote to Variable)해두고 재사용하기.

## **5\. 비교: Get Component vs Get Components (단수와 복수)**

* **Get Component by Class (단수):** 조건에 맞는 **첫 번째** 컴포넌트 하나만 반환합니다.  
* **Get Components by Class (복수):** 조건에 맞는 **모든** 컴포넌트를 찾아 **배열(Array)** 형태로 반환합니다. 액터에 동일한 종류의 컴포넌트가 여러 개 있을 때 사용합니다.

## **6\. 실제 활용 예시**

* **상호작용:** 플레이어가 바라본 액터(Hit Actor)에서 InventoryItem 컴포넌트를 찾아 아이템 데이터를 읽어올 때.  
* **시각 효과:** 특정 액터에 들어있는 모든 스태틱 메시의 색상을 한 번에 바꿀 때 (복수형 노드와 조합).  
* **카메라 제어:** 현재 활성화된 액터 내부의 CameraComponent를 찾아 화면을 전환할 때.
