# **Get Components by Class 노드 상세 가이드**

언리얼 엔진 블루프린트에서 Get Components by Class 노드는 액터의 구성 요소들을 프로그래밍적으로 탐색하고 제어할 때 핵심적인 역할을 합니다.

## **1\. 주요 역할 (Core Function)**

이 노드는 지정된 **Target(액터)** 내부를 검색하여, **Component Class** 핀에서 설정한 클래스(또는 그 자식 클래스)와 일치하는 모든 컴포넌트를 수집합니다. 결과물은 해당 컴포넌트들의 참조를 담은 \*\*배열(Array)\*\*로 출력됩니다.

## **2\. 노드 파라미터 설명**

| 핀 이름 | 타입 | 설명 |
| :---- | :---- | :---- |
| **Target** | Actor Reference | 검색 대상이 될 액터입니다. (기본값은 Self) |
| **Component Class** | Component Class Reference | 찾고자 하는 컴포넌트의 종류를 선택합니다 (예: StaticMeshComponent, PointLightComponent 등). |
| **Return Value** | Array of Objects | 클래스 필터에 걸러진 컴포넌트들의 리스트입니다. |

## **3\. 작동 원리**

1. **계층 구조 탐색**: Target으로 지정된 액터의 모든 컴포넌트 리스트를 확인합니다.  
2. **클래스 필터링**: 각 컴포넌트가 사용자가 지정한 Component Class와 일치하는지, 혹은 그 클래스를 상속받았는지 확인합니다.  
3. **배열 생성**: 조건에 맞는 컴포넌트들을 순서대로 배열에 담아 반환합니다. 만약 조건에 맞는 컴포넌트가 하나도 없다면 빈 배열을 반환합니다.

## **4\. 주요 사용 시나리오**

### **① 일괄 제어 (Batch Processing)**

하나의 액터 안에 여러 개의 조명(Point Light)이 있을 때, 모든 조명의 색상을 한꺼번에 바꾸고 싶다면 이 노드를 사용합니다.

* Get Components by Class (PointLightComponent) \-\> ForEachLoop \-\> Set Light Color

### **② 특정 기능을 가진 컴포넌트 검색**

액터에 어떤 컴포넌트가 포함되어 있는지 미리 알 수 없거나, 런타임에 추가된 컴포넌트를 찾아야 할 때 유용합니다. 예를 들어, 대미지를 입었을 때 액터 내의 모든 ParticleSystemComponent를 찾아 활성화하는 등의 처리가 가능합니다.

### **③ 특정 타입 존재 여부 확인**

배열의 Length 노드를 연결하여 0보다 큰지 확인하면, 해당 액터에 특정 타입의 컴포넌트가 존재하는지 판별하는 용도로 쓸 수 있습니다.

## **5\. 유사 노드와의 차이점**

* **GetComponentByClass**: (단수형) 조건에 맞는 **첫 번째** 컴포넌트 하나만 반환합니다. 특정 타입이 하나만 있을 것이 확실할 때 성능상 유리합니다.  
* **Get Components by Tag**: 클래스가 아니라 컴포넌트에 설정된 'Tag(태그)' 문자열을 기준으로 검색합니다.

## **6\. 사용 팁 및 주의사항**

* **캐스팅(Casting) 불필요**: 이 노드는 이미 사용자가 지정한 클래스 타입으로 배열을 반환하므로, 출력된 배열 요소를 사용할 때 별도의 Cast To 노드를 사용할 필요가 없습니다.  
* **비용 고려**: 컴포넌트가 아주 많은 액터에서 매 프레임(Tick)마다 이 노드를 호출하는 것은 성능에 좋지 않습니다. 가급적 BeginPlay에서 한 번 호출하여 변수에 저장해두고 사용하는 것을 권장합니다.  
* **자식 클래스 포함**: 예를 들어 SceneComponent를 검색 클래스로 설정하면, 이를 상속받은 StaticMesh, SpringArm, Camera 등이 모두 결과에 포함됩니다.

**요약하자면:** "이 액터 안에 있는 \*\*\[특정 종류\]\*\*의 컴포넌트들을 다 가져와서 나열해줘\!"라고 요청하는 노드입니다.
