# **Get Control Rotation 노드 완벽 이해하기**

언리얼 엔진에서 캐릭터의 움직임이나 카메라 시점을 제어할 때 가장 자주 사용되는 노드 중 하나인 **Get Control Rotation**에 대해 알아봅니다.

## **1\. 노드의 핵심 역할**

Get Control Rotation은 해당 폰(Pawn)을 소유하고 있는 \*\*컨트롤러(Controller)\*\*의 회전값을 반환합니다.

* **플레이어의 시선:** 플레이어가 마우스를 움직여 화면을 돌릴 때, 캐릭터의 몸체는 가만히 있어도 "컨트롤러"는 회전하고 있습니다. 이 값을 가져옵니다.  
* **의도의 방향:** 캐릭터가 실제 물리적으로 서 있는 방향(Actor Rotation)이 아니라, 플레이어가 **어디를 조준하거나 바라보고 있는지**에 대한 데이터를 제공합니다.

## **2\. Get Actor Rotation vs Get Control Rotation**

이 둘의 차이를 이해하는 것이 가장 중요합니다.

| 비교 항목 | Get Actor Rotation | Get Control Rotation |
| :---- | :---- | :---- |
| **대상** | 캐릭터의 물리적인 몸체 (Mesh/Capsule) | 캐릭터를 조종하는 추상적인 컨트롤러 |
| **기준** | 캐릭터의 발이 향한 방향 | 플레이어의 카메라/마우스가 향한 방향 |
| **주요 용도** | 캐릭터 전방 발사체 생성, 물리 충돌 판정 | **카메라 기준 이동**, 조준(Aiming), 시선 처리 |

## **3\. 주요 활용 사례**

### **① 카메라 방향으로 이동하기 (TPS/FPS)**

가장 흔한 사용법입니다. 플레이어가 'W'를 눌렀을 때, 캐릭터가 보는 방향으로 전진하게 만들려면 컨트롤러의 회전값이 필요합니다.

1. Get Control Rotation에서 값을 가져옵니다.  
2. Break Rotator를 통해 **Yaw(Z축)** 값만 추출합니다 (캐릭터가 하늘이나 땅으로 박히지 않게 하기 위함).  
3. Make Rotator로 다시 합친 후 Get Forward Vector를 연결합니다.  
4. 이 벡터를 Add Movement Input의 World Direction에 넣습니다.

### **② 조준 및 투사체 발사**

FPS 게임에서 총을 쏠 때, 총구의 방향보다 **카메라가 정중앙으로 바라보는 방향**으로 총알이 나가야 정확합니다. 이때 Get Control Rotation의 Forward Vector를 사용해 총알의 궤적을 결정합니다.

### **③ 컨트롤러 회전 사용 (Use Controller Rotation)**

디테일 패널이나 블루프린트 설정에서 Use Controller Rotation Yaw를 체크하면, 마우스를 돌릴 때마다 캐릭터 몸체(Actor)가 컨트롤러의 회전값에 동기화되어 함께 회전하게 됩니다.

## **4\. 주의사항**

* **Pawn 소유권:** 이 노드는 Pawn 클래스 내부에서 사용될 때 유효합니다. 만약 컨트롤러가 없는(Unpossessed) 상태에서 호출하면 기본값(0,0,0)을 반환할 수 있습니다.  
* **Pitch/Roll 제한:** 이동 로직을 짤 때는 주로 Yaw만 사용하세요. Pitch(위아래) 값을 이동 벡터에 그대로 포함하면 캐릭터가 땅 속으로 들어가려고 하거나 하늘로 날아오르려 할 수 있습니다.

## **요약**

Get Control Rotation은 \*\*"플레이어가 지금 어디를 보고 있는가?"\*\*에 대한 대답입니다. 캐릭터의 몸 방향이 아닌 **시선 방향**을 기준으로 무언가를 처리하고 싶을 때 반드시 사용해야 하는 노드입니다.
