# **언리얼 엔진 블루프린트: Get Controller 노드 이해하기**

언리얼 엔진에서 **Get Controller** 노드는 특정 \*\*폰(Pawn)\*\*이나 \*\*캐릭터(Character)\*\*를 현재 제어하고 있는 **컨트롤러(Controller)** 객체에 대한 참조를 가져오는 역할을 합니다.

## **1\. 기본 개념: 몸(Pawn)과 정신(Controller)**

언리얼 엔진의 아키텍처는 물리적인 형태와 로직을 분리합니다.

* **Pawn/Character (몸):** 월드에 배치된 모델, 애니메이션, 충돌 처리 등을 담당합니다.  
* **Controller (정신):** 플레이어의 입력이나 AI의 의사결정 로직을 담당합니다.

Get Controller는 이 "몸"에게 "지금 너를 조종하고 있는 정신이 누구니?"라고 묻는 것과 같습니다.

## **2\. 주요 역할 및 특징**

### **① 제어 주체 확인**

이 노드는 해당 폰이 **플레이어**에 의해 조종되고 있는지, 아니면 **AI**에 의해 조종되고 있는지 확인하는 시작점이 됩니다.

* 반환값은 Controller 오브젝트 참조입니다.  
* 실제 사용 시에는 Cast to PlayerController 또는 Cast to AIController를 통해 구체적인 기능을 사용합니다.

### **② 입력 및 UI 처리**

플레이어 컨트롤러는 입력(Input) 처리와 HUD/UI 표시를 담당합니다. 캐릭터 블루프린트 내에서 UI를 띄우거나 마우스 커서를 보이게 하려면 캐릭터의 컨트롤러를 가져와야 합니다.

### **③ 네트워크 (멀티플레이어)**

멀티플레이어 환경에서 특정 캐릭터가 '로컬 플레이어'인지 확인하거나, 서버 측에서 특정 캐릭터를 조종하는 클라이언트를 식별할 때 필수적입니다.

## **3\. Get Controller vs Get Player Controller**

초보자들이 가장 많이 헷갈려 하는 부분입니다.

| 구분 | Get Controller | Get Player Controller |
| :---- | :---- | :---- |
| **대상** | 특정 폰(Target)을 조종하는 컨트롤러 | 월드의 로컬 플레이어 (인덱스 기준) |
| **범위** | AI 컨트롤러와 플레이어 컨트롤러 모두 포함 | **플레이어 컨트롤러만** 포함 |
| **사용 위치** | 보통 폰/캐릭터 블루프린트 내부에서 사용 | 어디서든 사용 가능 (주로 싱글 플레이) |

* **Get Controller:** "이 캐릭터(나)를 조종하는 녀석을 가져와라." (AI일 수도 있음)  
* **Get Player Controller:** "이 게임을 하고 있는 0번 플레이어를 가져와라." (절대적 참조)

## **4\. 실전 활용 예시 (블루프린트 로직)**

1. **캐릭터 사망 시 입력 중지:**  
   * Get Controller \-\> Stop Movement 호출.  
2. **인벤토리 창 열기:**  
   * Get Controller \-\> Cast to PlayerController \-\> Create Widget (HUD 관련 로직).  
3. **AI 타겟 변경:**  
   * Get Controller \-\> Cast to AIController \-\> Blackboard 값 수정.

## **5\. 주의사항**

* **Unpossessed 상태:** 폰이 어떤 컨트롤러에 의해서도 조종되고 있지 않다면(Possess 되지 않았다면), 이 노드는 **Pending Kill** 또는 \*\*Null(None)\*\*을 반환합니다. 따라서 호출 후 Is Valid 노드로 체크하는 것이 안전합니다.  
* **서버/클라이언트:** 클라이언트에서는 자신이 조종하지 않는 다른 플레이어 캐릭터의 컨트롤러를 볼 수 없는 경우가 많습니다 (보안 및 최적화 이유).
