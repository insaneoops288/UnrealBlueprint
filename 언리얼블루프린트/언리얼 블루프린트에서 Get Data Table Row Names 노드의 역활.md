# **Get Data Table Row Names 노드 상세 가이드**

언리얼 엔진 블루프린트에서 **Get Data Table Row Names** 노드는 데이터 기반 시스템을 구축할 때 가장 기본이 되는 노드 중 하나입니다.

## **1\. 노드 개요**

이 노드는 지정된 **Data Table** 에셋을 스캔하여, 각 행의 첫 번째 열인 'Name' 항목들을 모두 수집해 **Name 타입의 배열**로 반환합니다.

## **2\. 입력 및 출력**

* **Data Table (입력)**: 읽으려는 대상 데이터 테이블을 선택합니다.  
* **Out Row Names (출력)**: 테이블에 존재하는 모든 행의 이름 리스트(Array)입니다.

## **3\. 대표적인 활용 패턴**

### **A. 모든 행 데이터 읽기 (전체 순회)**

데이터 테이블의 모든 내용을 로그로 찍거나, 게임 시작 시 메모리에 캐싱할 때 사용합니다.

1. Get Data Table Row Names 호출  
2. 반환된 배열을 ForEachLoop에 연결  
3. 루프의 Array Element를 Get Data Table Row 노드의 Row Name 핀에 연결  
4. 해당 행의 구조체(Struct) 데이터를 처리

### **B. 무작위 행 가져오기 (가챠/랜덤 드롭)**

1. Get Data Table Row Names 호출  
2. 출력 배열에서 Last Index 또는 Length 확인  
3. Random Integer in Range (0 \~ Last Index)로 무작위 인덱스 생성  
4. Get (Copy) 노드로 해당 인덱스의 이름을 추출  
5. 추출된 이름으로 실제 데이터 로드

### **C. 특정 데이터 존재 여부 확인**

배열의 Contains 노드를 사용하여 특정 이름의 행이 테이블에 존재하는지 프로그래밍적으로 빠르게 확인할 수 있습니다.

## **4\. 주의사항**

* **성능**: 데이터 테이블의 행이 수만 개 정도로 매우 많을 경우, 매 프레임(Tick)마다 이 노드를 호출하는 것은 성능에 좋지 않습니다. 가급적 BeginPlay에서 한 번 호출하여 변수에 저장해두고 사용하는 것을 권장합니다.  
* **빈 테이블**: 데이터 테이블이 비어있으면 빈 배열을 반환하므로, 이후 로직에서 Length가 0인지 체크하는 방어 로직이 있으면 좋습니다.

## **5\. 요약**

이 노드는 데이터 테이블의 \*\*"목차"\*\*를 가져오는 노드라고 이해하면 쉽습니다. 실제 내용을 보기 전에 어떤 페이지(행)들이 있는지 확인하는 용도로 사용됩니다.
