# **Get Data Table Row 노드 완벽 가이드**

언리얼 엔진의 **Get Data Table Row** 노드는 데이터 중심(Data-Driven) 설계를 가능하게 하는 핵심 노드입니다. 이 노드를 통해 외부 데이터(CSV, Excel 등)에서 가져온 테이블 정보를 게임 런타임 중에 자유롭게 호출할 수 있습니다.

## **1\. 노드의 주요 역할**

이 노드의 본질적인 역할은 \*\*"특정 데이터 테이블에서 내가 원하는 행(Row)의 데이터를 구조체(Struct) 형태로 추출하는 것"\*\*입니다.

예를 들어, 수백 개의 아이템 정보가 담긴 테이블에서 Potion\_01이라는 이름의 행을 찾아 그 아이템의 가격, 설명, 아이콘 정보를 가져올 때 사용합니다.

## **2\. 입력 및 출력 핀 설명**

### **입력 (Input)**

* **Data Table**: 참조할 데이터 테이블 에셋을 선택합니다.  
* **Row Name**: 찾고자 하는 행의 고유 이름(Name)을 입력하거나 변수로 연결합니다.

### **출력 (Output)**

* **Exec (White Arrows)**:  
  * **Row Found**: 해당 Row Name이 테이블에 존재하여 데이터를 성공적으로 가져왔을 때 실행됩니다.  
  * **Row Not Found**: 해당 이름의 행이 없을 때 실행됩니다. (예외 처리 가능)  
* **Out Row**: 테이블에서 가져온 실제 데이터입니다. 이 핀은 해당 데이터 테이블이 사용하는 **구조체(Struct)** 타입으로 자동 변경됩니다.

## **3\. 주요 특징 및 장점**

1. **데이터 기반 설계**: 하드코딩 대신 외부 데이터를 사용하므로 기획자가 엑셀에서 수치만 수정해도 게임에 즉시 반영됩니다.  
2. **구조체 자동 대응**: 선택한 데이터 테이블의 행 구조(Row Structure)에 따라 출력 핀의 타입이 유연하게 결정됩니다.  
3. **안전성**: 'Row Not Found' 실행 핀을 통해 잘못된 이름을 입력했을 때의 오류를 방지할 수 있습니다.

## **4\. 활용 예시**

### **아이템 시스템**

아이템 마스터 테이블에서 아이템 ID를 검색하여 해당 아이템의 공격력, 방어력, 메시(Mesh) 정보를 스폰 시점에 적용합니다.

### **다국어 대사 처리**

대화 시스템에서 현재 대화의 ID를 Row Name으로 입력하여 해당 언어에 맞는 텍스트 데이터를 화면에 출력합니다.

### **레벨 밸런싱**

몬스터의 등급(Common, Elite, Boss)에 따라 체력과 이동 속도 등을 데이터 테이블에서 조회하여 설정합니다.

## **5\. 주의사항 및 팁**

* **구조체 일치**: Get Data Table Row를 사용하려면 먼저 해당 테이블이 어떤 **Row Structure**를 사용하는지 알아야 합니다.  
* **성능**: 매 프레임(Tick)마다 호출하는 것보다는 필요한 시점(BeginPlay, 아이템 습득 시 등)에 한 번 호출하여 변수에 저장해두는 것이 성능상 유리합니다.  
* **분할(Break)**: 출력된 Out Row 핀에서 오른쪽 클릭 후 \*\*'Break \[구조체이름\]'\*\*을 선택하면 구조체 내부의 개별 변수(가격, 이름 등)를 따로 꺼내 쓸 수 있습니다.
