# **언리얼 엔진: Get Delta Time 노드의 역할과 활용**

언리얼 엔진에서 \*\*Delta Time(델타 타임)\*\*은 "이전 프레임이 끝나고 현재 프레임이 시작되기까지 걸린 시간"을 의미합니다. 단위는 초(Second)입니다.

## **1\. 왜 Delta Time이 필요한가? (프레임 독립성)**

게임은 실행되는 컴퓨터의 성능에 따라 초당 프레임 수(FPS)가 다릅니다.

* **컴퓨터 A:** 120 FPS (매우 빠름)  
* **컴퓨터 B:** 30 FPS (느림)

만약 Event Tick에서 사물을 프레임당 10cm씩 이동하도록 설정한다면:

* **컴퓨터 A:** 1초에 ![][image1] 이동  
* **컴퓨터 B:** 1초에 ![][image2] 이동

성능이 좋은 컴퓨터를 쓰는 플레이어가 더 빨리 움직이는 불공평한 상황이 발생합니다. 이를 해결하기 위해 **Delta Time**을 곱해줍니다.

## **2\. 수학적 원리**

이동 공식을 다음과 같이 변경합니다:

이동 거리 \= 속도(초당 이동량) \* Delta Time

결과적으로 어떤 성능의 컴퓨터에서도 **1초에 동일한 거리**를 이동하게 됩니다.

## **3\. 블루프린트에서의 사용법**

### **A. 일반적인 이동/회전**

Event Tick 노드에서 나오는 Delta Seconds 핀을 사용하거나, Get World Delta Seconds 노드를 호출하여 사용합니다.

1. **변수 선언:** MoveSpeed (예: 500.0)  
2. **연산:** MoveSpeed ![][image8] Delta Time  
3. **적용:** Add Local Offset 또는 Set Actor Location 등의 입력값으로 연결

### **B. 보간 (Interpolation)**

VInterp To 또는 RInterp To 노드를 사용할 때 Delta Time 입력 핀이 있습니다. 여기에 이 노드를 연결해야 부드러운 애니메이션이 프레임 속도에 영향을 받지 않고 일정하게 재생됩니다.

## **4\. 요약**

* **역할:** 프레임 간의 시간 간격을 반환.  
* **목적:** 하드웨어 성능(FPS)에 관계없이 게임 내 움직임이나 로직을 \*\*일정한 속도(시간 기반)\*\*로 실행하기 위함.  
* **공식:** 원하는 수치(초당) \* Delta Time \= 현재 프레임에 적용할 수치

**주의:** Event Tick이 아닌 곳(예: Begin Play나 일반 함수 내부)에서 Get World Delta Seconds를 사용할 때는 주의가 필요합니다. 해당 시점의 프레임 시간을 가져오기 때문에, 지속적인 변화를 줄 때는 반드시 매 프레임 호출되는 로직 내에서 사용해야 합니다.
