# **Get Display Name 노드 이해하기**

Get Display Name 노드는 언리얼 엔진 블루프린트에서 오브젝트의 이름을 식별할 때 가장 흔히 사용되는 노드 중 하나입니다.

## **1\. 주요 역할**

이 노드는 입력으로 받은 \*\*오브젝트(Object)\*\*의 "표시용 이름"을 **String(문자열)** 타입으로 반환합니다.

* **대상:** 액터(Actor), 스태틱 메시 컴포넌트, 변수, 데이터 에셋 등 모든 UObject 기반 클래스.  
* **출력:** 에디터의 월드 아웃라이너(World Outliner)나 디테일 패널에서 보이는 것과 유사한 이름을 반환합니다.

## **2\. 주요 사용 사례**

### **① 디버깅 (Debugging)**

가장 많이 사용되는 용도입니다. 특정 이벤트가 발생했을 때 어떤 액터에서 발생했는지 로그를 남길 때 유용합니다.

* 예: Print String 노드와 연결하여 "Collision detected with: \[Display Name\]"과 같이 출력.

### **② 개발자 도구 및 도구 제작**

에디터 내에서 작동하는 툴을 만들 때, 선택한 오브젝트의 이름을 기반으로 리스트를 만들거나 분류할 때 사용합니다.

## **3\. 에디터 vs 패키징 빌드 차이점 (중요\!)**

Get Display Name은 실행 환경에 따라 결과값이 달라질 수 있음을 반드시 인지해야 합니다.

| 환경 | 결과값 예시 |
| :---- | :---- |
| **언리얼 에디터** | BP\_Player\_C\_1, MyStone\_2 (아웃라이너에 보이는 이름과 유사) |
| **패키징 빌드 (실제 게임)** | BP\_Player\_C, MyStone (숫자 접미사가 생략되거나 클래스명 위주로 나옴) |

**경고:** 패키징된 게임 내에서는 에디터에서 보이던 고유 번호(Instance ID)가 사라지거나 이름 형식이 바뀔 수 있으므로, **이 노드에서 나온 문자열을 직접 비교(==)하여 게임 로직(문체 판별 등)을 짜는 것은 권장되지 않습니다.**

## **4\. Get Object Name vs Get Display Name**

* **Get Object Name:** 오브젝트의 내부 고유 이름(FName 기반)을 가져옵니다. 메모리 상의 실제 주소와 연관된 더 엄격한 이름을 반환합니다.  
* **Get Display Name:** 사용자가 읽기 편한 형태(String)로 변환된 이름을 가져옵니다.

## **5\. 결론 및 권장사항**

* **디버깅용**으로만 사용하세요.  
* 게임 플레이 로직(예: 이름이 'Boss'면 문을 연다)을 짤 때는 이 노드 대신 \*\*태그(Actor Tag)\*\*나 **인터페이스(Interface)**, 또는 **클래스 비교**를 사용하는 것이 훨씬 안전하고 전문적인 방법입니다.
