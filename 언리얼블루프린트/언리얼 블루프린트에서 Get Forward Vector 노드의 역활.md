# **언리얼 엔진 블루프린트: Get Forward Vector 노드 이해하기**

Get Forward Vector 노드는 특정 대상(액터 또는 컴포넌트)의 회전값을 기반으로, 해당 대상의 \*\*정면 방향을 가리키는 단위 벡터(Unit Vector)\*\*를 반환합니다.

## **1\. 주요 특징**

* **단위 벡터(Unit Vector):** 출력되는 벡터의 길이는 항상 **1.0**입니다. 이는 방향 정보만을 순수하게 전달하기 위함입니다.  
* **월드 좌표계(World Space):** 대상이 월드 상에서 어디를 회전해서 바라보고 있는지에 대한 방향을 월드 좌표 기준으로 알려줍니다.  
* **X축 기준:** 언리얼 엔진의 표준 좌표계에서 '앞(Forward)'은 기본적으로 **X축**입니다. 이 노드는 대상의 로컬 X축이 월드의 어느 방향을 향하고 있는지를 계산합니다.

## **2\. 작동 원리**

만약 캐릭터가 동쪽을 바라보고 있다면 Forward Vector는 (1, 0, 0)에 가까운 값을 가집니다. 캐릭터가 하늘을 보고 있다면 (0, 0, 1)에 가까운 값을 반환하게 됩니다.

수학적으로는 대상의 **회전(Rotation)** 값을 입력받아 삼각함수를 통해 방향 벡터를 추출하는 과정을 거칩니다.

## **3\. 주요 활용 사례**

### **① 캐릭터 이동 (Movement)**

가장 대표적인 사용처입니다. 플레이어가 'W' 키를 눌렀을 때, 캐릭터가 바라보는 방향으로 움직이게 하려면 이 노드가 필요합니다.

* **로직:** Get Control Rotation \-\> Get Forward Vector \-\> Add Movement Input의 'World Direction'에 연결.

### **② 발사체 생성 (Projectile Spawning)**

총을 쏘거나 마법 화살을 날릴 때, 어느 방향으로 날아가야 할지 결정할 때 사용합니다.

* **로직:** Spawn Actor from Class로 총알을 생성할 때, 총구(Muzzle)의 Forward Vector를 가져와 총알의 속도(Velocity)나 힘(Impulse)의 방향으로 설정합니다.

### **③ 라인 트레이스 (Line Tracing / Raycasting)**

캐릭터 정면에 상호작용 가능한 물체가 있는지 감지할 때 사용합니다.

* **로직:** Line Trace By Channel 노드의 시작점은 캐릭터 위치로, 끝점은 (캐릭터 위치 \+ (Forward Vector \* 사거리))로 계산하여 직선을 긋습니다.

## **4\. 다른 방향 노드와의 비교**

| 노드 이름 | 방향 | 기준 축 |
| :---- | :---- | :---- |
| **Get Forward Vector** | 앞 (Forward) | X축 |
| **Get Right Vector** | 오른쪽 (Right) | Y축 |
| **Get Up Vector** | 위 (Up) | Z축 |

## **5\. 실전 팁**

* **벡터 곱셈:** Forward Vector는 길이가 1이므로, 여기에 특정 숫자(Float)를 곱하면 원하는 거리만큼의 벡터를 만들 수 있습니다. (예: Forward Vector \* 500 \= 정면으로 500 유닛만큼의 거리)  
* **액터 vs 컴포넌트:** Get Actor Forward Vector는 액터 전체의 방향을, Get Forward Vector (컴포넌트 타겟)는 특정 컴포넌트(예: 카메라, 메쉬)의 방향을 가져옵니다. 캐릭터는 정면을 보더라도 카메라는 옆을 볼 수 있으므로 상황에 맞게 선택해야 합니다.

**요약:** Get Forward Vector는 \*\*"내가 지금 어디를 쳐다보고 있는가?"\*\*에 대한 답을 수학적 방향(벡터)으로 제공하는 노드입니다.
