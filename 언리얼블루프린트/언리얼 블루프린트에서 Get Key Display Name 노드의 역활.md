# **Get Key Display Name 노드 상세 가이드**

언리얼 엔진에서 Get Key Display Name 노드는 입력 시스템의 특정 키 데이터를 인간이 읽을 수 있는 친숙한 이름으로 변환하는 데 사용됩니다.

## **1\. 노드의 주요 역할**

이 노드는 Key 구조체를 입력받아 해당 키의 이름을 **로컬라이즈(현지화)된 텍스트** 형식으로 반환합니다.

* 예: LeftMouseButton \-\> **"Left Mouse Button"**  
* 예: Gamepad\_Special\_Right \-\> **"Menu"** (또는 컨트롤러 종류에 따른 적절한 이름)

## **2\. 입력 및 출력**

* **Input (Key):** 정보를 얻고자 하는 키 구조체입니다. (예: 키보드, 마우스, 게임패드 버튼 등)  
* **Output (Return Value):** 해당 키의 '표시용 이름'을 담은 **Text** 값입니다.

## **3\. 주요 활용 사례**

### **① UI에서 조작법 안내**

게임 내 튜토리얼이나 하단 안내 창에서 "공격을 하려면 \[Left Mouse Button\]을 누르세요"와 같은 문구를 동적으로 생성할 때 유용합니다.

* 플레이어가 키 바인딩을 변경하더라도, 변경된 키 값을 가져와 이 노드를 통과시키면 현재 바인딩된 키의 이름을 정확하게 출력할 수 있습니다.

### **② 키 바인딩 설정 메뉴**

옵션 메뉴에서 사용자가 어떤 키를 할당했는지 리스트로 보여줄 때 사용합니다. 내부적인 코드 네임(예: Enter) 대신 사용자에게 친숙한 이름(예: Enter)을 보여주기 위해 필수적입니다.

## **4\. Get Key Name vs Get Key Display Name**

두 노드는 비슷해 보이지만 중요한 차이점이 있습니다.

| 특징 | Get Key Name | Get Key Display Name |
| :---- | :---- | :---- |
| **반환 타입** | Name (식별자) | **Text (사용자 인터페이스용)** |
| **가독성** | 프로그래밍용 (공백 없음) | **사용자용 (공백 및 대문자 구분)** |
| **현지화** | 지원 안 함 | **지원 함 (다국어 대응 가능)** |
| **예시 결과** | LeftControl | **"Left Ctrl"** 또는 **"왼쪽 컨트롤"** |

## **5\. 실전 팁**

* **동적 텍스트 결합:** Format Text 노드와 함께 사용하여 "\[Key\]를 눌러 문을 여세요" 같은 문장을 만들면 매우 효과적입니다.  
* **아이콘 변환:** 실제 상용 게임에서는 텍스트 대신 버튼 아이콘(이미지)을 보여주는 경우가 많습니다. 이때는 이 노드의 결과값(Text)이나 Key 자체를 인덱스로 사용하여 '공통 UI 테이블'에서 해당 아이콘을 찾아오도록 로직을 구성합니다.

**요약:** Get Key Display Name은 기계적인 키 값을 **사용자에게 보여줄 예쁜 이름**으로 바꿔주는 필수적인 UI/UX용 노드입니다.
