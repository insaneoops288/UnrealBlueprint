# **Get Last Input Vector 노드 가이드**

언리얼 엔진의 CharacterMovementComponent 또는 Pawn에서 사용할 수 있는 **Get Last Input Vector** 노드는 유저가 컨트롤러(키보드, 조이스틱 등)를 통해 캐릭터에게 전달한 \*\*'마지막 이동 입력 방향'\*\*을 반환합니다.

## **1\. 노드의 핵심 역할**

이 노드는 캐릭터가 실제로 움직이고 있는지와 상관없이, \*\*플레이어가 어느 방향으로 가려고 하는지(의도)\*\*를 벡터 값으로 알려줍니다.

* **반환 값:** Vector 타입 (X, Y, Z 방향 성분)  
* **범위:** 보통 0에서 1 사이의 정규화된 방향 벡터를 반환합니다 (입력값의 세기에 따라 달라질 수 있음).

## **2\. Get Velocity(속도) 노드와의 결정적인 차이점**

이 부분을 이해하는 것이 가장 중요합니다.

| 비교 항목 | Get Last Input Vector | Get Velocity |
| :---- | :---- | :---- |
| **의미** | 플레이어의 **이동 의도** | 캐릭터의 **실제 물리적 속도** |
| **벽에 막혔을 때** | 유저가 키를 누르고 있다면 **방향 유지** | 물리적으로 멈춰있으므로 **0, 0, 0** |
| **공중 상태** | 입력 방향을 즉시 반환 | 중력 및 관성이 포함된 물리 속도 반환 |
| **주요 용도** | 애니메이션 블루프린트, 회전 처리 | 물리 연산, 낙하 대미지 계산 |

**예시:** 캐릭터가 벽을 향해 계속 'W' 키를 누르고 있다면?

* Get Velocity는 벽에 막혀 움직이지 못하므로 0을 반환합니다.  
* Get Last Input Vector는 유저가 앞으로 가려 하므로 (1, 0, 0\) 같은 전방 벡터를 계속 반환합니다.

## **3\. 주요 활용 사례**

### **① 캐릭터 회전 제어 (Orient Rotation to Movement)**

캐릭터가 이동하는 방향을 바라보게 만들 때 사용합니다. Get Velocity를 사용하면 캐릭터가 멈추는 순간 속도가 0이 되어 캐릭터 회전값이 기본값으로 돌아가는 문제가 생길 수 있지만, Input Vector를 활용하면 마지막 입력 방향을 기준으로 회전을 부드럽게 유지할 수 있습니다.

### **② 애니메이션 블루프린트 (Blend Spaces)**

캐릭터가 어느 방향으로 가려고 하는지에 따라 애니메이션을 블렌딩할 때 사용합니다. 예를 들어, 걷기 시작하는 순간(속도가 아직 붙지 않았을 때)에도 입력 벡터를 알면 즉시 올바른 방향의 출발 애니메이션을 재생할 수 있습니다.

### **③ 대시(Dash) 또는 스킬 방향 결정**

플레이어가 특정 스킬을 쓸 때 "현재 누르고 있는 방향으로 돌진"하게 만들고 싶다면, Get Last Input Vector를 가져와서 그 방향에 힘(Impulse)을 가하면 됩니다.

## **4\. 블루프린트 사용 시 팁**

1. **Is Falling 체크:** 캐릭터가 공중에 떠 있을 때 유저가 컨트롤을 하더라도 실제 물리적 이동은 적을 수 있습니다. 이때 Get Last Input Vector를 사용하면 공중에서도 유저가 조준하는 방향을 정확히 파악해 캐릭터의 시선을 처리할 수 있습니다.  
2. **Vector Length 확인:** Vector Length 노드를 연결하여 값이 0보다 큰지 확인하면, 현재 유저가 이동 키를 "누르고 있는지 아닌지"를 판별하는 조건문으로 쓸 수 있습니다.  
3. **Consume Input Vector:** Add Movement Input으로 입력된 벡터는 프레임마다 소비됩니다. Get Last Input Vector는 이번 프레임에 사용되기 직전 혹은 직후의 누적된 값을 가져오는 방식입니다.

## **5\. 요약**

Get Last Input Vector는 \*\*"플레이어가 지금 가고 싶어 하는 방향이 어디인가?"\*\*라는 질문에 답해주는 노드입니다. 실제 물리 현상(벽, 마찰, 무게)에 영향을 받지 않는 순수한 유저의 명령을 읽고 싶을 때 사용하세요.
