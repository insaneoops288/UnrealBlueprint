# **Get Last Input Vector 노드 완벽 이해하기**

언리얼 엔진의 Pawn 또는 Character 클래스에서 사용할 수 있는 **Get Last Input Vector** 노드는 플레이어의 "이동 의지"를 벡터 값으로 반환하는 함수입니다.

## **1\. 핵심 역할**

이 노드는 Add Movement Input 노드를 통해 전달된 **입력값들의 합계**를 반환합니다.

* **입력의 방향성:** 조이스틱이나 WASD 키보드 입력을 통해 캐릭터가 가고자 하는 방향을 0에서 1 사이의 크기를 가진 벡터로 나타냅니다.  
* **프레임 단위:** 마지막으로 처리된 프레임에서의 입력 값을 가져오며, 이동 컴포넌트(Character Movement Component)가 이 값을 소비하기 전의 상태를 참조합니다.

## **2\. Get Velocity vs Get Last Input Vector**

가장 흔히 혼동하는 두 노드의 차이점은 다음과 같습니다.

| 특징 | Get Velocity (속도) | Get Last Input Vector (입력 벡터) |
| :---- | :---- | :---- |
| **의미** | 캐릭터가 실제로 움직이고 있는 물리적인 속도 | 플레이어가 움직이려고 입력한 방향(의도) |
| **벽에 막혔을 때** | 값은 **0**에 가깝게 나옴 (움직이지 못하므로) | 값은 **1**(또는 입력값)로 유지됨 (계속 가려고 하므로) |
| **공중에 떴을 때** | 중력에 의한 낙하 속도 등이 포함됨 | 순수하게 키보드/스틱 입력값만 포함됨 |

## **3\. 주요 활용 사례**

### **① 캐릭터의 회전 제어 (Orient Rotation to Movement)**

캐릭터가 실제로 움직이기 전에도, 입력이 들어오는 즉시 해당 방향을 바라보게 하고 싶을 때 사용합니다. Get Velocity를 사용하면 벽에 막혔을 때 캐릭터가 회전을 멈추거나 떨리는 현상이 발생할 수 있지만, Get Last Input Vector를 사용하면 벽을 향해 계속 입력 중일 때 캐릭터가 그 방향을 안정적으로 응시합니다.

### **② 애니메이션 블루프린트 (ABP)**

* **Leaning (기울기):** 캐릭터가 갑자기 방향을 바꿀 때 몸을 기울이는 애니메이션을 구현할 때, 실제 가속도보다 입력 벡터를 참조하는 것이 더 즉각적인 반응을 보여줍니다.  
* **Start/Stop 애니메이션:** 이동 입력을 멈추는 순간을 정확히 감지하여 정지 애니메이션으로 전환할 때 유용합니다.

### **③ AI 판단 로직**

AI가 플레이어의 이동 방향을 예측하여 사격하거나 길을 막아야 할 때, 현재 속도보다 플레이어가 누르고 있는 입력 방향을 체크하는 것이 더 정확한 예측을 가능하게 합니다.

## **4\. 주의사항**

* Add Movement Input 노드를 사용하지 않고 Set Actor Location 등으로 강제 이동시키는 경우에는 이 노드에서 값을 얻을 수 없습니다.  
* 이동 컴포넌트가 활성화되어 있어야 하며, Consume Movement Input이 호출된 직후에는 값이 리셋될 수 있으므로 호출 순서(Tick 내의 타이밍)가 중요할 수 있습니다.

## **요약**

**Get Last Input Vector**는 "캐릭터가 지금 어디로 가고 있는가?"가 아니라 \*\*"사용자가 캐릭터를 어디로 보내려고 하는가?"\*\*를 묻는 노드입니다. 물리적인 충돌이나 마찰에 구애받지 않는 순수한 입력 데이터를 원할 때 선택하세요.
