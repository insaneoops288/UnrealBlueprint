# **Get Local Player Subsystem (Enhanced Input) 노드 가이드**

언리얼 엔진 5(UE5)에서 **향상된 입력(Enhanced Input)** 시스템을 사용할 때, 이 노드는 특정 로컬 플레이어의 입력 상태를 제어하는 '컨트롤 타워' 역할을 합니다.

## **1\. 핵심 역할: 입력 매핑 컨텍스트(IMC) 관리**

이 노드의 가장 주된 목적은 \*\*입력 매핑 컨텍스트(Input Mapping Context, IMC)\*\*를 플레이어에게 \*\*추가(Add)\*\*하거나 \*\*제거(Remove)\*\*하는 것입니다.

* **동적 입력 전환:** 게임 플레이 중 캐릭터가 차에 타거나, 메뉴 창을 열거나, 특정 아이템을 장착했을 때 입력 방식이 바뀌어야 합니다. 이때 이 서브시스템을 통해 새로운 IMC를 주입합니다.  
* **우선순위 제어:** 여러 개의 IMC가 동시에 활성화되어 있을 때, 어떤 입력이 먼저 처리될지 결정하는 우선순위(Priority)를 설정할 수 있습니다.

## **2\. 왜 '로컬 플레이어' 서브시스템인가?**

언리얼 엔진의 서브시스템(Subsystem)은 특정 수명 주기를 갖는 싱글톤 스타일의 객체입니다.

* **Local Player 기준:** 이 서브시스템은 각 로컬 플레이어(화면 하나를 사용하는 플레이어)마다 개별적으로 존재합니다.  
* **독립성:** 1번 플레이어가 사용하는 입력 설정이 2번 플레이어(로컬 멀티플레이 시)에게 영향을 주지 않도록 철저히 분리되어 관리됩니다.

## **3\. 주요 사용 함수 (연결 노드)**

이 노드로부터 선을 끌어내어 주로 사용하는 함수들은 다음과 같습니다.

| 함수명 | 역할 |
| :---- | :---- |
| **Add Mapping Context** | 특정 IMC를 플레이어에게 적용합니다. (가장 많이 사용) |
| **Remove Mapping Context** | 적용된 특정 IMC를 제거하여 해당 입력을 중단합니다. |
| **Clear All Mappings** | 현재 적용된 모든 입력 매핑을 한 번에 초기화합니다. |
| **Is Mapping Context Applied** | 특정 IMC가 현재 활성화 상태인지 확인합니다. |

## **4\. 블루프린트 구현 흐름 (Standard Workflow)**

일반적으로 게임이 시작될 때(BeginPlay) 입력을 활성화하기 위해 다음과 같은 순서로 노드를 구성합니다.

1. **Pawn/Character BeginPlay:** 캐릭터가 생성될 때 시작.  
2. **Get Controller:** 현재 캐릭터를 조종하는 컨트롤러를 가져옵니다.  
3. **Cast to Player Controller:** 로컬 플레이어 컨트롤러인지 확인합니다.  
4. **Get Local Player Subsystem (Enhanced Input):** 해당 컨트롤러의 로컬 플레이어로부터 서브시스템을 참조합니다.  
5. **Add Mapping Context:** 준비해둔 IMC\_Default 같은 자산을 연결하고 우선순위를 지정합니다.

## **5\. 주의사항**

* **Is Valid 체크:** 서브시스템을 호출하기 전에 컨트롤러나 로컬 플레이어가 유효한 상태인지 확인하는 것이 안전합니다.  
* **서버 vs 클라이언트:** 이 서브시스템은 **로컬 전용**입니다. 서버 측 로직에서 입력을 직접 수정하는 것이 아니라, 클라이언트가 자신의 로컬 플레이어 서브시스템을 통해 입력을 처리하고 그 결과를 서버로 보냅니다.

## **요약**

Get Local Player Subsystem (Enhanced Input)은 \*\*"이 플레이어가 지금 어떤 키 세팅(IMC)을 사용할지 결정할 권한을 가진 객체를 가져와라"\*\*는 명령과 같습니다. UE5에서 캐릭터의 입력을 구현하기 위한 필수적인 첫 단추입니다.
