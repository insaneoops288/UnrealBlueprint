# **Get Locomotion System 노드의 역할과 기능**

Get Locomotion System 노드는 주로 **Advanced Locomotion System (ALS)** V4 버전이나 이를 기반으로 한 프로젝트에서 볼 수 있는 노드입니다. 이 노드는 캐릭터의 이동(Locomotion)과 관련된 모든 상태 정보와 함수에 접근할 수 있게 해주는 **인터페이스(Interface)** 또는 **컴포넌트 게터(Getter)** 역할을 합니다.

### **1\. 핵심 역할: 상태 정보의 관문**

캐릭터가 단순히 이동하는 것을 넘어, 현재 어떤 방식으로 걷고 있는지, 어떤 자세를 취하고 있는지에 대한 데이터를 가져오거나 수정할 때 사용합니다.

* **상태 확인**: 캐릭터가 현재 걷고 있는지(Walking), 뛰고 있는지(Running), 아니면 전력 질주 중인지(Sprinting) 확인합니다.  
* **자세 제어**: 서 있는 상태(Standing)인지, 앉아 있는 상태(Crouching)인지 구분합니다.  
* **오버레이 상태**: 캐릭터가 무기를 들고 있는지, 가방을 메고 있는지 등 상체 애니메이션의 레이어를 결정하는 정보를 가져옵니다.

### **2\. 주요 반환 데이터 (ALS 기준)**

이 노드를 통해 연결된 객체(보통 BPI\_LocomotionSystem 인터페이스)는 다음과 같은 변수와 열거형(Enum)에 접근하게 해줍니다.

| 변수/상태명 | 설명 |
| :---- | :---- |
| **Gait** | 이동 속도 단계를 나타냅니다 (Walk, Run, Sprint). |
| **Stance** | 캐릭터의 자세를 나타냅니다 (Standing, Crouching). |
| **Rotation Mode** | 캐릭터가 바라보는 방향 로직을 결정합니다 (Velocity Direction, Looking Direction). |
| **Overlay State** | 특정 애니메이션 포즈를 덮어씌울 때 사용합니다 (Default, Rifle, Torch 등). |
| **Movement State** | 현재 이동 중인지, 공중에 떠 있는지 등의 물리적 상태를 나타냅니다. |

### **3\. 사용 예시 (Blueprint)**

1. **속도 제한 구현**: Get Locomotion System → Set Gait를 호출하여 특정 상황(예: 조준 시)에서 강제로 'Walk' 상태로 고정할 수 있습니다.  
2. **애니메이션 동기화**: 애니메이션 블루프린트(AnimBP) 내에서 캐릭터의 실제 로코모션 상태를 참조하여, 그에 맞는 애니메이션 데이터(블렌드 스페이스 등)를 출력할 때 사용합니다.  
3. **UI 업데이트**: 현재 캐릭터의 자세(서기/앉기) 아이콘을 화면에 표시하기 위해 이 노드에서 데이터를 가져옵니다.

### **4\. 주의사항**

* **커스텀 플러그인 확인**: 만약 본인의 프로젝트가 ALS가 아님에도 이 노드가 있다면, 팀 내에서 제작한 **Locomotion Component**가 추가되어 있거나 해당 인터페이스가 캐릭터에 구현(Implemented)되어 있는지 확인해야 합니다.  
* **인터페이스 구현**: 이 노드를 사용하려면 해당 액터(Actor)가 반드시 Locomotion 관련 인터페이스를 상속받고 있어야 하며, 그렇지 않으면 None을 반환하여 오류가 발생할 수 있습니다.

### **요약**

Get Locomotion System은 캐릭터의 \*\*"어떻게 움직이는가"\*\*에 대한 모든 복합적인 정보를 한곳에서 관리하고 외부(다른 블루프린트나 UI)에서 쉽게 접근할 수 있도록 도와주는 **중간 다리** 역할을 합니다.
