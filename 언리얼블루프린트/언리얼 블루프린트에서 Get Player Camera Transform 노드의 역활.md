# **Get Player Camera Transform 노드 상세 가이드**

Get Player Camera Transform 노드는 게임 월드 내에서 플레이어의 시점(카메라)이 어디에 있고, 어느 방향을 향하고 있는지를 파악하는 데 가장 기본이 되는 노드입니다.

## **1\. 노드의 주요 역할**

이 노드는 지정된 플레이어 인덱스(기본값 0)의 **활성화된 카메라**를 찾아서 그 카메라의 **월드 트랜스포름(World Transform)** 값을 반환합니다.

* **Location (위치):** 카메라의 월드 좌표값 (XYZ).  
* **Rotation (회전):** 카메라가 바라보는 방향 (Roll, Pitch, Yaw).  
* **Scale (크기):** 카메라의 크기값 (일반적으로 1,1,1).

## **2\. 주요 입력 및 출력**

* **Input (Player Index):** 어떤 플레이어의 카메라 정보를 가져올지 결정합니다. 싱글 플레이어 게임에서는 보통 0을 사용합니다.  
* **Output (Return Value):** 카메라의 전체 Transform 구조체를 반환합니다. 필요한 경우 이 핀에서 마우스 오른쪽 버튼을 클릭해 \*\*'구조체 핀 분할(Split Struct Pin)'\*\*을 선택하면 위치, 회전, 스케일을 각각 따로 사용할 수 있습니다.

## **3\. 실무 활용 사례**

### **① 라인 트레이스 (Line Trace / 레이캐스팅)**

FPS나 TPS 게임에서 총을 쏘거나 사물과 상호작용할 때 필수적입니다.

* **시작 지점:** 카메라의 Location.  
* **끝 지점:** Location \+ (카메라의 Forward Vector \* 사거리).  
* 이를 통해 화면 정중앙(조준점)이 가리키는 물체를 정확히 판별할 수 있습니다.

### **② 발사체 또는 이펙트 스폰**

플레이어의 시선 방향으로 수류탄을 던지거나 마법 화살을 발사할 때 사용합니다.

* Spawn Actor from Class 노드의 Spawn Transform에 이 노드의 값을 연결하면 플레이어 눈 위치에서 발사체가 나갑니다.

### **③ 카메라 기준 좌표 계산**

특정 UI 요소를 카메라 앞 1미터 지점에 배치하고 싶을 때, 카메라의 트랜스포름을 기준으로 상대적인 위치를 계산할 수 있습니다.

## **4\. 유사한 노드와 차이점**

| 노드 이름 | 특징 |
| :---- | :---- |
| **Get Player Camera Transform** | 가장 간편하게 카메라의 위치와 회전값을 가져옴. |
| **Get Player Camera Manager** | 카메라 매니저 객체 자체를 가져옴. 카메라 페이드, 셰이크, FOV 변경 등 더 복잡한 제어에 사용. |
| **Get Control Rotation** | 카메라 방향이 아닌 '폰(Pawn)의 컨트롤러'가 회전하는 방향을 가져옴 (캐릭터가 보는 방향과 일치하지 않을 수 있음). |

## **5\. 팁**

* **트랜스포름 분할:** 노드의 출력 핀에서 마우스 우클릭 후 Split Struct Pin을 누르면 Location과 Rotation 핀이 따로 나와서 코드가 훨씬 깔끔해집니다.  
* **멀티플레이어:** 서버/클라이언트 환경에 따라 각 플레이어의 인덱스를 정확히 지정해야 의도한 카메라 정보를 얻을 수 있습니다.
