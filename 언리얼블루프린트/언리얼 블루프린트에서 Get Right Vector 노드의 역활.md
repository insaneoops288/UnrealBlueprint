# **Get Right Vector 노드 이해하기**

언리얼 엔진에서 **Get Right Vector** 노드는 액터(Actor)나 컴포넌트의 회전값(Rotation)을 바탕으로, 해당 물체의 로컬 좌표계 기준 \*\*오른쪽(Right)\*\*이 월드 좌표계에서 어느 방향인지를 계산해주는 노드입니다.

## **1\. 핵심 개념**

* **방향성**: 물체의 오른쪽 방향을 가리킵니다.  
* **단위 벡터**: 결과값은 항상 길이가 **1**인 정규화된(Normalized) 벡터입니다.  
* **좌표계**: 언리얼 엔진은 오른손 좌표계(공식적으로는 왼손 좌표계를 기반으로 한 독자적 방식이지만, 축 기준으로는 X: 앞, Y: 오른쪽, Z: 위)를 사용합니다. 따라서 로컬 Y축 방향이 곧 Right Vector입니다.

## **2\. 주요 특징**

* **회전 연동**: 액터가 회전하면 Get Right Vector 값도 그에 맞춰 변합니다. 예를 들어 캐릭터가 북쪽을 보면 오른쪽 벡터는 동쪽을 가리키고, 캐릭터가 서쪽을 보면 오른쪽 벡터는 북쪽을 가리킵니다.  
* **수학적 배경**: 액터의 World Rotation 행렬에서 Y축 열(Column) 데이터를 추출한 것과 같습니다.

## **3\. 자주 사용되는 사례**

### **① 캐릭터 좌우 이동 (Strafing)**

캐릭터가 옆으로 걷게 만들 때 가장 많이 사용됩니다.

* **입력**: 축 입력(Axis Input) 값 (예: \-1.0 \~ 1.0)  
* **로직**: Get Right Vector 결과값에 입력값을 곱한 뒤, Add Movement Input 노드의 'World Direction'에 연결합니다.

### **② 물체 옆에 스폰하기**

캐릭터의 오른쪽에 동료 캐릭터나 아이템을 생성하고 싶을 때 사용합니다.

* **로직**: Get Actor Location \+ (Get Right Vector \* 거리값)을 계산하여 새로운 위치를 구합니다.

### **③ 라인 트레이스 (Line Trace)**

캐릭터의 오른쪽 시야에 장애물이 있는지 체크할 때 사용합니다.

* **로직**: Start는 캐릭터 위치, End는 Start \+ (Get Right Vector \* 탐지 거리)로 설정합니다.

## **4\. 다른 벡터 노드와의 비교**

| 노드 이름 | 방향 | 로컬 축 |
| :---- | :---- | :---- |
| **Get Forward Vector** | 앞 (Front) | X축 |
| **Get Right Vector** | 오른쪽 (Right) | Y축 |
| **Get Up Vector** | 위 (Up) | Z축 |

## **5\. 주의사항**

* **반대 방향(왼쪽)**: 왼쪽 방향 벡터가 필요하다면 Get Right Vector 값에 **\-1**을 곱하면 됩니다.  
* **월드 vs 로컬**: 이 노드는 액터의 현재 회전을 반영한 '월드 공간상의 방향'을 반환합니다. 단순히 로컬 (0, 1, 0)을 의미하는 것이 아니라, "현재 이 물체가 돌아가 있는 상태에서 오른쪽이 어디인가?"를 알려줍니다.

**팁**: 캐릭터 블루프린트에서 제어 중인 방향(Control Rotation)을 기준으로 좌우 이동을 구현하고 싶다면, Get Actor Right Vector 대신 Get Control Rotation에서 Break Rotator를 한 뒤, Yaw값만 사용해 Rotation from X Vector 또는 Get Forward/Right Vector를 직접 계산하기도 합니다. (경사면에서 땅을 뚫고 들어가는 것을 방지하기 위함)
