# **Get Typed Pawn 노드 완벽 가이드**

언리얼 엔진의 **Common UI** 플러그인을 사용하다 보면 자주 접하게 되는 Get Typed Pawn 노드는 효율적인 UI 로직 작성을 위해 필수적인 도구입니다.

## **1\. 노드의 정의 및 역할**

Get Typed Pawn은 해당 위젯을 소유하고 있는 로컬 플레이어의 \*\*Pawn(폰)\*\*을 가져오되, 노드 설정에서 지정한 **특정 클래스(Class) 타입으로 즉시 반환**해주는 역할을 합니다.

* **기존 방식:** Get Owning Player Pawn → Cast To MyCharacter  
* **Get Typed Pawn 방식:** Get Typed Pawn (MyCharacter 선택) → 결과값 즉시 사용

## **2\. 주요 특징**

### **① 자동 형변환 (Implicit Casting)**

노드 자체에서 Pawn Class를 선택할 수 있는 드롭다운 메뉴가 있습니다. 여기서 원하는 클래스(예: BP\_PlayerCharacter)를 선택하면, 출력 핀의 데이터 타입이 해당 클래스로 고정됩니다. 덕분에 별도의 **Cast To** 노드를 연결할 필요가 없어 그래프가 훨씬 깔끔해집니다.

### **② Common UI 전용**

이 노드는 일반적인 위젯이 아닌, CommonUserWidget이나 CommonActivatableWidget 등 **Common UI 플러그인** 기반의 클래스 내부에서 사용하도록 최적화되어 있습니다.

### **③ 로컬 플레이어 컨텍스트**

UI는 보통 로컬 플레이어와 상호작용합니다. 이 노드는 위젯의 'Owning Player'를 자동으로 추적하여 해당 플레이어가 제어 중인 폰을 찾아줍니다.

## **3\. 노드 구성 요소**

* **Pawn Class (입력/설정):** 가져오고자 하는 구체적인 캐릭터/폰 클래스를 선택합니다.  
* **Return Value (출력):** 선택한 클래스 타입으로 형변환된 객체 인스턴스를 반환합니다. 형변환에 실패하거나 폰이 존재하지 않으면 Null을 반환합니다.

## **4\. 왜 사용하는가? (장점)**

1. **가독성:** 블루프린트 와이어가 복잡해지는 '스파게티 코드'를 방지합니다.  
2. **유지보수:** 캐스팅 로직이 노드 하나에 통합되어 있어 관리가 쉽습니다.  
3. **Common UI 워크플로우:** Common UI 프레임워크 내에서 제공하는 다양한 입력 처리 및 포커스 시스템과 완벽하게 호환됩니다.

## **5\. 사용 시 주의사항**

* **플러그인 활성화:** 이 노드를 사용하려면 프로젝트 설정의 플러그인 메뉴에서 **Common UI**가 활성화되어 있어야 합니다.  
* **Null 체크:** 게임 시작 직후나 캐릭터가 사망하여 폰이 파괴된 상태에서는 Null을 반환할 수 있으므로, 항상 Is Valid 노드로 확인하는 습관이 필요합니다.

## **6\. 일반 Get Player Pawn과의 차이점 비교**

| 기능 | Get Player Pawn | Get Typed Pawn |
| :---- | :---- | :---- |
| **반환 타입** | 일반 Pawn (Object) | 사용자 지정 클래스 (Typed) |
| **추가 노드** | 별도의 Cast 노드 필요 | Cast 노드 불필요 |
| **소속** | 엔진 기본 엔진 (Engine) | Common UI 플러그인 |
| **주 사용처** | 일반적인 게임 로직 | Common UI 기반 위젯 로직 |

**요약하자면:** Get Typed Pawn은 UI에서 플레이어 캐릭터의 정보(체력, 스탯 등)를 실시간으로 반영할 때, **코드를 간결하게 유지하면서 특정 캐릭터 타입에 바로 접근하게 해주는 편리한 도구**입니다.
