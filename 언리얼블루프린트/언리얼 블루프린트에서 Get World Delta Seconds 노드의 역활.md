# **Get World Delta Seconds 노드의 역할과 이해**

언리얼 엔진에서 Get World Delta Seconds 노드는 \*\*이전 프레임에서 현재 프레임까지 경과한 시간(초 단위)\*\*을 반환합니다.

## **1\. 핵심 정의**

* **단위:** 초 (Seconds, 예: 60FPS인 경우 약 0.0166초)  
* **역할:** 컴퓨터 사양이나 그래픽 부하에 따라 변하는 프레임 속도와 상관없이, 게임 내 로직(이동, 회전, 수치 계산 등)이 **실제 시간**에 맞춰 일정하게 작동하도록 만듭니다.

## **2\. 왜 사용해야 하나요? (프레임 독립성)**

게임을 만들 때 "1프레임당 10cm 이동"하도록 코딩했다고 가정해 봅시다.

* **컴퓨터 A (100 FPS):** 1초에 100번 실행 → **1,000cm 이동**  
* **컴퓨터 B (30 FPS):** 1초에 30번 실행 → **300cm 이동**

이처럼 프레임에 의존하면 저사양 PC 사용자는 캐릭터가 느리게 움직이는 불공평한 상황이 발생합니다. 이를 해결하기 위해 Delta Seconds를 곱합니다.

**공식:** 원하는 속도(초당) \* Delta Seconds

## **3\. 블루프린트 활용 예시**

### **① 캐릭터 이동 (Constant Movement)**

틱(Tick) 이벤트에서 캐릭터를 이동시킬 때 사용합니다.

* **노드 구성:** Get Actor Forward Vector \-\> Vector \* Float (여기에 Delta Seconds와 Speed를 곱함) \-\> Add Actor World Offset  
* **결과:** 1초에 정확히 Speed 값만큼 이동하게 됩니다.

### **② 수치 보간 (Interpolation)**

FInterp To 또는 VInterp To 노드에서 사용됩니다.

* **Delta Time 핀:** 여기에 Get World Delta Seconds를 연결합니다.  
* **역할:** 프레임이 튀더라도 부드럽고 일정한 속도로 목표 수치에 도달하게 도와줍니다.

### **③ 타이머 및 쿨다운 (Custom Timers)**

특정 변수에 매 프레임 Delta Seconds를 더해서 시간을 측정할 수 있습니다.

* **로직:** CurrentTime \= CurrentTime \+ Get World Delta Seconds  
* **결과:** CurrentTime이 1.0이 되면 실제 시간으로 1초가 흐른 것입니다.

## **4\. 요약 및 주의사항**

| 특징 | 설명 |
| :---- | :---- |
| **일관성** | 모든 기기에서 동일한 게임 속도를 보장함 |
| **Tick 이벤트** | Event Tick 노드에는 기본적으로 Delta Seconds 핀이 포함되어 있어 굳이 이 노드를 따로 호출하지 않아도 됨 |
| **일시정지** | 게임이 일시정지(Paused)되면 World Delta Seconds는 0이 되어 로직이 멈춤 |

**결론:** "프레임당 처리"를 "시간당 처리"로 바꿔주는 마법의 숫자라고 이해하시면 됩니다. 이동이나 지속적인 변화가 일어나는 모든 Tick 로직에는 반드시 이 값을 곱해주어야 합니다.
