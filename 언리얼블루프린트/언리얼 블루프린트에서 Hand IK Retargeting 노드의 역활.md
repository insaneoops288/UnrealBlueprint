# **Hand IK Retargeting 노드 상세 분석**

언리얼 엔진의 애니메이션 시스템에서 Hand IK Retargeting 노드는 리타겟팅된 애니메이션이 실제 캐릭터의 골격 구조에 맞춰 손의 위치를 정확하게 유지하도록 돕는 역할을 합니다.

## **1\. 주요 역할 (The Core Role)**

이 노드의 핵심 목적은 \*\*"소스 캐릭터의 손 위치(IK 본)를 타겟 캐릭터의 실제 손 위치(FK 본)와 일치시키는 것"\*\*입니다.

* **위치 오차 보정:** 캐릭터마다 팔의 길이나 어깨 넓이가 다르기 때문에, A 캐릭터의 애니메이션을 B 캐릭터에게 적용하면 손이 공중에 뜨거나 몸 안으로 파고드는 현상이 발생합니다. 이 노드는 이를 계산하여 보정합니다.  
* **IK 본(IK Bone) 업데이트:** 언리얼 엔진의 마네킹 스켈레톤에는 ik\_hand\_l, ik\_hand\_r과 같은 전용 IK 본이 있습니다. 이 노드는 리타겟팅 프로세스 중에 이러한 IK 본들을 실제 손 뼈(hand\_l, hand\_r)의 위치에 맞게 정렬합니다.

## **2\. 왜 필요한가? (Use Cases)**

리타겟팅(Retargeting)만으로는 해결되지 않는 물리적 접촉 문제를 해결하기 위해 사용됩니다.

1. **양손 무기 파지:** 소총을 들고 있는 애니메이션을 리타겟팅할 때, 팔이 짧은 캐릭터는 왼손이 총의 덮개에 닿지 않을 수 있습니다. 이 노드를 통해 IK 본을 보정하면 손이 정확히 총 위치에 고정됩니다.  
2. **벽 짚기 또는 상호작용:** 캐릭터가 벽을 짚는 동작에서 손바닥이 벽면과 정확히 밀착되어야 할 때 사용합니다.  
3. **애니메이션 품질 향상:** 단순히 뼈의 회전값만 전달하는 것이 아니라, 최종적인 "목표 위치"를 보정함으로써 애니메이션의 안정감을 줍니다.

## **3\. 작동 원리 (How it Works)**

Hand IK Retargeting 노드는 보통 애니메이션 블루프린트의 **AnimGraph**에서 리타겟팅 노드 다음에 위치합니다.

* **입력:** 리타겟팅된 포즈(Retargeted Pose).  
* **프로세스:**  
  1. 타겟 캐릭터의 현재 손 위치(FK)를 확인합니다.  
  2. 해당 캐릭터의 IK 본(ik\_hand\_l/r) 위치를 FK 본 위치로 이동시킵니다.  
  3. 필요에 따라 오프셋을 적용하여 미세 조정합니다.  
* **결과:** IK 본이 실제 손의 위치와 완벽하게 동기화된 포즈를 출력합니다.

## **4\. 노드 설정 주요 옵션**

노드를 선택하면 디테일 패널에서 다음과 같은 설정을 조정할 수 있습니다.

| 옵션 | 설명 |
| :---- | :---- |
| **IK Bones to Move** | 위치를 보정할 IK 본들의 리스트를 지정합니다 (예: ik\_hand\_l, ik\_hand\_r). |
| **Retargeting Source** | 리타겟팅의 기준이 되는 소스 자산을 선택합니다. |
| **Alpha** | 보정 효과의 강도를 조절합니다 (0.0 이면 효과 없음, 1.0 이면 100% 적용). |

## **5\. 주의사항**

* **IK 셋업 필수:** 이 노드가 제대로 작동하려면 스켈레톤에 반드시 IK 본(ik\_hand\_root, ik\_hand\_l 등)이 올바르게 설정되어 있어야 하며, IK 릭(IK Rig)에서도 해당 본들이 정의되어 있어야 합니다.  
* **실행 순서:** 보통 IK Retargeter 노드 뒤에 배치하여, 리타겟팅이 완료된 직후에 최종 보정 단계로 사용하는 것이 일반적입니다.

### **요약**

Hand IK Retargeting 노드는 **캐릭터의 체형 차이로 발생하는 손 위치의 어색함을 없애고, 무기 파지나 사물 상호작용 시 손이 정확한 위치에 가도록 IK 본을 강제로 정렬해주는 보정 도구**입니다.
