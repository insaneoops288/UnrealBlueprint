# **Unreal Engine: Has All Tags 노드 완벽 가이드**

언리얼 엔진의 **Gameplay Tags** 시스템에서 Has All Tags 노드는 복합적인 조건을 체크할 때 사용되는 핵심 노드입니다.

## **1\. 노드의 핵심 역할**

Has All Tags 노드는 \*\*"대상(Tag Container)이 내가 제시한 여러 개의 태그(Other Container)를 모두 포함하고 있는가?"\*\*를 판단합니다.

* **논리 구조**: AND 연산 (모두 참이어야 결과가 참)  
* **결과**: Boolean (True 또는 False)

## **2\. 주요 입력 핀 (Input Pins)**

1. **Tag Container**: 검사 대상이 되는 태그들의 묶음입니다. (예: 캐릭터가 현재 가지고 있는 버프/상태 이상 태그들)  
2. **Other Container**: 확인하고자 하는 '조건' 태그들의 묶음입니다. 이 안에 넣은 태그가 Tag Container에 **전부** 들어있어야 합니다.

## **3\. 실전 예시: 스킬 발동 조건**

예를 들어, 어떤 강력한 마법 스킬을 사용하기 위해 캐릭터가 **'마나 충전'** 상태이면서 동시에 **'분노'** 상태여야 한다고 가정해 봅시다.

* **비교 대상 (Tag Container)**: 캐릭터의 현재 태그들  
  * 상태 A: {State.HasMana, State.Berserk, State.Grounded}  
  * 상태 B: {State.HasMana, State.Grounded}  
* **체크 조건 (Other Container)**: {State.HasMana, State.Berserk}

**결과:**

* **상태 A**의 경우: 두 태그를 모두 가지고 있으므로 True 반환 \-\> **스킬 발동 가능**  
* **상태 B**의 경우: State.Berserk가 없으므로 False 반환 \-\> **스킬 발동 불가**

## **4\. 'Has Any Tag' 노드와의 차이점**

가장 많이 혼동하는 노드인 Has Any Tag와 비교하면 이해가 빠릅니다.

| 노드 이름 | 논리 | 설명 |
| :---- | :---- | :---- |
| **Has All Tags** | **AND** | 제시한 태그가 **모두** 있어야 함. (엄격한 조건) |
| **Has Any Tag** | **OR** | 제시한 태그 중 **하나라도** 있으면 됨. (느슨한 조건) |

## **5\. 활용 팁**

* **계층 구조 지원**: State.Buff라는 태그를 체크할 때, 대상이 State.Buff.SpeedUp을 가지고 있다면 이 역시 포함된 것으로 간주합니다 (부모 태그 관계).  
* **최적화**: 단순히 Branch 노드와 연결하여 복잡한 if 문을 대신할 수 있어 블루프린트 구조가 매우 깔끔해집니다.  
* **데이터 에셋 활용**: 아이템의 속성이나 퀘스트 완료 조건을 태그로 관리할 때 매우 유용합니다.

## **6\. 요약**

Has All Tags는 \*\*"전부 다 있어?"\*\*라고 묻는 노드입니다. 특정 동작을 위해 여러 가지 선행 조건이 동시에 충족되어야 할 때 사용하세요.
