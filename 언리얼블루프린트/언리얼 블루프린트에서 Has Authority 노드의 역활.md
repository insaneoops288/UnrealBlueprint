언리얼 블루프린트에서 Has Authority의 역활?

언리얼 엔진의 블루프린트에서 Has Authority 노드는 멀티플레이어 게임 제작 시 **"현재 이 코드를 실행 중인 컴퓨터가 이 액터에 대한 제어 권한(서버)을 가지고 있는가?"**를 확인하는 가장 핵심적인 노드입니다.

쉽게 말해, 서버와 클라이언트를 구분하는 분기점 역할을 합니다.

1. 주요 역할 및 출력 (Output)
이 노드는 실행 핀 하나를 받아 두 개의 실행 경로로 나눕니다.

Authority (권한 있음): 현재 코드가 **서버(Server)**에서 실행 중임을 의미합니다. 게임의 중요한 데이터 수정, 물리 연산, 액터 스폰 등은 보통 여기서 처리합니다.

Remote (원격): 현재 코드가 **클라이언트(Client)**에서 실행 중임을 의미합니다. 주로 시각적인 효과(FX), 사운드, UI 업데이트 등 플레이어에게 직접 보여지는 부분을 처리합니다.

2. 왜 사용해야 하나요?
멀티플레이어 환경에서는 모든 클라이언트가 자기 마음대로 게임의 상태를 바꾸면 안 되기 때문입니다.

보안 (치트 방지): 플레이어가 자신의 체력을 100으로 바꾸는 코드를 작성했더라도, Has Authority를 통해 서버에서만 실행되도록 제한하면 클라이언트가 값을 조작하는 것을 막을 수 있습니다.

데이터 일관성: 서버가 "진실"을 들고 있어야 합니다. 아이템 획득, 점수 계산 등 게임의 흐름에 영향을 주는 로직은 서버에서만 실행되어야 모든 플레이어에게 동일한 결과가 전달됩니다.

중복 실행 방지: 예를 들어, 서버와 클라이언트 모두에서 "몬스터 소환" 코드가 실행되면 몬스터가 두 마리 생길 수 있습니다. Has Authority를 사용해 서버에서만 한 번 소환되도록 관리합니다.

3. 주요 활용 사례
구분	Authority (서버)에서 주로 하는 일	Remote (클라이언트)에서 주로 하는 일
게임 로직	체력 감소, 아이템 획득, 점수 추가	UI(HUD) 업데이트, 로컬 애니메이션 재생
액터 관리	무기 발사체 스폰, 적 AI 제어	파티클 이펙트 생성, 사운드 재생
변수 관리	Replicated 변수 값 수정	서버에서 복제된 변수 값을 화면에 표시
4. 실전 예시 (체력 아이템 예시)
캐릭터가 체력 아이템에 닿았을 때의 로직입니다.

Overlap 이벤트 발생

Has Authority 노드 연결

Authority 경로: 캐릭터의 '체력' 변수를 +20 증가 시킴 (서버가 값을 바꿈으로써 모든 클라이언트에 복제됨)

Remote 경로: "아이템 획득!" 이라는 사운드를 재생하거나 화면에 메시지를 띄움 (해당 클라이언트에게만 보여짐)

팁: Has Authority는 보통 Switch Has Authority라는 이름의 노드로 검색하여 사용합니다. 이 노드는 실행 핀(Execution Pin)을 서버와 클라이언트로 깔끔하게 나눠줍니다.

블루프린트에서 네트워크 복제(Replication)를 다루신다면, 이 노드는 선택이 아닌 필수입니다. 어떤 기능을 서버에 맡기고 어떤 기능을 클라이언트에 맡길지 결정하는 첫 단추라고 보시면 됩니다!
