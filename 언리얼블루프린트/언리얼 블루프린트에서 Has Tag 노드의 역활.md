# **언리얼 엔진 Has Tag 노드 완전 정복**

언리얼 엔진에서 **Has Tag** 노드는 객체의 속성을 복잡한 캐스팅(Casting) 없이 빠르게 확인하기 위한 핵심적인 도구입니다. 주로 **Actor Has Tag**와 **Gameplay Tag Has Tag** 두 가지 방식으로 나뉩니다.

## **1\. Actor Has Tag (액터 태그 확인)**

가장 일반적으로 사용되는 노드입니다. 모든 액터(Actor)는 기본적으로 Tags라는 문자열 배열(String Array)을 가지고 있습니다.

### **💡 주요 역할**

* **객체 식별**: Overlapping Actor가 플레이어인지, 적인지, 혹은 특정 아이템인지 확인할 때 사용합니다.  
* **캐스팅 방지**: Cast To PlayerCharacter 같은 무거운 노드를 사용하기 전에, 태그로 먼저 걸러내어 성능을 최적화할 수 있습니다.

### **🛠 사용 방법**

1. **태그 설정**: 액터의 디테일(Details) 패널에서 Tags 항목을 찾아 "Player" 또는 "Enemy" 같은 이름을 추가합니다.  
2. **블루프린트 호출**:  
   * Actor Has Tag 노드를 배치합니다.  
   * **Target**: 확인할 액터 연결.  
   * **Tag**: 설정했던 태그 이름(String) 입력.  
   * **ReturnValue**: Branch 노드에 연결하여 로직 분기.

## **2\. Gameplay Tag Has Tag (게임플레이 태그 확인)**

더 전문적이고 체계적인 시스템인 **Gameplay Tags** 환경에서 사용됩니다.

### **💡 주요 역할**

* **계층 구조 확인**: "Status.Debuff.Stun" 같은 계층형 태그를 관리할 때 사용합니다.  
* **광범위한 체크**: 부모 태그만 확인해도 자식 태그를 포함하는지 판별할 수 있습니다 (예: "Status" 태그를 가졌는지 체크하면 그 하위인 "Stun"도 감지 가능).

## **3\. 실전 활용 예시**

### **A. 충돌 시 플레이어 확인 (Trigger Box)**

트리거 박스에 무언가 부딪혔을 때, 그것이 장애물인지 플레이어인지 구분해야 합니다.

* **Logic**: OnComponentBeginOverlap \-\> Other Actor 연결 \-\> Actor Has Tag (Tag: "Player") \-\> Branch \-\> True일 때만 이벤트 발생.

### **B. 데미지 시스템**

폭발 범위 내에 있는 액터들 중 '데미지를 입지 않는 객체'를 제외하고 싶을 때 사용합니다.

* **Logic**: Get All Actors with Tag로 특정 그룹만 가져오거나, 개별 액터에게 Has Tag (Tag: "Invincible")가 있는지 검사하여 False인 경우에만 데미지 적용.

## **4\. 왜 Has Tag를 사용하나요? (장점)**

1. **유연성**: 클래스 구조를 변경하지 않고도 에디터에서 태그만 추가하면 로직을 바로 적용할 수 있습니다.  
2. **성능**: Cast To 노드는 해당 클래스의 모든 정보를 메모리에 로드하려고 시도하지만, 태그 체크는 단순한 문자열 비교이므로 훨씬 가볍습니다.  
3. **다중 분류**: 하나의 액터는 여러 개의 태그를 가질 수 있습니다. (예: 한 액터가 "Enemy", "Boss", "FireElement" 태그를 동시에 가질 수 있음)

## **⚠️ 주의사항**

* **대소문자 구분**: 기본적으로 문자열 비교이므로 오타에 주의해야 합니다.  
* **매직 넘버/스트링**: 코드나 블루프린트 곳곳에 직접 문자열을 입력하면 나중에 수정이 어렵습니다. 자주 쓰는 태그는 변수로 관리하는 것이 좋습니다.
