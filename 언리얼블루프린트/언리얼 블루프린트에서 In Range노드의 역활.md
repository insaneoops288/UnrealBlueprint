# **언리얼 엔진 블루프린트: In Range 노드 이해하기**

In Range 노드는 수학적 비교 연산을 간소화해 주는 노드로, 특정 값이 주어진 범위 안에 있는지 판별할 때 사용됩니다.

## **1\. 노드의 주요 역할**

기본적으로 "A가 B보다 크거나 같고, C보다 작거나 같은가?"라는 복잡한 논리 구조를 하나의 노드로 합쳐놓은 것입니다. 주로 \*\*Float(실수)\*\*와 **Integer(정수)** 타입에서 사용됩니다.

## **2\. 입력 및 출력 핀 설명**

| 핀 이름 | 타입 | 설명 |
| :---- | :---- | :---- |
| **Value** | Float / Int | 검사하려는 대상 숫자입니다. |
| **Min** | Float / Int | 범위의 최소값입니다. |
| **Max** | Float / Int | 범위의 최대값입니다. |
| **Inclusive Min** | Boolean | 체크 시 최소값(Min)을 포함할지 여부입니다. (기본값: True) |
| **Inclusive Max** | Boolean | 체크 시 최대값(Max)을 포함할지 여부입니다. (기본값: True) |
| **Return Value** | Boolean | 값이 범위 내에 있으면 True, 없으면 False를 반환합니다. |

## **3\. 주요 활용 사례**

### **① 캐릭터 체력(HP) 상태 체크**

캐릭터의 체력이 특정 구간(예: 0% \~ 30%)에 있을 때 '위험 상태' UI를 표시하거나 경고음을 재생하고 싶을 때 사용합니다.

* **Value**: 현재 체력  
* **Min**: 0  
* **Max**: 30  
* **결과**: True일 때 붉은색 화면 효과 발생

### **② 거리 기반 상호작용**

플레이어와 아이템 사이의 거리가 상호작용 가능한 적정 범위(예: 100\~200 유닛 사이)에 있는지 확인할 때 유용합니다.

### **③ 입력 값 제한 및 검증**

조이스틱이나 마우스의 입력 값이 특정 데드존(Deadzone)을 벗어나 유효한 범위 안에 들어왔는지 판별할 때 사용합니다.

## **4\. In Range vs Clamp 차이점**

초보 개발자들이 자주 혼동하는 Clamp 노드와 비교하면 역할이 명확해집니다.

* **In Range**: "범위 안에 있는가?"라고 **질문**하고 Yes/No로 답합니다. (판단용)  
* **Clamp**: "범위 밖으로 나가지 못하게 **고정**하라"고 명령합니다. (수치 수정용)

## **5\. 팁: Inclusive 옵션 활용**

* Inclusive Min/Max가 True이면: Min \<= Value \<= Max (이상/이하)  
* Inclusive Min/Max가 False이면: Min \< Value \< Max (초과/미만)

이 옵션을 통해 경계값 처리를 별도의 비교 노드 없이 간편하게 조절할 수 있습니다.
