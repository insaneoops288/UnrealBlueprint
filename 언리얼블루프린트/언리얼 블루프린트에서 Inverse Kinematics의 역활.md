# **언리얼 엔진 블루프린트에서의 Inverse Kinematics (IK) 역할**

\*\*Inverse Kinematics (IK)\*\*는 말단 장치(End Effector, 예: 손이나 발)의 위치와 회전값을 먼저 결정한 후, 그에 맞춰 상위 관절(부모 뼈대)들의 각도를 역으로 계산하여 배치하는 방식입니다.

### **1\. IK의 핵심 역할**

#### **① 지형 적응 (Foot Placement)**

가장 많이 사용되는 사례입니다. 캐릭터가 경사로나 계단을 오를 때, 발이 공중에 떠 있거나 땅을 뚫고 들어가는 현상을 방지합니다.

* **역할:** 캐릭터의 캡슐 컴포넌트 위치와 상관없이, 실제 지형의 높이에 맞춰 발바닥의 위치를 보정합니다.

#### **② 환경 상호작용 (Reaching & Grabbing)**

캐릭터가 특정 물체를 잡거나 만져야 할 때 유용합니다.

* **역할:** 애니메이션 소스(FBX)에는 없던 특정 위치(예: 문 손잡이, 벽)로 캐릭터의 손을 정확히 이동시킵니다. 물체의 위치가 실시간으로 변해도 손이 따라가게 할 수 있습니다.

#### **③ 시선 처리 및 조준 (Look-at & Aiming)**

캐릭터의 머리나 무기가 타겟을 바라보게 합니다.

* **역할:** 적 캐릭터나 특정 사물의 위치를 추적하여 목 관절이나 척추 관절을 변형시켜 항상 대상을 바라보게 만듭니다.

#### **④ 절차적 애니메이션 (Procedural Animation)**

미리 제작된 애니메이션 파일의 한계를 극복합니다.

* **역할:** 물리 엔진과 결합하여 캐릭터가 넘어질 때 손으로 땅을 짚거나, 가방을 메고 있을 때 어깨 끈의 위치를 조정하는 등 동적인 움직임을 생성합니다.

### **2\. FK(Forward Kinematics)와의 차이**

| 구분 | FK (전방 운동학) | IK (역운동학) |
| :---- | :---- | :---- |
| **계산 방향** | 뿌리(Root) → 끝(End) | 끝(End) → 뿌리(Root) |
| **제어 방식** | 어깨, 팔꿈치 회전값을 직접 입력 | 손목이 도달할 최종 좌표(Vector)를 입력 |
| **주요 사용** | 기본적인 달리기, 걷기 애니메이션 | 발 접지, 물체 집기, 시선 처리 |

### **3\. 언리얼 블루프린트에서의 구현 방식**

언리얼 엔진에서는 주로 다음 두 가지 경로로 IK를 제어합니다.

#### **A. 애니메이션 블루프린트 (Anim Blueprint)**

AnimGraph 내에서 전용 노드를 사용하여 실시간 보정을 수행합니다.

* **Two Bone IK:** 팔이나 다리와 같이 두 개의 뼈로 구성된 관절 구조에 적합한 가벼운 노드입니다.  
* **FABRIK:** 여러 개의 관절이 연결된 체인(예: 긴 꼬리, 촉수)을 제어할 때 사용합니다.  
* **CCDIK:** 특정 타겟을 향해 관절을 구부릴 때 유용한 알고리즘입니다.

#### **B. 컨트롤 리그 (Control Rig) \- 권장 방식**

UE4 후반부와 UE5에서 도입된 최신 방식으로, 훨씬 강력하고 유연합니다.

* **특징:** 애니메이션 블루프린트 밖에서 캐릭터의 뼈대를 직접 제어하는 로직을 짭니다.  
* **Full Body IK (FBIK):** 몸 전체의 균형을 유지하면서 특정 부위를 IK로 제어할 수 있는 고성능 솔루션입니다.

### **4\. 실무 활용 팁**

1. **Trace (레이캐스트) 활용:** 블루프린트의 LineTraceByChannel을 사용하여 발 밑 지형의 높이와 노멀(Normal)값을 구한 뒤, 이를 IK 노드의 타겟 위치로 전달합니다.  
2. **Alpha(블렌딩) 조절:** IK 효과를 너무 강하게 주면 애니메이션이 부자연스러워질 수 있습니다. Alpha 값을 조절하여 기존 애니메이션과 IK 보정값을 적절히 섞어주는 것이 중요합니다.  
3. **Pole Vector 설정:** 무릎이나 팔꿈치가 꺾이는 방향을 결정하는 'Pole Vector'를 정확히 설정해야 관절이 뒤집히는 현상을 막을 수 있습니다.

### **요약**

언리얼 블루프린트에서 IK는 \*\*"캐릭터를 살아있는 환경 속에 고정시키는 풀(Glue)"\*\*과 같습니다. 미리 만들어진 애니메이션이 모든 상황에 대응할 수 없기 때문에, IK를 통해 실시간으로 캐릭터의 포즈를 수정하여 게임의 퀄리티를 높입니다.
