# **Is Local Player Controller 노드 완벽 이해**

언리얼 엔진의 네트워크 환경에서 이 노드는 "현재 이 코드가 실행되는 환경이 이 플레이어의 주인인가?"를 판별하는 핵심적인 역할을 합니다.

## **1\. 기본 개념**

이 노드는 Controller 오브젝트를 타겟으로 하며, 해당 컨트롤러가 \*\*로컬 플레이어(Local Player)\*\*에 의해 제어되고 있다면 True를, 네트워크를 통해 넘어온 \*\*원격 플레이어(Remote Player)\*\*의 컨트롤러라면 False를 반환합니다.

* **로컬(Local):** 내 모니터 앞에 앉아 있는 내가 직접 조작하는 환경.  
* **원격(Remote):** 멀티플레이어 게임에서 다른 컴퓨터에 있는 사람이 조작하는 환경.

## **2\. 왜 필요한가요? (네트워크 복제)**

멀티플레이어 게임을 만들 때, 서버에는 접속한 모든 플레이어의 Player Controller가 존재합니다. 하지만 클라이언트(플레이어의 컴퓨터)에는 **오직 자기 자신의 Player Controller만 존재**합니다.

만약 어떤 로직이 모든 플레이어 컨트롤러를 대상으로 실행된다면, 내가 아닌 다른 사람의 데이터까지 내 화면에 영향을 줄 수 있습니다. 이를 방지하기 위해 "이게 정말 내 컨트롤러가 맞는지" 확인하는 과정이 필요합니다.

## **3\. 주요 활용 사례**

### **① UI 및 HUD 표시**

HUD(체력바, 인벤토리 등)는 화면을 보고 있는 '로컬 플레이어'에게만 보여야 합니다.

* **로직:** BeginPlay에서 컨트롤러가 로컬인지 확인한 후, True일 때만 Create Widget을 호출합니다.

### **② 카메라 및 포스트 프로세스 효과**

대미지를 입었을 때 화면이 붉게 변하는 효과는 해당 플레이어에게만 나타나야 합니다.

* **로직:** 대미지 이벤트 발생 시 Is Local Player Controller를 체크하여 화면 효과를 적용합니다.

### **③ 입력 처리 (Input Handling)**

멀티플레이어 환경에서 다른 플레이어의 캐릭터가 내 키보드 입력에 반응하면 안 됩니다.

* 보통 캐릭터(Pawn) 단에서는 Is Locally Controlled를 사용하지만, 컨트롤러 단의 로직에서는 이 노드를 사용하여 입력을 제한합니다.

## **4\. 유사 노드와 차이점**

| 노드명 | 대상(Target) | 주요 용도 |
| :---- | :---- | :---- |
| **Is Local Player Controller** | Controller | 이 컨트롤러가 로컬 사용자의 것인지 확인 |
| **Is Locally Controlled** | Pawn / Character | 이 캐릭터가 현재 로컬 기기에서 제어되고 있는지 확인 |
| **Switch Has Authority** | Actor | 현재 실행 중인 기기가 서버(Authority)인지 클라이언트(Remote)인지 확인 |

## **5\. 블루프린트 구성 예시**

\[Event AnyDamage\] \-\> \[Get Controller\] \-\> \[Is Local Player Controller\]  
                                               |  
                                        \[Branch (True)\] \-\> \[Play Camera Shake\]

*설명: 대미지를 받았을 때, 이 컨트롤러가 로컬 플레이어의 것이라면 카메라 흔들림 효과를 재생합니다.*

**팁:** 싱글 플레이어 게임에서는 항상 한 명의 플레이어만 존재하므로 이 노드는 항상 True를 반환합니다. 따라서 주로 **멀티플레이어 개발** 시 필수적으로 사용하게 됩니다.
