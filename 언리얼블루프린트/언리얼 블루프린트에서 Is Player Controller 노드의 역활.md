# **Unreal Engine: Is Player Controller 노드 이해하기**

언리얼 엔진 아키텍처에서 Controller는 Pawn을 빙의(Possess)하여 조종하는 보이지 않는 행위자입니다. 이때 조종 주체가 사람인지 AI인지 구분하는 것이 개발 과정에서 매우 중요합니다.

## **1\. 노드의 핵심 역할**

Is Player Controller 노드는 입력받은 객체(Target)가 **Player Controller** 클래스(또는 그 자식 클래스)의 인스턴스인지 확인하여 Boolean (True/False) 값을 반환합니다.

* **True**: 조종 주체가 사람(로컬 또는 네트워크 플레이어)임.  
* **False**: 조종 주체가 AI이거나, 해당 객체가 컨트롤러가 아님.

## **2\. 주요 사용 시나리오**

### **A. AI와 플레이어의 로직 분리**

하나의 Character 블루프린트를 플레이어와 AI가 공유해서 사용하는 경우, 특정 행동(예: UI 표시, 특수 입력 처리)은 플레이어에게만 적용되어야 합니다.

* **예시**: 데미지를 입었을 때 화면에 붉은 오버레이를 띄우는 로직은 Is Player Controller가 True일 때만 실행하도록 구성합니다.

### **B. 멀티플레이어 환경에서의 판별**

서버 코드 내에서 어떤 액션의 주체가 실제 플레이어인지 확인하여 점수를 부여하거나 통계를 기록할 때 사용합니다.

### **C. 게임 모드 로직**

GameMode에서 모든 컨트롤러를 순회하며 플레이어만 찾아내어 특정 버프를 주거나 위치를 이동시킬 때 유용합니다.

## **3\. 블루프린트 구성 예시**

보통 Controller 레퍼런스를 가져와서 이 노드에 연결합니다.

1. **Get Controller**: 현재 폰(Pawn)이 소유한 컨트롤러를 가져옵니다.  
2. **Is Player Controller**: 해당 컨트롤러를 타겟으로 연결합니다.  
3. **Branch**: 반환된 Boolean 값으로 로직을 분기합니다.

\[Get Controller\] \-\> \[Is Player Controller\] \-\> \[Branch\]  
                                                   |  
                                         (True) \-\> \[플레이어 전용 로직\]  
                                         (False) \-\> \[AI 전용 로직\]

## **4\. 주의사항**

* **대상 타입**: 이 노드는 Actor 타입을 입력으로 받지만, 내부적으로는 Controller인지 확인합니다. 만약 컨트롤러가 아닌 일반 액터를 넣으면 항상 False를 반환합니다.  
* **Is Local Player Controller와의 차이점**:  
  * Is Player Controller: 단순히 '사람'이 조종하는지 체크 (네트워크상의 다른 플레이어도 포함).  
  * Is Local Player Controller: 현재 이 게임이 돌아가고 있는 '내 기기'의 플레이어인지 체크.

## **5\. 결론**

Is Player Controller는 \*\*"이 조종 주체가 사람인가?"\*\*라는 질문에 답하는 노드입니다. 게임 내 시스템에서 플레이어와 AI를 차별화해야 하는 모든 순간에 가장 기본적으로 사용되는 필수 노드 중 하나입니다.
