# **Is Player Controller 노드의 역할과 활용**

언리얼 엔진의 블루프린트 시스템에서 Is Player Controller 노드는 객체의 타입을 식별하는 데 사용되는 **형변환(Casting)의 간소화 버전** 혹은 **조건부 필터** 역할을 합니다.

## **1\. 주요 역할**

이 노드는 입력으로 받은 타겟(Object)이 APlayerController 클래스 또는 그 하위 클래스의 인스턴스인지를 판별합니다.

* **참(True):** 해당 객체가 실제 사람이 조작하는 플레이어 컨트롤러일 때.  
* **거짓(False):** 해당 객체가 AI 컨트롤러이거나, 컨트롤러가 아닌 다른 액터(Pawn, Actor 등)일 때.

## **2\. Get Player Controller와의 차이점**

초보 개발자들이 가장 많이 혼동하는 부분입니다.

| 노드 이름 | 역할 | 결과값 |
| :---- | :---- | :---- |
| **Get Player Controller** | 월드에 존재하는 특정 인덱스(예: 0번 플레이어)의 컨트롤러 **참조를 가져옴** | Player Controller 객체 |
| **Is Player Controller** | 이미 가지고 있는 참조가 플레이어 컨트롤러인지 **확인함** | Boolean (True/False) |

## **3\. 실무 활용 사례**

### **① AI와 플레이어 구분 (Pawn 내부 로직)**

캐릭터(Pawn)가 대미지를 입었을 때, 나를 때린 존재가 사람인지 AI인지에 따라 다른 처리를 하고 싶을 때 사용합니다.

* **로직:** Get Controller \-\> Is Player Controller \-\> Branch  
* 플레이어라면 UI에 킬 마크를 띄우고, AI라면 무시하는 식의 처리가 가능합니다.

### **② 멀티플레이어 로컬 확인**

네트워크 환경에서 특정 컨트롤러가 현재 내 컴퓨터(Local)에서 실행 중인 플레이어의 것인지 확인할 때 유용합니다. (주로 Is Local Player Controller와 병행해서 사용됩니다.)

### **③ 반복문 내 필터링**

Get All Actors of Class 등으로 모든 컨트롤러를 불러온 뒤, 그중에서 AI를 제외하고 실제 플레이어들에게만 특정 메시지나 이벤트를 보내고 싶을 때 필터로 사용합니다.

## **4\. 노드 사용 팁**

* **입력 핀:** 기본적으로 'Target' 핀에 Controller 참조를 연결해야 정확한 결과를 얻을 수 있습니다.  
* **순수 함수:** 이 노드는 실행 핀(하얀색 화살표)이 없는 **Pure Function**입니다. 따라서 값이 필요할 때마다 연결해서 사용할 수 있으며 성능에 큰 부하를 주지 않습니다.
