# **Is State Blending Out 노드 이해하기**

언리얼 엔진(Unreal Engine)의 애니메이션 블루프린트(AnimBP) 내 \*\*상태 머신(State Machine)\*\*에서 이 노드는 현재 상태의 생명 주기를 파악하는 데 매우 중요한 역할을 합니다.

## **1\. 기본 정의**

Is State Blending Out 노드는 현재 활성화된 애니메이션 상태가 트랜지션 조건이 충족되어 **다음 상태로 블렌딩(Blending)을 시작했는지 여부**를 Boolean(true/false) 값으로 반환합니다.

* **True**: 현재 상태가 종료되기 시작했고, 설정된 블렌드 시간(Blend Time) 동안 다음 애니메이션으로 서서히 전환되고 있는 상태입니다.  
* **False**: 현재 상태가 안정적으로 재생 중이거나, 아직 트랜지션이 시작되지 않은 상태입니다.

## **2\. 주요 역할 및 용도**

### **① 애니메이션 종료 시점의 로직 실행**

특정 상태가 끝날 때 딱 한 번만 실행되어야 하는 로직이 있을 때 유용합니다. 예를 들어, 발차기 공격 상태가 끝나고 대기(Idle) 상태로 돌아가기 시작할 때 특정 변수를 초기화하거나 사운드를 재생하고 싶을 때 사용합니다.

### **② 조건부 노드 업데이트 제어**

상태가 블렌딩 아웃되는 동안에는 애니메이션의 가중치(Weight)가 점차 줄어듭니다. 이때 불필요한 계산을 방지하거나, 캐릭터의 특정 부위(IK 등) 보정을 중단시키는 트리거로 활용할 수 있습니다.

### **③ 시각 효과(VFX) 및 사운드 동기화**

애니메이션이 완전히 끝나기 직전(블렌딩 아웃 시작 시점)에 파티클을 제거하거나 사운드를 페이드 아웃시키는 용도로 사용됩니다.

## **3\. 작동 메커니즘 예시**

만약 **'JumpStart'** 상태에서 **'JumpLoop'** 상태로 넘어가는 트랜지션의 **Blend Time**이 0.2초로 설정되어 있다면:

1. 'JumpStart' 애니메이션이 재생되는 동안에는 노드가 false를 반환합니다.  
2. 트랜지션 조건이 충족되는 순간, Is State Blending Out은 즉시 true가 됩니다.  
3. 이후 0.2초 동안 두 애니메이션이 섞이는 과정 내내 true를 유지합니다.  
4. 블렌딩이 완전히 끝나고 'JumpLoop' 상태가 100% 가중치를 가지게 되면, 이전 상태의 이 노드는 더 이상 참조되지 않습니다.

## **4\. 실전 블루프린트 활용 팁**

### **트랜지션 그래프(Transition Graph)에서의 활용**

트랜지션 노드 자체에서 \*\*"결과에 도달하기 전 남은 시간"\*\*을 체크하는 Time Remaining (Ratio) 노드와 함께 사용하여, 애니메이션이 거의 끝나갈 때쯤 미리 다음 상태를 준비하는 로직을 짤 때 유용합니다.

### **주의 사항**

* 이 노드는 **해당 상태(State)의 내부 그래프**나 관련 노드 내에서만 유효한 맥락을 가집니다.  
* **Is State Blending In** 노드와는 반대 개념입니다. (In은 상태가 시작될 때 true)

## **5\. 요약**

| 기능 | 설명 |
| :---- | :---- |
| **반환 타입** | Boolean (true 또는 false) |
| **핵심 시점** | 트랜지션 시작 직후 \~ 블렌딩 완료 전까지 |
| **추천 상황** | 상태 종료 시 변수 초기화, 종료 이펙트 트리거, 애니메이션 흐름 제어 |

이 노드를 잘 활용하면 애니메이션 간의 연결을 더욱 부드럽게 만들고, 상태 변화에 따른 복잡한 게임 로직을 애니메이션 블루프린트 내에서 깔끔하게 관리할 수 있습니다.
