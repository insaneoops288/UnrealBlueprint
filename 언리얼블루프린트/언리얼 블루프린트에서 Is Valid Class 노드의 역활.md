# **언리얼 엔진 블루프린트: Is Valid Class 노드 가이드**

Is Valid Class 노드는 클래스 유형의 변수(Class Reference)를 다룰 때 안정성을 확보하기 위한 필수적인 검사 도구입니다.

## **1\. 주요 역할**

* **비어 있음(None) 확인**: 클래스 변수가 아무것도 할당되지 않은 'None' 상태인지, 아니면 특정 클래스(예: BP\_Player, StaticMeshActor 등)로 올바르게 설정되어 있는지 확인합니다.  
* **런타임 에러 방지**: 클래스 정보가 없는 상태에서 Spawn Actor from Class나 Create Widget 같은 노드를 실행하면 게임이 크래시되거나 오류 로그가 발생합니다. 이를 방지하기 위해 사전에 검사합니다.

## **2\. 입력 및 출력**

* **Input (Class)**: 확인할 클래스 참조 변수 (TSubclassOf 형식 등).  
* **Return Value (Boolean)**:  
  * True: 유효한 클래스가 할당됨.  
  * False: 클래스가 지정되지 않음(None).

## **3\. 언제 사용하는가?**

### **① 액터 스폰 전 (Spawn Actor from Class)**

가장 흔한 사례입니다. 아이템을 드롭하거나 총알을 발사할 때, 스폰할 액터 클래스가 변수로 지정되어 있다면 스폰하기 직전에 이 노드로 검사합니다.

* **상황**: 인벤토리 시스템에서 아이템을 버릴 때, 해당 아이템의 'Class To Spawn' 변수가 설정되어 있는지 확인.

### **② 위젯 생성 전 (Create Widget)**

UI 위젯을 동적으로 생성할 때, 생성할 위젯 클래스가 유효한지 확인합니다.

### **③ 데이터 테이블/설정 값 로드 후**

데이터 테이블에서 클래스 정보를 가져왔을 때, 데이터가 누락되었거나 잘못되었을 가능성이 있으므로 사용 전 검사합니다.

## **4\. Is Valid (오브젝트) vs Is Valid Class 비교**

| 구분 | Is Valid (Object) | Is Valid Class |
| :---- | :---- | :---- |
| **확인 대상** | 월드에 존재하는 **객체(Instance)** | 클래스 **정의(Blueprint/C++ Class)** |
| **비유** | 구워진 붕어빵이 먹을 수 있는 상태인가? | 붕어빵을 구울 **틀**이 준비되었는가? |
| **사용 사례** | 변수에 저장된 캐릭터가 파괴되었는지 확인 | 스폰할 액터 종류가 선택되었는지 확인 |

## **5\. 실전 팁**

* **Pure Node**: Is Valid Class는 실행 핀(흰색 선)이 없는 순수 노드입니다. Branch 노드와 함께 사용하거나 Select 노드 등의 조건으로 활용하기 좋습니다.  
* **기본값 주의**: 클래스 변수를 생성한 후 디테일 패널에서 기본값(Default Value)을 설정하지 않으면 기본적으로 'None'이 됩니다. 이때 검사 없이 사용하면 문제가 발생하므로 반드시 체크하는 습관이 중요합니다.

**요약하자면:** Is Valid Class는 \*\*"내가 지금 이 클래스로 무언가를 만들(Spawn/Create) 준비가 되었는가?"\*\*를 묻는 안전장치입니다.
