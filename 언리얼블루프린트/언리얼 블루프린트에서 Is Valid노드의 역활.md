# **언리얼 블루프린트: Is Valid 노드 가이드**

언리얼 엔진에서 객체 지향 프로그래밍을 할 때, 변수가 참조하는 객체가 항상 존재한다는 보장은 없습니다. **Is Valid** 노드는 이러한 불확실성을 해결하여 게임의 안정성을 높여주는 필수 노드입니다.

## **1\. Is Valid 노드의 핵심 역할**

Is Valid 노드는 입력받은 \*\*객체 참조(Object Reference)\*\*가 다음의 조건을 만족하는지 확인합니다.

* **메모리에 할당되었는가?** (변수가 비어있지 않은가)  
* **파괴 대기 중(Pending Kill)이 아닌가?** (Destroy Actor 등으로 인해 제거되는 중이 아닌가)

만약 유효하지 않은(Not Valid) 객체의 함수를 호출하거나 변수에 접근하려고 하면, 언리얼은 \*\*"Accessed None"\*\*이라는 런타임 에러를 발생시키며 로직이 중단되거나 게임이 크래시될 수 있습니다.

## **2\. Is Valid 노드의 두 가지 형태**

### **① 실행형 Is Valid (Question Mark 아이콘)**

가장 흔히 사용되는 형태로, 실행 흐름(Execution Flow)을 분기합니다.

* **입력:** 객체 참조 (Object)  
* **출력 (Is Valid):** 객체가 정상적일 때 실행할 로직 연결  
* **출력 (Is Not Valid):** 객체가 없거나 파괴되었을 때 실행할 로직 연결  
* **장점:** Branch 노드를 따로 쓸 필요 없이 바로 흐름을 나눌 수 있어 그래프가 깔끔해집니다.

### **② 순수형 Is Valid (함수형)**

Boolean(True/False) 값을 반환하는 노드입니다.

* **입력:** 객체 참조 (Object)  
* **출력:** Return Value (Boolean)  
* **사용처:** Branch 노드와 함께 사용하거나, Select 노드, 혹은 복합적인 조건식(AND, OR)이 필요할 때 사용합니다.

## **3\. 언제 반드시 사용해야 하나요?**

1. **지연 실행 후 (Delay):** Delay 노드 이후에는 그 사이 객체가 파괴되었을 가능성이 있으므로 체크가 필요합니다.  
2. **다른 액터 참조 시:** 다른 액터(예: 타겟 액터, 플레이어 캐릭터)를 변수에 담아두고 쓸 때, 그 액터가 게임 도중 사라질 수 있다면 반드시 체크해야 합니다.  
3. **컴포넌트 접근 시:** 동적으로 생성하거나 제거하는 컴포넌트에 접근할 때 필요합니다.  
4. **BeginPlay 직후:** 특정 객체가 스폰되는 순서에 따라 아직 초기화가 안 되었을 수 있습니다.

## **4\. 실전 팁**

* **오른쪽 클릭 메뉴:** 블루프린트의 변수 노드 위에서 마우스 오른쪽 버튼을 클릭하면 **"Convert to Validated Get"** 메뉴가 나타납니다. 이를 선택하면 일반 Get 노드가 '실행형 Is Valid' 노드와 합쳐진 형태로 변해 공간을 절약할 수 있습니다.  
* **가비지 컬렉션:** 언리얼의 가비지 컬렉터가 객체를 정리하기 전, Destroy가 호출된 직후에도 Is Valid는 False를 반환하므로 안전하게 로직을 방어할 수 있습니다.

## **요약**

**Is Valid는 "이 객체 지금 써도 안전하니?"라고 묻는 것과 같습니다.** "Accessed None" 에러를 예방하기 위해, 확신이 없는 모든 객체 참조 앞에는 Is Valid를 습관적으로 사용하는 것이 좋습니다.
