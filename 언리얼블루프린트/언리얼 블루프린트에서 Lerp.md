# **Unreal Engine Blueprint: Lerp (Linear Interpolation) 가이드**

Lerp 노드는 두 값 사이를 선형적으로 보간하여 결과값을 반환합니다. 애니메이션, UI 효과, 물리적 이동 등 거의 모든 부드러운 전환 효과에 필수적으로 사용됩니다.

### **2\. 블루프린트에서의 Lerp 종류**

언리얼 엔진은 데이터 타입에 따라 다양한 Lerp 노드를 제공합니다.

1. **Lerp (Float):** 일반적인 숫자 보간 (예: 투명도, 크기 조절).  
2. **Lerp (Vector):** 공간상의 좌표 보간 (예: A 지점에서 B 지점으로 이동).  
3. **Lerp (Rotator):** 회전값 보간 (예: 캐릭터가 서서히 특정 방향을 보게 함).  
   * *주의:* 회전은 'Shortest Path' 옵션을 체크해야 가장 가까운 방향으로 회전합니다.  
4. **Lerp (Linear Color):** 색상 보간 (예: 낮에서 밤으로 변하는 하늘 색상).

### **3\. 실전 활용 예시: Timeline과 Lerp의 조합**

가장 흔한 패턴은 **Timeline** 노드를 사용하여 0에서 1까지 변하는 Float 값을 생성하고, 이를 Lerp의 Alpha에 연결하는 방식입니다.

#### **예: 부드럽게 열리는 문 (Door)**

1. **Timeline 생성:** 0초에 값 0, 1초에 값 1을 갖는 커브를 만듭니다.  
2. **Lerp (Rotator) 배치:**  
   * **A:** 문이 닫힌 각도 (0, 0, 0\)  
   * **B:** 문이 열린 각도 (0, 0, 90\)  
   * **Alpha:** Timeline의 출력값 연결  
3. **Update:** SetRelativeRotation 노드를 호출하여 매 프레임 문을 회전시킵니다.

### **4\. 자주 하는 실수와 팁**

#### **① Alpha 값의 범위**

Alpha는 보통 **0.0 \~ 1.0** 사이여야 합니다. 만약 1.0보다 큰 값을 넣으면 B보다 더 멀리 가버리는 'Extrapolation(외삽)' 현상이 발생합니다. 이를 방지하려면 Clamp 노드를 사용하거나 Lerp 노드 내부의 Clamp 옵션을 확인하세요.

#### **② Lerp vs InterpTo**

* **Lerp:** 우리가 직접 Alpha 값을 조절해야 합니다 (보통 Timeline과 함께 사용).  
* **FInterpTo / VInterpTo:** 'Delta Time'과 'Interp Speed'를 사용하여 현재 값에서 목표 값까지 **자동으로** 부드럽게 다가갑니다. 매 프레임(Tick) 실행되는 로직에 더 적합합니다.

#### **③ 색상 보간 시 주의점**

Lerp (Linear Color)를 사용할 때, 빛의 밝기(HDR)가 포함된 색상을 보간하면 중간 단계에서 예상보다 훨씬 밝거나 어두운 색이 나올 수 있습니다. 이때는 Linear Color 타입을 정확히 사용하고 있는지 확인해야 합니다.
