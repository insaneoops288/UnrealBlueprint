# **Link Anim Class Layers 노드의 역할과 활용**

언리얼 엔진 4.24 버전부터 도입된 **애니메이션 블루프린트 레이어(Animation Blueprint Layers)** 시스템은 복잡한 애니메이션 그래프를 모듈화하고 관리하기 위해 만들어졌습니다. **Link Anim Class Layers** 노드는 이 시스템의 중추적인 역할을 합니다.

## **1\. 핵심 역할: "동적 로직 교체"**

이 노드의 가장 큰 역할은 **메인 애니메이션 블루프린트**가 사전에 정의된 인터페이스를 통해 **별도의 애니메이션 레이어 클래스**를 실시간으로 연결하거나 해제할 수 있도록 하는 것입니다.

* **인터페이스 기반**: 특정 '애니메이션 레이어 인터페이스'를 상속받은 클래스라면 무엇이든 메인 AnimBP에 연결할 수 있습니다.  
* **런타임 변경**: 게임 플레이 중 캐릭터가 무기를 바꾸거나 상태(부상, 분노 등)가 변했을 때, 그에 맞는 애니메이션 로직으로 즉시 교체할 수 있습니다.

## **2\. 작동 방식 (Workflow)**

Link Anim Class Layers를 사용하기 위한 일반적인 단계는 다음과 같습니다.

1. **Animation Layer Interface 생성**: 어떤 기능을 가질지 정의합니다. (예: FullBody\_Override, UpperBody\_Layer)  
2. **Main AnimBP에서 인터페이스 추가**: 메인 블루프린트의 Class Settings에서 해당 인터페이스를 추가하고, 애님 그래프 내에서 레이어 노드를 배치합니다.  
3. **Layer Class 생성**: 인터페이스를 구현하는 별도의 애님 블루프린트 클래스를 만듭니다. (예: Rifle\_AnimLayer, Pistol\_AnimLayer)  
4. **노드 호출**: 캐릭터 블루프린트나 메인 AnimBP 이벤트 그래프에서 Link Anim Class Layers 노드를 실행하여 특정 클래스를 레이어에 할당합니다.

## **3\. 주요 장점**

### **1\) 모듈화 및 협업**

과거에는 하나의 거대한 AnimBP 안에 수많은 스테이트 머신(State Machine)과 블렌딩 로직을 넣어야 했습니다. 레이어를 사용하면 무기별, 캐릭터별 애니메이션 로직을 별도 파일로 분리할 수 있어 여러 명의 애니메이터가 동시에 작업하기 용이합니다.

### **2\) 메모리 최적화**

모든 무기의 애니메이션 로직을 하나의 AnimBP에 넣으면 사용하지 않는 무기의 데이터까지 메모리에 로드될 수 있습니다. 레이어를 사용하면 **현재 필요한 로직만 링크**하여 메모리 사용량을 줄일 수 있습니다.

### **3\) 코드 재사용성**

한 번 만든 '이동 레이어(Locomotion Layer)'나 '무기 레이어'를 다른 캐릭터(Skeletal Mesh가 호환되는 경우)에게도 쉽게 재사용할 수 있습니다.

## **4\. Sub Anim Instance와의 차이**

| 구분 | Sub Anim Instance | Link Anim Class Layers |
| :---- | :---- | :---- |
| **연결 방식** | 그래프 내에 고정됨 (Static) | 실행 중에 바꿀 수 있음 (Dynamic) |
| **유연성** | 컴파일 타임에 결정됨 | 런타임에 결정됨 |
| **주요 용도** | 복잡한 그래프를 부분별로 나눌 때 | 무기 교체, 장비에 따른 모션 변경 등 |

## **5\. 실전 예시 (Weapon System)**

캐릭터가 **검**을 들고 있다가 **활**로 바꿀 때의 상황입니다.

1. 캐릭터 블루프린트에서 무기 교체 이벤트 발생.  
2. Get Anim Instance를 통해 메인 AnimBP를 가져옴.  
3. **Link Anim Class Layers** 노드를 호출.  
4. In Class 핀에 BP\_Bow\_AnimationLayer 클래스를 입력.  
5. 이제 메인 AnimBP 내의 WeaponLayer 지점에서 활 관련 애니메이션 로직이 작동함.

## **6\. 주의 사항**

* **인터페이스 일치**: 링크하려는 클래스는 반드시 메인 AnimBP가 사용하는 애니메이션 레이어 인터페이스를 구현하고 있어야 합니다.  
* **초기화 비용**: 너무 빈번하게(매 프레임 등) 링크를 시도하면 성능 저하가 발생할 수 있으므로, 상태가 바뀔 때 한 번만 호출하는 것이 좋습니다.
