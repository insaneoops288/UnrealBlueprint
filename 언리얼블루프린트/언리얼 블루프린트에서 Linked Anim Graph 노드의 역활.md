# **언리얼 엔진 Linked Anim Graph 노드의 역할과 활용**

**Linked Anim Graph** 노드는 하나의 애니메이션 블루프린트(AnimBP) 내에서 다른 애니메이션 블루프린트나 인터페이스의 그래프를 참조하여 실행할 수 있게 해주는 기능을 합니다. 이를 통해 거대한 애니메이션 로직을 여러 개의 작은 단위로 쪼개어 관리할 수 있습니다.

### **1\. 주요 역할 (Core Roles)**

#### **① 모듈화 및 재사용성 (Modularity & Reusability)**

가장 큰 역할은 애니메이션 로직의 **부품화**입니다. 예를 들어, '걷기/뛰기' 같은 공통 로직은 메인 AnimBP에 두고, '칼을 휘두르는 동작'이나 '총을 쏘는 동작'은 별도의 AnimBP로 만든 뒤 필요할 때만 Linked Anim Graph로 연결하여 사용할 수 있습니다.

#### **② 협업 효율성 (Team Collaboration)**

하나의 거대한 AnimBP 파일에서 여러 명이 작업하면 컨플릭트(Conflict)가 발생하기 쉽습니다. 하지만 로직을 Linked Anim Graph로 분리하면, 한 명은 '이동 로직'을 수정하고 다른 한 명은 '전투 로직' 파일을 수정하는 식으로 동시 작업이 가능해집니다.

#### **③ 성능 최적화 (Performance)**

Linked Anim Graph는 별도의 인스턴스로 동작할 수 있어, 복잡한 로직을 병렬로 처리하는 데 유리합니다. 또한, 상황에 따라 특정 그래프를 동적으로 교체(Dynamic Linking)함으로써 불필요한 노드 연산을 줄일 수 있습니다.

### **2\. 작동 방식**

Linked Anim Graph를 사용하는 방법은 크게 두 가지입니다.

1. **상수(Static) 연결**: 특정 애니메이션 블루프린트 클래스를 직접 지정하여 해당 그래프를 실행합니다.  
2. **인터페이스(Interface) 연결**: 애니메이션 레이어 인터페이스(Animation Layer Interface)를 사용하여, 런타임에 어떤 AnimBP를 연결할지 동적으로 결정합니다.

### **3\. 주요 활용 사례**

* **무기 시스템**: 장착한 무기의 종류(검, 활, 총 등)에 따라 각각 다른 AnimBP를 제작하고, 캐릭터의 메인 AnimBP에 있는 Linked Anim Graph 노드에 현재 무기에 맞는 그래프를 할당합니다.  
* **탑승물/가탈물**: 캐릭터가 말에 타거나 차량에 탑승했을 때, 탑승 전용 애니메이션 로직을 가진 그래프를 연결합니다.  
* **복잡한 캐릭터 부분**: 망토, 머리카락, 혹은 복잡한 기계 팔과 같이 특정 부위에만 적용되는 물리/애니메이션 로직을 분리하여 관리합니다.

### **4\. Sub-Anim Instance와의 차이점**

| 구분 | Sub-Anim Instance | Linked Anim Graph (Layer) |
| :---- | :---- | :---- |
| **유연성** | 구조가 고정적임 | 인터페이스를 통해 동적 교체 가능 |
| **데이터 흐름** | 입력/출력 핀을 수동으로 연결 | 인터페이스 기반으로 자동 매핑 가능 |
| **용도** | 단순한 하위 로직 포함 | 복잡한 시스템의 모듈화 및 확장 |

### **5\. 요약**

**Linked Anim Graph**는 애니메이션 블루프린트의 "함수" 또는 "외부 라이브러리"와 같습니다. 프로젝트의 규모가 커질수록 메인 그래프의 복잡도를 낮추고 유지보수성을 높이기 위해 반드시 숙지해야 하는 강력한 노드입니다.
