# **언리얼 엔진 Linked Anim Layer 상세 가이드**

\*\*Linked Anim Layer(링크된 애니메이션 레이어)\*\*는 애니메이션 블루프린트(AnimBP)의 로직을 여러 자산으로 분리하여 관리할 수 있게 해주는 강력한 도구입니다. 이 기능은 특히 대규모 프로젝트나 다양한 무기/상태 시스템이 필요한 게임에서 필수적으로 사용됩니다.

### **1\. 핵심 역할: 애니메이션의 모듈화 (Modularity)**

가장 큰 역할은 하나의 거대한 애니메이션 블루프린트가 모든 상황(걷기, 뛰기, 칼 휘두르기, 총 쏘기 등)을 처리하지 않도록 **로직을 쪼개는 것**입니다.

* **인터페이스 기반 설계:** Animation Layer Interface 자산을 통해 "이 레이어는 입력 데이터를 받아 이런 출력을 내보낸다"라는 규격만 정해둡니다.  
* **분리된 구현:** 메인 AnimBP는 "여기에 공격 애니메이션이 들어올 자리다"라고 노드(Linked Anim Layer)만 배치하고, 실제 공격 로직은 별도의 AnimBP에서 작성합니다.

### **2\. 주요 기능 및 장점**

#### **① 동적 교체 (Runtime Swapping)**

게임 플레이 도중 캐릭터가 장착한 아이템에 따라 애니메이션 로직을 즉시 바꿀 수 있습니다.

* **예시:** 캐릭터가 '검'을 들면 검용 레이어 로직을 링크하고, '활'을 들면 활용 레이어 로직으로 교체합니다. 메인 AnimBP의 기본 이동(Locomotion) 로직은 그대로 유지하면서 특정 부분만 갈아 끼우는 방식입니다.

#### **② 협업 효율성**

여러 명의 애니메이터나 개발자가 동시에 작업하기 좋습니다.

* 한 명은 메인 이동 시스템을 수정하고, 다른 한 명은 특정 무기의 공격 레이어만 독립된 파일에서 작업할 수 있어 파일 충돌(Conflict)이 줄어듭니다.

#### **③ 재사용성**

한 번 만들어둔 애니메이션 레이어(예: 부상당한 상태의 걸음걸이 레이어)를 규격만 맞다면 여러 캐릭터 AnimBP에서 재사용할 수 있습니다.

### **3\. 작동 워크플로우**

1. **애니메이션 레이어 인터페이스 생성:** 사용할 레이어의 이름과 입력/출력 핀을 정의합니다.  
2. **메인 AnimBP 설정:** 인터페이스를 추가하고, 애님 그래프 내에 Linked Anim Layer 노드를 배치하여 위치를 잡습니다.  
3. **레이어 AnimBP 구현:** 인터페이스를 구현하는 별도의 애님 블루프린트를 만들어 실제 애니메이션 노드들을 배치합니다.  
4. **런타임 링크:** 블루프린트에서 Link Anim Class Layers 함수를 호출하여 메인 AnimBP에 특정 레이어 클래스를 할당합니다.

### **4\. Linked Anim Graph와 차이점**

| 특징 | Linked Anim Graph | Linked Anim Layer |
| :---- | :---- | :---- |
| **성격** | 단순히 다른 애님 그래프를 호출 | 인터페이스를 통한 동적 교체 |
| **유연성** | 고정된 그래프를 호출할 때 유리 | 상황에 따라 로직을 갈아 끼울 때 유리 |
| **주사용처** | 서브 그래프 관리 | 무기 시스템, 상태별 애니메이션 변화 |

### **요약**

Linked Anim Layer 노드는 메인 애니메이션 블루프린트 안에 \*\*"나중에 결정될 애니메이션 로직이 들어갈 예약석"\*\*을 만드는 노드입니다. 이를 통해 복잡한 스파게티 코드를 방지하고, 확장성 높은 애니메이션 시스템을 구축할 수 있습니다.
