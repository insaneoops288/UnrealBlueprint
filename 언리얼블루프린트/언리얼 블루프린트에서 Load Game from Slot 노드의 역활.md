# **Load Game from Slot 노드 상세 설명**

언리얼 엔진에서 게임의 진행 상태(점수, 위치, 인벤토리 등)를 로컬 저장 장치에서 읽어올 때 사용하는 가장 기본적인 노드입니다.

## **1\. 주요 역할**

이 노드는 하드디스크나 SSD에 저장된 특정 '슬롯'의 파일을 찾아 메모리로 읽어들인 뒤, 이를 **SaveGame 객체** 형태로 반환합니다.

## **2\. 입력 파라미터 (Inputs)**

* **Slot Name (String):** 불러올 저장 파일의 이름입니다. 저장할 때 사용했던 이름과 정확히 일치해야 합니다. (예: "PlayerSave\_01")  
* **User Index (Int):** 여러 사용자가 한 기기를 사용할 경우 구분하기 위한 인덱스입니다. 보통 단일 플레이어 게임에서는 0을 사용합니다.

## **3\. 출력 파라미터 (Outputs)**

* **Return Value (SaveGame Object Reference):** 성공적으로 파일을 읽었다면 해당 데이터를 담고 있는 객체를 반환합니다. 실패할 경우(파일이 없을 때 등) None을 반환합니다.

## **4\. 표준적인 사용 흐름 (Workflow)**

단순히 노드 하나만 사용하는 것이 아니라, 보통 다음과 같은 3단계 과정을 거칩니다.

### **1단계: 저장 데이터 존재 확인**

먼저 Does Save Game Exist 노드를 사용하여 해당 슬롯 이름의 파일이 실제로 존재하는지 확인하는 것이 안전합니다.

### **2단계: 데이터 로드**

데이터가 있다면 Load Game from Slot 노드를 호출합니다.

### **3단계: 형 변환 (Casting)**

반환된 객체는 범용적인 Save Game 타입이므로, 우리가 직접 만든 변수들이 들어있는 **커스텀 SaveGame 클래스**로 Cast To \[MySaveGame\] 노드를 통해 형 변환을 해주어야 합니다. 그래야 내부에 저장된 변수(HP, Level 등)에 접근할 수 있습니다.

## **5\. 블루프린트 구성 예시 (의사 코드)**

1. **Does Save Game Exist** (SlotName: "SlotA")  
2. **Branch** (True인 경우)  
3. **Load Game from Slot** (SlotName: "SlotA")  
4. **Cast to MySaveGame\_C** (성공 시)  
5. **Get Variables** (저장되어 있던 변수 값을 꺼내어 현재 게임 캐릭터나 컨트롤러에 적용)

## **6\. 주의사항**

* **반환값 확인:** 파일이 손상되었거나 없을 경우 None이 반환될 수 있으므로, 항상 Cast 노드나 Is Valid 노드를 통해 검증하는 습관이 중요합니다.  
* **슬롯 이름 관리:** 문자열(String)로 관리되므로 오타에 주의해야 합니다. 보통 변수나 열거형으로 관리하는 것이 좋습니다.  
* **비동기 로드:** 저장 데이터가 매우 클 경우 게임이 순간적으로 멈출 수 있습니다. 이럴 때는 Async Load Game from Slot 노드를 사용하는 것을 권장합니다.
