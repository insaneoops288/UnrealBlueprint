# **언리얼 엔진 블루프린트: Print String 노드의 역할**

언리얼 엔진에서 흔히 '로그를 남긴다'고 할 때 사용하는 노드는 **Print String**입니다. 이 노드는 게임 실행 중에 특정 정보를 화면이나 출력 로그(Output Log)에 표시하여 프로그램이 의도대로 작동하는지 확인하는 역할을 합니다.

## **1\. 주요 역할**

### **① 실행 흐름 확인 (Flow Control)**

특정 이벤트(예: 버튼 클릭, 충돌 발생)가 실제로 일어났는지 확인하고 싶을 때 사용합니다. "이벤트 발생\!"이라는 문구를 출력하게 설정하면, 해당 로직이 실행되었는지 즉각적으로 알 수 있습니다.

### **② 변수 값 모니터링**

게임 중 실시간으로 변하는 값(체력, 좌표, 점수 등)을 확인하는 데 유용합니다. In String 핀에 변수를 연결하면 현재 값이 화면에 출력됩니다.

### **③ 신속한 디버깅**

복잡한 중단점(Breakpoint) 설정 없이도, 로직의 중간중간에 배치하여 어느 단계에서 오류가 발생하는지 빠르게 파악할 수 있습니다.

## **2\. 노드 상세 옵션 (Details)**

Print String 노드의 화살표를 눌러 확장하면 다음과 같은 세부 설정을 조정할 수 있습니다:

| 옵션 | 설명 |
| :---- | :---- |
| **In String** | 화면이나 로그에 출력할 실제 문자열(텍스트)입니다. |
| **Print to Screen** | 체크 시, 게임 화면(Viewport) 왼쪽 상단에 텍스트를 띄웁니다. |
| **Print to Log** | 체크 시, 에디터 하단의 '출력 로그(Output Log)' 창에 기록을 남깁니다. |
| **Text Color** | 화면에 표시될 텍스트의 색상을 지정합니다. (중요도에 따라 빨강, 초록 등으로 구분 가능) |
| **Duration** | 화면에 텍스트가 머무는 시간(초 단위)입니다. |
| **Key** | 동일한 'Key' 이름을 가진 로그가 들어오면 기존 로그를 덮어씁니다. (값이 계속 변하는 데이터를 볼 때 화면이 지저분해지는 것을 방지) |

## **3\. 실무 팁**

* **로그와 화면 출력의 차이**: Print to Screen은 일시적이지만, Print to Log는 에디터 세션이 유지되는 동안 기록이 남으므로 나중에 다시 확인하기 좋습니다.  
* **Development Only**: 이 노드는 'Development' 빌드에서는 작동하지만, 최종 출시용인 'Shipping' 빌드에서는 자동으로 제외되므로 성능 걱정 없이 디버깅 용도로 사용할 수 있습니다.  
* **문자열 결합**: Append 노드와 함께 사용하면 "현재 체력: " \+ "80"과 같이 고정 텍스트와 변수 값을 합쳐서 더 명확한 로그를 볼 수 있습니다.

## **4\. 요약**

**Print String**은 개발자의 눈과 귀가 되어주는 노드입니다. 코드가 보이지 않는 게임 실행 환경에서 무슨 일이 일어나고 있는지 알려주는 가장 빠르고 간편한 방법입니다.
