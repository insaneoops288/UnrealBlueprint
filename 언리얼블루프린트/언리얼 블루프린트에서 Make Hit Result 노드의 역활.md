# **Make Hit Result 노드 완벽 가이드**

언리얼 엔진의 블루프린트에서 Make Hit Result 노드는 충돌 정보를 담는 데이터 꾸러미인 Hit Result 구조체를 **수동으로 생성**하는 역할을 합니다.

## **1\. 주요 역할 (Role)**

### **데이터 구조화 (Packing Data)**

충돌 지점, 노멀(방향), 히트된 액터 등 흩어져 있는 개별 변수들을 하나의 Hit Result 타입으로 묶어줍니다. Hit Result를 입력값으로 받는 함수나 이벤트에 데이터를 전달하기 위해 필수적입니다.

### **가상 충돌 시뮬레이션 (Simulating Hits)**

엔진의 물리 시스템(Line Trace 등)을 거치지 않고도, 로직상 "여기에서 충돌이 일어난 것으로 간주하겠다"는 처리를 할 때 사용합니다. 예를 들어, 커스텀 발사체 로직이나 특정 좌표를 기반으로 한 피격 판정을 직접 만들 때 유용합니다.

### **데이터 수정 및 재조립 (Modification)**

Break Hit Result 노드로 기존의 충돌 정보를 분해한 뒤, 특정 값(예: 위치나 노멀)만 살짝 수정하여 다시 합칠 때 사용합니다.

## **2\. 주요 입력 항목 (Input Pins)**

이 노드에는 매우 많은 핀이 있지만, 실전에서 자주 쓰이는 핵심 항목은 다음과 같습니다.

| 항목 | 설명 |
| :---- | :---- |
| **Blocking Hit** | 실제로 무언가에 부딪혔는지 여부 (Boolean) |
| **Location** | 레이(Ray)가 이동 중 충돌체 표면에 닿았을 때의 위치 |
| **Impact Point** | 실제 물리적 접촉이 일어난 지점 (대부분 Location과 비슷하지만 미세하게 다를 수 있음) |
| **Normal** | 충돌한 표면의 법선 벡터 (튕겨 나갈 방향 등을 계산할 때 사용) |
| **Phys Mat** | 충돌한 표면의 물리 재질 (발소리, 파티클 종류 결정 시 중요) |
| **Hit Actor** | 충돌한 대상 액터 참조 |
| **Hit Component** | 충돌한 구체적인 컴포넌트 |
| **Bone Name** | (스켈레탈 메시인 경우) 부딪힌 뼈의 이름 |

## **3\. 실전 활용 예시**

### **A. Apply Point Damage와 함께 사용**

Apply Point Damage 함수는 입력값으로 Hit Info (Hit Result)를 요구합니다. 만약 라인 트레이스를 쓰지 않고 커스텀한 폭발 로직으로 데미지를 전달하고 싶다면, Make Hit Result를 사용해 충돌 지점과 방향을 직접 입력해 주어야 합니다.

### **B. 네트워크 동기화 및 데이터 전달**

서버와 클라이언트 간에 충돌 위치를 공유할 때, 개별 벡터값들 대신 Hit Result 하나로 통일해서 함수 인자(Parameter)로 넘기면 관리가 훨씬 효율적입니다.

### **C. 디버깅 및 테스트**

특정 위치에서 충돌이 일어났을 때의 반응을 테스트하고 싶을 때, 강제로 Make Hit Result에 값을 채워넣어 로직이 정상 작동하는지 확인할 수 있습니다.

## **4\. 요약: Break vs Make**

* **Break Hit Result**: 덩어리진 충돌 정보를 **풀어서** 개별 정보(위치, 액터 등)를 꺼낼 때 사용. (가장 많이 쓰임)  
* **Make Hit Result**: 개별 정보를 **합쳐서** 충돌 정보 덩어리를 만들 때 사용.

Make Hit Result는 주로 **시스템 간에 데이터를 넘겨주거나, 물리 엔진의 결과를 커스텀하게 조작하고 싶을 때** 사용하는 고급 노드라고 이해하시면 됩니다.
