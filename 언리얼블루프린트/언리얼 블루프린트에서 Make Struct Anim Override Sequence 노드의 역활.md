# **Make Struct Anim Override Sequence 노드 가이드**

언리얼 엔진 5의 최신 애니메이션 시스템(특히 **AnimNext**)에서는 애니메이션 데이터를 더욱 유연하고 구조적으로 관리하기 위해 '구조체(Struct)'를 적극적으로 활용합니다. Make Struct Anim Override Sequence 노드는 그 중심에 있는 데이터 생성 도구입니다.

## **1\. 노드의 주요 역할**

이 노드는 단일 애니메이션 시퀀스(Anim Sequence)를 **애니메이션 오버라이드 시스템이 인식할 수 있는 구조체 형식**으로 포장(Wrapping)하는 역할을 합니다.

* **데이터 형식 변환**: 단순한 UAnimSequence 객체 참조를 오버라이드 로직에서 처리 가능한 AnimOverrideSequence 구조체로 변환합니다.  
* **동적 애니메이션 교체**: 게임 실행 도중 특정 조건(예: 부상 상태, 무기 장착 등)에 따라 기본 애니메이션을 다른 애니메이션으로 갈아끼울 때 사용됩니다.  
* **매개변수 전달**: 단순히 시퀀스만 전달하는 것이 아니라, 오버라이드 시 필요한 추가 설정값들을 함께 묶어서 전달할 수 있도록 설계되어 있습니다.

## **2\. 주요 입력 및 출력 항목**

보통 이 노드는 다음과 같은 구성을 가집니다.

| 항목 | 유형 | 설명 |
| :---- | :---- | :---- |
| **Sequence** | Anim Sequence (Object Reference) | 오버라이드하여 실제로 재생할 애니메이션 에셋을 선택합니다. |
| **(추가 옵션)** | Float / Bool 등 | 엔진 버전에 따라 재생 속도, 루프 여부 등의 추가 파라미터가 포함될 수 있습니다. |
| **Return Value** | Anim Override Sequence (Struct) | 생성된 오버라이드용 구조체 데이터입니다. |

## **3\. 왜 이 노드를 사용하는가?**

기존의 애니메이션 블루프린트(AnimBP) 방식에서는 모든 상태 변화를 '스테이트 머신(State Machine)'이나 '블렌드 스페이스(Blend Space)'에 하드코딩해야 했습니다. 하지만 이 구조체를 활용하면 다음과 같은 장점이 있습니다.

1. **확장성**: 수십 개의 무기 종류가 있더라도, 각 무기마다 AnimBP를 새로 만들 필요 없이 애니메이션 구조체 데이터만 교체하면 됩니다.  
2. **데이터 기반 설계 (Data-Driven)**: 데이터 테이블(Data Table)이나 에셋 유틸리티를 통해 애니메이션 리스트를 관리하고, 이를 Make Struct 노드를 통해 실시간으로 주입할 수 있습니다.  
3. **AnimNext와의 호환성**: 언리얼의 차세대 애니메이션 그래프 시스템인 AnimNext에서는 노드 간 데이터 전달 시 이러한 구조체 형식을 표준으로 사용합니다.

## **4\. 실전 활용 시나리오**

1. **캐릭터 부상 시스템**: 캐릭터의 체력이 일정 이하로 떨어지면, Make Struct Anim Override Sequence를 통해 '걷기' 시퀀스를 '절뚝거리며 걷기' 시퀀스로 할당하여 오버라이드 변수에 저장합니다.  
2. **무기별 모션 변경**: 검을 들었을 때와 총을 들었을 때의 기본 대기(Idle) 포즈를 데이터 테이블에서 가져와 이 노드로 구조체를 만든 뒤, 애니메이션 그래프에 전달합니다.

## **5\. 주의사항**

* 이 노드는 주로 **애니메이션 레이어 인터페이스**나 **AnimNext 그래프**와 함께 사용될 때 진가를 발휘합니다. 일반적인 간단한 애니메이션 재생에는 Play Animation 노드만으로 충분할 수 있습니다.  
* 해당 노드가 보이지 않는다면 관련 플러그인(AnimNext 등)이 활성화되어 있는지 확인해야 합니다.
