# **언리얼 엔진 블루프린트: MakeArray 노드 상세 가이드**

MakeArray 노드는 블루프린트 그래프 내에서 개별적인 데이터 항목들을 모아 하나의 \*\*배열(Array)\*\*로 합쳐주는 역할을 합니다. 변수로 미리 배열을 만들어 두지 않고도, 로직 중간에 즉석에서 리스트를 생성해야 할 때 매우 유용합니다.

## **1\. 주요 역할 및 특징**

* **즉석 배열 생성:** 변수 패널에서 배열 변수를 따로 선언하지 않아도, 노드 연결만으로 임시 배열을 만들 수 있습니다.  
* **가변 입력 핀:** 노드 왼쪽의 Add Pin (+) 버튼을 눌러 원하는 만큼 입력 항목을 늘릴 수 있습니다.  
* **데이터 타입 자동 지정:** 첫 번째 핀에 연결되는 데이터의 타입(예: Integer, Float, Actor Reference 등)에 따라 배열 전체의 타입이 결정됩니다.

## **2\. 사용 방법**

1. **노드 호출:** 그래프 빈 곳에서 우클릭 후 Make Array를 검색하여 배치합니다.  
2. **핀 추가:** 노드 하단의 Add Pin을 클릭하여 배열에 포함할 요소의 개수를 설정합니다.  
3. **데이터 연결:** 각 핀에 배열에 넣고 싶은 값이나 변수를 연결합니다.  
4. **결과 활용:** 오른쪽의 출력 핀(Array)을 배열을 입력으로 받는 다른 노드(예: ForEachLoop, Contains, Get 등)에 연결합니다.

## **3\. 기술적 주의사항**

### **타입 일관성 (Type Consistency)**

MakeArray 노드에 연결되는 모든 입력 값은 **동일한 타입**이어야 합니다. 예를 들어, 0번 핀에 정수(Integer)를 연결했다면 1번 핀에 실수(Float)를 직접 연결할 수 없으며, 형변환(Conversion)이 필요합니다.

### **메모리 및 성능**

* MakeArray는 호출될 때마다 해당 요소들을 가진 **새로운 임시 배열**을 생성합니다.  
* 매우 큰 배열을 매 프레임(Tick)마다 MakeArray로 만드는 것은 성능에 좋지 않을 수 있으므로, 내용이 변하지 않는다면 변수로 캐싱(Caching)하여 사용하는 것이 좋습니다.

## **4\. 활용 예시**

* **특정 액터 그룹 순회:** 여러 개의 액터 변수를 하나로 묶어 ForEachLoop로 한꺼번에 로직을 처리할 때 사용합니다.  
* **랜덤 선택:** 여러 값 중 하나를 무작위로 뽑고 싶을 때, MakeArray로 묶은 뒤 Random Array Item 노드를 연결합니다.  
* **함수 인자 전달:** 배열을 입력 파라미터로 받는 함수나 이벤트에 여러 값을 한꺼번에 보낼 때 유용합니다.

## **5\. 요약**

| 특징 | 내용 |
| :---- | :---- |
| **입력** | 다수의 단일 데이터 항목 (가변적) |
| **출력** | 입력된 항목들이 순서대로 담긴 단일 배열 |
| **장점** | 별도의 변수 선언 없이 로직 중간에 리스트 생성 가능 |
| **제한** | 모든 입력 핀의 데이터 타입이 동일해야 함 |
