# **Map Range Clamped 노드 완벽 이해하기**

언리얼 엔진의 Map Range Clamped 노드는 한 범위의 값을 다른 범위의 값으로 \*\*비례적으로 매핑(Mapping)\*\*하고, 결과값이 범위를 벗어나지 않도록 \*\*제한(Clamping)\*\*하는 함수입니다.

## **1\. 노드 구조와 파라미터**

이 노드는 총 5개의 입력 핀을 가집니다.

| 입력 항목 | 설명 |
| :---- | :---- |
| **Value** | 변환하고자 하는 현재 입력값입니다. |
| **In Range A** | 원래 범위의 시작점 (최솟값)입니다. |
| **In Range B** | 원래 범위의 끝점 (최대값)입니다. |
| **Out Range A** | 변환될 결과 범위의 시작점입니다. |
| **Out Range B** | 변환될 결과 범위의 끝점입니다. |

## **2\. 작동 원리 (비례식)**

기본적으로 이 노드는 다음과 같은 수학적 비례를 계산합니다.

1. **입력 단계**: Value가 In Range A와 B 사이에서 어느 정도 위치(백분율)에 있는지 계산합니다.  
2. **출력 단계**: 그 백분율을 그대로 Out Range A와 B 사이에 적용하여 결과값을 산출합니다.  
3. **클램핑(Clamped) 단계**: 만약 계산된 결과값이 Out Range A보다 작거나 Out Range B보다 크다면, 해당 범위를 벗어나지 않도록 강제로 고정합니다.

### **예시**

* **입력 범위**: 0 \~ 100  
* **출력 범위**: 0 \~ 1.0  
* **Value가 50일 때**: 입력 범위의 딱 중간이므로, 출력값은 **0.5**가 됩니다.  
* **Value가 150일 때**: 입력 범위(100)를 넘었지만, **Clamped** 노드이므로 출력값은 최대치인 **1.0**에 고정됩니다.

## **3\. 실무 활용 사례**

### **① 캐릭터 체력 게이지 (UI)**

* **상황**: 체력이 0 \~ 500인데, UI Progress Bar는 0.0 \~ 1.0 사이의 값만 받습니다.  
* **설정**:  
  * In Range A/B: 0 / 500  
  * Out Range A/B: 0 / 1.0  
* **결과**: 체력이 250이면 바가 절반(0.5) 차오릅니다.

### **② 거리 기반 효과 (사운드/이펙트)**

* **상황**: 적과의 거리가 가까워질수록 화면을 더 붉게 만들고 싶을 때.  
* **설정**:  
  * In Range A/B: 1000(멀 때) / 200(가까울 때)  
  * Out Range A/B: 0.0(안 붉음) / 1.0(매우 붉음)  
* **특징**: In Range A가 B보다 커도 작동합니다(역전 매핑).

## **4\. Map Range Unclamped와의 차이점**

* **Map Range Clamped**: 결과값이 무조건 Out Range A와 B 사이로 제한됩니다. 안전하지만 유연성은 적습니다.  
* **Map Range Unclamped**: 입력값이 범위를 벗어나면 출력값도 비례해서 범위를 뚫고 나갑니다. (예: 0\~100을 0\~1로 매핑 시 200을 넣으면 2.0이 나옴)

## **5\. 요약: 언제 사용하나요?**

**"A라는 기준의 숫자를 B라는 기준의 숫자로 바꾸고 싶은데, 절대 정해진 범위를 넘어가면 안 될 때"** 사용합니다.

수학적으로 복잡한 비례식을 세울 필요 없이 노드 하나로 해결할 수 있어 인터페이스(UI), 카메라 흔들림 강도, 사운드 볼륨 조절 등 거의 모든 블루프린트 로직에서 필수적으로 사용됩니다.
