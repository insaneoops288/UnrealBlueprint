# **Map Range Clamped 노드 완벽 이해하기**

언리얼 엔진 블루프린트에서 **Map Range Clamped** 노드는 수학적 계산을 시각적으로 단순화해주는 핵심 노드 중 하나입니다. 한 마디로 정의하자면 \*\*"A 범위에 속한 값을 B 범위의 비율에 맞춰 변환하되, 범위를 넘어서지 않게 해주는 도구"\*\*입니다.

### **1\. 주요 핀(Pin) 설명**

노드는 크게 5개의 입력 핀과 1개의 출력 핀으로 구성됩니다.

| 이름 | 타입 | 역할 |
| :---- | :---- | :---- |
| **Value** | Float | 변환하고자 하는 실제 입력 값입니다. |
| **In Range A** | Float | 입력 범위의 시작점입니다. |
| **In Range B** | Float | 입력 범위의 끝점입니다. |
| **Out Range A** | Float | 변환될 결과 범위의 시작점입니다. (Value가 In A일 때 이 값이 출력됨) |
| **Out Range B** | Float | 변환될 결과 범위의 끝점입니다. (Value가 In B일 때 이 값이 출력됨) |
| **Return Value** | Float | 변환이 완료된 최종 결과값입니다. |

### **2\. "Clamped"의 의미**

이 노드의 가장 큰 특징은 이름 뒤에 붙은 **Clamped**입니다.

* **Map Range Unclamped:** 입력값이 In Range를 벗어나면 출력값도 Out Range를 비례해서 벗어납니다.  
* **Map Range Clamped:** 입력값이 In Range를 벗어나더라도, 출력값은 무조건 Out Range A와 Out Range B 사이의 값으로 고정됩니다.

**예시:** In(0\~100) \-\> Out(0\~1) 설정 시

* 입력값이 **150**이라면?  
  * Unclamped: 1.5 출력  
  * **Clamped: 1.0 출력 (최대치로 고정)**

### **3\. 실무 활용 사례**

#### **① 캐릭터 체력바 (HP Bar)**

캐릭터의 체력이 0에서 500일 때, UI 머티리얼이나 프로그레스 바는 보통 0.0 \~ 1.0 사이의 값을 사용합니다.

* **Value:** 현재 체력 (예: 250\)  
* **In Range:** 0 / 500  
* **Out Range:** 0.0 / 1.0  
* **결과:** 0.5 (절반)

#### **② 거리에 따른 불투명도 (Opacity)**

플레이어가 특정 물체에 가까워질수록 물체가 투명해지게 만들고 싶을 때 사용합니다.

* **Value:** 플레이어와 물체 사이의 거리  
* **In Range:** 200 (가까움) / 1000 (멂)  
* **Out Range:** 0.0 (투명) / 1.0 (불투명)  
* **효과:** 200유닛보다 가까워지면 완전히 투명(0.0)하게 유지되고, 그 사이에서는 서서히 나타납니다.

#### **③ 조이스틱/마우스 감도 조절**

입력 장치의 물리적 수치를 게임 내 회전 값으로 변환할 때 유용합니다.

* **Value:** 축 입력값 (-1.0 \~ 1.0)  
* **In Range:** \-1.0 / 1.0  
* **Out Range:** \-90.0 / 90.0 (도 단위 회전)

### **4\. 수학적 원리 (참고용)**

이 노드는 내부적으로 다음과 같은 공식을 수행합니다.

1. **정규화 (Normalize):** (Value \- In Range A) / (In Range B \- In Range A)  
2. **보간 (Lerp):** Out Range A \+ (정규화된 값) \* (Out Range B \- Out Range A)  
3. **제한 (Clamp):** 결과를 Out Range 최소/최대값 사이로 가둠.

### **5\. 팁**

* **역전 가능:** Out Range A에 1.0을 넣고 Out Range B에 0.0을 넣으면 값이 반대로 뒤집힙니다. (거리가 멀어질수록 작아지게 만드는 등)  
* **성능:** 복잡한 If 문이나 Clamp 노드를 여러 개 연결할 필요 없이 이 노드 하나로 해결되므로 가독성과 유지보수에 매우 좋습니다.
