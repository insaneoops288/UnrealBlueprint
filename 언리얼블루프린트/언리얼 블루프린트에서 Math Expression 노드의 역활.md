# **Math Expression 노드의 역할과 활용**

언리얼 엔진 블루프린트에서 **Math Expression** 노드는 여러 개의 수학 연산 노드(더하기, 빼기, 곱하기 등)를 하나의 노드로 결합하여 복잡한 공식을 간단하게 표현할 수 있게 해줍니다.

## **1\. 주요 역할**

### **코드 가독성 향상 (스파게티 코드 방지)**

일반적으로 복잡한 공식을 블루프린트로 구현하면 수많은 사칙연산 노드와 와이어가 얽혀 소위 '스파게티' 상태가 되기 쉽습니다. Math Expression 노드는 이를 한 줄의 텍스트로 해결하여 그래프를 깔끔하게 유지합니다.

### **자동 핀 생성**

표현식에 변수 이름을 입력하면, 노드에 해당 이름의 **입력 핀이 자동으로 생성**됩니다. 예를 들어 (A \+ B) \* C라고 입력하면 A, B, C라는 이름의 입력 핀이 즉시 나타납니다.

### **서브 그래프 생성**

노드를 더블 클릭하면 내부적으로 생성된 연산 그래프를 확인할 수 있습니다. 엔진이 텍스트를 분석해 최적화된 노드 구조를 자동으로 만들어주는 방식입니다.

## **2\. 사용 예시**

예를 들어, 캐릭터의 체력 비율을 계산하고 특정 계수를 곱하는 공식이 있다고 가정해 봅시다.

* **일반 방식:** CurrentHP 노드 → MaxHP 노드와 연결된 Division 노드 → Float \* Float 노드 연결  
* **Math Expression 방식:** 노드에 (CurrentHP / MaxHP) \* Modifier 입력 끝.

## **3\. 지원하는 기능 및 문구**

Math Expression 노드는 단순 사칙연산 외에도 다양한 수학 함수를 지원합니다.

* **기본 연산:** \+, \-, \*, /, %(나머지)  
* **삼각 함수:** sin, cos, tan, asin, acos 등  
* **수학 상수:** PI, e  
* **기타 함수:** abs(절대값), pow(제곱), sqrt(제곱근), log, min, max, clamp

## **4\. 사용 시 주의사항**

1. **변수 이름 중복:** 블루프린트 내에 이미 존재하는 변수 이름과 표현식 내의 이름을 동일하게 사용하면 자동으로 연결을 시도하지만, 때로는 혼동을 줄 수 있으므로 명확한 이름을 사용하는 것이 좋습니다.  
2. **논리 연산 제한:** 주로 수학적 계산에 최적화되어 있어, Branch 같은 조건부 로직이나 복잡한 객체 참조보다는 **수치 계산**에 집중해서 사용하는 것이 효과적입니다.  
3. **결과 타입:** 입력값의 타입(Float, Integer 등)에 따라 결과 타입이 결정됩니다.

## **5\. 결론**

Math Expression 노드는 \*\*"복잡한 공식을 인간이 읽기 쉬운 텍스트로 작성하고, 블루프린트 그래프를 간결하게 유지하는 도구"\*\*입니다. 수학 연산이 3개 이상 겹치기 시작한다면 이 노드를 사용하는 것을 적극 권장합니다.
