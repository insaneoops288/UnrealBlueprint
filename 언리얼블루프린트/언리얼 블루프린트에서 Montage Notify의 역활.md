# **언리얼 엔진 몽타주 노티파이(Montage Notify) 상세 가이드**

애니메이션 몽타주 내에서 사용되는 **Montage Notify**는 애니메이션의 시각적 흐름과 게임의 논리적 실행을 연결하는 핵심 요소입니다.

## **1\. 몽타주 노티파이의 특징**

일반적인 Anim Notify는 별도의 블루프린트 클래스를 만들거나 애님 인스턴스(Anim Instance)에서 이벤트를 받아야 하지만, Montage Notify는 **몽타주를 재생시킨 노드에서 직접 이벤트를 처리**할 수 있게 설계되었습니다.

* **Notify Name 사용**: 노티파이에 이름을 부여하여, 하나의 몽타주 안에서 여러 개의 노티파이를 구분할 수 있습니다. (예: "Hit", "SpawnEffect", "EnableCombo")  
* **즉각적인 반응**: 몽타주 재생 노드(Play Montage)의 실행 흐름(Execution Flow) 상에서 바로 로직을 이어갈 수 있어 가독성이 좋습니다.

## **2\. 사용 방법 (Workflow)**

1. **몽타주 에디터**:  
   * 노티파이 트랙에서 우클릭 \-\> Add Notify... \-\> Montage Notify 선택.  
   * 디테일 패널의 **Notify Name** 항목에 고유한 이름(예: AttackImpact)을 입력합니다.  
2. **블루프린트 그래프**:  
   * Play Montage and Wait 노드를 배치합니다.  
   * On Notify Begin 핀에 로직을 연결합니다.  
   * Notify Name 핀에서 나오는 값을 Switch on Name 또는 Equal (Name) 노드로 확인하여 원하는 로직을 실행합니다.

## **3\. 일반 Anim Notify vs Montage Notify 비교**

| 구분 | 일반 Anim Notify | Montage Notify |
| :---- | :---- | :---- |
| **이벤트 위치** | 애님 인스턴스(Anim Instance) | 몽타주 재생 노드(Play Montage) |
| **재사용성** | 여러 애니메이션에서 공유 가능 | 해당 몽타주 재생 로직에 국한됨 |
| **주요 용도** | 사운드 재생, 파티클 생성 등 범용적 효과 | 공격 판정, 콤보 입력 창 활성화 등 로직 제어 |
| **클래스 생성** | 필요함 (선택 사항) | 필요 없음 (이름만 지정) |

## **4\. 실전 활용 예시: 콤보 공격**

캐릭터가 3단 공격을 할 때, 1단 공격 애니메이션이 끝나기 직전에 "다음 입력을 받을 수 있는 구간"을 설정하고 싶다면 Montage Notify를 해당 구간의 시작과 끝에 배치합니다. 블루프린트에서는 이 신호를 받아 '입력 가능 플래그'를 켜고 끄는 식으로 구현할 수 있습니다.

**주의사항**: Play Montage 노드가 아닌 일반적인 애니메이션 시퀀스 재생 중에는 Montage Notify가 작동하지 않습니다. 반드시 **몽타주(Montage)** 자산 내에서 사용해야 합니다.
