# **Multi Sphere Trace By Channel 노드 상세 가이드**

Multi Sphere Trace By Channel은 지정된 시작점부터 끝점까지 구체 모양의 충돌 영역을 이동시키며, 설정한 \*\*콜리전 채널(Collision Channel)\*\*에 반응하는 모든 물체를 찾아 배열(Array) 형태로 반환하는 노드입니다.

## **1\. 주요 역할**

* **범위 내 다수 대상 감지**: 단일 히트(Single Hit)가 아닌, 경로 상에 있는 모든 유효한 대상을 한 번에 찾습니다.  
* **공간 쿼리**: 레이저(Line Trace)보다 넓은 면적을 체크해야 할 때 유용합니다.  
* **필터링**: 특정 채널(예: Visibility, Camera, Pawn 등)에 설정된 물체만 선별적으로 감지합니다.

## **2\. 주요 입력 파라미터 (Inputs)**

| 파라미터 | 설명 |
| :---- | :---- |
| **Start / End** | 구체 트레이스가 시작되고 끝나는 3D 공간의 좌표입니다. |
| **Radius** | 트레이스에 사용할 구체의 반지름입니다. 이 값이 클수록 더 넓은 범위를 체크합니다. |
| **Trace Channel** | 어떤 콜리전 채널을 대상으로 할지 결정합니다. (예: Pawn 채널로 설정하면 캐릭터들만 감지) |
| **Actors to Ignore** | 트레이스 결과에서 제외할 액터들의 목록입니다. (자기 자신을 제외할 때 자주 사용) |
| **Draw Debug Type** | 화면에 트레이스 경로를 시각적으로 표시합니다. (None, For One Frame, For Duration, Persistent) |

## **3\. 주요 출력 파라미터 (Outputs)**

| 파라미터 | 설명 |
| :---- | :---- |
| **Return Value** | 하나라도 부딪힌 물체가 있다면 True, 없으면 False를 반환합니다. |
| **Out Hits** | 부딪힌 모든 물체들에 대한 정보를 담은 **Hit Result 구조체 배열**입니다. |

## **4\. Single Sphere Trace와의 차이점**

* **Single Sphere Trace**: 경로 상에서 **가장 먼저 부딪힌 첫 번째 물체**의 정보만 가져오고 종료됩니다.  
* **Multi Sphere Trace**: 경로 상에서 부딪힌 **모든 물체**를 배열에 담습니다.  
  * *주의*: 벽 뒤에 숨은 적까지 모두 감지하고 싶을 때 주로 사용됩니다.

## **5\. 실전 활용 사례**

1. **광역 데미지(AoE) 판정**: 폭발이 일어났을 때 특정 반경 내에 있는 모든 적을 찾아 데미지를 입힐 때 사용합니다.  
2. **복합 장애물 감지**: 얇은 풀숲이나 투명한 벽 등을 관통하여 그 뒤에 있는 실제 목표물까지 모두 찾아야 할 때 사용합니다.  
3. **근접 공격 판정**: 캐릭터가 휘두르는 칼의 궤적을 두꺼운 구체 형태로 그려서, 궤적에 걸리는 모든 적을 타격할 때 유용합니다.

## **6\. 블루프린트 구성 팁**

* **For Each Loop 활용**: Out Hits 배열을 For Each Loop 노드에 연결하여, 감지된 각 액터에 대해 순차적으로 로직(예: 데미지 적용, 이펙트 생성)을 실행합니다.  
* **Break Hit Result**: 배열의 요소를 분해하여 Hit Actor, Hit Location, Hit Normal 등의 세부 정보를 추출할 수 있습니다.
