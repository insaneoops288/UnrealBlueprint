# **Unreal Engine Blueprint: Normalize Axis 노드 완벽 이해**

Normalize Axis 노드는 입력된 부동 소수점(Float) 각도 값을 받아, 이를 기하학적으로 동일하지만 수치적으로는 **\-180.0 \~ 180.0** 범위 내에 있는 값으로 재계산하여 반환합니다.

## **1\. 주요 역할 (Core Role)**

회전값은 360도를 주기로 반복됩니다. 예를 들어, **270도**는 반대 방향으로 회전했을 때 **\-90도**와 위치가 같습니다.

하지만 프로그래밍 연산 시 270이라는 숫자와 \-90이라는 숫자는 큰 차이를 만듭니다. Normalize Axis는 이를 하나의 표준 범위(-180\~180)로 통일하여 수학적 계산을 용이하게 만듭니다.

### **수치 변환 예시**

| 입력 값 (Input) | 결과 값 (Output) | 설명 |
| :---- | :---- | :---- |
| **190.0** | **\-170.0** | 180도를 넘었으므로 반대 방향의 음수 값으로 변환 |
| **270.0** | **\-90.0** | 시계 방향 270도는 반시계 방향 90도와 동일 |
| **450.0** | **90.0** | 한 바퀴(360도)를 제외한 나머지 값 |
| **\-200.0** | **160.0** | \-180도보다 작으므로 양수 범위로 변환 |

## **2\. 왜 사용하는가? (Why use it?)**

### **① 최단 경로 회전 계산**

캐릭터가 현재 170도를 바라보고 있는데, 목표 방향이 \-170도라고 가정해 봅시다.

* **Normalize를 안 할 경우:** 숫자의 차이가 340도이므로, 캐릭터가 먼 길을 돌아 거의 한 바퀴를 회전합니다.  
* **Normalize를 할 경우:** 170도와 \-190도(변환된 값) 사이의 차이를 계산하거나, 두 값의 차이를 다시 Normalize 하면 아주 짧은 거리(20도)만 회전하면 된다는 것을 알 수 있습니다.

### **② 수치 오버플로우 방지**

회전값이 계속 누적되어 1000도, 5000도와 같이 커지는 것을 방지하고 항상 일정한 범위 내에서 관리할 수 있게 해줍니다.

## **3\. 주요 활용 사례 (Common Use Cases)**

1. **회전 오차(Delta) 계산:** 두 회전 값의 차이를 구할 때 사용합니다. (예: Target Rotation \- Current Rotation 결과를 Normalize Axis에 연결)  
2. **카메라 Pitch 제한:** 카메라가 위아래로 회전할 때 값이 180도를 넘어가서 뒤집히는 현상을 제어하기 위해 사용합니다.  
3. **조준 보정 (Aim Assist):** 적과의 각도 차이를 계산하여 조준선을 보정할 때, 가장 가까운 회전 방향을 판단하기 위해 필수적입니다.

## **4\. 유사 노드와 차이점**

* **Normalize Axis:** 단순 Float 값(각도) 하나를 처리합니다.  
* **UKismetMathLibrary::NormalizeRotator:** Rotator(Roll, Pitch, Yaw 전체)를 입력받아 각 축을 모두 \-180\~180 범위로 정규화합니다.  
* **Vector Normalization (Normalize):** 각도가 아니라 **벡터의 길이(Magnitude)를 1로** 만드는 노드이므로 전혀 다른 기능입니다. 혼동하지 않도록 주의해야 합니다.

### **요약**

Normalize Axis는 \*\*"각도를 사람이 이해하기 쉽고 수학적으로 안전한 \-180 \~ 180도 사이의 값으로 깔끔하게 정리해 주는 필터"\*\*라고 이해하시면 됩니다.
