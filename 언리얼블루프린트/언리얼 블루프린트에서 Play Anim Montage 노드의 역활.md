# **Play Anim Montage 노드의 역할과 기능**

언리얼 엔진의 **Play Anim Montage** 노드는 스켈레탈 메시 컴포넌트에서 **애니메이션 몽타주(Animation Montage)** 자산을 재생하는 데 사용됩니다. 단순히 애니메이션을 트는 것을 넘어, 상태 머신(State Machine) 외부에서 특정 로직에 따라 애니메이션을 강제로 실행하거나 제어할 때 필수적입니다.

## **1\. 주요 역할 (Core Roles)**

### **① 상태 머신(State Machine) 우회 및 중단**

일반적인 이동(걷기, 뛰기)은 애니메이션 블루프린트의 상태 머신에서 처리되지만, 공격, 피격, 스킬 사용과 같은 일회성 액션은 상태 머신을 복잡하게 만들 수 있습니다. Play Anim Montage를 사용하면 현재 상태 머신의 애니메이션을 일시적으로 덮어쓰고 몽타주를 재생할 수 있습니다.

### **② 슬롯(Slot) 기반의 블렌딩**

몽타주는 내부적으로 **슬롯(Slot)** 시스템을 사용합니다. 예를 들어 'UpperBody' 슬롯에 공격 몽타주를 할당하면, 하체는 달리고 있더라도 상체만 공격 애니메이션을 재생하는 식의 **레이어드 애니메이션** 구현이 가능합니다.

### **③ 섹션(Section) 제어**

몽타주는 여러 개의 '섹션'으로 나뉠 수 있습니다. 이 노드를 통해 특정 섹션부터 재생을 시작하거나, 콤보 공격 시스템에서 다음 섹션으로 점프하는 등의 정교한 흐름 제어가 가능합니다.

### **④ 루트 모션(Root Motion) 지원**

캐릭터의 실제 위치가 애니메이션에 따라 이동해야 하는 경우(예: 구르기, 돌진), 몽타주는 루트 모션 데이터를 가장 효과적으로 처리합니다.

## **2\. 입력 파라미터 (Input Pins)**

| 파라미터 | 설명 |
| :---- | :---- |
| **In Anim Montage** | 재생할 몽타주 자산을 선택합니다. |
| **In Play Rate** | 재생 속도입니다. (1.0은 정상 속도, 2.0은 2배속) |
| **In Start Section Name** | 몽타주 내부의 특정 섹션 이름부터 재생하고 싶을 때 입력합니다. (비워두면 처음부터 재생) |
| **Skeletal Mesh Component** | 애니메이션을 재생할 대상 메시입니다. (기본적으로 'Mesh'가 연결됨) |

## **3\. 출력 및 반환값 (Outputs)**

* **Return Value (Float):** 재생될 애니메이션의 총 길이(초)를 반환합니다. 만약 몽타주 재생에 실패하면 0.0을 반환합니다.  
* **실행 핀:** 노드가 호출되는 즉시 다음 로직으로 넘어갑니다. (애니메이션이 끝날 때까지 기다리지 않습니다.)

## **4\. Play Animation 노드 vs Play Anim Montage**

| 특징 | Play Animation (일반) | Play Anim Montage |
| :---- | :---- | :---- |
| **주 목적** | 단순한 일회성 애니메이션 재생 | 복잡한 시퀀스, 콤보, 섹션 제어 |
| **제어력** | 낮음 (그냥 재생만 함) | 높음 (섹션 이동, 슬롯 블렌딩 가능) |
| **애니 블루프린트** | 애니 블루프린트 설정을 무시할 수 있음 | 애니 블루프린트 내 **Slot** 노드와 상호작용 |
| **비트(Notify)** | 사용 가능하지만 제한적임 | 몽타주 노티파이를 통해 정교한 타이밍 조절 가능 |

## **5\. 실무 활용 팁**

1. **슬롯 배치:** 애니메이션 블루프린트의 애님 그래프(AnimGraph) 내에 반드시 Slot 'DefaultSlot' (또는 지정한 슬롯 이름) 노드가 배치되어 있어야 몽타주가 화면에 출력됩니다.  
2. **중단 처리:** 새로운 몽타주를 재생하면 기존에 재생 중이던 몽타주는 설정된 'Blend Out' 시간에 따라 부드럽게 중단됩니다.  
3. **콤보 시스템:** Montage Set Next Section 노드와 함께 사용하여 사용자의 입력 타이밍에 따라 다음 공격 동작으로 연결하는 로직을 짤 때 핵심적으로 사용됩니다.

### **결론**

Play Anim Montage는 캐릭터의 \*\*능동적인 액션(공격, 상호작용, 스킬)\*\*을 구현할 때 가장 먼저 고려해야 할 노드입니다. 상태 머신의 복잡도를 낮추면서도 강력한 제어 기능을 제공합니다.
