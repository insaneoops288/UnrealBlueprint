# **언리얼 엔진 Property Access (프로퍼티 액세스) 가이드**

프로퍼티 액세스는 외부 객체(예: 캐릭터, 폰)의 변수나 함수 결과값에 안전하고 빠르게 접근하기 위해 사용되는 내부 시스템입니다. 주로 \*\*애니메이션 블루프린트(AnimBP)\*\*에서 성능 최적화와 멀티스레딩 지원을 위해 핵심적인 역할을 수행합니다.

## **1\. 핵심 역할**

### **① 성능 최적화 (Fast Path 지원)**

기존의 방식은 Event Blueprint Update Animation 이벤트에서 캐스팅(Cast)을 하고 변수를 가져와 로컬 변수에 저장하는 과정을 거칩니다. 이 과정은 매 프레임 \*\*블루프린트 가상 머신(VM)\*\*을 실행시키므로 CPU 비용이 발생합니다.

* **역할:** 프로퍼티 액세스를 사용하면 VM을 거치지 않고 데이터를 직접 복사하는 **패스트 패스(Fast Path)** 최적화가 활성화되어 CPU 부하를 크게 줄입니다.

### **② 스레드 안전성 (Thread Safety) 보장**

언리얼 엔진은 성능을 위해 애니메이션 업데이트를 게임 스레드가 아닌 별도의 \*\*워커 스레드(Worker Thread)\*\*에서 병렬로 처리합니다.

* **역할:** 게임 스레드에 있는 캐릭터의 변수에 직접 접근하면 스레드 간 충돌(Race Condition)이 발생할 수 있습니다. 프로퍼티 액세스는 데이터를 안전하게 '스냅샷' 찍듯이 가져오거나 스레드 세이프한 방식으로 연결하여 크래시를 방지합니다.

### **③ 그래프 단순화 및 유지보수**

여러 노드를 거쳐 변수를 전달하는 대신, 노드 자체의 핀에 직접 변수를 바인딩할 수 있습니다.

* **역할:** 이벤트 그래프에 복잡한 로직을 작성하지 않고도 애니메이션 그래프 내의 노드(예: 블렌드 스페이스 등)에서 필요한 데이터를 즉시 참조할 수 있어 그래프가 훨씬 깔끔해집니다.

## **2\. 작동 방식 (Binding)**

애니메이션 블루프린트 노드의 디테일 패널에서 변수 옆의 **\[바인딩(Bind)\]** 드롭다운을 통해 설정할 수 있습니다.

* **자동(Automatic):** 상황에 따라 스레드 세이프 여부를 판단하여 최적의 위치에서 평가합니다.  
* **스레드 세이프:** 워커 스레드에서 데이터를 읽어옵니다.  
* **논리적 연결:** 단순 변수뿐만 아니라 GetVelocity().Size()와 같은 체인 형태의 접근도 지원하며, 이를 내부적으로 최적화된 코드로 변환하여 실행합니다.

## **3\. 언제 사용해야 하는가?**

1. **애니메이션 블루프린트를 사용할 때:** 거의 필수적으로 권장됩니다. 이벤트 그래프에서의 Set 노드 사용을 최소화하고 프로퍼티 액세스로 대체하세요.  
2. **멀티스레드 애니메이션 업데이트를 사용할 때:** Thread Safe Update Animation 함수 내에서 외부 데이터를 참조해야 할 때 반드시 사용해야 합니다.  
3. **최적화가 필요한 대규모 프로젝트:** 수많은 NPC나 캐릭터가 등장하는 게임에서 애니메이션 부하를 줄이는 데 결정적인 역할을 합니다.

## **4\. 요약**

| 구분 | 기존 방식 (Event Graph) | 프로퍼티 액세스 방식 |
| :---- | :---- | :---- |
| **실행 스레드** | 주로 게임 스레드 (단일) | 워커 스레드 (병렬 가능) |
| **성능** | 블루프린트 VM 비용 발생 | 패스트 패스(최적화) 적용 |
| **안정성** | 스레드 경합 위험 존재 | 스레드 세이프 보장 |
| **코드 형태** | 캐스팅 및 수많은 연결선 | 직접 바인딩 또는 전용 노드 |

**주의:** 프로퍼티 액세스는 읽기 전용에 가깝습니다. 데이터를 "가져오는(Get)" 용도로 설계되었으므로, 복잡한 논리 연산이나 상태 변경(Set)은 여전히 별도의 로직이 필요할 수 있습니다.
