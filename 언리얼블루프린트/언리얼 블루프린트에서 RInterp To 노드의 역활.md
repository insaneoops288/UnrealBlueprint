# **RInterp To 노드 상세 가이드**

RInterp To (Rotation Interpolate To) 노드는 언리얼 엔진에서 회전값을 부드럽게 전환하기 위해 사용되는 수학적 유틸리티 노드입니다.

## **1\. 주요 역할 (Role)**

이 노드의 가장 큰 역할은 \*\*"현재의 회전 상태에서 목표하는 회전 상태로 매 프레임 조금씩 이동시켜 부드러운 회전 효과를 만드는 것"\*\*입니다.

### **핵심 특징**

* **최단 경로 회전 (Shortest Path):** 단순히 숫자 값을 더하는 것이 아니라, 360도 체계에서 가장 가까운 방향으로 회전하도록 내부적으로 계산합니다 (예: 350도에서 10도로 갈 때 340도를 거꾸로 도는 것이 아니라 20도만큼 앞으로 회전).  
* **프레임 독립성:** Delta Time을 입력받아 하드웨어 성능(FPS)에 상관없이 일정한 속도로 회전하도록 설계되었습니다.  
* **지수적 감쇠 (Ease Out):** 목표값에 가까워질수록 회전 속도가 자연스럽게 줄어들어 부드럽게 멈추는 효과가 있습니다.

## **2\. 입력 및 출력 핀 설명**

| 핀 이름 | 타입 | 설명 |
| :---- | :---- | :---- |
| **Current** | Rotator | 현재 액터나 컴포넌트의 회전값입니다. |
| **Target** | Rotator | 도달하고자 하는 최종 목적지 회전값입니다. |
| **Delta Time** | Float | 이전 프레임과 현재 프레임 사이의 시간 간격입니다. (보통 Get World Delta Seconds 연결) |
| **Interp Speed** | Float | 보간 속도입니다. 값이 클수록 목표치에 더 빨리 도달합니다. |
| **Return Value** | Rotator | 계산된 이번 프레임의 새로운 회전값입니다. 이를 Set Actor Rotation 등에 연결합니다. |

## **3\. 작동 원리**

이 노드는 단독으로 한 번 실행해서는 효과를 볼 수 없습니다. **Tick** 이벤트나 **Timeline**처럼 매 프레임 호출되는 흐름 안에서 사용해야 합니다.

1. 매 프레임 Current 값을 가져옵니다.  
2. Interp Speed와 Delta Time을 계산하여 Target 방향으로 조금 이동한 새로운 값을 출력합니다.  
3. 다음 프레임에 이 '새로운 값'이 다시 Current로 입력되면서 목표에 도달할 때까지 반복됩니다.

## **4\. 실전 활용 예시**

### **A. 캐릭터가 타겟을 부드럽게 바라보게 하기**

* Find Look at Rotation 노드로 목표 방향을 계산한 뒤, 이를 Target에 넣고 RInterp To를 거쳐 캐릭터의 회전값을 업데이트하면 고개가 즉각 휙 돌아가지 않고 스르륵 돌아가게 됩니다.

### **B. 부드럽게 열리는 문**

* 문이 닫힌 상태(0도)와 열린 상태(90도) 사이를 보간하여 문이 부드럽게 열리는 연출을 할 때 사용합니다.

## **5\. RInterp To vs RInterp To Constant**

* **RInterp To:** 목표에 가까워질수록 속도가 느려집니다. (부드러운 느낌)  
* **RInterp To Constant:** 목표에 도달할 때까지 **일정한 속도**로 회전합니다. (기계적인 회전이 필요할 때 사용)

## **6\. 주의사항**

* **Current 입력:** 반드시 "현재 실제 회전값"을 매 프레임 업데이트해서 넣어줘야 합니다. 고정된 값을 넣으면 회전이 일어나지 않습니다.  
* **Speed 설정:** 속도가 0이면 회전하지 않습니다. 보통 2\~10 사이의 값을 테스트하며 조정하는 것이 좋습니다.
