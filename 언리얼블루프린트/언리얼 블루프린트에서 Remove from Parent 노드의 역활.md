# **Unreal Engine: Remove from Parent 노드 이해하기**

Remove from Parent 노드는 언리얼 엔진의 UMG(Unreal Motion Graphics) 시스템에서 특정 위젯을 **부모 위젯이나 뷰포트(Viewport)로부터 분리**하는 역할을 합니다.

## **1\. 주요 역할**

### **① 위젯 제거 (Detachment)**

이 노드가 호출되면 해당 위젯은 더 이상 화면에 보이지 않게 됩니다.

* **뷰포트에서 제거:** Add to Viewport를 통해 화면에 직접 붙인 위젯을 제거합니다.  
* **부모 컨테이너에서 제거:** Canvas Panel, Vertical Box, Scroll Box 등의 자식으로 포함되어 있던 위젯을 해당 컨테이너에서 빼냅니다.

### **② 가비지 컬렉션(GC) 대상 지정**

Remove from Parent를 호출한다고 해서 위젯이 메모리에서 즉시 삭제되는 것은 아닙니다.

* 해당 위젯을 참조(Reference)하고 있는 변수가 더 이상 없다면, 언리얼 엔진의 \*\*가비지 컬렉터(Garbage Collector)\*\*가 추후 메모리에서 이를 정리합니다.  
* 만약 변수에 위젯을 저장해 두었다면, 화면에서는 사라지지만 데이터는 메모리에 남아 있어 나중에 다시 화면에 붙일 수 있습니다.

## **2\. 작동 방식**

이 노드는 위젯 블루프린트 내부에서 Self를 타겟으로 호출하거나, 외부(예: Character, Player Controller)에서 특정 위젯 변수를 타겟으로 호출합니다.

### **블루프린트 예시 흐름**

1. **위젯 생성:** Create Widget 노드로 위젯 인스턴스 생성.  
2. **화면 표시:** Add to Viewport 노드로 화면에 출력.  
3. **제거 로직:** 특정 버튼 클릭(On Clicked) 또는 이벤트 발생 시 Remove from Parent 실행.

## **3\. 주요 활용 사례**

* **메뉴 닫기:** 'Esc' 키를 누르거나 '닫기' 버튼을 눌러 일시정지 메뉴나 인벤토리 창을 닫을 때.  
* **HUD 전환:** 게임 플레이 중 특정 상황(컷신 등)에서 현재 HUD를 숨겨야 할 때.  
* **아이템 슬롯 삭제:** 인벤토리에서 아이템을 버리거나 사용할 때 해당 아이템 위젯을 리스트에서 제거할 때.

## **4\. 주의사항 (Tips)**

### **Is Valid 체크**

이미 제거되었거나 생성되지 않은 위젯에 대해 Remove from Parent를 호출하면 오류가 발생할 수 있습니다. 호출 전 Is Valid 노드를 사용하여 위젯이 존재하는지 확인하는 습관이 중요합니다.

### **가시성(Visibility)과의 차이**

* **Set Visibility (Hidden/Collapsed):** 위젯은 부모 공간에 그대로 남아 있지만 단순히 보이지 않게 하거나 자리만 차지하지 않게 합니다. 다시 보여줄 때 비용이 적습니다.  
* **Remove from Parent:** 위젯을 구조 자체에서 완전히 빼버립니다. 다시 보여주려면 다시 Add to Viewport를 하거나 새로 생성해야 할 수도 있습니다.

### **변수 관리**

위젯을 제거한 후 다시 사용할 계획이 없다면, 해당 위젯을 담고 있던 변수를 Null(비어있음)로 설정해 주는 것이 메모리 관리에 도움이 됩니다.

## **요약**

Remove from Parent는 \*\*"이 UI를 화면(혹은 부모)에서 떼어내라"\*\*는 명령입니다. UI의 생명 주기를 관리하는 가장 기본적인 노드 중 하나입니다.
