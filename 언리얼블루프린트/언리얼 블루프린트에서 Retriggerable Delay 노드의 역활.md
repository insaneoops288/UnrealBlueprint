# **언리얼 블루프린트: Retriggerable Delay 노드 가이드**

Retriggerable Delay 노드는 이름 그대로 **"다시 트리거(실행)할 수 있는 지연"** 노드입니다. 지정된 시간 동안 실행을 지연시키지만, 지연 시간 도중에 노드가 다시 호출되면 타이머가 처음부터 다시 시작됩니다.

## **1\. 핵심 동작 원리**

### **일반 Delay vs Retriggerable Delay**

| 기능 | Delay (일반 지연) | Retriggerable Delay (재트리거 지연) |
| :---- | :---- | :---- |
| **중복 실행 시** | 이미 타이머가 돌아가고 있다면, 추가 실행 요청을 무시합니다. | 이미 타이머가 돌아가고 있더라도, **타이머를 0으로 초기화**하고 다시 카운트를 시작합니다. |
| **비유** | "30초만 기다려"라고 말한 뒤, 중간에 다시 말해도 처음 약속한 30초가 지나야 움직입니다. | "30초만 기다려"라고 말했는데, 10초 뒤에 다시 말하면 그 시점부터 다시 30초를 기다립니다. |

## **2\. 주요 특징**

1. **타이머 리셋**: 노드의 입력 핀(In)에 신호가 들어올 때마다 Duration에 설정된 시간으로 타이머가 초기화됩니다.  
2. **Completed 핀**: 설정된 시간이 온전히 흐른 뒤에만 Completed 핀으로 실행 신호가 나갑니다.  
3. **잠재적 노드 (Latent Node)**: 시계 아이콘이 붙어 있는 노드로, 실행이 완료될 때까지 시간이 걸리는 비동기 노드입니다.

## **3\. 대표적인 활용 사례**

### **① 입력 스팸 방지 (Anti-Spam)**

플레이어가 버튼을 연타할 때, 마지막 입력으로부터 일정 시간이 지난 후에만 특정 동작을 수행하고 싶을 때 사용합니다.

* **예시**: 인벤토리 정렬 버튼을 연타할 때, 연타가 멈추고 0.5초 뒤에 실제 정렬 로직이 실행되도록 설정.

### **② UI 자동 숨기기**

플레이어가 아이템을 획득하거나 특정 행동을 했을 때 알림 UI를 띄우고, 일정 시간 동안 아무 변화가 없으면 UI를 닫을 때 유용합니다.

* **예시**: 대미지를 입을 때마다 체력바를 표시하고 Retriggerable Delay(3초)를 연결. 계속 대미지를 입으면 체력바가 계속 유지되다가, 공격이 멈추고 3초 뒤에 사라짐.

### **③ 콤보 시스템의 타임아웃**

플레이어의 공격 콤보 상태를 유지하다가, 일정 시간 내에 다음 공격이 입력되지 않으면 초기화하는 로직에 적합합니다.

### **④ 자동 저장 (Auto-Save)**

플레이어가 설정을 변경할 때마다 매번 저장하지 않고, 변경이 멈춘 후 1\~2초 뒤에 한 번만 저장하도록 구현할 수 있습니다.

## **4\. 블루프린트 구성 예시**

\[커스텀 이벤트: ShowDamageUI\]  
       |  
\[체력바 UI 위젯 표시\]  
       |  
\[Retriggerable Delay (Duration: 2.0)\] \--- (2초 동안 재입력이 없으면) \---\> \[체력바 UI 숨기기\]

*위 구조에서 ShowDamageUI 이벤트가 1초 간격으로 계속 호출된다면, Retriggerable Delay는 계속 0으로 리셋되므로 UI는 절대 숨겨지지 않고 화면에 계속 유지됩니다.*

## **5\. 주의사항**

* **반복 루프 주의**: Completed 핀을 다시 노드의 입력에 연결하면 무한 루프가 발생할 수 있습니다. (물론 지연 시간이 있으므로 즉각적인 크래시는 나지 않지만 의도하지 않은 반복이 생길 수 있음)  
* **정교한 타이머**: 아주 정밀한 시간 제어가 필요하거나 중간에 남은 시간을 계산해야 한다면 Set Timer by Event 또는 Set Timer by Function Name 노드를 사용하는 것이 더 제어하기 쉽습니다.
