# **언리얼 엔진 네트워크 역할: ROLE\_AutonomousProxy**

멀티플레이어 게임에서 모든 액터(Actor)는 네트워크 상에서 자신의 위치와 상태를 결정하는 \*\*'역할(NetRole)'\*\*을 가집니다. 그중 ROLE\_AutonomousProxy는 클라이언트 측의 사용자 경험을 결정짓는 핵심적인 요소입니다.

## **1\. 정의 (Definition)**

ROLE\_AutonomousProxy는 \*\*"내가 조종하고 있는 나 자신의 캐릭터"\*\*를 의미합니다.

* **서버(Server):** 해당 액터에 대해 ROLE\_Authority (권한)를 가집니다.  
* **소유 클라이언트(Owning Client):** 해당 액터에 대해 ROLE\_AutonomousProxy 역할을 가집니다.  
* **다른 클라이언트(Other Clients):** 해당 액터에 대해 ROLE\_SimulatedProxy (시뮬레이션 대리자) 역할을 가집니다.

## **2\. 주요 역할과 특징**

### **① 클라이언트 측 예측 (Client-Side Prediction)**

가장 중요한 역할입니다. 만약 캐릭터가 움직일 때 서버의 응답(Packet)을 기다린 후 움직인다면, 플레이어는 자신의 입력과 화면상의 움직임 사이에 지연(Lag)을 느끼게 됩니다.

* AutonomousProxy는 서버의 허락을 기다리지 않고 **로컬에서 즉시 입력을 처리**하여 움직임을 보여줍니다.  
* 이후 서버에 자신의 위치를 보고하고, 만약 서버의 결과와 다를 경우 서버의 데이터를 바탕으로 위치를 보정(Reconciliation)받습니다.

### **② 입력 처리 (Input Handling)**

이 역할을 가진 액터만 사용자로부터 입력을 직접 받아 서버로 보낼 수 있는 권한을 가집니다. 로컬 플레이어의 컨트롤러(PlayerController)가 이 액터를 '빙의(Possess)'하고 있을 때 이 상태가 됩니다.

### **③ RPC (Remote Procedure Call) 실행**

Server RPC를 호출할 수 있는 자격이 주어집니다. 언리얼에서 클라이언트가 서버에 무언가를 요청하려면, 해당 클라이언트가 소유권을 가진(즉, AutonomousProxy 상태인) 액터를 통해서만 가능합니다.

## **3\. 블루프린트에서의 활용 예시**

블루프린트에서 GetLocalRole 노드를 사용해 현재 액터의 역할을 확인할 수 있습니다.

* **분기 처리 (Switch on ENetRole):**  
  * Authority: 서버에서 실행할 로직 (데미지 계산, 점수 갱신 등)  
  * Autonomous Proxy: 로컬 플레이어에게만 보여줄 로직 (화면 흔들림, 로컬 UI 업데이트, 즉각적인 사운드 재생 등)  
  * Simulated Proxy: 다른 플레이어들이 보는 내 모습 (부드러운 위치 보간 등)

## **4\. 요약 비교**

| 역할 이름 | 의미 | 발생 위치 | 주요 특징 |
| :---- | :---- | :---- | :---- |
| **ROLE\_Authority** | 권한자 | 서버 | 게임의 절대적인 진실. 데이터의 최종 결정권. |
| **ROLE\_AutonomousProxy** | 자율 대리자 | 소유 클라이언트 | 로컬 입력 즉시 반응(예측). 서버와 통신하며 보정받음. |
| **ROLE\_SimulatedProxy** | 시뮬레이션 대리자 | 나머지 클라이언트 | 서버가 보내준 위치 데이터를 바탕으로 부드럽게 흉내만 냄. |

## **5\. 주의사항**

* AutonomousProxy는 오직 **플레이어가 소유한 액터**에게만 존재합니다. AI가 조종하는 적 캐릭터는 클라이언트 측에서 모두 SimulatedProxy입니다.  
* 게임 개발 시 "내가 보는 내 캐릭터"와 "남이 보는 내 캐릭터"의 시각적 효과를 다르게 주어야 할 때 이 노드를 통한 체크가 필수적입니다.
