# **Save Game to Slot 노드의 역할 및 활용 가이드**

언리얼 엔진에서 **Save Game to Slot** 노드는 게임의 데이터를 로컬 저장 장치(HDD/SSD)에 파일 형태로 영구적으로 저장하는 핵심적인 역할을 합니다.

## **1\. 주요 역할**

이 노드는 메모리(RAM)에 존재하는 **Save Game 객체**의 데이터를 바이너리 파일로 변환하여 지정된 \*\*슬롯 이름(Slot Name)\*\*으로 저장합니다. 이를 통해 게임을 종료한 후 다시 실행하더라도 이전의 진행 상태, 캐릭터 능력치, 아이템 목록 등을 그대로 유지할 수 있습니다.

## **2\. 노드의 입력 파라미터 (Input Pins)**

| 파라미터 | 설명 |
| :---- | :---- |
| **Save Game Object** | 저장할 데이터가 담긴 SaveGame 클래스의 인스턴스(객체)입니다. |
| **Slot Name** | 저장 파일의 이름입니다 (예: "SaveSlot\_01"). 이 이름을 통해 나중에 데이터를 다시 불러옵니다. |
| **User Index** | 여러 명의 사용자가 한 기기를 사용할 경우 구분하기 위한 인덱스입니다 (일반적으로 0 사용). |

## **3\. 일반적인 저장 워크플로우**

데이터를 성공적으로 저장하기 위해서는 보통 다음과 같은 단계를 거칩니다.

1. **Create Save Game Object**: 저장 전용 클래스(SaveGame을 상속받은 클래스)의 객체를 생성합니다.  
2. **데이터 설정**: 생성된 객체의 변수(HP, 골드, 레벨 등)에 현재 게임 상태의 값을 대입합니다.  
3. **Save Game to Slot**: 설정된 객체를 이 노드에 연결하여 실제 파일로 저장합니다.

## **4\. 주의사항 및 팁**

### **동기 vs 비동기 저장**

* **Save Game to Slot**: 저장 작업이 완료될 때까지 게임의 메인 스레드가 일시적으로 멈출 수 있습니다 (동기 방식). 데이터 양이 적을 때는 문제가 없으나, 양이 많으면 '렉'이 발생할 수 있습니다.  
* **Async Save Game to Slot**: 게임 성능에 영향을 주지 않고 백그라운드에서 저장하는 **비동기** 버전입니다. 최신 게임 개발에서는 이 노드 사용을 더 권장합니다.

### **저장 위치**

* 저장된 파일은 보통 프로젝트 폴더 내 Saved/SaveGames 폴더에 .sav 확장자로 생성됩니다.  
* 패키징된 게임의 경우 사용자의 AppData 폴더 내에 저장됩니다.

### **데이터 구조 설계**

* SaveGame 클래스 내부에 저장할 변수들을 미리 정의해두어야 합니다. 변수가 'Editable'이 아니더라도 저장 프로세스에는 지장이 없습니다.

## **5\. 요약**

Save Game to Slot은 \*\*"현재 메모리에 있는 데이터를 파일로 구워내는 작업"\*\*을 담당합니다. 이 노드가 호출되는 순간, 지정한 슬롯 이름으로 데이터가 디스크에 물리적으로 기록됩니다.
