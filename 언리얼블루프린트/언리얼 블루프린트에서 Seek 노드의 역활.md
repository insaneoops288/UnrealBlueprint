# **언리얼 엔진 블루프린트 Seek 노드 역할 분석**

언리얼 엔진에서 **Seek** 노드는 문맥(Context)에 따라 크게 세 가지 주요 역할을 수행합니다. 가장 일반적인 용도는 미디어 파일의 재생 위치를 변경하는 것입니다.

## **1\. 미디어 플레이어(Media Player)에서의 Seek**

미디어 프레임워크에서 Seek 노드는 비디오나 오디오 파일의 **재생 시점을 특정 시간(Timestamp)으로 강제 이동**시키는 역할을 합니다.

* **주요 기능:** 영상의 특정 구간으로 '건너뛰기'를 할 때 사용합니다.  
* **입력 파라미터:** \* **Target:** 대상이 되는 Media Player 오브젝트 레퍼런스.  
  * **Time:** 이동하고자 하는 시점 (Timespan 구조체 사용).  
* **활용 예시:**  
  * 플레이어가 '다시 보기' 버튼을 눌렀을 때 영상을 0초로 되돌림.  
  * 특정 이벤트 발생 시 영상의 중간 지점부터 재생.

## **2\. 오디오(Audio) 및 타임라인(Timeline)에서의 역할**

엄밀히 말하면 타임라인에는 Seek이라는 이름의 노드 대신 **Set New Time** 또는 **Set Playback Position** 노드가 같은 역할을 수행합니다.

* **타임라인:** 애니메이션의 특정 프레임으로 즉시 이동하여 실행 위치를 변경합니다.  
* **오디오 컴포넌트:** Set Playback Position 노드를 통해 사운드 큐나 웨이브 파일의 특정 시간대부터 소리가 나도록 조절합니다.

## **3\. AI 조향 행동(Steering Behaviors)에서의 Seek**

AI 플러그인이나 커스텀 AI 시스템을 사용할 때 **Seek**은 AI가 특정 타겟을 향해 **직선으로 추적하며 이동하는 알고리즘**을 의미합니다.

* **원리:** 1\. 타겟 위치 \- 현재 위치를 계산하여 방향 벡터를 구합니다.  
  2\. 해당 방향으로 최대 속도(Max Speed)를 적용합니다.  
  3\. 현재 속도와의 차이를 계산하여 부드러운 회전과 가속(Steering Force)을 생성합니다.  
* **결과:** AI가 타겟을 향해 부드럽게 방향을 틀며 쫓아가는 움직임을 보입니다.

## **4\. 요약 및 주의사항**

| 구분 | 주요 역할 | 비고 |
| :---- | :---- | :---- |
| **Media Player** | 영상/음향 재생 위치 변경 | 가장 일반적인 'Seek' 노드의 형태 |
| **Timeline** | 애니메이션 재생 시점 강제 지정 | Set New Time 노드 사용 |
| **AI Steering** | 타겟을 향한 추적 이동 로직 | 주로 벡터 연산이나 플러그인에서 제공 |

### **💡 팁:**

미디어 플레이어에서 Seek 노드를 사용할 때는 **'Seekable'** 여부를 먼저 확인해야 합니다. 스트리밍 미디어의 경우 특정 구간으로의 Seek이 제한될 수 있으므로, Can Seek 노드로 확인 후 실행하는 것이 안전합니다.
