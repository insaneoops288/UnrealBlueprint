# **언리얼 엔진 블루프린트: Select Float 노드 완벽 이해하기**

Select Float 노드는 특정 조건(Boolean)에 따라 두 개의 실수(Float) 값 중 하나를 선택하여 출력하는 **조건부 선택 노드**입니다. 프로그래밍의 '삼항 연산자'(condition ? A : B)와 동일한 역할을 합니다.

## **1\. 노드의 기본 구조**

이 노드는 다음과 같은 핀으로 구성되어 있습니다.

| 핀 이름 | 유형 | 역할 |
| :---- | :---- | :---- |
| **Pick A** | Boolean (입력) | True이면 A를, False이면 B를 선택하도록 결정하는 스위치입니다. |
| **A** | Float (입력) | Pick A가 **True**일 때 출력될 값입니다. |
| **B** | Float (입력) | Pick A가 **False**일 때 출력될 값입니다. |
| **Return Value** | Float (출력) | 선택된 결과값입니다. |

## **2\. 주요 역할 및 장점**

### **1\) 그래프 단순화 (Clean Graph)**

일반적으로 조건에 따라 변수 값을 정하려면 Branch 노드를 사용하고 True와 False 경로에 각각 Set 노드를 배치해야 합니다. 하지만 Select Float를 사용하면 한 줄의 흐름으로 값을 결정할 수 있어 노드 그래프가 훨씬 깔끔해집니다.

### **2\) 인라인 계산 가능**

함수의 입력 파라미터나 다른 계산 노드의 입력 핀에 직접 연결할 수 있어, 중간에 실행 흐름(Execution Line)을 끊지 않고도 동적인 값 할당이 가능합니다.

## **3\. 실전 활용 예시**

### **예시 A: 달리기 속도 조절**

캐릭터가 '달리기' 버튼을 눌렀는지 여부에 따라 속도 값을 다르게 적용할 때 사용합니다.

* **Pick A**: IsRunning? (부울 변수)  
* **A**: 600.0 (달리기 속도)  
* **B**: 300.0 (걷기 속도)  
* **결과**: CharacterMovement의 MaxWalkSpeed에 연결.

### **예시 B: 체력에 따른 UI 투명도**

체력이 낮을 때만 경고 UI를 표시하고 싶을 때 유용합니다.

* **Pick A**: Health \< 20.0 (비교 연산)  
* **A**: 1.0 (불투명)  
* **B**: 0.0 (투명)  
* **결과**: 위젯의 Render Opacity에 연결.

## **4\. Branch 노드와의 차이점**

| 구분 | Branch 노드 | Select Float 노드 |
| :---- | :---- | :---- |
| **목적** | 전체적인 로직의 **실행 흐름** 제어 | 특정 **데이터 값**의 결정 |
| **특징** | 실행 핀(하얀 화살표)이 있음 | 실행 핀이 없으며 순수하게 값만 전달 |
| **사용 시점** | "이 일을 할까 저 일을 할까?" 결정 시 | "이 값을 쓸까 저 값을 쓸까?" 결정 시 |

## **5\. 팁: 확장성**

언리얼 엔진에는 Select Float 외에도 데이터 타입에 따라 다양한 Select 노드가 존재합니다.

* Select Int, Select Vector, Select Color, Select Object 등  
* 만약 선택지가 2개(True/False)가 아니라 여러 개라면, **Wildcard Select** 노드를 사용하여 Enum(열거형)이나 Int 값에 따라 3개 이상의 값 중 하나를 고를 수도 있습니다.

**요약:** Select Float는 "만약 \~라면 A 값을, 아니면 B 값을 내보내라"는 명령을 실행 흐름을 끊지 않고 수행하는 효율적인 도구입니다.
