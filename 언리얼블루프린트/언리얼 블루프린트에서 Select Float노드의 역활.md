언리얼 블루프린트에서 **Select Float** 노드는 불리언(Boolean) 조건에 따라 **두 개의 실수(Float) 값 중 하나를 선택**하여 출력해 주는 '결정 노드'입니다.

프로그래밍의 \*\*삼항 연산자($condition ? A : B$)\*\*와 똑같은 역할을 하며, 로직의 흐름을 끊지 않고 값만 교체하고 싶을 때 매우 유용합니다.

## ---

**1\. 노드의 구조와 작동 원리**

* **Pick A (입력 \- Boolean):** 어떤 값을 선택할지 결정하는 스위치입니다.  
  * **False(체크 해제):** **A** 핀에 연결된 값을 출력합니다.  
  * **True(체크 선택):** **B** 핀에 연결된 값을 출력합니다.  
* **A (입력 \- Float):** 조건이 거짓일 때 사용할 값입니다.  
* **B (입력 \- Float):** 조건이 참일 때 사용할 값입니다.  
* **Return Value (출력):** 선택된 최종 실수 값입니다.

## ---

**2\. 왜 사용하나요? (Branch 노드와의 차이)**

가장 큰 이유는 \*\*"그래프를 깔끔하게 유지하기 위해서"\*\*입니다.

* **Branch 노드:** 실행 흐름(하얀색 선) 자체를 두 갈래로 나눕니다. 로직 자체가 완전히 달라질 때 사용합니다.  
* **Select Float 노드:** 실행 흐름은 그대로 두고, 특정 핀에 들어갈 **'데이터'만** 상황에 맞게 바꿔줍니다.

**비유:** \> \* **Branch:** "배가 고프면 식당으로 가고, 안 고프면 집으로 가라." (가는 길 자체가 바뀜)

* **Select:** "식당에 갔는데, 돈이 있으면 '비싼 메뉴'를 고르고 없으면 '싼 메뉴'를 골라라." (식당에 간다는 행동은 같지만 선택하는 물건만 바뀜)

## ---

**3\. 주요 활용 사례**

### **① 걷기/달리기 속도 조절**

캐릭터의 Max Walk Speed를 설정할 때, IsRunning 변수에 따라 값을 다르게 주고 싶을 때 사용합니다.

* **Pick A:** IsRunning 변수 연결  
* **A (False):** 300.0 (걷기 속도)  
* **B (True):** 600.0 (달리기 속도)  
* 이 결과를 Character Movement의 속도 설정 핀에 바로 꽂으면 됩니다.

### **② 대미지 계산 (크리티컬 판정)**

공격 시 크리티컬 확률에 당첨되었는지에 따라 대미지 배율을 다르게 적용할 때 유용합니다.

* **Pick A:** IsCritical? (확률 계산 결과)  
* **A (False):** 1.0 (일반 배율)  
* **B (True):** 2.0 (두 배 대미지)

### **③ UI 투명도 제어**

특정 아이템을 마우스로 가리키고(Hover) 있는지에 따라 불투명도를 조절할 때 사용합니다.

* **Pick A:** IsHovered  
* **A (False):** 0.5 (반투명)  
* **B (True):** 1.0 (불투명)

## ---

**4\. 실무 팁: Select 노드의 확장성**

사실 **Select** 노드는 Float뿐만 아니라 모든 데이터 타입에 존재합니다. (Select Int, Select Color, Select Vector, Select Asset 등)

* 만약 3개 이상의 값 중에서 골라야 한다면 어떻게 할까요?  
* **Wildcard Select** 노드를 사용하면 됩니다. 입력 핀에 \*\*열거형(Enum)\*\*을 연결하면, 열거형에 정의된 항목 개수만큼 입력 핀이 자동으로 생성되어 훨씬 복잡한 선택 로직을 구현할 수 있습니다.

**요약**

**Select Float**은 "조건에 따라 이 숫자 아니면 저 숫자를 쓰겠다"라고 결정할 때, **Branch 노드 없이도 깔끔하게 값을 전달**해 주는 효율적인 도구입니다.
