# **Unreal Engine: Select Rotator 노드 상세 가이드**

Select Rotator 노드는 특정 조건(Boolean 또는 Enum 등)에 따라 입력된 여러 **Rotator(Pitch, Yaw, Roll)** 값 중 하나를 선택하여 결과값으로 내보내는 "선택기" 역할을 합니다.

## **1\. 주요 역할**

* **조건부 값 결정**: "만약 A라면 이 회전값을, 아니라면 저 회전값을 사용하라"는 로직을 수행합니다.  
* **실행 흐름 간소화**: Branch 노드를 사용하여 실행선(흰색 선)을 두 갈래로 나누지 않고도, 하나의 노드 안에서 데이터를 선택할 수 있어 블루프린트가 훨씬 깔끔해집니다.

## **2\. 노드 구조 (Inputs & Outputs)**

### **입력 (Inputs)**

* **Index (또는 Pick A / B)**: 어떤 값을 선택할지 결정하는 기준입니다. 보통 Boolean(True/False) 타입을 사용하지만, Select 노드 자체는 Integer나 Enum을 연결하여 확장할 수도 있습니다.  
* **A / False**: 조건이 거짓(False)일 때 출력할 Rotator 값입니다.  
* **B / True**: 조건이 참(True)일 때 출력할 Rotator 값입니다.

### **출력 (Outputs)**

* **Return Value**: 선택된 최종 Rotator 값입니다.

## **3\. 왜 사용하는가? (Branch 노드와의 차이)**

| 기능 | Branch \+ Set Rotation | Select Rotator \+ Set Rotation |
| :---- | :---- | :---- |
| **로직 방식** | 실행 흐름(Execution Flow) 제어 | 데이터 흐름(Data Flow) 제어 |
| **복잡도** | 노드가 많아지고 실행선이 꼬이기 쉬움 | 노드 하나로 해결되어 구조가 단순함 |
| **가독성** | 어떤 경로로 실행되는지 파악하기 좋음 | 어떤 "값"이 전달되는지 파악하기 좋음 |

## **4\. 실전 활용 예시**

### **예시 1: 캐릭터의 방향 전환**

캐릭터가 왼쪽을 보고 있을 때와 오른쪽을 보고 있을 때의 회전값을 미리 지정해두고, IsMovingLeft 변수(Boolean)를 Index에 연결하여 즉각적으로 타겟 회전값을 결정할 수 있습니다.

### **예시 2: 문(Door)의 개폐 상태**

* **A (False)**: 문이 닫힌 회전값 (0, 0, 0\)  
* **B (True)**: 문이 90도 열린 회전값 (0, 0, 90\)  
* **Index**: bIsOpen 변수 연결

이렇게 선택된 값을 RInterp To 노드나 Timeline의 목적지로 연결하여 부드러운 애니메이션을 구현할 때 자주 사용됩니다.

## **5\. 팁**

* 블루프린트에서 일반 Select 노드를 배치한 후, 입력 핀에 Rotator 변수를 드래그해서 연결하면 자동으로 Select Rotator 형태로 타입이 고정됩니다.  
* 단순히 두 값만 비교하는 것이 아니라, Enum(열거형)을 연결하면 상태(상, 하, 좌, 우 등)에 따른 다중 선택도 가능합니다.
