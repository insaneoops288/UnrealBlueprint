# **언리얼 블루프린트 Select 노드 완벽 가이드**

언리얼 엔진의 **Select** 노드는 입력받은 **Index(정수)** 값에 따라 준비된 여러 옵션 중 하나를 선택하여 결과로 반환하는 데이터 선택 노드입니다.

## **1\. 기본 개념**

Select 노드는 "여러 개의 데이터 바구니 중 몇 번째 바구니를 꺼낼 것인가?"를 결정하는 역할을 합니다.

* **Index (입력):** 어떤 옵션을 선택할지 결정하는 정수(Integer) 값입니다. (0부터 시작)  
* **Option 0, 1, 2... (입력):** 실제로 출력될 후보 값들입니다.  
* **Return Value (출력):** 선택된 옵션의 값이 출력됩니다.

## **2\. 주요 특징**

### **① 와일드카드(Wildcard) 속성**

Select 노드는 처음 생성했을 때는 타입을 가지지 않지만, 옵션 핀에 특정 데이터 타입(Float, Vector, Actor Reference 등)을 연결하는 순간 해당 타입으로 고정됩니다. 즉, 숫자뿐만 아니라 거의 모든 데이터를 선택할 수 있습니다.

### **② 확장성**

노드를 우클릭하거나 디테일 패널을 통해 \*\*'Option 추가'\*\*를 할 수 있습니다. 이를 통해 2개 이상의 수많은 선택지를 하나로 관리할 수 있습니다.

### **③ 실행 핀이 없음 (Pure Node)**

Select 노드는 실행선(하얀색 선)이 연결되지 않는 **Pure Node**입니다. 데이터가 필요할 때 즉시 계산되어 값을 전달하므로 로직이 깔끔해집니다.

## **3\. 활용 예시**

### **예시 A: 무기 교체 시스템**

* **Index:** 현재 장착 중인 무기 번호 (0: 칼, 1: 총, 2: 활)  
* **Options:** 각 무기의 스태틱 메쉬(Static Mesh)  
* **결과:** 무기 번호가 바뀔 때마다 캐릭터 손바닥에 부착된 메쉬가 자동으로 변경됩니다.

### **예시 B: 난이도별 데미지 설정**

* **Index:** 게임 난이도 (0: 쉬움, 1: 보통, 2: 어려움)  
* **Options:** 50.0, 100.0, 200.0 (Float 값)  
* **결과:** 현재 난이도 변수만 넘겨주면 적이 주는 데미지를 즉시 결정할 수 있습니다.

## **4\. Enum(열거형)과의 조합 (강력 추천)**

Select 노드의 진가는 \*\*Enum(열거형)\*\*과 함께 사용할 때 나타납니다.

1. 정수(Int) 대신 **Enum 변수**를 Index 핀에 연결합니다.  
2. 노드의 핀 이름이 자동으로 Enum의 항목 이름(Idle, Run, Attack 등)으로 바뀝니다.  
3. 숫자로 관리할 때 생기는 실수(0이 뭐였지?)를 방지하고 가독성을 극대화할 수 있습니다.

## **5\. Select vs Switch On Int 차이점**

| 특징 | Select 노드 | Switch On Int 노드 |
| :---- | :---- | :---- |
| **목적** | \*\*값(Data)\*\*을 선택하여 반환 | \*\*실행 흐름(Flow)\*\*을 분기 |
| **실행선** | 없음 (Pure) | 있음 (Impure) |
| **반환값** | 있음 | 없음 |
| **용도** | 변수 설정, 세팅값 변경 | 로직의 갈래를 나눌 때 |

## **요약**

**Select** 노드는 복잡한 Branch나 Switch 문을 사용하지 않고도, 특정 조건에 맞는 **값**을 효율적으로 뽑아낼 때 사용하는 "데이터 스위치"입니다. 로직을 훨씬 간결하고 읽기 쉽게 만들어줍니다.
