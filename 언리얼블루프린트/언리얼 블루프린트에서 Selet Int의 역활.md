언리얼 블루프린트에서 Selet Int의 역활?

언리얼 엔진 블루프린트에서 Select 노드(그 중에서도 Index 입력이 Integer인 경우)는 한마디로 **"조건에 따라 여러 값 중 하나를 골라주는 스마트한 선택기"**라고 할 수 있습니다.

복잡한 Branch(If/Else) 노드를 여러 개 연결하는 것보다 훨씬 깔끔하게 코드를 짤 수 있게 도와주는 아주 고마운 노드죠.

1. Select Int의 핵심 역할
이 노드의 역할은 아주 단순합니다.

입력(Index): 정수(Integer) 값을 받습니다. (예: 0, 1, 2...)

선택지(Option): 각 숫자(Index)에 대응하는 결과값들을 미리 적어둡니다.

출력(Return Value): 입력된 숫자에 해당하는 결과값을 뱉어냅니다.

[!TIP]
마치 자판기와 같습니다. **"1번을 누르면 콜라, 2번을 누르면 사이다, 3번을 누르면 주스가 나온다"**는 로직을 노드 하나로 해결하는 것이죠.

2. 왜 사용하나요? (장점)
특징	설명
코드 가독성	Branch 노드를 3~4개씩 줄줄이 연결(Cascading)할 필요가 없어 그래프가 깨끗해집니다.
유연성	노드를 우클릭해서 Add Option Pin을 누르면 선택지를 얼마든지 늘릴 수 있습니다.
타입 가변성	출력값은 정수뿐만 아니라 Float, String, Vector, Actor Reference 등 원하는 모든 데이터 타입으로 바꿀 수 있습니다.
3. 실전 활용 예시
예를 들어, 게임의 난이도에 따라 적의 공격력을 설정한다고 가정해 봅시다.

Index: DifficultyLevel (0: 쉬움, 1: 보통, 2: 어려움) 변수를 연결합니다.

Option 0: 10.0 (쉬움 난이도 공격력)

Option 1: 25.0 (보통 난이도 공격력)

Option 2: 50.0 (어려움 난이도 공격력)

Return Value: 이 값을 적의 Damage 변수에 세팅합니다.

이렇게 하면 "만약 0이면 10을 주고, 아니면 다시 만약 1이면..." 같은 복잡한 비교 로직 없이 단 한 번의 선택으로 끝납니다.

4. 주의할 점
정확한 인덱스: 입력되는 정수가 옵션에 없는 숫자(예: 옵션은 0~2인데 입력이 5인 경우)라면, 기본적으로 마지막 옵션을 출력하거나 에러를 방지하기 위해 인덱스 범위를 미리 체크하는 것이 좋습니다.

와일드카드: 처음에 Select 노드를 꺼내면 결과 핀이 회색(Wildcard)인데, 여기에 아무 값이나 먼저 연결하면 그 데이터 타입으로 고정됩니다.
