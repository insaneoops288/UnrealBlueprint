언리얼 블루프린트에서 **Select Int** 노드는 특정 조건(Boolean)이나 인덱스(Index)에 따라 **여러 개의 정수(Integer) 값 중 하나를 골라 출력**하는 역할을 합니다.

Select Float과 마찬가지로 로직의 흐름(실행 선)을 쪼개지 않고, \*\*"상황에 맞는 숫자 데이터만 쏙 뽑아서 전달"\*\*하고 싶을 때 사용합니다.

## ---

**1\. 노드의 작동 방식**

**Select Int** 노드는 크게 두 가지 방식으로 변신합니다.

### **① Boolean 기반 (A 아니면 B)**

가장 단순한 형태입니다. Pick A 핀에 Boolean 변수를 연결하면, **False일 때 Option 0**을, **True일 때 Option 1**을 출력합니다.

### **② Index/Enum 기반 (여러 개 중 하나)**

Index 핀에 \*\*정수(Int)\*\*나 \*\*열거형(Enum)\*\*을 연결하면, 해당 숫자나 상태에 맞는 옵션 칸이 자동으로 활성화됩니다.

* 인덱스가 0이면 0번 칸의 값을, 1이면 1번 칸의 값을 내보냅니다.  
* \*\*\[+\] (Add Pin)\*\*을 눌러 선택지를 무한정 늘릴 수 있습니다.

## ---

**2\. 왜 사용하나요? (Branch vs Select)**

복잡한 Branch나 Switch on Int 노드를 주렁주렁 매달지 않아도 되기 때문에 **그래프가 훨씬 깔끔해집니다.**

* **Branch (노가다):** "만약 난이도가 쉬움이면 10, 보통이면 20, 어려움이면 30을 변수에 저장해라." (실행 선이 복잡해짐)  
* **Select (효율적):** "난이도(Enum)를 넣으면 그에 맞는 숫자(10, 20, 30)를 바로 뱉어라." (실행 선 하나로 끝남)

## ---

**3\. 주요 활용 사례**

### **① 아이템 보상 수량 결정**

상자의 등급에 따라 나오는 골드의 양을 다르게 설정할 때 유용합니다.

* **Index:** 상자 등급 (0: 나무, 1: 은, 2: 금)  
* **Option 0:** 100  
* **Option 1:** 500  
* **Option 2:** 2000

### **② 무작위 인덱스 추출**

여러 개의 아이템 ID 중 하나를 랜덤하게 뽑아 넘겨줄 때 사용합니다.

* Random Integer in Range 노드 결과를 Index에 연결하고, 각 옵션에 아이템 고유 번호를 적어두면 간단한 랜덤 드랍 시스템이 완성됩니다.

### **③ 콤보 단계 제어**

현재 공격이 몇 번째 콤보인지에 따라 애니메이션 인덱스나 대미지 수치를 다르게 전달할 때 사용합니다.

## ---

**4\. 실무 팁: Enum(열거형)과 찰떡궁합**

**Select Int** 노드의 진가는 \*\*Enum(열거형)\*\*과 연결할 때 발휘됩니다.

1. 직접 만든 Enum(예: E\_StoryChapter)을 Index 핀에 드래그해서 꽂습니다.  
2. 그러면 0, 1, 2 숫자가 아니라 **'Chapter\_01', 'Chapter\_02'** 같은 이름으로 입력 칸이 바뀝니다.  
3. 코드를 읽을 때 "아, 이 챕터일 때는 이 정수 값을 쓰는구나\!"라고 한눈에 파악할 수 있어 유지보수가 매우 쉬워집니다.

---

**요약**

**Select Int**는 "여러 개의 정수 후보 중 **현재 상황에 딱 맞는 숫자 하나만 골라주는 데이터 필터**"입니다.

혹시 지금 프로젝트에서 캐릭터의 능력치나 아이템 번호를 관리하는 로직을 짜고 계신가요? 이 노드와 Enum을 조합하면 복잡한 조건문 없이도 아주 깔끔한 설계를 할 수 있습니다. 더 궁금한 노드나 상황이 있다면 언제든 물어봐 주세요\!
