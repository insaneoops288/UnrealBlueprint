# **Unreal Engine: Sequence Evaluator (by time) 노드 분석**

**Sequence Evaluator**는 애니메이션 블루프린트의 애니메이션 그래프(AnimGraph) 내에서 사용되는 노드로, 애니메이션 시퀀스의 특정 프레임을 "수동으로" 선택하여 출력할 때 사용됩니다.

## **1\. 핵심 역할**

일반적인 'Play Sequence' 노드는 시간이 흐름에 따라 자동으로 애니메이션을 재생하지만, **Sequence Evaluator**는 사용자가 제공하는 Explicit Time (명시적 시간) 값에 고정된 포즈를 반환합니다. 즉, 애니메이션의 재생 제어권을 로직(블루프린트 변수 등)으로 완전히 가져오는 노드입니다.

## **2\. 주요 입력 및 설정 항목**

### **Explicit Time (명시적 시간)**

* **타입:** Float  
* **설명:** 애니메이션 시퀀스 내에서 추출하고 싶은 정확한 시간(초 단위)입니다.  
* **활용:** 이 값을 변수와 연결하여 애니메이션을 앞뒤로 스크러빙(Scrubbing)하거나 특정 시점에 멈추게 할 수 있습니다.

### **Sequence (시퀀스)**

* **타입:** Anim Sequence Asset  
* **설명:** 평가할 대상이 되는 애니메이션 에셋을 지정합니다.

### **Teleport to Explicit Time**

* **타입:** Boolean  
* **설명:** True일 경우, 이전 시간과의 보간을 무시하고 즉시 해당 시간의 포즈로 점프합니다. 루트 모션(Root Motion)이나 특정 본(Bone)의 연속성이 중요하지 않은 경우 주로 켭니다.

### **Reinitialization Behavior**

* **설명:** 노드가 다시 활성화될 때 애니메이션을 처음부터 다시 계산할지, 아니면 이전 상태를 유지할지 결정합니다.

## **3\. 주요 활용 사례**

1. **변수에 따른 포즈 변화 (Blending by Variable)**  
   * 예: 활시위를 당기는 동작. Explicit Time을 '활시위 당김 정도(0.0 \~ 1.0)' 변수와 매핑하여, 사용자가 버튼을 누르는 정도에 따라 애니메이션 프레임을 정교하게 제어할 수 있습니다.  
2. **수동 애니메이션 블렌딩**  
   * 타임라인이나 인터폴레이션(fInterp)을 통해 Explicit Time을 부드럽게 변경하여, 복잡한 로직에 따른 애니메이션 재생을 구현합니다.  
3. **절차적 애니메이션 보조**  
   * 발 위치 보정(IK)이나 특정 각도에 따른 몸의 기울기 등을 애니메이션 시퀀스의 특정 구간에 미리 만들어두고, 각도 값을 시간 값으로 환산하여 포즈를 가져옵니다.

## **4\. Play Sequence 노드와의 차이점**

| 특징 | Play Sequence | Sequence Evaluator |
| :---- | :---- | :---- |
| **시간 흐름** | 자동으로 흐름 (Delta Time 기반) | 사용자가 직접 입력 (Explicit Time) |
| **제어 방식** | 재생/정지/루프 위주 | 특정 프레임 정밀 타격/스캔 |
| **주 용도** | 일반적인 걷기, 달리기, 대기 동작 | 로직 기반의 정밀한 포즈 제어 |

## **5\. 팁: 성능 최적화**

Sequence Evaluator는 매 틱마다 특정 시간을 계산하여 포즈를 생성하므로, 단순히 루프되는 애니메이션에는 적합하지 않습니다. **상태 머신(State Machine)** 내부에서 특정 조건에서만 동작하도록 설계하거나, 필요한 경우에만 Explicit Time을 업데이트하는 것이 성능에 유리합니다.
