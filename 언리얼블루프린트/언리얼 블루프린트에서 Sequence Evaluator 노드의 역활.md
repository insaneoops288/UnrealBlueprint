# **Unreal Engine: Sequence Evaluator 노드 가이드**

애니메이션 블루프린트의 애님 그래프(AnimGraph)에서 애니메이션 시퀀스를 배치하면 기본적으로 **Sequence Player** 상태로 생성됩니다. 이를 우클릭하여 **Convert to Sequence Evaluator**로 변환했을 때의 역할과 특징은 다음과 같습니다.

### **1\. 주요 역할: "수동 프레임 제어"**

**Sequence Evaluator**의 핵심은 애니메이션이 시간에 따라 자동으로 흐르게 두지 않고, **개발자가 특정 시간(Explicit Time)을 직접 지정하여 해당 포즈를 출력**하게 만드는 것입니다.

* **Sequence Player:** 매 프레임마다 Delta Time만큼 시간이 자동으로 경과하며 애니메이션이 재생됩니다. (Play/Loop 방식)  
* **Sequence Evaluator:** 외부에서 입력된 Explicit Time 값에 해당하는 단일 프레임 포즈를 고정적으로 출력합니다. (Scrubbing/Static 방식)

### **2\. 핵심 파라미터**

노드를 선택했을 때 디테일 패널이나 노드 핀에서 볼 수 있는 주요 설정입니다.

* **Explicit Time (명시적 시간):** 출력하고 싶은 애니메이션의 시점(초 단위)입니다. 이 값을 변수로 제어하여 애니메이션을 앞뒤로 훑거나 특정 위치에 멈춰 세울 수 있습니다.  
* **Should Loop:** (드물게 사용되지만) 시간이 애니메이션 길이를 초과할 때 루프 여부를 결정합니다.  
* **Teleport to Explicit Time:** true일 경우, 시간 값이 급격하게 변해도 보간 없이 즉시 해당 프레임으로 이동합니다.

### **3\. 언제 사용하는가? (활용 사례)**

#### **① 변수에 따른 애니메이션 제어**

게임 플레이 데이터(예: 캐릭터의 체력, 충전 게이지, 조준 각도 등)를 애니메이션의 진행도와 동기화시킬 때 사용합니다.

* *예: 활 시위를 당기는 정도를 마우스 입력값(0\~1)에 따라 애니메이션의 0초\~1초 구간으로 매핑.*

#### **② 애니메이션 브라우징 및 스크러빙(Scrubbing)**

UI 슬라이더를 움직여 캐릭터의 애니메이션을 앞뒤로 돌려보는 기능을 구현할 때 필수적입니다.

#### **③ 복잡한 블렌딩 로직**

두 애니메이션 사이의 특정 포즈만을 추출하여 고정시킨 뒤, 다른 포즈와 섞고 싶을 때 사용합니다. 일반 플레이어는 계속 재생되어 포즈가 변하지만, Evaluator는 특정 시점에 고정할 수 있기 때문입니다.

#### **④ 성능 최적화**

애니메이션의 업데이트 로직(Update Rate Optimization 등)을 탈 필요가 없는 단순한 포즈 고정의 경우, Sequence Player보다 연산 비용이 약간 더 저렴합니다.

### **4\. 사용 방법**

1. **AnimGraph**에서 애니메이션 시퀀스 노드를 배치합니다.  
2. 해당 노드를 **우클릭**합니다.  
3. 컨텍스트 메뉴에서 **Convert to Sequence Evaluator**를 선택합니다.  
4. 노드에 생긴 **Explicit Time** 핀에 float 변수를 연결하여 제어합니다.  
   * *팁: 다시 자동 재생으로 돌리고 싶다면 우클릭 후 **Convert to Sequence Player**를 누르면 됩니다.*

### **요약 비교**

| 기능 | Sequence Player | Sequence Evaluator |
| :---- | :---- | :---- |
| **재생 방식** | 자동 (Time-based) | 수동 (Input-based) |
| **주요 입력** | Play Rate (재생 속도) | Explicit Time (특정 시점) |
| **상태** | 애니메이션이 흐름 | 특정 프레임에 고정 가능 |
| **적합한 상황** | 걷기, 뛰기, 공격 등 일반 동작 | 게이지 연동, 정지된 포즈, 정밀 제어 |
