# **Set Actor Rotation 노드 완벽 이해하기**

언리얼 엔진 블루프린트에서 Set Actor Rotation은 액터의 방향을 설정하는 가장 기본적인 노드 중 하나입니다. 이 노드의 핵심 역할과 주요 파라미터에 대해 알아봅니다.

## **1\. 주요 역할**

Set Actor Rotation 노드는 선택한 액터의 회전값을 **월드 좌표계(World Coordinate System)** 기준으로 설정합니다. 즉, 액터가 어디에 있든 상관없이 게임 세상의 절대적인 북쪽, 남쪽 등을 기준으로 회전시킵니다.

* **절대값 설정**: 현재 회전값에 더하는 것이 아니라, 입력한 값으로 '변경'합니다.  
* **월드 기준**: 부모 컴포넌트나 로컬 좌표가 아닌 게임 월드 전체의 0,0,0 좌표계를 따릅니다.

## **2\. 노드 파라미터(입력단)**

| 파라미터명 | 타입 | 설명 |
| :---- | :---- | :---- |
| **Target** | Actor Object Reference | 회전시킬 대상 액터입니다. 보통 Self가 기본값입니다. |
| **New Rotation** | Rotator | 액터에게 부여할 새로운 회전값(Roll, Pitch, Yaw)입니다. |
| **Teleport Physics** | Boolean | 체크 시, 물리(Physics) 시뮬레이션 중인 오브젝트가 회전할 때 속도를 유지하면서 순간이동하듯 회전합니다. 체크하지 않으면 물리 엔진이 회전 중 충돌을 계산하여 튕겨 나갈 수 있습니다. |

## **3\. 주요 특징 및 주의사항**

### **상대 회전과의 차이**

* **Set Actor Rotation**: 월드 기준 (예: "지도의 북쪽을 봐라")  
* **Set Actor Relative Rotation**: 부모(Parent) 기준 (예: "자동차 몸체가 회전할 때 바퀴도 그에 맞춰 회전해라")  
* **Add Actor Local Rotation**: 현재 자신의 방향 기준 (예: "지금 보는 방향에서 왼쪽으로 10도 돌아라")

### **좌표계 (Rotator)**

언리얼 엔진의 Rotator는 다음 세 가지 축을 사용합니다:

* **X (Roll)**: 앞뒤 축을 기준으로 회전 (옆으로 기울기)  
* **Y (Pitch)**: 좌우 축을 기준으로 회전 (위아래 끄덕이기)  
* **Z (Yaw)**: 상하 축을 기준으로 회전 (좌우 도리도리 \- 가장 많이 사용됨)

## **4\. 자주 사용하는 패턴**

### **A. 특정 타겟 바라보기**

Find Look at Rotation 노드와 조합하여 캐릭터가 항상 플레이어를 바라보게 만들 때 자주 사용합니다.

* **Start**: 내 위치  
* **Target**: 플레이어 위치  
* **Return Value** \-\> Set Actor Rotation의 **New Rotation**에 연결

### **B. 부드러운 회전 (Interpolation)**

RInterp To 노드와 함께 사용하여 딱딱하게 변하는 것이 아니라 스르륵 부드럽게 회전하도록 구현합니다.

\[이벤트 틱\] \-\> \[RInterp To\] \-\> \[Set Actor Rotation\]

## **5\. 요약**

Set Actor Rotation은 \*\*"이 액터를 월드 기준으로 정확히 이 각도로 맞춰라"\*\*라고 명령할 때 사용하는 노드입니다. 물리 법칙을 무시하고 강제로 방향을 고정해야 할 때나, 생성(Spawn) 직후 초기 방향을 설정할 때 필수적으로 사용됩니다.
