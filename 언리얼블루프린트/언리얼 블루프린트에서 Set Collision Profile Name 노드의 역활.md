# **Set Collision Profile Name 노드 완벽 이해하기**

언리얼 엔진에서 **Set Collision Profile Name** 노드는 PrimitiveComponent(스태틱 메시, 스켈레탈 메시, 콜리전 박스 등)의 충돌 설정을 런타임 중에 쉽고 빠르게 변경할 수 있도록 돕는 도구입니다.

## **1\. 핵심 역할**

일반적으로 충돌 설정을 변경하려면 '충돌 활성화 여부(Collision Enabled)', '오브젝트 타입(Object Type)', '채널별 응답(Ignore, Overlap, Block)' 등을 개별적으로 설정해야 합니다.

**Set Collision Profile Name**은 이 모든 복잡한 설정을 하나의 \*\*프로필 이름(Preset Name)\*\*으로 묶어서 한꺼번에 적용합니다.

## **2\. 주요 파라미터**

* **Target**: 충돌 설정을 변경할 Primitive Component입니다.  
* **In Collision Profile Name**: 적용할 프로필의 이름입니다. (예: NoCollision, BlockAll, OverlapAll, Pawn 등)

## **3\. 왜 이 노드를 사용하는가?**

1. **관리의 효율성**: 여러 개의 노드(Set Collision Enabled, Set Collision Response to Channel 등)를 나열할 필요 없이 단 하나의 노드로 끝낼 수 있습니다.  
2. **일관성**: 프로젝트 설정에서 정의한 프리셋을 그대로 사용하므로, 프로젝트 전체의 충돌 로직을 일관되게 유지할 수 있습니다.  
3. **가독성**: 블루프린트 그래프가 훨씬 깔끔해집니다.

## **4\. 실전 활용 예시**

* **아이템 획득**: 바닥에 떨어진 무기를 캐릭터가 집었을 때, 무기가 더 이상 지형이나 캐릭터와 충돌하지 않도록 프로필을 NoCollision으로 바꿉니다.  
* **무적 상태/유령 효과**: 캐릭터가 특정 아이템을 먹었을 때 적이나 장애물을 통과하도록 프로필을 OverlapAll 또는 커스텀 프로필로 변경합니다.  
* **문(Door) 상태**: 문이 열려있을 때는 캐릭터가 지나갈 수 있게 하고, 잠겼을 때는 BlockAll로 변경하여 길을 막습니다.

## **5\. 사용자 지정 프로필 설정 방법**

언리얼 엔진에는 기본 프로필 외에도 사용자가 직접 규칙을 정한 프로필을 추가할 수 있습니다.

1. **Project Settings**(프로젝트 세팅)으로 이동합니다.  
2. **Engine \> Collision** 섹션을 찾습니다.  
3. **Presets** 항목에서 New... 버튼을 눌러 새로운 프로필을 생성합니다.  
4. 여기서 정의한 이름을 블루프린트의 **In Collision Profile Name** 칸에 입력하면 작동합니다.

## **6\. 주의사항**

* **이름 오타**: 프로필 이름은 'Name' 타입이므로 철자가 하나라도 틀리면 기본값으로 돌아가거나 의도치 않은 동작을 할 수 있습니다. (대소문자 구분 및 정확한 스펠링 확인 필수)  
* **컴포넌트 유형**: 이 노드는 반드시 PrimitiveComponent를 타겟으로 해야 합니다. 일반 액터(Actor) 변수를 직접 연결할 수 없으며, 액터 내부의 컴포넌트를 참조해야 합니다.

### **요약**

**Set Collision Profile Name**은 "충돌 규칙 묶음(프리셋)"을 통째로 갈아끼우는 노드입니다. 복잡한 충돌 연산을 단순화하고 프로젝트를 체계적으로 관리하고 싶을 때 반드시 사용해야 하는 필수 노드입니다.
