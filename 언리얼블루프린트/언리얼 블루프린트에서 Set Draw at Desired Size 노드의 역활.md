# **Set Draw at Desired Size 노드 가이드**

언리얼 엔진의 블루프린트에서 **Set Draw at Desired Size** 노드는 주로 \*\*위젯 컴포넌트(Widget Component)\*\*의 렌더링 방식을 제어하는 데 사용됩니다.

## **1\. 노드의 핵심 역할**

이 노드는 위젯 컴포넌트가 3D 월드에서 그려질 때, 그 크기를 사용자가 수동으로 입력한 고정 값(Draw Size)이 아니라, \*\*UMG 위젯 내부의 실제 콘텐츠 크기(Desired Size)\*\*에 맞춰 자동으로 조정할지 여부를 결정합니다.

* **True (체크됨):** 위젯 내부의 텍스트, 이미지 등 콘텐츠의 길이에 따라 위젯 컴포넌트의 쿼드(Quad) 크기가 실시간으로 변합니다.  
* **False (체크 해제됨):** Draw Size 변수에 설정된 고정된 해상도(예: 500x500)로 위젯이 그려집니다.

## **2\. 주요 특징 및 작동 원리**

### **💡 콘텐츠 기반의 동적 크기 조절**

만약 위젯 안에 이름표(Name Tag)가 있고, 이름이 짧을 때와 길 때가 다르다면 이 기능을 사용합니다. 이름이 길어지면 위젯 컴포넌트의 3D 평면도 자동으로 옆으로 길어집니다.

### **💡 픽셀 완벽도 (Pixel Perfection)**

고정된 Draw Size를 사용할 경우, 실제 위젯 내용물은 작은데 드로우 사이즈만 크면 UI가 불필요하게 늘어지거나 여백이 생길 수 있습니다. Draw at Desired Size를 켜면 딱 필요한 만큼만 렌더링 타겟을 생성하므로 시각적으로 더 깔끔합니다.

### **💡 주의사항: 피벗(Pivot) 설정**

크기가 동적으로 변하기 때문에, 위젯 컴포넌트의 **Pivot** 설정이 중요합니다.

* Pivot이 (0.5, 0.5)라면 중심을 기준으로 크기가 늘어납니다.  
* 왼쪽 정렬을 원한다면 Pivot을 (0, 0.5) 등으로 설정해야 UI가 한쪽 방향으로 자연스럽게 확장됩니다.

## **3\. 언제 사용하나요?**

1. **가변 텍스트 UI:** 캐릭터 머리 위의 이름표나 대화창처럼 글자 수에 따라 배경 크기가 변해야 하는 경우.  
2. **동적 리스트:** 아이템 슬롯이 추가됨에 따라 세로로 길어지는 3D 인벤토리 화면.  
3. **툴팁(Tooltip):** 마우스를 올렸을 때 나타나는 설명창이 내용에 따라 크기가 달라져야 할 때.

## **4\. 블루프린트 사용 예시**

1. **위젯 컴포넌트 참조:** 블루프린트 컴포넌트 패널에서 위젯 컴포넌트를 그래프로 드래그합니다.  
2. **노드 호출:** Set Draw at Desired Size 노드를 검색하여 배치합니다.  
3. **실행 핀 연결:** 보통 BeginPlay나 특정 UI 업데이트 로직 직후에 연결하여 설정을 변경합니다.

## **5\. 요약 비교**

| 기능 | Draw at Desired Size (OFF) | Draw at Desired Size (ON) |
| :---- | :---- | :---- |
| **크기 결정** | Draw Size에 입력한 수치 고정 | UMG 위젯 내부 레이아웃 크기 |
| **유연성** | 낮음 (콘텐츠가 잘리거나 여백 발생) | 높음 (내용물에 딱 맞게 조절됨) |
| **주요 용도** | 고정된 키패드, 고정 메뉴판 | 이름표, 동적 상태창, 툴팁 |
