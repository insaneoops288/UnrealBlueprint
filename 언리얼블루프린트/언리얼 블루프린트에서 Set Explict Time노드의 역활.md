# **Unreal Engine: Set Explicit Time 노드 이해하기**

언리얼 엔진의 블루프린트에서 Set Explicit Time 노드는 \*\*미디어 플레이어(Media Player)\*\*가 재생 중인 미디어의 현재 위치를 특정 시간으로 즉시 이동시키거나 고정할 때 사용됩니다.

## **1\. 노드의 주요 역할**

이 노드의 핵심 기능은 미디어 소스의 재생 헤드(Playback Head)를 개발자가 지정한 \*\*정확한 시간(Timespan)\*\*으로 설정하는 것입니다. 일반적인 Seek 노드와 유사해 보이지만, 미디어 소스의 상태를 보다 명시적으로 제어하고자 할 때 주로 사용됩니다.

## **2\. 주요 파라미터 (Inputs)**

* **Target (Media Player Object Reference):** 제어하고자 하는 미디어 플레이어 변수입니다.  
* **Time (Timespan):** 이동하고자 하는 목표 시간입니다. (예: 5초 지점, 1분 지점 등)  
* **In Immediate (Boolean):** \- True: 명령 즉시 해당 프레임으로 렌더링을 업데이트합니다. (성능 비용이 발생할 수 있지만 즉각적인 반응이 필요할 때 사용)  
  * False: 다음 미디어 업데이트 사이클까지 기다렸다가 이동합니다.

## **3\. Seek 노드와의 차이점**

기본적으로 두 노드 모두 미디어의 시간을 이동시키는 '탐색(Seeking)' 기능을 수행하지만, 맥락상 다음과 같은 차이가 있습니다.

| 특징 | Seek | Set Explicit Time |
| :---- | :---- | :---- |
| **일반적 용도** | 사용자가 슬라이더를 옮기거나 특정 구간으로 건너뛸 때 | 시스템적으로 특정 프레임을 강제하거나 동기화할 때 |
| **정밀도** | 미디어 소스의 키프레임 등에 영향을 받을 수 있음 | 지정된 시간에 대한 보다 명시적인 접근 |
| **사용 맥락** | 일반적인 비디오 재생 제어 | 시퀀서(Sequencer) 연동이나 특수 효과 동기화 |

## **4\. 주요 활용 사례**

### **A. 시퀀서(Sequencer)와의 동기화**

레벨 시퀀스(Level Sequence) 내에서 미디어 트랙을 사용할 때, 엔진의 타임라인과 비디오의 프레임을 1:1로 일치시켜야 하는 경우가 있습니다. 이때 Set Explicit Time을 사용하여 현재 시퀀서의 시간에 맞춰 비디오의 프레임을 강제로 맞출 수 있습니다.

### **B. 인터랙티브 스크러빙(Scrubbing)**

UI 슬라이더를 마우스로 드래그하여 비디오를 실시간으로 탐색할 때, 사용자의 입력에 따라 미디어 플레이어의 시간을 실시간으로 갱신하여 "화면이 멈추지 않고 부드럽게 이동하는 느낌"을 줄 수 있습니다.

### **C. 특정 이벤트 시점 이동**

게임 내에서 특정 트리거를 밟았을 때 비디오가 즉시 10초 지점부터 재생되어야 한다면, Set Explicit Time을 통해 위치를 잡고 Play 명령을 내립니다.

## **5\. 주의사항**

1. **미디어 열기 상태 확인:** 미디어 소스가 완전히 열리지 않은 상태(Open Source 직후 등)에서 이 노드를 호출하면 작동하지 않을 수 있습니다. OnMediaOpened 이벤트 이후에 호출하는 것이 안전합니다.  
2. **성능:** In Immediate를 True로 설정하고 매 프레임(Tick)마다 호출하면 미디어 디코더에 과부하가 걸려 프레임 드랍이 발생할 수 있습니다.  
3. **포맷 지원:** 모든 미디어 코덱이나 컨테이너가 정밀한 타임 탐색을 지원하는 것은 아닙니다. 고정 프레임 레이트(CFR) 미디어를 사용하는 것이 가장 안정적입니다.

### **요약**

Set Explicit Time은 미디어 플레이어에게 \*\*"지금 당장 이 시간대로 가\!"\*\*라고 명시적으로 명령하는 노드입니다. 정밀한 영상 제어나 다른 시스템과의 동기화가 필요할 때 유용하게 활용됩니다.
