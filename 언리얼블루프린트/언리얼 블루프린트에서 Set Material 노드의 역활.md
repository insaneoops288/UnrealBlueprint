# **Set Material 노드의 역할과 활용**

언리얼 엔진 블루프린트에서 **Set Material** 노드는 게임 플레이 도중 객체의 외형을 프로그래밍 방식으로 변경할 때 가장 기본적으로 사용되는 노드입니다.

## **1\. 핵심 역할**

Set Material 노드는 특정 **Primitive Component**(Static Mesh, Skeletal Mesh 등)의 지정된 머티리얼 슬롯에 새로운 머티리얼을 할당합니다. 즉, 에디터에서 미리 설정된 머티리얼을 무시하고, 로직에 따라 실시간으로 외형을 바꿀 수 있게 해줍니다.

## **2\. 노드 구성 요소 (Input Pins)**

* **Target (대상):** 머티리얼을 변경할 메시 컴포넌트를 연결합니다. (예: 캐릭터의 Mesh, 월드의 Static Mesh)  
* **Element Index (엘리먼트 인덱스):** 메시가 가진 머티리얼 슬롯 번호입니다.  
  * 0부터 시작하며, 메시가 여러 개의 머티리얼 구역으로 나뉘어 있을 때 특정 구역만 바꾸고 싶다면 해당 인덱스 번호를 지정해야 합니다.  
* **Material (머티리얼):** 새롭게 적용하고 싶은 머티리얼 또는 머티리얼 인스턴스 자산을 연결합니다.

## **3\. 주요 활용 사례**

1. **상태 변화 표현:**  
   * 캐릭터가 대미지를 입었을 때 일시적으로 빨간색 머티리얼로 변경.  
   * 아이템을 획득했을 때 반짝이는 효과가 있는 머티리얼로 교체.  
2. **커스터마이징:**  
   * 플레이어의 선택에 따라 옷이나 무기의 색상/재질을 변경.  
3. **상호작용 피드백:**  
   * 마우스 커서를 올렸을 때(OnBeginCursorOver) 하이라이트 머티리얼을 적용하여 강조.

## **4\. 주의사항 및 팁**

### **인덱스 확인**

메시 에디터에서 해당 메시가 몇 개의 슬롯을 가지고 있는지, 내가 바꾸려는 부분의 인덱스가 몇 번인지 반드시 확인해야 합니다. 잘못된 인덱스를 넣으면 아무런 변화가 없거나 엉뚱한 곳이 바뀔 수 있습니다.

### **Set Material vs Dynamic Material Instance**

단순히 머티리얼 자체를 '교체'하는 것이라면 Set Material을 쓰면 됩니다. 하지만 머티리얼 내부의 **파라미터(색상 값, 텍스처 수치 등)만 실시간으로 부드럽게 조절**하고 싶다면, Create Dynamic Material Instance 노드를 사용하여 인스턴스를 먼저 만든 후 그 결과값을 Set Material로 한 번 적용해준 뒤 파라미터를 조절하는 방식이 더 효율적입니다.

### **성능 고려**

프레임마다 너무 자주 머티리얼을 교체하는 것은 드로우 콜(Draw Call)에 영향을 줄 수 있으므로, 필요한 시점에만(이벤트 발생 시) 호출하는 것이 좋습니다.

**요약:** Set Material은 "이 물체의 옷(머티리얼)을 지금 당장 다른 것으로 갈아입혀라"라는 명령을 내리는 노드라고 이해하시면 됩니다.
