# **언리얼 엔진: Set Owner 노드의 역할과 활용**

Set Owner 노드는 언리얼 엔진에서 한 액터가 다른 액터에 의해 '소유'되도록 설정하는 기능을 합니다. 이는 단순한 계층 구조를 넘어, 네트워크 복제(Replication)와 입력 시스템에서 매우 중요한 의미를 갖습니다.

## **1\. 주요 역할**

### **① 네트워크 복제 및 권한 (Networking & Replication)**

멀티플레이어 게임에서 Set Owner는 해당 액터의 \*\*연결 소유권(Connection Ownership)\*\*을 결정합니다.

* **서버와 클라이언트:** 특정 클라이언트의 PlayerController를 액터의 Owner로 설정하면, 서버는 해당 액터가 그 클라이언트와 연관되어 있다고 판단합니다.  
* **조건부 복제:** Cond\_OwnerOnly 같은 복제 조건을 사용할 때, 소유자에게만 데이터가 전송되도록 제어할 수 있습니다.  
* **RPC (Remote Procedure Calls):** 액터에서 Run on Owning Client RPC를 실행하려면 반드시 해당 액터의 Owner가 유효한 PlayerController여야 합니다.

### **② 입력 처리 (Input Handling)**

언리얼의 입력 시스템은 기본적으로 PlayerController \-\> Pawn \-\> Owned Actors 순서로 흐릅니다.

* 특정 액터(예: 인벤토리 아이템, 장착된 무기)가 플레이어의 입력을 직접 받게 하려면, 해당 플레이어의 Pawn이나 Controller를 Owner로 설정해야 합니다.

### **③ 시각적 제어 (Owner No See / Only Owner See)**

액터 컴포넌트의 렌더링 옵션 중에는 소유자에게만 보이거나, 소유자에게만 보이지 않게 하는 설정이 있습니다.

* **Owner No See:** 1인칭 게임에서 플레이어 자신의 몸체가 카메라를 가리지 않도록 설정할 때 사용합니다.  
* **Only Owner See:** 플레이어 본인에게만 보이는 UI 요소나 가이드라인 등을 구현할 때 활용합니다.

## **2\. 주요 사용 사례**

| 사례 | 설명 |
| :---- | :---- |
| **무기 시스템** | 캐릭터가 무기를 집었을 때, 무기의 Owner를 캐릭터(Pawn)로 설정하여 발사 입력이나 리플리케이션을 관리합니다. |
| **발사체 (Projectiles)** | 총알을 소환할 때 발사한 캐릭터를 Owner로 설정하면, 나중에 '누가 죽였는가'를 판별하거나 팀킬 방지 로직을 짤 때 유용합니다. |
| **차량 및 탑승물** | 플레이어가 차량에 탑승할 때 차량의 Owner를 플레이어로 변경하여 제어권을 획득합니다. |

## **3\. 주의사항**

1. **서버 전용:** Set Owner는 반드시 \*\*서버(Server/Authority)\*\*에서 호출해야 합니다. 클라이언트에서 호출하면 서버의 상태와 동기화되지 않아 무시됩니다.  
2. **수명 주기:** Owner로 설정된 액터가 파괴(Destroy)된다고 해서 소유된 액터가 자동으로 파괴되지는 않지만, 논리적인 연결 고리가 끊기게 됩니다.  
3. **Null 설정:** New Owner 핀을 비워두고 호출하면 소유권을 해제할 수 있습니다.

## **4\. 블루프린트 구성 예시**

\[Event Interaction\] \-\> \[Set Owner (Target: Weapon, New Owner: PlayerPawn)\]

* 위와 같이 설정하면 이제 Weapon 액터 내에서 Get Owner를 호출했을 때 PlayerPawn을 반환하며, 플레이어의 연결 권한을 공유하게 됩니다.
