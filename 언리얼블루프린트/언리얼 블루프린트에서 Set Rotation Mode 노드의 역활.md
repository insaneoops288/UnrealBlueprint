# **언리얼 블루프린트: Set Rotation Mode 노드의 역할**

Set Rotation Mode는 캐릭터가 이동하거나 가만히 있을 때, **몸의 방향(Rotation)을 무엇에 맞출 것인가**를 설정하는 노드입니다. 주로 캐릭터 무브먼트 컴포넌트(Character Movement Component)와 연동되어 캐릭터의 시각적 방향을 제어합니다.

## **1\. 주요 역할 (Role)**

이 노드의 핵심은 캐릭터의 **상태에 따라 회전 로직을 실시간으로 교체**하는 것입니다. 일반적인 캐릭터 게임에서는 크게 두 가지 모드를 오갑니다.

### **A. Velocity Direction (이동 방향 기준)**

* **설명:** 캐릭터가 현재 움직이고 있는 방향(Velocity)을 향해 몸을 돌립니다.  
* **사용 예시:** 3인칭 어드벤처 게임에서 캐릭터가 달릴 때, 카메라 방향과 상관없이 이동하는 쪽을 바라보게 할 때 사용합니다.  
* **내부 동작:** bOrientRotationToMovement 옵션이 활성화된 것과 유사하게 작동합니다.

### **B. Looking Direction (바라보는 방향 기준 / 스트레이프)**

* **설명:** 캐릭터가 컨트롤러(또는 카메라)가 바라보는 방향을 계속 유지합니다.  
* **사용 예시:** FPS/TPS 슈팅 게임에서 조준(Aiming) 중일 때나, 옆으로 걷기(Strafe)를 할 때 사용합니다. 캐릭터는 앞을 보면서 옆이나 뒤로 이동할 수 있게 됩니다.  
* **내부 동작:** bUseControllerDesiredRotation 옵션이 활성화된 것과 유사하게 작동합니다.

## **2\. 왜 이 노드를 사용하는가?**

기본적인 Character Movement Component에도 회전 관련 변수들이 있지만, Set Rotation Mode를 사용하는 이유는 다음과 같습니다.

1. **상태 관리의 편의성:** "조준 모드", "비전투 모드" 등의 열거형(Enum) 값을 통해 복잡한 회전 변수 여러 개를 한 번에 제어할 수 있습니다.  
2. **부드러운 전환:** 단순히 체크박스를 켜고 끄는 것이 아니라, 모드 변경 시 보간(Interpolation)을 통해 캐릭터가 부드럽게 방향을 돌리도록 설계된 경우가 많습니다.  
3. **애니메이션 연동:** Rotation Mode가 무엇이냐에 따라 애니메이션 블루프린트(ABP)에서 Orientation Warping이나 Stride Warping 같은 기술을 적용할지 결정하는 기준이 됩니다.

## **3\. 대표적인 활용 시나리오 (Lyra 등)**

| 모드 이름 | 캐릭터의 행동 | 적합한 게임 장르 |
| :---- | :---- | :---- |
| **Velocity Direction** | 조이스틱을 미는 방향으로 캐릭터가 즉시 회전함. | 오픈월드 탐험, 플랫폼 게임 |
| **Looking Direction** | 카메라가 도는 대로 캐릭터도 같이 회전함 (게걸음 가능). | 슈팅 게임(TPS), 하드코어 액션 |
| **Aiming** | Looking Direction과 유사하지만 더욱 정밀하게 컨트롤러 회전에 동기화됨. | 정밀 조준 시 |

## **4\. 블루프린트 구현 팁**

이 노드를 호출할 때는 보통 다음과 같은 흐름을 가집니다.

1. **이벤트 발생:** 유저가 우클릭(조준)을 누름.  
2. **노드 호출:** Set Rotation Mode를 호출하고 인자를 Looking Direction으로 설정.  
3. **결과:** 캐릭터가 즉시 카메라 앞을 바라보며 조준 포즈를 취함.  
4. **이벤트 해제:** 우클릭을 떼면 다시 Velocity Direction으로 복구.

### **주의사항**

Set Rotation Mode는 언리얼 엔진의 **기본(Base) 엔진 노드가 아닐 수 있습니다.** 많은 경우 CharacterMovementComponent를 상속받아 커스텀하게 만든 함수이거나, 특정 플러그인에서 제공하는 기능입니다. 만약 노드가 검색되지 않는다면, 사용 중인 캐릭터 클래스나 컴포넌트에 해당 함수가 정의되어 있는지 확인해야 합니다.
