# **Set Sequence with Inertial Blending 상세 분석**

언리얼 엔진(UE4/UE5)의 애니메이션 블루프린트에서 제공되는 이 노드는 **관성 블렌딩(Inertial Blending)** 기술을 사용하여 현재 재생 중인 애니메이션 시퀀스를 새로운 시퀀스로 즉각적으로 교체하는 역할을 합니다.

## **1\. 관성 블렌딩(Inertial Blending)이란?**

기존의 \*\*표준 블렌딩(Linear/Smooth)\*\*은 전환 구간 동안 '이전 애니메이션'과 '다음 애니메이션' 두 개를 동시에 계산하여 섞는 방식입니다. 반면, **관성 블렌딩**은 물리 기반의 포스트 프로세스 방식입니다.

* **작동 방식:** 전환이 시작되는 순간, 이전 애니메이션의 포즈, 속도, 가속도 정보를 캡처합니다. 그 후 이전 애니메이션은 즉시 중단하고, 다음 애니메이션을 재생하면서 캡처된 운동 에너지를 수학적으로 감쇠시켜 목표 포즈로 자연스럽게 이동시킵니다.  
* **핵심 철학:** "이전 애니메이션을 계속 돌릴 필요 없이, 그 물리적 관성만 가져와서 다음 애니메이션에 얹는다."

## **2\. 노드의 주요 역할 및 파라미터**

이 노드는 보통 **애니메이션 블루프린트의 이벤트 그래프** 또는 **Linked Anim Graph** 내의 로직에서 호출됩니다.

| 파라미터 | 설명 |
| :---- | :---- |
| **Sequence** | 새로 재생할 대상 애니메이션 시퀀스입니다. |
| **Inertial Blend Time** | 관성 블렌딩이 지속될 시간(초)입니다. 짧을수록 빠르고 딱딱하며, 길수록 부드럽습니다. |
| **Blend Profile** | 본(Bone)별로 가중치를 다르게 설정한 프로파일이 있다면 지정할 수 있습니다. |

## **3\. 관성 블렌딩의 장점**

1. **성능 최적화 (CPU 이득):**  
   * 표준 블렌딩은 전환 중에 두 애니메이션 시퀀스를 모두 업데이트(Tick)해야 합니다.  
   * 관성 블렌딩은 전환 시작 즉시 소스 애니메이션을 꺼버리므로, 한 번에 하나의 애니메이션만 계산하면 됩니다. 수많은 캐릭터가 등장하는 게임에서 큰 성능 차이를 만듭니다.  
2. **부드러운 시각적 효과:**  
   * 물리적 관성을 고려하기 때문에, 갑작스러운 방향 전환이나 포즈 변화 시 "미끄러지는" 듯한 느낌 없이 묵직하고 자연스러운 전환이 일어납니다.  
3. **복잡한 상태 머신 단순화:**  
   * 수많은 전이(Transition) 선을 그릴 필요 없이 로직에 따라 노드 하나로 시퀀스를 교체할 수 있습니다.

## **4\. 필수 요구 사항: Inertialization 노드**

Set Sequence with Inertial Blending 노드를 사용하려면 반드시 \*\*애니메이션 그래프(AnimGraph)\*\*에 **Inertialization** 노드가 배치되어 있어야 합니다.

* **위치:** 포즈가 최종적으로 출력되기 직전(Output Pose 전) 또는 캐시된 포즈 뒤에 배치합니다.  
* **역할:** 그래프 상류에서 발생한 "관성 블렌딩 요청"을 실제로 계산하여 최종 포즈에 적용하는 '처리 장치' 역할을 합니다. 이 노드가 없으면 블렌딩이 작동하지 않고 애니메이션이 툭툭 끊기게 됩니다.

## **5\. 사용 예시 (Best Practice)**

* **상태 변화가 잦은 경우:** 캐릭터의 발걸음이 왼쪽, 오른쪽, 대기 상태 등으로 빠르게 전환될 때 유용합니다.  
* **피격 리액션:** 어떤 동작을 하다가 갑자기 피격 애니메이션으로 넘어갈 때, 이전 동작의 운동량을 유지하며 자연스럽게 꺾이는 연출이 가능합니다.  
* **성능이 중요한 모바일 게임:** 다수의 NPC 애니메이션을 처리할 때 CPU 부하를 획기적으로 줄여줍니다.

## **6\. 주의사항**

* **포즈 차이가 극심할 때:** 완전히 반대되는 포즈(예: 누워 있다가 갑자기 서기)로 관성 블렌딩을 길게 잡으면 관성이 너무 커서 캐릭터의 몸이 비정상적으로 꼬여 보일 수 있습니다. 이럴 때는 블렌드 타임을 매우 짧게 설정해야 합니다.
