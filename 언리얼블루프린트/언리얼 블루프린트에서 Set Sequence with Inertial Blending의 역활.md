언리얼 블루프린트 Set Sequence with Inertial Blending?

언리얼 엔진에서 Set Sequence with Inertial Blending (또는 이와 유사한 관성 블렌딩 방식)은 애니메이션 시퀀스를 교체할 때 물리적인 관성을 계산하여 포즈를 부드럽게 연결하는 고성능 블렌딩 기술입니다.

기존의 전통적인 크로스페이드(Standard Blend) 방식과 무엇이 다른지, 그리고 어떻게 사용하는지 핵심 위주로 정리해 드릴게요.

1. 관성 블렌딩(Inertial Blending)이란?
전통적인 크로스페이드는 두 애니메이션을 동시에 재생하며 가중치를 조절하지만, 관성 블렌딩은 전환이 시작되는 순간 이전 포즈의 속도와 가속도를 계산하여 대상 포즈로 밀어넣는 방식입니다.

성능 우위: 두 애니메이션을 동시에 계산할 필요가 없어 연산 비용이 저렴합니다.

자연스러운 흐름: 갑작스러운 방향 전환이나 멈춤 동작에서 물리적으로 더 자연스러운 느낌을 줍니다.

2. 사용을 위한 필수 준비 단계
이 노드나 기능을 제대로 사용하려면 반드시 **애님 그래프(AnimGraph)**에 관성화를 처리할 노드가 포함되어 있어야 합니다.

Inertialization 노드 배치: 애님 그래프의 최종 포즈(Output Pose) 바로 직전에 Inertialization 노드를 연결하세요.

노드 설정: Sequence Player 노드를 선택한 후, 디테일 패널에서 블렌딩 방식을 Inertial로 설정하거나 관련 함수를 호출합니다.

3. 주요 파라미터 설명
Set Sequence with Inertial Blending 노드(또는 관련 함수)에서 자주 쓰이는 파라미터는 다음과 같습니다.

파라미터	설명
Sequence	새로 재생할 타겟 애니메이션 시퀀스입니다.
Blend Time	전환에 소요되는 시간입니다. (보통 0.1~0.2초의 짧은 시간이 효과적입니다.)
Inertialization Request	관성 블렌딩을 트리거합니다. 이 요청이 있어야 Inertialization 노드가 작동합니다.
4. 장단점 비교
특징	표준 블렌드 (Standard)	관성 블렌드 (Inertial)
CPU 비용	높음 (두 애니메이션 동시 계산)	낮음 (하나만 계산)
포즈 일치	두 애니메이션이 비슷해야 자연스러움	포즈 차이가 커도 물리적으로 보정됨
사용 사례	부드러운 상태 변화 (걷기 ↔ 달리기)	빠른 반응이 필요한 동작 (피격, 방향 전환)
⚠️ 주의사항
Inertialization 노드 누락: 애님 그래프에 Inertialization 노드가 없으면 블렌딩이 작동하지 않고 툭 끊기듯(Snap) 바뀝니다.

너무 긴 블렌드 시간: 관성 블렌딩은 물리적 감쇠를 시뮬레이션하므로, 0.5초 이상의 긴 블렌딩에는 적합하지 않을 수 있습니다.
