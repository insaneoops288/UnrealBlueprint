# **Set Sequence 노드의 역할과 활용**

언리얼 엔진의 시퀀서(Sequencer) 시스템에서 Set Sequence 노드는 동적으로 연출을 교체할 때 핵심적인 역할을 합니다.

## **1\. 주요 역할**

Set Sequence 노드는 **Target**(보통 Level Sequence Actor 또는 Sequence Player)이 참조하고 있는 \*\*Sequence 에셋(.uasset)\*\*을 실시간으로 변경합니다.

* **할당(Assignment):** 특정 플레이어가 어떤 시나리오(시퀀스)를 재생할지 결정합니다.  
* **재사용성:** 하나의 Level Sequence Actor를 레벨에 배치해두고, 상황에 따라 A 커트신, B 커트신을 갈아 끼우며 재생할 수 있게 해줍니다.

## **2\. 노드의 구성 (Input Pins)**

보통 다음과 같은 핀으로 구성됩니다.

| 핀 이름 | 역할 |
| :---- | :---- |
| **Target** | 시퀀스를 설정할 대상입니다. (예: Level Sequence Actor 컴포넌트나 변수) |
| **In Sequence** | 새롭게 할당할 Level Sequence 에셋을 선택합니다. |

## **3\. 왜 사용하나요? (사용 시나리오)**

### **① 동적 커트신 교체**

플레이어의 선택에 따라 다른 엔딩 시퀀스를 보여줘야 할 때, 여러 개의 액터를 배치하는 대신 하나의 플레이어에 Set Sequence로 에셋만 교체하여 메모리를 절약하고 관리를 단순화할 수 있습니다.

### **② 데이터 기반 연출**

데이터 테이블(Data Table)에 시퀀스 에셋 경로를 저장해두고, 게임 로직에 따라 해당 데이터를 읽어와 시퀀스를 설정하고 재생할 때 필수적입니다.

## **4\. 주의사항 (중요\!)**

1. **재생 상태:** Set Sequence를 호출한다고 해서 시퀀스가 자동으로 \*\*재생(Play)\*\*되는 것은 아닙니다. 설정을 마친 후 반드시 Play 또는 Play Looping 노드를 별도로 호출해야 합니다.  
2. **바인딩 문제:** 시퀀스 에셋이 바뀌면 내부의 '액터 바인딩(Actor Binding)' 정보가 유효하지 않을 수 있습니다. 만약 새 시퀀스가 다른 액터들을 제어해야 한다면, Set Binding 노드를 통해 새롭게 대상을 지정해줘야 할 수도 있습니다.  
3. **초기화:** 재생 중인 상태에서 Set Sequence를 호출하면 예기치 않은 그래픽 버그가 발생할 수 있으므로, 가급적 Stop을 먼저 호출하거나 재생 전 단계에서 설정하는 것이 좋습니다.

## **5\. 블루프린트 흐름 예시**

1. **Event BeginPlay** (또는 상호작용 이벤트)  
2. **Get Level Sequence Actor** (레벨에 배치된 액터 참조)  
3. **Set Sequence** (새로운 시나리오 에셋 연결)  
4. **Play** (재생 시작)
