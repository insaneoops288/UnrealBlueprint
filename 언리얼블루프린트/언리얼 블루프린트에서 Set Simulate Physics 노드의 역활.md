# **Set Simulate Physics 노드의 역할과 사용법**

언리얼 엔진의 블루프린트에서 Set Simulate Physics 노드는 특정 컴포넌트(주로 Static Mesh나 Skeletal Mesh)가 엔진의 물리 엔진(Chaos Physics 등)의 영향을 받을지 여부를 실시간으로 결정하는 역할을 합니다.

## **1\. 핵심 역할**

이 노드의 주요 기능은 \*\*"물리 시뮬레이션의 활성화/비활성화"\*\*입니다.

* **True(활성화):** 해당 오브젝트가 중력의 영향을 받아 아래로 떨어지고, 다른 물체와 충돌했을 때 튕겨 나가거나 구르는 등 물리 법칙에 따라 반응합니다.  
* **False(비활성화):** 오브젝트가 공중에 고정되거나 애니메이션/코드에 의해서만 움직이게 됩니다. 물리 법칙에 의한 자유 낙하나 충돌 반응은 무시됩니다.

## **2\. 노드 파라미터 구성**

* **Target (Primitive Component):** 물리를 적용할 대상 컴포넌트입니다. (예: 스태틱 메시, 스켈레탈 메시 등)  
* **Simulate (Boolean):** \* 체크됨: 물리 시뮬레이션 시작.  
  * 체크 해제: 물리 시뮬레이션 중단.

## **3\. 사용 시 필수 조건 (Pre-requisites)**

이 노드가 정상적으로 작동하려면 다음 조건들이 충족되어야 합니다.

1. **Mobility(모빌리티) 설정:** 해당 컴포넌트의 모빌리티가 \*\*Moveable(무버블)\*\*로 설정되어 있어야 합니다. Static(스테틱)이나 Stationary(스테이셔너리)인 경우 물리 시뮬레이션이 불가능합니다.  
2. **Collision(콜리전) 설정:** 오브젝트에 콜리전(충돌체)이 설정되어 있어야 하며, 콜리전 프리셋이 **Physics Actor** 혹은 물리 반응이 가능한 설정이어야 합니다.  
3. **Root Component 고려:** 블루프린트 액터의 루트가 아닌 하위 컴포넌트에 물리를 적용하면, 해당 컴포넌트가 액터 본체에서 떨어져 나가 독자적으로 움직일 수 있습니다.

## **4\. 실전 활용 사례**

### **A. 아이템 드랍 시스템**

플레이어가 인벤토리에서 아이템을 버릴 때, 아이템 액터를 생성하고 Set Simulate Physics를 True로 설정하면 아이템이 자연스럽게 바닥으로 떨어집니다.

### **B. 캐릭터 사망 (Ragdoll)**

캐릭터가 사망했을 때, 스켈레탈 메시 컴포넌트에 대해 이 노드를 실행하면 캐릭터가 물리 법칙에 따라 털썩 쓰러지는 '래그돌(Ragdoll)' 상태가 됩니다.

### **C. 상호작용 오브젝트**

평소에는 고정되어 있던 문이나 상자가 플레이어의 공격을 받았을 때만 물리적으로 반응하게 하려면, 공격 받는 시점에 이 노드를 호출하여 물리를 켭니다.

## **5\. 주의사항 및 팁**

* **성능:** 너무 많은 오브젝트가 동시에 Simulate Physics를 활성화하면 CPU 성능에 부담을 줄 수 있습니다. 필요하지 않을 때는 다시 끄는 것이 좋습니다.  
* **네트워크 복제:** 멀티플레이어 환경에서는 물리 시뮬레이션의 위치값이 서버와 클라이언트 간에 차이가 날 수 있으므로, 적절한 Replicate 설정이나 위치 보정이 필요합니다.  
* **중력 제어:** 물리를 켠 상태에서 공중에 떠 있게 하고 싶다면, Set Enable Gravity 노드를 함께 사용하여 중력만 따로 끌 수도 있습니다.

**요약하자면,** Set Simulate Physics는 정적인 물체를 동적인 '물리 객체'로 바꾸어 주는 스위치와 같습니다.
