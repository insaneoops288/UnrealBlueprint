# **Set Skeletal Mesh 노드 개요**

언리얼 엔진 블루프린트에서 **Set Skeletal Mesh** 노드는 **Skeletal Mesh Component**가 참조하는 스켈레탈 메시 애셋을 실시간으로 교체하는 역할을 합니다.

## **1\. 주요 역할**

* **캐릭터 코스튬 교체:** 플레이어가 상점에서 옷을 갈아입거나 장비를 장착할 때 외형을 바꿉니다.  
* **변신 시스템:** 캐릭터가 특정 아이템을 먹고 다른 생명체나 강화된 모습으로 변할 때 사용합니다.  
* **모듈형 캐릭터 구성:** 머리, 몸통, 다리 등 파츠가 나누어진 캐릭터에서 특정 부위의 메시만 교체할 때 활용합니다.  
* **최적화:** 멀리 있는 캐릭터의 메시를 더 단순한 메시로 수동 교체하거나, 상황에 따라 다른 퀄리티의 메시를 보여줄 때 사용합니다.

## **2\. 노드 입력 항목 (Input Pins)**

| 입력 이름 | 타입 | 설명 |
| :---- | :---- | :---- |
| **Target** | Skeletal Mesh Component | 메시를 변경할 대상 컴포넌트입니다. (보통 캐릭터 블루프린트의 'Mesh' 컴포넌트) |
| **New Mesh** | Skeletal Mesh Object Reference | 새로 할당할 스켈레탈 메시 애셋을 선택합니다. |
| **Reinit Pose** | Boolean | True인 경우, 메시가 바뀔 때 애니메이션 포즈를 초기 상태(A-Pose/T-Pose)로 재설정합니다. 기본값은 True입니다. |

## **3\. 작동 원리 및 특징**

1. **런타임 변경:** 게임이 실행 중인 상태에서 즉각적으로 외형이 바뀝니다.  
2. **기존 데이터 유지:** 컴포넌트에 설정된 트랜스포름(위치, 회전, 크기)이나 부모-자식 관계는 유지된 채로 '알맹이(메시)'만 바뀝니다.  
3. **애니메이션 영향:** 새로운 메시로 바뀌면 이전 메시에서 재생 중이던 애니메이션 시퀀스나 몽타주가 중단될 수 있습니다.

## **4\. 주의사항 (매우 중요)**

### **A. 스켈레톤(Skeleton) 호환성**

가장 중요한 점은 **기존 메시와 새로운 메시가 동일한 스켈레톤을 공유해야 한다**는 것입니다.

* 만약 스켈레톤이 다르면 애니메이션이 정상적으로 작동하지 않거나 캐릭터가 기괴하게 찌그러질 수 있습니다.  
* 스켈레톤이 다른 메시로 교체해야 한다면, **Set Anim Instance Class** 노드를 함께 사용하여 해당 메시에 맞는 애니메이션 블루프린트도 같이 교체해줘야 합니다.

### **B. 애니메이션 블루프린트 재설정**

Set Skeletal Mesh를 호출하면 가끔 애니메이션 블루프린트 인스턴스가 초기화되거나 연결이 끊기는 경우가 발생할 수 있습니다. 메시 교체 후 애니메이션이 멈춘다면 Set Anim Instance Class 노드로 다시 한번 애니메이션 블루프린트를 지정해주는 것이 안전합니다.

### **C. 소켓(Socket) 및 부착물**

기존 메시의 특정 소켓에 무기나 아이템을 붙여놓았다면, 메시가 바뀌었을 때 새로운 메시에도 **동일한 이름의 소켓**이 있어야 부착물이 올바른 위치에 유지됩니다.

## **5\. 활용 예시 (블루프린트 로직)**

1. **이벤트 발생:** (예: 'E' 키 입력)  
2. **조건 체크:** (예: 현재 입고 있는 옷이 'A'인가?)  
3. **노드 실행:** Set Skeletal Mesh 호출 \-\> New Mesh 칸에 'B' 옷 메시 할당.  
4. **후처리:** 필요한 경우 Set Anim Instance Class로 애니메이션 보정.

**요약하자면:** Set Skeletal Mesh는 캐릭터의 **'피부'나 '외형'을 갈아끼우는 노드**이며, 이때 뼈대(Skeleton)가 호환되는지를 확인하는 것이 가장 큰 관건입니다.
