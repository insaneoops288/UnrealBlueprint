# **Set SkeletalMesh Asset 노드 상세 가이드**

언리얼 엔진의 블루프린트에서 Set SkeletalMesh Asset 노드는 캐릭터나 오브젝트의 형태를 실시간으로 변경할 때 사용하는 핵심 노드입니다.

## **1\. 노드의 주요 역할**

이 노드는 **Skeletal Mesh Component**를 타겟으로 삼아, 해당 컴포넌트가 렌더링하고 계산하는 '스켈레탈 메쉬 에셋'을 다른 것으로 즉시 교체합니다.

* **동적 외형 변경:** 게임 플레이 도중 캐릭터의 옷, 갑옷, 무기 또는 캐릭터 전체 모델을 바꿀 수 있습니다.  
* **메쉬 최적화:** 상황에 따라 더 단순한 메쉬로 교체하거나, 보이지 않는 거리에서 메쉬를 제거/교체하는 용도로 활용됩니다.

## **2\. 주요 입력 파라미터**

* **Target (Skeletal Mesh Component):** 메쉬를 바꿀 대상 컴포넌트입니다.  
* **New Mesh (Skeletal Mesh Object Reference):** 새로 할당할 스켈레탈 메쉬 에셋입니다.  
* **Reinit Pose (Boolean):** \* True (기본값): 메쉬가 바뀔 때 애니메이션 포즈를 초기화합니다.  
  * False: 현재의 포즈를 최대한 유지하려고 시도합니다 (스켈레톤 구조가 동일할 때 유용).

## **3\. 실전 활용 사례**

### **① 캐릭터 커스터마이징 (Armor/Outfits)**

사용자가 인벤토리에서 갑옷을 착용하면, 해당 부위의 스켈레탈 메쉬를 새 에셋으로 교체합니다.

* *예: '기본 신체' 메쉬에서 '강철 갑옷' 메쉬로 변경*

### **② 캐릭터 변신 또는 진화**

특정 아이템을 먹거나 레벨업 시 캐릭터의 외형 전체를 다른 스켈레탈 메쉬로 교체하여 시각적 변화를 줍니다.

### **③ 모듈형 캐릭터 (Modular Characters)**

머리, 몸통, 팔, 다리를 각각의 Skeletal Mesh Component로 분리한 뒤, 각 부위별로 Set SkeletalMesh Asset을 호출하여 조합형 캐릭터를 만듭니다.

## **4\. 기술적 주의사항 (중요)**

### **A. 스켈레톤(Skeleton) 호환성**

교체하려는 New Mesh가 기존 컴포넌트에서 사용 중인 **Animation Blueprint**와 호환되는 **Skeleton**을 공유해야 합니다. 스켈레톤이 다르면 애니메이션이 재생되지 않거나 모델이 비정상적으로 뒤틀릴 수 있습니다.

### **B. 애니메이션 블루프린트 재설정**

메쉬를 교체한 후 애니메이션이 멈추는 경우가 있습니다. 이 경우 Set Anim Instance Class 노드를 함께 호출하여 애니메이션 블루프린트를 다시 지정해 주어야 할 때가 많습니다.

### **C. 소켓(Socket) 및 부착물**

기존 메쉬의 소켓에 무언가(칼, 총 등)를 붙여놓았다면, 새 메쉬에도 **동일한 이름의 소켓**이 있어야 부착물이 위치를 유지합니다. 소켓 이름이 다르면 부착물이 캐릭터의 원점(Root)으로 이동해 버릴 수 있습니다.

### **D. 성능 고려**

런타임에 큰 용량의 메쉬 에셋을 빈번하게 교체하면 프레임 드랍(Hitch)이 발생할 수 있습니다. 필요한 메쉬는 미리 로드(Async Load)해 두는 것이 좋습니다.

## **5\. 요약**

Set SkeletalMesh Asset은 \*\*"외형(Mesh)을 실시간으로 갈아끼우는 명령어"\*\*입니다. 하지만 단순히 껍데기만 바꾸는 것이 아니라, 그와 연결된 **애니메이션, 소켓, 물리 설정** 등을 함께 고려해야 완벽한 구현이 가능합니다.
