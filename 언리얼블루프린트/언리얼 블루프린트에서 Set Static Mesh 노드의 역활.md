# **Set Static Mesh 노드 상세 가이드**

언리얼 엔진 블루프린트에서 Set Static Mesh 노드는 게임 플레이 도중 오브젝트의 외형(모델)을 실시간으로 변경할 때 사용하는 핵심적인 노드입니다.

## **1\. 노드의 주요 역할**

이 노드는 **Static Mesh Component**를 타겟으로 삼아, 해당 컴포넌트에 할당된 **Static Mesh 자산**을 새로운 것으로 갈아끼웁니다.

* **변경 전**: A라는 모양의 바위 메시  
* **노드 실행 후**: B라는 모양의 나무 메시 (동일한 컴포넌트 위치에서 모델만 변경됨)

## **2\. 입력 항목 (Inputs)**

* **Target (스태틱 메시 컴포넌트)**: 메시를 바꿀 대상 컴포넌트입니다. (예: 액터 내부의 'Mesh' 컴포넌트)  
* **New Mesh (스태틱 메시 오브젝트 레퍼런스)**: 새로 적용하고 싶은 스태틱 메시 자산을 선택합니다.

## **3\. 주요 활용 사례**

### **① 아이템 장착 및 교체**

캐릭터가 손에 들고 있는 무기나 장비가 스태틱 메시일 경우, 다른 아이템을 습득했을 때 외형을 즉시 바꿀 수 있습니다.

* 예: 철검(Iron Sword) 메시를 황금검(Gold Sword) 메시로 교체

### **② 상태 변화 시각화 (파괴 등)**

오브젝트가 대미지를 입었을 때 '온전한 상태의 메시'를 '부서진 상태의 메시'로 교체하여 시각적 효과를 줍니다.

* 예: 멀쩡한 창문 \-\> 깨진 창문

### **③ 커스터마이징 시스템**

플레이어가 캐릭터의 모자, 안경, 가방 등 부착물의 외형을 선택할 때 사용합니다.

### **④ 상호작용 피드백**

퍼즐 게임 등에서 버튼을 눌렀을 때 장치의 모양이 변하거나, 스위치의 상태가 바뀌는 것을 표현할 때 유용합니다.

## **4\. 주의사항 및 팁**

### **성능 (Performance)**

* 이 노드는 호출될 때 렌더링 상태를 업데이트하므로, **매 프레임(Tick)마다 호출하는 것은 피해야 합니다.** 메시가 실제로 바뀌어야 하는 순간에만 한 번 실행하는 것이 좋습니다.

### **콜리전 (Collision)**

* 메시를 교체하면 해당 메시가 가지고 있는 새로운 콜리전 데이터가 적용됩니다. 하지만 컴포넌트 자체의 콜리전 설정(Query Only, Physics Only 등)은 유지되므로, 메시 크기가 크게 변할 경우 물리 엔진 동작을 확인해야 합니다.

### **재질 (Materials)**

* Set Static Mesh로 메시를 바꾸면, 기본적으로 **새로운 메시 자산에 설정되어 있는 기본 재질**이 입혀집니다. 만약 특정 재질을 강제로 유지하거나 바꾸고 싶다면, 이 노드 다음에 Set Material 노드를 추가로 사용해야 합니다.

## **5\. 블루프린트 구성 예시**

1. **Event BeginPlay** 혹은 **Custom Event** 발생  
2. **Set Static Mesh** 노드 연결  
3. **Target**에 Static Mesh Component 드래그 앤 드롭  
4. **New Mesh** 핀에 원하는 메시 에셋 할당
