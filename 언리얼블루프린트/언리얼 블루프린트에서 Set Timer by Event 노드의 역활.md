# **Set Timer by Event 노드 가이드**

언리얼 엔진 블루프린트에서 Set Timer by Event 노드는 특정 \*\*이벤트(Event)\*\*를 지정된 \*\*시간(Time)\*\*이 흐른 뒤에 호출하도록 예약하는 역할을 합니다.

## **1\. 노드의 주요 역할**

1. **지연 실행 (Delay Execution):** 특정 로직을 즉시 실행하지 않고, 몇 초 뒤에 한 번만 실행하고 싶을 때 사용합니다.  
2. **반복 실행 (Looping):** Tick과 비슷하지만, 더 최적화된 방식으로 일정한 간격(예: 1초마다, 0.5초마다)으로 로직을 반복 실행합니다.  
3. **시간 제어 (Time Management):** 타이머 핸들(Timer Handle)을 통해 실행 중인 타이머를 일시 정지, 재개 또는 중단할 수 있습니다.

## **2\. 주요 파라미터 (입력 항목)**

* **Event:** 실행될 커스텀 이벤트나 함수를 연결합니다. (위쪽의 빨간색 사각형 핀)  
* **Time:** 이벤트가 실행될 때까지의 대기 시간(초 단위)입니다.  
* **Looping:** 이 옵션을 체크하면 Time 간격마다 이벤트가 무한히 반복됩니다. 체크하지 않으면 한 번만 실행됩니다.  
* **Initial Start Delay:** 타이머가 처음 시작될 때만 적용되는 추가 대기 시간입니다.  
* **Initial Start Delay Variance:** 첫 실행 대기 시간에 무작위성(오차)을 부여하고 싶을 때 사용합니다.

## **3\. 출력 항목 (Return Value)**

* **Timer Handle:** 이 노드는 **타이머 핸들(Timer Handle)** 구조체를 반환합니다. 이 핸들은 매우 중요한데, 나중에 이 타이머를 멈추거나(Clear and Invalidate Timer by Handle), 남은 시간을 확인하거나, 일시 정지할 때 이 값을 참조해야 하기 때문입니다.

## **4\. 왜 Tick 대신 타이머를 쓰나요?**

* **성능 최적화:** Event Tick은 매 프레임마다 호출되므로 CPU 소모가 큽니다. 반면 타이머는 필요한 주기에만 실행되므로 훨씬 효율적입니다.  
* **제어의 용이성:** 특정 상황(예: 캐릭터 사망)에서 로직을 멈추기가 훨씬 쉽습니다.

## **5\. 실무 활용 예시**

### **1\) 총기 연사 (Full Auto Fire)**

* 마우스를 클릭했을 때 Set Timer by Event를 호출 (Looping 체크).  
* Time에 연사 속도(예: 0.1초)를 입력.  
* 연결된 이벤트에서 총알 발사 로직 수행.  
* 마우스를 뗐을 때 Clear Timer로 중단.

### **2\) 체력 자동 회복 (HP Regen)**

* 전투 중이 아닐 때 1초마다 체력을 5씩 회복하도록 설정.

### **3\) 일정 시간 후 파괴 (Delayed Destruction)**

* 아이템을 먹은 뒤 3초 후에 아이템 메시가 사라지게 할 때.

## **6\. 주의사항**

* **핸들 관리:** 반복(Looping) 타이머를 생성했다면, 반드시 필요 없을 때 Clear and Invalidate Timer by Handle 노드를 호출하여 꺼주어야 합니다. 그렇지 않으면 메모리와 성능에 불필요한 영향을 줄 수 있습니다.  
* **0초 입력:** Time에 0을 입력하면 다음 프레임에 바로 실행되거나 에러가 발생할 수 있으니 주의해야 합니다.
