# **Set Timer by Event 노드 상세 가이드**

언리얼 엔진에서 **Set Timer by Event**는 시간에 기반한 로직을 처리할 때 사용하는 핵심 노드입니다. 실행 흐름(Execution Line)을 잠시 멈추는 Delay 노드와 달리, 백그라운드에서 시간을 측정하며 지정된 '이벤트'를 트리거합니다.

## **1\. 노드의 주요 역할**

1. **지연 실행 (Delayed Execution):** 특정 시간이 지난 후 단 한 번 이벤트를 실행합니다.  
2. **반복 실행 (Looping):** 설정한 시간 간격마다 이벤트를 무한히 반복해서 실행합니다. (예: 초당 체력 회복, 총기 연사)  
3. **비동기 처리:** 타이머가 돌아가는 동안 다른 블루프린트 로직은 멈추지 않고 계속 실행됩니다.

## **2\. 주요 입력 및 출력 핀 설명**

### **입력 (Inputs)**

* **Event (Delegate):** 실행하고자 하는 **Custom Event**나 **Function**을 연결합니다. 이벤트의 빨간색 사각형 핀을 이 곳에 드래그하여 연결합니다.  
* **Time:** 이벤트를 실행하기까지의 시간(초 단위, Float)입니다.  
* **Looping:** 체크하면 Time에 설정된 간격마다 이벤트가 반복됩니다. 체크 해제 시 1회만 실행됩니다.  
* **Initial Start Delay:** 타이머가 처음 시작될 때만 적용되는 추가 지연 시간입니다.  
* **Initial Start Delay Variance:** 첫 실행 지연 시간에 무작위성을 부여합니다.

### **출력 (Outputs)**

* **Return Value (Timer Handle):** 생성된 타이머에 대한 \*\*'핸들(일종의 리모컨)'\*\*을 반환합니다. 이 핸들을 변수로 저장해두면 나중에 해당 타이머를 강제로 멈추거나, 일시정지하거나, 남은 시간을 확인하는 데 사용할 수 있습니다.

## **3\. Delay 노드와의 결정적 차이점**

| 특징 | Delay | Set Timer by Event |
| :---- | :---- | :---- |
| **흐름 제어** | 실행 라인을 해당 노드에서 멈춤 | 실행 라인은 즉시 통과됨 (백그라운드 실행) |
| **반복성** | 루프를 직접 그려야 함 | Looping 옵션 하나로 해결 |
| **취소/조절** | 중간에 취소하기 어려움 | Timer Handle을 통해 언제든 중단 가능 |
| **성능** | 단순 지연에 적합 | 지속적인 시스템 로직에 적합 |

## **4\. 주요 활용 사례**

1. **무기 연사 (Full Auto Fire):**  
   * 마우스를 클릭하면 타이머 시작(Looping 활성화).  
   * Time에 연사 속도 설정.  
   * 마우스를 떼면 Clear and Invalidate Timer by Handle을 사용하여 사격 중지.  
2. **도트 데미지 (Damage over Time):**  
   * 독에 걸렸을 때 1초마다 체력을 깎는 이벤트를 반복 실행.  
   * 5초 뒤에 타이머를 해제하는 별도의 로직 구성.  
3. **UI 갱신:**  
   * 0.1초마다 서버 상태나 플레이어의 거리를 체크하여 UI 텍스트 업데이트.

## **5\. 관리 팁 (Best Practices)**

* **변수화:** Return Value인 Timer Handle은 반드시 \*\*변수로 승격(Promote to Variable)\*\*시켜 저장하는 것이 좋습니다. 그래야 나중에 Clear Timer by Handle 노드로 타이머를 안전하게 종료할 수 있습니다.  
* **중복 실행 방지:** 특정 조건에서 타이머를 새로 시작하기 전에 이미 돌아가고 있는 동일한 타이머 핸들이 유효한지(Is Timer Active by Handle) 확인하는 로직을 추가하면 버그를 줄일 수 있습니다.

## **6\. 관련 노드**

* **Clear and Invalidate Timer by Handle:** 타이머를 멈추고 핸들을 무효화합니다.  
* **Pause Timer by Handle:** 타이머를 일시정지합니다.  
* **Unpause Timer by Handle:** 일시정지된 타이머를 다시 시작합니다.  
* **Get Timer Remaining Time by Handle:** 이벤트 실행까지 남은 시간을 확인합니다.
