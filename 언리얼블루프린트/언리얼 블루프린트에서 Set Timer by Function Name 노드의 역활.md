# **Set Timer by Function Name 노드 상세 가이드**

언리얼 엔진에서 **Set Timer by Function Name** 노드는 코드의 실행 흐름을 시간 기반으로 제어할 때 사용됩니다. Tick 이벤트를 사용하지 않고도 특정 로직을 주기적으로 실행할 수 있어 성능 최적화에도 중요한 역할을 합니다.

## **1\. 주요 역할**

* **지연 실행:** 특정 시간(초)이 지난 후 함수를 단 한 번 실행합니다.  
* **반복 실행 (Looping):** 일정 간격마다 함수를 계속해서 호출합니다 (예: 1초마다 체력 회복).  
* **Tick 대용:** 매 프레임 실행될 필요가 없는 로직을 낮은 빈도로 실행하여 CPU 부하를 줄입니다.

## **2\. 노드 파라미터 상세 설명**

| 입력 핀 (Input Pin) | 설명 |
| :---- | :---- |
| **Object** | 함수가 포함되어 있는 오브젝트입니다. 보통 Self를 연결합니다. |
| **Function Name** | 호출하려는 함수의 **정확한 이름**을 문자열(String)로 입력합니다. (오타 주의) |
| **Time** | 함수가 호출되기까지의 대기 시간(초 단위)입니다. |
| **Looping** | 체크하면 Time 간격으로 함수를 무한 반복 호출합니다. |
| **Initial Start Delay** | 첫 번째 실행에만 적용되는 추가 지연 시간입니다. |
| **Initial Start Delay Variance** | 첫 실행 지연 시간에 무작위성(랜덤)을 부여합니다. |

## **3\. 출력: Timer Handle (타이머 핸들)**

이 노드는 **Timer Handle** 구조체를 반환합니다. 이 핸들은 생성된 타이머의 '고유 ID'와 같으며, 다음과 같은 작업에 사용됩니다:

* **Pause Timer:** 타이머 일시 정지  
* **Unpause Timer:** 타이머 재개  
* **Clear Timer by Handle:** 타이머를 중단하고 제거 (가장 많이 사용)  
* **Is Timer Active:** 타이머가 현재 작동 중인지 확인

## **4\. 사용 시 주의사항**

1. **이름 정확도:** 함수 이름을 문자열로 직접 입력하기 때문에, 함수 이름을 바꾸면 이 노드의 텍스트도 반드시 수정해야 합니다. (이 단점 때문에 최근에는 Set Timer by Event 노드를 더 선호하기도 합니다.)  
2. **함수 조건:** 호출하려는 함수는 **입력 파라미터(Input Parameters)가 없어야 합니다.** 인자가 있는 함수는 이 노드로 호출할 수 없습니다.  
3. **성능:** 매 프레임 동작하는 Tick 대신, 0.1초나 0.5초 간격의 타이머를 사용하면 게임 성능을 크게 개선할 수 있습니다.

## **5\. 활용 예시**

* **무기 연사:** 공격 버튼을 누르면 0.1초 간격으로 Fire 함수를 반복 실행하고, 버튼을 떼면 핸들을 사용해 타이머를 해제합니다.  
* **상태 효과:** 독 데미지를 1초마다 5번 입히고 타이머를 종료합니다.  
* **UI 업데이트:** 현재 점수나 서버 상태를 2초마다 갱신합니다.
