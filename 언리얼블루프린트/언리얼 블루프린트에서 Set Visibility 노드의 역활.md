# **언리얼 블루프린트: Set Visibility 노드 상세 가이드**

Set Visibility 노드는 언리얼 엔진 블루프린트에서 \*\*씬 컴포넌트(Scene Component)\*\*의 시각적 표시 여부를 제어하는 데 사용됩니다. 캐릭터의 장비를 교체하거나, 퍼즐 요소를 나타나게 하는 등 게임의 동적인 시각 변화를 줄 때 필수적인 노드입니다.

## **1\. 노드의 핵심 역할**

이 노드의 주요 목적은 특정 컴포넌트가 **렌더링(Rendering) 엔진에 의해 그려질지 말지**를 결정하는 것입니다.

* **Visible (Checked):** 컴포넌트가 화면에 나타납니다.  
* **Hidden (Unchecked):** 컴포넌트가 화면에서 사라집니다.

## **2\. 주요 입력 파라미터 (Inputs)**

| 항목 | 설명 |
| :---- | :---- |
| **Target** | 가시성을 변경할 대상을 지정합니다. 주로 Static Mesh, Skeletal Mesh, Particle System, Widget Component 등이 연결됩니다. |
| **New Visibility** | 체크박스 형태의 Boolean 값입니다. 체크하면 보이고, 해제하면 숨깁니다. |
| **Propagate to Children** | 자식 컴포넌트에게도 동일한 설정을 적용할지 여부입니다. True일 경우 해당 컴포넌트에 붙어있는 모든 하위 요소들도 함께 보이거나 숨겨집니다. |

## **3\. 주요 특징 및 주의사항**

### **① 렌더링 vs 콜리전 (Collision)**

* **중요:** Set Visibility는 오직 **시각적인 모습**만 제어합니다.  
* 컴포넌트를 숨기더라도 해당 컴포넌트의 **콜리전(충돌 판정)은 여전히 남아 있을 수 있습니다.**  
* 만약 물리적으로도 사라지게 하고 싶다면 Set Collision Enabled 노드를 함께 사용하거나, 아예 Set Actor Hidden in Game을 사용해야 합니다.

### **② 성능 최적화**

* 화면에 보이지 않는(Hidden) 객체는 드로우 콜(Draw Call)을 발생시키지 않으므로 성능 최적화에 도움이 됩니다.  
* 하지만 메모리에는 여전히 로드되어 있으므로, 완전히 필요 없는 객체라면 Destroy Actor를 고려해야 합니다.

### **③ Hidden in Game과의 차이**

* 디테일 패널의 Hidden in Game 옵션은 초기 상태를 설정하는 것이고, Set Visibility 노드는 게임 플레이 도중 실시간으로 이 값을 변경하는 수단입니다.

## **4\. 실제 활용 예시**

1. **아이템 획득 효과:** 플레이어가 아이템에 닿았을 때 아이템 메쉬를 숨기고 파티클 효과만 재생함.  
2. **무기 교체 시스템:** 캐릭터가 들고 있는 여러 무기 중 현재 사용하지 않는 무기들은 Set Visibility를 꺼두고, 장착한 무기만 켬.  
3. **UI 위젯 컴포넌트:** 캐릭터 머리 위의 체력 바나 이름을 상황에 따라(예: 전투 중일 때만) 표시함.  
4. **레이더/미니맵:** 특정 구역에 진입했을 때만 가이드 화살표를 나타나게 함.

## **5\. 팁: Toggle Visibility**

가시성을 단순히 켜고 끄는 것이 목적이라면 Set Visibility 대신 **Toggle Visibility** 노드를 사용할 수 있습니다. 이 노드는 현재 상태가 '보임'이면 '숨김'으로, '숨김'이면 '보임'으로 자동으로 반전시켜 줍니다.
