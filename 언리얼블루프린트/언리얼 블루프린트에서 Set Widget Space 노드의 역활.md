# **Set Widget Space 노드 상세 분석**

언리얼 엔진의 블루프린트에서 Set Widget Space 노드는 **Widget Component**의 Widget Space 속성을 런타임 중에 변경할 때 사용합니다. 이 설정은 UI가 게임의 3D 공간에 존재할지, 아니면 플레이어의 2D 화면 위에 그려질지를 결정합니다.

## **1\. 주요 역할 및 옵션**

이 노드는 EWidgetSpace 열거형(Enum) 값을 입력으로 받으며, 두 가지 모드를 제공합니다.

### **A. World (월드 스페이스)**

* **개념:** UI를 실제 3D 월드 내의 한 오브젝트처럼 취급합니다.  
* **특징:**  
  * **거리감:** 카메라와 위젯 사이의 거리에 따라 크기가 커지거나 작아집니다.  
  * **가려짐(Occlusion):** 다른 3D 액터(벽, 캐릭터 등) 뒤로 가면 UI가 가려집니다.  
  * **회전:** 액터의 회전 값에 영향을 받습니다.  
  * **용도:** 게임 내 컴퓨터 모니터, 상호작용 가능한 키오스크, 벽에 붙은 메뉴판 등 '다이아제틱(Diegetic) UI'에 적합합니다.

### **B. Screen (스크린 스페이스)**

* **개념:** 위젯의 위치는 3D 월드 좌표를 추적하지만, 실제 렌더링은 플레이어의 2D 화면(Viewport) 위에 오버레이로 그려집니다.  
* **특징:**  
  * **항상 정면:** 카메라가 어디를 보든 항상 플레이어를 정면으로 바라봅니다 (Billboarding 효과).  
  * **고정 크기:** 일반적으로 거리에 상관없이 일정한 해상도를 유지하려고 합니다 (설정에 따라 조절 가능).  
  * **최상단 렌더링:** 월드 오브젝트에 의해 가려지지 않고 화면 가장 앞쪽에 표시됩니다.  
  * **용도:** 캐릭터 머리 위의 닉네임, HP 바, 퀘스트 마커, 아이템 이름표 등에 적합합니다.

## **2\. World vs Screen 비교표**

| 특성 | World Space | Screen Space |
| :---- | :---- | :---- |
| **렌더링 위치** | 3D 월드 좌표 내 | 2D 뷰포트 상단 |
| **거리 영향** | 거리에 따라 크기 변화 | 항상 일정한 선명도 유지 |
| **깊이 테스트** | 다른 물체에 의해 가려짐 | 항상 화면 앞에 표시됨 |
| **라이팅 영향** | 적용 가능 (Emissive 등) | 라이팅 영향 없음 |
| **퍼포먼스** | 상대적으로 무거움 (Post-process 등 영향) | 상대적으로 가벼움 |

## **3\. 블루프린트 사용 예시**

보통 다음과 같은 상황에서 Set Widget Space를 사용합니다.

1. **상태 변화에 따른 전환:**  
   * 평소에는 캐릭터 머리 위에 **Screen Space**로 HP바를 띄우다가, 특정 연출(컷신 등)에서 UI가 배경 뒤로 숨어야 할 때 **World Space**로 변경할 수 있습니다.  
2. **가독성 확보:**  
   * 멀리 있는 오브젝트의 UI가 너무 작아서 안 보일 때, 런타임에 **Screen Space**로 전환하여 화면에 크게 표시되도록 강제할 수 있습니다.

## **4\. 주의사항**

* **해상도 문제:** World Space일 때는 위젯 컴포넌트의 Draw Size 설정이 실제 월드 크기에 영향을 미치므로 적절한 픽셀 밀도를 설정해야 글자가 깨지지 않습니다.  
* **상호작용:** Screen Space 위젯은 마우스 클릭 등을 처리할 때 Widget Interaction Component 설정이 월드 방식과 약간 다를 수 있으므로 확인이 필요합니다.  
* **VR/AR:** VR 환경에서는 Screen Space UI가 사용자의 눈에 너무 가깝게 붙어 어지러움을 유발할 수 있으므로, 대부분 **World Space** 사용을 권장합니다.
