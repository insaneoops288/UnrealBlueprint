# **Unreal Engine: Set Widget Space 노드 가이드**

언리얼 엔진의 \*\*Widget Component(위젯 컴포넌트)\*\*에서 사용하는 Set Widget Space 노드는 위젯이 렌더링되는 "공간"을 정의합니다. 이 설정에 따라 UI가 게임 세계의 일부처럼 보일지, 아니면 화면에 달라붙은 2D 요소처럼 보일지가 결정됩니다.

## **1\. 주요 역할 및 옵션**

이 노드는 Widget Space라는 열거형(Enum) 값을 입력받으며, 두 가지 옵션이 존재합니다.

### **① World (월드 공간)**

* **특징**: 위젯이 실제 3D 월드 좌표계 내의 한 물체처럼 존재합니다.  
* **시각적 효과**:  
  * 카메라와의 거리에 따라 크기가 작아지거나 커집니다 (원근감 적용).  
  * 다른 3D 액터 뒤로 숨겨질 수 있습니다 (오클루전 적용).  
  * 라이팅이나 포스트 프로세스의 영향을 받을 수 있습니다.  
* **활용**: 게임 내 컴퓨터 모니터 화면, 벽에 붙은 메뉴판, VR 인터페이스 등.

### **② Screen (스크린 공간)**

* **특징**: 위젯이 월드에 배치된 위치를 추적하지만, 실제 렌더링은 플레이어의 화면(Viewport) 위 2D 레이어에 그려집니다.  
* **시각적 효과**:  
  * 카메라와 멀어져도 설정된 해상도 크기를 그대로 유지합니다.  
  * 항상 다른 3D 물체보다 위에 그려집니다 (Z-Order 무시).  
  * 3D 회전값이 적용되지 않고 항상 카메라를 정면으로 응시합니다.  
* **활용**: 캐릭터 머리 위의 체력바(HP Bar), 플레이어 이름표, 퀘스트 마커 등.

## **2\. 왜 실시간으로 변경하나요? (Set 노드의 활용)**

에디터 디테일 패널에서도 설정할 수 있지만, 블루프린트에서 Set Widget Space 노드를 사용하는 이유는 **상황에 따른 가시성 변화** 때문입니다.

* **상호작용 시 전환**: 멀리 있을 때는 캐릭터 머리 위에 2D로 보이던 이름표(Screen)가, 캐릭터에게 가까이 다가가 대화를 시작하면 실제 3D 월드의 메뉴(World)로 자연스럽게 전환되도록 연출할 수 있습니다.  
* **최적화**: 멀리 있는 NPC의 UI는 World 공간에서 작게 렌더링하여 연산 부하를 줄이고, 가까운 대상만 Screen으로 표시하여 가독성을 높이는 방식입니다.

## **3\. 주의사항 및 팁**

1. **해상도 차이**: Screen 공간은 위젯 블루프린트의 디자인 해상도를 그대로 따르지만, World 공간은 위젯 컴포넌트의 Draw Size 설정에 따라 화질이 결정됩니다. World 모드에서 글자가 흐리다면 Draw Size를 키우고 컴포넌트의 Scale을 줄여야 합니다.  
2. **입력 처리**: World 공간 위젯은 플레이어 컨트롤러의 Show Mouse Cursor와 위젯 컴포넌트의 Receive Hardware Input 설정이 중요합니다. 반면 Screen 공간 위젯은 일반적인 Add to Viewport와 유사하게 동작합니다.  
3. **성능**: 화면에 수많은 Screen Space 위젯이 떠 있으면 드로우 콜(Draw Call)이 증가할 수 있으므로, 거리에 따른 Hidden in Game 처리가 권장됩니다.

## **4\. 블루프린트 구성 예시**

1. **Target**: 수정하려는 Widget Component를 연결합니다.  
2. **New Space**: World 또는 Screen 중 원하는 모드를 선택합니다.  
3. **실행 타이밍**: 보통 BeginPlay나 특정 이벤트(예: 타겟팅 모드 진입) 발생 시 실행합니다.
