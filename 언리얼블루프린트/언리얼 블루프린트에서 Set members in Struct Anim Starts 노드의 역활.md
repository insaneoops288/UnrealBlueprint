# **Set members in Struct 노드의 역할과 활용**

언리얼 엔진에서 **Set members in Struct** 노드는 구조체 변수의 전체 내용을 교체하지 않고, **원하는 특정 멤버(내부 변수)만 골라서 값을 수정**할 때 사용합니다.

## **1\. 노드의 핵심 기능**

일반적으로 구조체의 값을 변경하려면 구조체를 분해(Break)한 뒤, 다시 합치는(Make) 과정을 거쳐야 합니다. 하지만 Set members in Struct 노드를 사용하면 다음과 같은 장점이 있습니다.

* **선택적 수정**: 노드를 선택한 후 '디테일(Details)' 패널에서 수정하고 싶은 변수만 체크하여 노출시킬 수 있습니다.  
* **가독성 향상**: 불필요한 'Break/Make' 노드 연결을 줄여 블루프린트 그래프를 깔끔하게 유지합니다.  
* **참조(Reference) 전달**: 구조체 변수가 'By Reference'로 전달된 경우, 원본 데이터를 직접 수정하는 효율적인 처리가 가능합니다.

## **2\. 애니메이션 블루프린트(AnimBP)에서의 역할**

애니메이션 시스템, 특히 UE5의 **Thread Safe(스레드 안전)** 환경에서 이 노드는 자주 사용됩니다.

### **애니메이션 노드 속성 수정**

UE5에서는 Update Animation 대신 Blueprint Thread Safe Update Animation 함수를 권장합니다. 이때 특정 애니메이션 노드(예: Blend Space, Play Sequence 등)의 속성을 변경하려면 해당 노드의 참조를 구조체 형태로 받아와서 수정해야 합니다.

* **역할**: 애니메이션 상태(State)나 노드 참조 구조체 내부의 값(예: 재생 비율, 블렌드 가중치, 활성화 여부 등)을 실시간으로 업데이트합니다.  
* **예시**: 특정 조건이 충족되었을 때 애니메이션 시퀀스의 시작 지점이나 재생 속도를 담은 구조체 멤버를 이 노드로 변경하여 시각적 효과를 제어합니다.

## **3\. 사용 방법**

1. **구조체 연결**: 수정하려는 구조체 변수를 노드의 왼쪽 'Struct (By Ref)' 핀에 연결합니다.  
2. **멤버 선택**: 노드를 클릭합니다.  
3. **디테일 패널**: 오른쪽 'Details' 패널에서 수정하고자 하는 변수의 체크박스를 선택합니다.  
4. **값 입력**: 노드에 나타난 핀에 새로운 값을 입력하거나 변수를 연결합니다.

## **4\. 주의사항**

* **By Ref vs By Value**: 이 노드는 연결된 구조체가 '참조(By Ref)'인 경우 원본을 수정하지만, 단순히 '값(By Value)'으로 복사된 데이터를 연결하면 해당 노드 내의 로컬 복사본만 수정되고 원본 변수는 변하지 않을 수 있습니다.  
* **애니메이션 스레드**: 애니메이션 그래프 내에서 직접 구조체를 수정할 때는 스레드 안전성(Thread Safety)을 항상 확인해야 하며, 가급적 속성 바인딩(Property Binding)이나 스레드 안전 함수 내에서 처리하는 것이 좋습니다.

## **5\. 결론**

"Anim Starts"와 같은 구조체에서 이 노드를 사용한다는 것은, **애니메이션이 시작되거나 실행 중인 상태에서 특정 설정값(시작 시간, 루프 여부, 트랜지션 데이터 등)을 동적으로, 그리고 깔끔하게 변경하기 위함**이라고 이해하시면 됩니다.
