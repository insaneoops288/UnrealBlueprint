# **Should Distance Match Stop 노드의 역할**

언리얼 엔진의 **거리 매칭(Distance Matching)** 시스템 내에서 Should Distance Match Stop 노드는 캐릭터의 물리적 상태와 애니메이션 데이터를 비교하여, "지금 애니메이션을 정지 시퀀스로 전환하면 목표 지점에서 발 미끄러짐 없이 딱 멈출 수 있는가?"를 계산하는 함수입니다.

## **1\. 핵심 기능**

이 노드는 보통 \*\*캐릭터 무브먼트 컴포넌트(Character Movement Component)\*\*의 감속도(Deceleration)와 현재 속도를 기반으로 다음을 수행합니다:

* **정지 예측:** 현재 속도로 이동 중일 때, 완전히 멈추기까지 남은 거리(Distance to Stop)를 계산합니다.  
* **전환 시점 결정:** 남은 거리가 특정 임계값(Threshold)보다 작아지거나, 정지 애니메이션이 커버할 수 있는 거리 안에 들어왔을 때 True를 반환합니다.

## **2\. 왜 사용하는가? (발 미끄러짐 방지)**

전통적인 애니메이션 방식은 속도가 0이 되면 단순히 Idle 상태로 블렌딩하지만, 이는 발이 지면에서 미끄러지는 현상(Foot Sliding)을 유발합니다.

Should Distance Match Stop을 사용하면:

1. 캐릭터가 실제 물리적으로 멈추기 직전에 정지 애니메이션을 미리 재생합니다.  
2. 애니메이션의 재생 속도를 남은 거리에 동기화하여 발이 지면에 붙어 있는 것처럼 보이게 합니다.

## **3\. 주요 입력 파라미터**

보통 다음과 같은 데이터를 입력받습니다:

* **Distance to Location:** 목표 지점(또는 정지 예상 지점)까지의 거리.  
* **Velocity:** 캐릭터의 현재 속도.  
* **Deceleration:** 캐릭터 무브먼트 설정상의 감속 값.  
* **Stop Threshold:** 정지 상태로 간주할 최소 거리 값.

## **4\. 일반적인 사용 흐름 (Workflow)**

1. **State Machine:** 'Run' 상태에서 'Stop' 상태로 가는 트랜지션 로직에 이 노드를 배치합니다.  
2. **Logic:** Should Distance Match Stop이 True를 반환하면 정지 애니메이션으로 전환합니다.  
3. **Distance Matching:** 정지 애니메이션 내에서 실제 남은 거리를 기반으로 애니메이션 포즈를 샘플링하여 정밀한 정지를 구현합니다.

## **5\. 요약**

Should Distance Match Stop은 **"물리적 감속 거리"와 "애니메이션의 정지 거리"를 일치시키기 위한 트리거**입니다. 고품질의 캐릭터 이동 시스템을 구축할 때 필수적인 노드 중 하나입니다.
