# **SpawnActorFromClass 노드 상세 가이드**

언리얼 엔진 블루프린트에서 SpawnActorFromClass 노드는 런타임(게임 실행 중)에 특정 클래스의 **액터(Actor) 인스턴스를 월드에 생성**하는 역할을 합니다.

## **1\. 주요 역할 및 기능**

* **동적 생성**: 미리 배치된 액터 외에, 필요에 따라 실시간으로 적, 투사체, 아이템, 이펙트 등을 생성합니다.  
* **클래스 정의**: 어떤 종류의 액터를 만들지 Class 핀을 통해 지정합니다.  
* **위치 및 회전 설정**: 생성될 액터의 월드 내 위치, 회전, 스케일을 결정합니다.

## **2\. 주요 입력 핀 (Inputs)**

| 핀 이름 | 설명 |
| :---- | :---- |
| **Class** | 생성할 액터의 블루프린트 클래스 또는 C++ 클래스를 선택합니다. |
| **Spawn Transform** | 액터가 생성될 위치(![][image1]), 회전(![][image2]), 크기(![][image3]) 정보를 담은 트랜스포머입니다. |
| **Collision Handling Override** | 생성 위치에 다른 물체가 있을 경우의 처리 방식입니다. (예: 무조건 생성, 조정 후 생성, 생성 실패 등) |
| **Owner** | 생성된 액터의 소유자를 지정합니다. 네트워크 복제(Replication) 시 중요합니다. |
| **Instigator** | 생성 원인(주로 대미지를 가한 폰/캐릭터)을 지정합니다. |

## **3\. 출력 핀 (Outputs)**

* **Return Value**: 생성된 액터의 \*\*객체 참조(Object Reference)\*\*를 반환합니다. 이를 통해 생성 직후 변수를 설정하거나 함수를 호출하는 등 즉각적인 조작이 가능합니다. 생성에 실패하면 None을 반환합니다.

## **4\. 일반적인 사용 예시**

1. **발사체 발사**: 플레이어가 총을 쏠 때 총구 위치(![][image4])에서 탄환 액터를 생성합니다.  
2. **적 스폰 시스템**: 특정 구역에 플레이어가 진입하면 지정된 스폰 포인트에서 적 인스턴스를 생성합니다.  
3. **아이템 드랍**: 몬스터가 사망할 때 해당 위치에서 전리품 상자나 아이템 액터를 생성합니다.

## **5\. 주의사항 및 팁**

### **1\) 유효성 검사 (Is Valid)**

액터 생성은 충돌이나 메모리 문제로 실패할 수 있습니다. 반환된 값을 사용하기 전에 반드시 Is Valid 노드로 체크하는 것이 안전합니다.

### **2\) 지연 생성 (Expose on Spawn)**

생성할 액터 내부의 변수에 **'Expose on Spawn(스폰 시 노출)'** 옵션을 체크해두면, SpawnActor 노드 자체에 해당 변수를 입력할 수 있는 핀이 나타납니다. 이는 생성과 동시에 초기 데이터를 전달할 때 매우 유용합니다.

### **3\) 퍼포먼스**

수많은 액터를 한꺼번에 스폰하는 작업은 CPU에 부하를 줄 수 있습니다. 총알처럼 빈번하게 생성/파괴되는 객체는 **객체 풀링(Object Pooling)** 기법을 고려하는 것이 좋습니다.
