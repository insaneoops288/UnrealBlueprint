언리얼 블루프린트에서 TMap타입에서 Find노드의 역활?

언리얼 엔진 블루프린트에서 TMap(맵) 데이터 타입의 Find 노드는 특정 **키(Key)**를 사용해 그에 연결된 **값(Value)**을 찾아내는 역할을 합니다.

쉽게 말해, 사전에서 단어(Key)를 찾아 그 의미(Value)를 가져오는 것과 같습니다.

1. 주요 기능
Find 노드는 입력한 키가 맵 안에 존재하는지 확인하고, 존재한다면 해당 값을 출력합니다.

2. 노드의 구성 (핀 설명)
Target (Input): 탐색할 대상이 되는 TMap 변수를 연결합니다.

Key (Input): 찾고자 하는 특정 키 값을 입력합니다. (예: 아이템 이름, ID 등)

Value (Output): 키를 찾았을 때 반환되는 값입니다.

Return Value (Output, Boolean): 가장 중요한 부분 중 하나입니다. 키를 성공적으로 찾았으면 True, 찾지 못했으면 False를 반환합니다.

3. 사용 시 주의사항 및 팁
성공 여부 확인 필수: Find 노드 바로 다음에 Branch(분기) 노드를 연결하여 Return Value가 True인지 먼저 확인하는 것이 좋습니다. 키가 없는데 Value를 바로 사용하려고 하면, 기본값(Default Value)이 출력되거나 의도치 않은 논리 오류가 발생할 수 있습니다.

참조 반환: 블루프린트의 Find 노드는 해당 값의 복사본이 아닌 **참조(Reference)**를 반환하는 경우가 많습니다. 즉, 반환된 값을 수정하면 맵 안에 들어있는 실제 데이터도 변경될 수 있습니다 (변수 설정에 따라 다를 수 있음).
