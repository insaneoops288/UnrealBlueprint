언리얼 블루프린트에서 \*\*TMap(맵)\*\*은 '키(Key)'와 '값(Value)'이 한 쌍으로 묶여 저장되는 데이터 보관함입니다. 여기서 **Find** 노드는 \*\*"특정 키를 제시했을 때, 그에 연결된 값을 찾아와라"\*\*라고 명령하는 역할을 합니다.

쉽게 비유하자면, 수많은 사물함이 있는 복도에서 \*\*'사물함 번호(Key)'\*\*를 넣으면 그 안에 들어있는 \*\*'물건(Value)'\*\*을 꺼내 주는 것과 같습니다.

## ---

**1\. 노드의 구조**

* **Target (입력):** 검색 대상이 될 TMap 변수입니다.  
* **Key (입력):** 찾고자 하는 고유한 식별자입니다. (예: 아이템 이름, 플레이어 ID 등)  
* **Value (출력):** 키를 찾는 데 성공했을 때 반환되는 실제 데이터입니다.  
* **Return Value (출력 \- Boolean):** 가장 중요한 핀입니다. **찾기에 성공하면 True, 실패하면 False**를 반환합니다.

## ---

**2\. 왜 사용하나요? (장점)**

### **① 압도적인 검색 속도**

배열(Array)에서 특정 항목을 찾으려면 첫 번째 칸부터 마지막 칸까지 일일이 확인해야 하지만(ForEachLoop), TMap의 Find는 내부적으로 **해시(Hash)** 알고리즘을 사용하기 때문에 데이터가 수만 개가 있어도 **거의 즉시** 원하는 값을 찾아냅니다.

### **② 직관적인 데이터 관리**

숫자 인덱스(0, 1, 2...) 대신 문자열이나 오브젝트 자체를 키로 쓸 수 있어 코드가 훨씬 읽기 편해집니다.

* *배열:* "인벤토리의 5번째 칸에 뭐가 있지?"  
* *TMap:* "\*\*'포션'\*\*이라는 아이템의 \*\*'상세 정보'\*\*가 뭐지?"

## ---

**3\. 주요 활용 사례**

### **📂 인벤토리 시스템 (Item ID → Item Data)**

플레이어가 아이템을 습득했을 때, 아이템의 이름(Key)을 Find 노드에 넣어서 해당 아이템의 공격력, 설명, 아이콘 정보(Value)를 즉시 불러올 때 사용합니다.

### **🏆 점수판 (Player Name → Score)**

특정 플레이어의 이름을 넣으면 그 플레이어가 현재 기록 중인 점수를 가져오는 로직을 짤 때 유용합니다.

### **🛠️ 데이터 테이블 캐싱**

엑셀(Data Table) 데이터를 게임 시작 시 TMap에 담아두면, 게임 도중 아주 빠르게 데이터를 조회할 수 있습니다.

## ---

**4\. ⚠️ 주의사항: "반드시 불리언(Boolean)을 먼저 확인하세요\!"**

Find 노드를 쓸 때 가장 많이 하는 실수는 \*\*"값이 당연히 있을 것"\*\*이라고 가정하는 것입니다. 만약 존재하지 않는 키를 검색하면 Find 노드는 기본값(0 혹은 None)을 출력하지만, 로직이 꼬일 수 있습니다.

* **올바른 방법:** Find 노드 바로 뒤에 **Branch** 노드를 연결하여, Return Value가 **True**일 때만 결과 값을 사용하도록 설계해야 안전합니다.

---

**요약**

**TMap Find** 노드는 "데이터가 아무리 많아도 \*\*이름표(Key)\*\*만 알면 \*\*내용물(Value)\*\*을 순식간에 찾아주는 초고속 검색 도구"입니다.

혹시 지금 인벤토리 시스템이나 캐릭터 스테이터스를 관리하기 위해 TMap을 사용 중이신가요? Find 노드 외에도 데이터를 추가하거나(Add), 삭제하는(Remove) 노드들에 대해서도 궁금한 점이 있으신가요?
