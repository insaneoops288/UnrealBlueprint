# **To Text (Name) 노드의 역할과 이해**

언리얼 엔진에서 문자열을 다루는 타입은 크게 **Name**, **String**, **Text** 세 가지가 있습니다. To Text (Name) 노드는 내부 식별용인 Name을 사용자 화면에 표시하기 위한 Text로 변환할 때 사용합니다.

### **1\. 주요 역할: "내부 ID를 사용자 화면용으로 변환"**

* **UI 표시:** 언리얼의 UI 시스템인 UMG(위젯)에서 텍스트 블록(Text Block)은 오직 **Text** 타입만 입력받습니다. 캐릭터 이름이나 아이템 이름이 Name 변수로 저장되어 있다면, 이를 화면에 그리기 위해 반드시 이 노드를 거쳐야 합니다.  
* **로컬라이징(다국어 지원) 준비:** Text 타입은 언리얼 엔진의 로컬라이제이션 시스템과 연동되는 유일한 타입입니다. Name은 번역되지 않지만, 이를 Text로 변환하면 번역 데이터베이스에 활용할 수 있는 상태가 됩니다.

### **2\. 왜 Name을 바로 쓰지 않고 변환하나요?**

언리얼 엔진에서 각 타입의 용도가 다르기 때문입니다.

| 타입 | 주요 용도 | 특징 |
| :---- | :---- | :---- |
| **Name** | 태그, 소켓 이름, 본 이름, 맵 이름 등 **내부 식별** | 대소문자 구분이 없고 매우 가벼움 (에셋 참조 등에 최적화) |
| **String** | 프로그래밍적 문자열 조작 (자르고 붙이기 등) | 유연하지만 메모리 사용량이 많고 로컬라이징 불가 |
| **Text** | **사용자에게 보여주는 글자** | 로컬라이징(번역) 지원, 숫자/날짜 형식화 기능 포함 |

### **3\. 노드 사용 시 주의사항**

* **성능:** Name에서 Text로 변환하는 과정은 단순한 복사보다 약간의 비용이 발생합니다. 하지만 매 프레임 수만 번 호출하는 것이 아니라면 UI 업데이트 시 사용하는 것은 전혀 문제가 되지 않습니다.  
* **대소문자:** Name 타입은 원래 대소문자를 구분하지 않으므로, 변환된 Text는 Name 변수에 저장되었던 당시의 형태를 유지하게 됩니다.

### **4\. 실제 활용 예시**

1. **아이템 이름 출력:** 데이터 테이블에 아이템 ID를 Name으로 저장해 두었다가, 플레이어가 아이템을 획득했을 때 화면 상단에 "철검(Name \-\> Text)을 획득했습니다\!"라고 띄울 때 사용합니다.  
2. **디버깅:** 특정 소켓의 이름이나 액터의 태그가 무엇인지 Print String이 아닌 UI 슬롯에 실시간으로 표시하고 싶을 때 변환하여 연결합니다.

### **요약**

To Text (Name) 노드는 **시스템 내부에서 쓰이는 가벼운 '이름(Name)'을 사용자가 읽을 수 있는 '글자(Text)'로 바꾸는 연결 고리**입니다.
