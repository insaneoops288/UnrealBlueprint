# **언리얼 엔진 VInterp To 노드 분석**

**VInterp To** (Vector Interpolation To) 노드는 현재의 벡터 값에서 목표로 하는 벡터 값까지 매 프레임마다 일정 속도로 부드럽게 이동시키는 역할을 합니다. 주로 오브젝트의 위치(Location)나 크기(Scale)를 자연스럽게 변경할 때 사용됩니다.

## **1\. 노드 입력 파라미터 (Inputs)**

| 항목 | 타입 | 설명 |
| :---- | :---- | :---- |
| **Current** | Vector | 현재 오브젝트의 위치나 상태 값을 연결합니다. (보통 Get Actor Location 등과 연결) |
| **Target** | Vector | 최종적으로 도달하고자 하는 목표 지점의 값입니다. |
| **Delta Time** | Float | 이전 프레임과 현재 프레임 사이의 시간 간격입니다. (주로 Get World Delta Seconds나 Tick의 Delta Time 연결) |
| **Interp Speed** | Float | 보간 속도입니다. 값이 클수록 목표 지점에 더 빨리 도달합니다. |

## **2\. 작동 원리**

VInterp To는 단순한 직선 보간(Lerp)과 달리 \*\*프레임 독립적(Frame-rate Independent)\*\*으로 동작하도록 설계되었습니다.

1. **동적 이동**: 매 프레임 호출될 때마다 Current에서 Target 방향으로 Interp Speed와 Delta Time을 곱한 수치만큼 벡터를 밀어줍니다.  
2. **부드러운 감속**: 목표 지점에 가까워질수록 이동 거리가 미세하게 줄어들어 자연스럽게 멈추는 효과를 줍니다.  
3. **프레임 보정**: Delta Time을 사용하기 때문에 컴퓨터의 사양(FPS)에 상관없이 동일한 물리적 시간 동안 동일한 거리를 이동하게 됩니다.

## **3\. 주요 활용 사례**

### **① 부드러운 카메라 추적**

카메라가 캐릭터를 즉각적으로 따라가지 않고, 약간의 지연(Lag)을 두고 부드럽게 따라오게 만들 때 사용합니다.

### **② 오브젝트 이동 (Floating/Moving)**

문이 열리거나, 발판이 이동할 때 Timeline 노드 없이 간단하게 구현하고 싶을 때 유용합니다.

### **③ 조준점(Crosshair) 보정**

무기의 반동으로 인해 벌어진 조준점을 다시 중앙으로 부드럽게 모아줄 때 벡터 값을 조정하여 사용합니다.

## **4\. 실전 사용 팁**

* **반드시 Tick이나 타이머에 연결하세요**: 이 노드는 한 번 실행해서 결과가 나오는 것이 아니라, 목표에 도달할 때까지 **지속적으로(매 프레임)** 호출되어야 합니다.  
* **Current 값 업데이트**: Current 핀에는 반드시 "실시간으로 변하고 있는 현재 값"이 들어가야 합니다. 고정된 변수 값을 넣으면 이동하지 않습니다.  
* **Interp Speed 0**: 속도 값을 0으로 설정하면 Current 값이 변하지 않고 그대로 유지됩니다.

## **5\. 유사 노드들**

* **FInterp To**: 부동 소수점(Float) 값을 보간 (예: 라이트 밝기, 투명도)  
* **RInterp To**: 회전(Rotation) 값을 보간 (예: 캐릭터의 회전, 문 회전)  
* **CInterp To**: 컬러(Color) 값을 보간
