# **Unreal Engine Blueprint: Vector Is Zero 노드 분석**

언리얼 엔진 블루프린트에서 Vector Is Zero 노드는 입력받은 벡터 값이 \*\*(0, 0, 0)\*\*인지 확인하여 부울(Boolean) 값을 반환하는 비교 노드입니다.

## **1\. 기본 기능**

* **입력 (Vector):** 검사하려는 벡터 값 (예: 캐릭터의 속도, 방향, 위치 오프셋 등).  
* **출력 (Boolean):** \- 벡터의 X, Y, Z 모든 성분이 정확히 0.0이면 True를 반환합니다.  
  * 하나라도 0이 아니면 False를 반환합니다.

## **2\. 주요 활용 사례**

### **① 이동 멈춤 감지**

캐릭터나 액터의 **Velocity(속도)** 벡터를 이 노드에 연결하여, 현재 액터가 완전히 멈춰 있는 상태인지 확인할 수 있습니다.

* 예: Get Velocity \-\> Vector Is Zero \-\> Branch (True라면 대기 애니메이션 재생)

### **② 유효하지 않은 방향 벡터 체크**

발사체의 진행 방향이나 대상을 향한 벡터가 생성되지 않았을 때(0인 경우) 오류를 방지하기 위해 사용합니다.

* 예: 노멀라이즈(Normalize) 하기 전 벡터가 0인지 확인하여 'Division by Zero' 에러 방지.

### **③ 입력값 확인**

게임 패드나 조이스틱의 입력값(Axis Value)을 벡터로 합쳤을 때, 플레이어의 입력이 없는 상태인지 판단할 때 유용합니다.

## **3\. 주의사항 및 'Vector Is Nearly Zero'와의 차이**

부동 소수점(Floating Point) 계산을 사용하는 게임 엔진 특성상, 물리 연산 결과값이 0.000001처럼 아주 미세하게 0이 아닌 경우가 발생할 수 있습니다. 이때 Vector Is Zero는 False를 반환하여 의도치 않은 동작을 유발할 수 있습니다.

따라서 더 안정적인 프로그래밍을 위해 아래 노드를 더 자주 사용합니다:

| 노드 이름 | 특징 |
| :---- | :---- |
| **Vector Is Zero** | 정확히 0이어야 함. (엄격한 비교) |
| **Vector Is Nearly Zero** | 지정한 **Tolerance(허용 오차)** 범위 내에 있으면 0으로 간주. (권장 방식) |

### **팁: Is Nearly Zero 사용 권장**

대부분의 이동이나 물리 체크 시에는 Vector Is Nearly Zero 노드를 사용하고, Tolerance 값에 아주 작은 값(예: 0.0001)을 입력하는 것이 버그를 줄이는 좋은 방법입니다.

## **4\. 요약**

Vector Is Zero는 \*\*"이 벡터에 아무런 값도 들어있지 않은가?"\*\*를 묻는 가장 직관적인 방법입니다. 하지만 물리 연산이 포함된 값이라면 오차 범위를 허용하는 Nearly Zero 계열 노드와 함께 상황에 맞춰 선택하는 것이 중요합니다.
