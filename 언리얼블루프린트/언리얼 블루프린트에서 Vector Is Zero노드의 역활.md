언리얼 블루프린트에서 Vector Is Zero노드의 역활?


언리얼 엔진 블루프린트에서 Vector Is Zero 노드는 말 그대로 입력받은 벡터(X, Y, Z)의 값이 0인지 확인하여 그 결과를 참(True) 또는 거짓(False)으로 반환하는 비교 노드입니다.

이 노드의 주요 역할과 특징을 정리해 드릴게요.

1. 주요 역할
정지 상태 확인: 캐릭터의 속도(Velocity) 벡터가 0인지 확인하여 캐릭터가 완전히 멈췄는지 체크할 때 자주 사용합니다.

유효성 검사: 어떤 계산 결과로 나온 방향 데이터가 존재하는지, 혹은 입력값이 없는 상태인지를 판단할 때 쓰입니다.

2. 핵심 파라미터: Tolerance (허용 오차)
이 노드에서 가장 중요한 부분은 Tolerance 값입니다.

컴퓨터의 부동 소수점 연산 특성상, 물리 연산이나 복잡한 계산을 거치면 실제로는 정지해 있어도 벡터 값이 (0.000001, 0, 0) 처럼 아주 미세한 값을 가질 때가 많습니다.

Tolerance 값을 설정하면(기본값은 보통 아주 작은 수), 벡터의 각 성분이 이 오차 범위 안에만 들어오면 **"0이다(True)"**라고 판정해 줍니다. 덕분에 수학적으로 완벽한 0이 아니더라도 실질적인 정지 상태를 쉽게 감지할 수 있습니다.

3. 활용 예시
애니메이션 전환: Get Velocity 노드의 결과값을 Vector Is Zero에 연결하고, 결과가 True이면 'Idle(대기)' 애니메이션을, False이면 'Run(달리기)' 애니메이션을 재생하도록 로직을 짤 수 있습니다.

발사체 로직: 총알의 진행 방향 벡터가 0이라면 발사되지 않도록 방지하는 예외 처리에 사용합니다.

4. 유사 노드와 차이점
Equal (Vector == Vector): 두 벡터가 똑같은지 비교합니다. 한쪽을 (0,0,0)으로 두면 Vector Is Zero와 비슷하지만, Vector Is Zero는 '0인지 확인'하는 용도에 특화되어 있고 오차 범위(Tolerance) 설정이 직관적이라는 장점이 있습니다.

요약하자면: "이 벡터가 실질적으로 아무런 힘이나 방향이 없는 0의 상태인가?"를 판단할 때 사용하는 아주 유용한 유틸리티 노드입니다.
