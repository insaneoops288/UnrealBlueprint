언리얼 블루프린트 for loop와 for each loop의 차이?

언리얼 엔진 블루프린트에서 For Loop와 For Each Loop는 모두 반복 작업을 수행하지만, **'무엇을 기준으로 반복하느냐'**에 결정적인 차이가 있습니다.

쉽게 요약하자면, **For Loop는 '숫자'**를 세고, **For Each Loop는 '목록(배열)'**을 훑는다고 생각하면 됩니다.

1. For Loop (포 루프)
특정 **숫자 범위(인덱스)**만큼 반복하고 싶을 때 사용합니다.

주요 입력: First Index (시작 숫자), Last Index (끝 숫자)

특징: 배열이 없어도 실행할 수 있습니다. 예를 들어 "1부터 10까지 10번 실행해라" 같은 명령에 적합합니다.

주요 출력: Index (현재 몇 번째 반복인지 알려주는 정수 값)

언제 쓰나요?

특정 동작을 정확히 N번 반복하고 싶을 때 (예: 총알을 5발 연사하기)

특정 숫자 범위 내의 계산이 필요할 때

아이템을 특정 간격으로 10개 배치하고 싶을 때

2. For Each Loop (포 이치 루프)
배열(Array) 안에 들어있는 아이템의 개수만큼 반복하고 싶을 때 사용합니다.

주요 입력: Array (대상 배열)

특징: 배열의 길이를 자동으로 계산합니다. 배열에 아이템이 5개면 5번, 100개면 100번 알아서 돌아갑니다.

주요 출력: * Array Element: 현재 순서에 해당하는 실제 데이터(아이템)

Array Index: 현재 순서의 번호 (0부터 시작)

언제 쓰나요?

인벤토리에 있는 모든 아이템의 이름을 출력하고 싶을 때

게임 내에 존재하는 모든 적(Enemy)의 체력을 10씩 깎고 싶을 때

선택된 여러 액터들의 위치를 한꺼번에 옮기고 싶을 때

3. 핵심 차이점 비교
특징	For Loop	For Each Loop
반복 기준	정수 범위 (Start ~ End)	배열의 크기 (Array Size)
주요 대상	단순 횟수 반복	데이터 집합(배열) 처리
자동 인지	개발자가 숫자를 직접 지정해야 함	배열에 담긴 개수를 스스로 파악함
데이터 접근	인덱스(숫자)만 제공함	**실제 데이터(Element)**를 바로 뽑아줌
💡 팁: 중단하고 싶을 때는?
두 노드 모두 with Break 버전이 존재합니다 (For Loop with Break, For Each Loop with Break).
반복문 실행 도중 특정 조건을 만족했을 때(예: 찾으려는 아이템을 발견했을 때) Break 핀에 신호를 주면 즉시 반복을 멈출 수 있습니다.
