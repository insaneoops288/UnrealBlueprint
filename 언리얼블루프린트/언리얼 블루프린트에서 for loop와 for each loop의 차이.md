언리얼 블루프린트에서 **For Loop**와 **For Each Loop**는 모두 반복 작업을 수행하지만, \*\*"무엇을 기준으로 반복하느냐"\*\*에 따라 사용법이 완전히 다릅니다.

쉽게 말해, **숫자**를 세며 반복할 것인지, \*\*목록(배열)\*\*을 뒤지며 반복할 것인지의 차이입니다.

## ---

**1\. For Loop (숫자 기반 반복)**

특정 횟수만큼 로직을 반복하고 싶을 때 사용합니다. 시작 숫자와 끝 숫자를 직접 지정합니다.

* **입력:** First Index (시작), Last Index (끝)  
* **작동:** 시작 번호부터 끝 번호까지 숫자를 하나씩 올리며 실행합니다.  
* **비유:** "1번 사물함부터 10번 사물함까지 순서대로 가서 문을 열어봐."

### **주요 활용 사례**

* 특정 개수(예: 10개)만큼 아이템을 랜덤하게 스폰할 때.  
* 숫자 $i$를 활용해 수학적인 계산을 반복해야 할 때 (예: 계단식 좌표 계산).  
* 배열과는 상관없이 단순히 "N번 실행"이 필요할 때.

## ---

**2\. For Each Loop (배열 기반 반복)**

**배열(Array)** 변수 안에 들어있는 항목의 개수만큼 자동으로 반복합니다. 배열 안의 '실제 내용물'을 바로 꺼내 쓸 수 있어 매우 편리합니다.

* **입력:** Array (배열 데이터)  
* **작동:** 배열의 첫 번째 칸(0번)부터 마지막 칸까지 돌며, 매번 \*\*그 칸에 들어있는 값(Array Element)\*\*을 줍니다.  
* **비유:** "이 바구니 안에 든 사과들을 하나씩 꺼내서 썩었는지 확인해봐."

### **주요 활용 사례**

* 인벤토리 배열을 돌며 특정 아이템이 있는지 찾을 때.  
* Get All Actors of Class로 가져온 액터 목록 전체에 특정 명령을 내릴 때.  
* 리스트에 있는 모든 적의 체력을 10%씩 깎을 때.

## ---

**3\. 한눈에 비교하기**

| 특징 | For Loop | For Each Loop |
| :---- | :---- | :---- |
| **반복 기준** | **정해진 숫자 범위** (Index) | **배열의 크기** (Array Size) |
| **필수 입력** | 시작 인덱스, 끝 인덱스 | 배열 (Array) |
| **주요 출력** | 현재 순서 (Index) | **배열 안의 실제 값**, 인덱스 |
| **유연성** | 횟수를 마음대로 정할 수 있음 | 배열 내용물에 바로 접근 가능 |
| **사용 난이도** | 직접 범위를 계산해야 함 | 매우 직관적이고 편함 |

## ---

**4\. 어떤 상황에 무엇을 쓸까요?**

### **상황 A: "맵에 나무를 50그루 심고 싶다."**

👉 **For Loop**가 적합합니다. 0부터 49까지 돌리면서 나무를 스폰하면 됩니다.

### **상황 B: "내 주변에 있는 모든 적(Enemy)을 파괴하고 싶다."**

👉 **For Each Loop**가 적합합니다. Get All Actors...로 적 목록(배열)을 뽑은 뒤, 이를 하나씩 꺼내 Destroy Actor를 실행하면 됩니다.

### **상황 C: "배열의 10번째부터 20번째까지만 확인하고 싶다."**

👉 이럴 땐 **For Loop**를 씁니다. 10(Start)에서 20(End)까지 돌리면서, 배열의 Get 노드에 인덱스를 넣어 값을 가져오면 됩니다.

---

**💡 팁: Loop with Break**

두 노드 모두 **"with Break"** 버전이 있습니다. 반복 중간에 특정 조건(예: 아이템을 찾았을 때)이 만족되면 즉시 반복을 중단하고 빠져나오고 싶을 때 사용합니다.
