# **언리얼 엔진 블루프린트: 순수 함수 (Pure Function)**

블루프린트에서 함수는 크게 **순수(Pure)** 함수와 **비순수(Impure)** 함수로 나뉩니다. 이 문서에서는 순수 함수의 특징과 올바른 사용법을 설명합니다.

## **1\. 순수 함수란?**

순수 함수는 **데이터의 상태를 변경하지 않는(Read-only)** 함수입니다.

* **실행 핀(Execution Pins)이 없음**: 하얀색 화살표 입력/출력이 없습니다.  
* **데이터 흐름에 의해 호출**: 출력 핀에 연결된 노드가 실행될 때만 호출되어 값을 전달합니다.  
* **사이드 이펙트(Side Effect) 없음**: 변수 값을 수정하거나, 액터를 스폰하거나, 파괴하는 등의 동작을 포함해서는 안 됩니다.

## **2\. 순수 함수 vs 비순수 함수 비교**

| 특징 | 비순수 함수 (Impure) | 순수 함수 (Pure) |
| :---- | :---- | :---- |
| **실행 핀** | 있음 (순차적 실행 필요) | 없음 (데이터 필요 시 호출) |
| **노드 색상** | 파란색 (기본) | 녹색 |
| **주요 용도** | 로직 수행, 상태 변경, 타이머 등 | 수학 계산, Get 함수, 상태 확인 |
| **호출 시점** | 실행 흐름이 도달했을 때 | 출력 값이 다른 노드에서 필요할 때 |

## **3\. 순수 함수를 만드는 방법**

### **블루프린트 에디터에서**

1. 함수를 생성합니다.  
2. **디테일(Details)** 패널의 **상속/기능(Function)** 섹션으로 이동합니다.  
3. **Pure** 체크박스를 체크합니다.

### **C++에서**

C++ 함수를 블루프린트 순수 함수로 노출하려면 BlueprintPure 지정자를 사용합니다.

// 헤더 파일 (.h)  
UFUNCTION(BlueprintPure, Category \= "Math")  
float GetCalculatePower(float Base, float Multiplier) const;

*참고: const 키워드를 함께 사용하는 것이 관례이며, 이는 해당 함수가 클래스의 멤버 변수를 수정하지 않음을 보장합니다.*

## **4\. 실행 메커니즘과 주의사항 (중요)**

순수 함수는 일반 함수와 실행 방식이 다르기 때문에 주의가 필요합니다.

### **재계산 (Re-evaluation)**

순수 함수의 출력 핀이 **여러 개의 노드에 연결되어 있다면**, 그 노드들이 실행될 때마다 순수 함수 내부의 로직이 **매번 다시 실행**됩니다.

* 만약 순수 함수 내부에서 복잡한 계산을 수행한다면 성능 저하의 원인이 될 수 있습니다.  
* 이 경우, 변수에 값을 한 번 저장(Cache)한 뒤 사용하는 것이 좋습니다.

### **값의 일관성**

순수 함수 내부에 Random Float 같은 노드가 들어있다면, 출력 핀을 연결한 곳마다 서로 다른 랜덤 값을 받을 가능성이 있습니다. 실행 시점마다 로직이 다시 돌아가기 때문입니다.

## **5\. 언제 사용해야 하나요?**

* **Getter 함수**: 특정 변수의 값을 가공해서 가져올 때.  
* **수학적 계산**: 더하기, 빼기, 벡터 연산 등.  
* **조건 확인**: IsAlive?, HasWeapon? 등 Boolean 결과를 반환할 때.  
* **형변환(Casting)**: 특정 타입으로 변환하여 데이터만 확인할 때.

### **요약**

순수 함수는 \*\*"세상에 영향을 주지 않고 오직 질문에 답만 하는 함수"\*\*라고 이해하면 쉽습니다. 실행 핀을 줄여 블루프린트를 깔끔하게 만들 수 있지만, 호출 횟수에 따른 성능과 값의 변화를 항상 염두에 두어야 합니다.
