# **Actor Has Tag 노드 완벽 가이드**

언리얼 엔진에서 Actor Has Tag 노드는 액터를 식별하고 분류하는 가장 쉽고 빠른 방법 중 하나입니다.

## **1\. 노드의 주요 역할**

이 노드는 입력받은 **Actor**의 Tags 배열을 검색하여, 사용자가 입력한 \*\*Tag(Name 타입)\*\*와 일치하는 값이 있는지 확인합니다.

* **입력 (Input):** Target (Actor Object Reference), Tag (Name)  
* **출력 (Output):** Return Value (Boolean \- 성공 시 True, 실패 시 False)

## **2\. 왜 사용하는가? (활용 사례)**

### **① 특정 대상 필터링 (Trigger Volume)**

트리거 박스에 무언가 부딪혔을 때, 그것이 '플레이어'인지 '적'인지 '아이템'인지 구분할 때 사용합니다.

* 예: OnComponentBeginOverlap 이벤트 발생 시, 상대 액터(Other Actor)가 "Player" 태그를 가졌는지 확인하여 문을 열어줌.

### **② 데미지 시스템**

특정 공격이 특정 진영에게만 데미지를 입히도록 설정할 때 유용합니다.

* 예: 폭발 범위 안에 액터가 있어도 "Team\_A" 태그를 가진 액터는 데미지를 무시함.

### **③ 오브젝트 상호작용**

월드에 배치된 수많은 액터 중 상호작용이 가능한 오브젝트만 골라낼 때 사용합니다.

* 예: 라인트레이스(Line Trace)에 맞은 액터가 "Interactable" 태그를 가졌을 때만 상호작용 UI를 띄움.

## **3\. 설정 방법 (액터에 태그 추가하기)**

1. 레벨에 배치된 **액터**를 선택하거나 **블루프린트 클래스**를 엽니다.  
2. **Details(디테일)** 패널에서 Tag라고 검색합니다.  
3. **Actor** 섹션 아래의 **Tags** 배열에 \+ 버튼을 눌러 원하는 문자열(예: Enemy, KeyItem)을 입력합니다.  
   * *주의: Component Tags와 Actor Tags는 다르므로 반드시 Actor 섹션의 Tags를 확인하세요.*

## **4\. '액터 태그' vs '게임플레이 태그'**

질문에서 언급하신 \*\*게임플레이 태그(Gameplay Tag)\*\*와는 약간의 차이가 있습니다.

| 구분 | Actor Has Tag (기본) | Gameplay Tags (고급) |
| :---- | :---- | :---- |
| **데이터 타입** | 단순 문자열 (FName) | 계층 구조 데이터 (Data Driven) |
| **복잡성** | 매우 쉬움 (설정 불필요) | 프로젝트 설정에서 관리 필요 |
| **특징** | 오타 발생 시 찾기 힘듦 | 계층형(Parent.Child) 관리가 가능하며 강력함 |
| **확인 노드** | Actor Has Tag | Has Matching Gameplay Tag 등 |

## **5\. 사용 시 주의사항**

1. **대소문자 구분 없음:** Player와 player는 동일하게 인식됩니다.  
2. **오타 주의:** 문자열을 직접 입력하므로 오타가 나면 작동하지 않습니다.  
3. **성능:** 매우 자주 호출되는 Tick 서비스 등에서 수백 개의 태그를 매번 검사하는 것은 권장되지 않으나, 일반적인 이벤트(Overlap, Hit 등)에서는 성능 영향이 거의 없습니다.

### **요약**

Actor Has Tag는 **"너 누구니?"** 혹은 \*\*"너 이 그룹 소속이니?"\*\*라고 물어보는 체크포인트 역할을 합니다. 복잡한 캐스팅(Cast To...) 없이도 간단하게 액터의 성격을 파악할 수 있는 매우 유용한 노드입니다.
