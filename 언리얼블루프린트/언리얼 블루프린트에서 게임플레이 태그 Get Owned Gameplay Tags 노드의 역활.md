# **GetOwnedGameplayTags 노드 상세 가이드**

GetOwnedGameplayTags 노드는 게임플레이 태그 시스템(Gameplay Tag System)을 사용하는 액터나 컴포넌트에서 현재 활성화된 태그들을 조회할 때 사용되는 가장 기본적인 함수입니다.

## **1\. 핵심 역할**

이 노드의 주된 목적은 \*\*"이 오브젝트가 지금 어떤 상태(태그)인가?"\*\*를 확인하는 것입니다. 해당 오브젝트(Self 또는 타겟)가 가지고 있는 Gameplay Tag Container를 반환하며, 여기에는 해당 시점에 할당된 모든 태그가 포함되어 있습니다.

## **2\. 작동 조건 (Interface)**

이 노드를 사용하기 위해서는 대상 오브젝트가 \*\*IGameplayTagAssetInterface\*\*를 구현(Implementation)하고 있어야 합니다.

* **C++:** IGameplayTagAssetInterface를 상속받아 GetOwnedGameplayTags 함수를 오버라이드해야 합니다.  
* **Blueprint:** 해당 인터페이스를 추가하고 기능을 구현해야 노드에서 정상적으로 값을 가져올 수 있습니다.  
* **GAS (Gameplay Ability System):** AbilitySystemComponent를 사용하는 경우, 기본적으로 이 인터페이스 기능을 내장하고 있어 태그 조회가 매우 용이합니다.

## **3\. 입력과 출력**

* **Target (입력):** 태그를 확인할 대상 오브젝트 (보통 IGameplayTagAssetInterface를 구현한 액터나 컴포넌트).  
* **Tag Container (출력):** 대상이 보유한 모든 태그가 담긴 컨테이너(FGameplayTagContainer)를 반환합니다.

## **4\. 주요 활용 사례**

### **① 특정 태그 보유 여부 확인 (Has Tag)**

반환된 컨테이너를 Has Tag 또는 Has Any Tags 노드와 연결하여, 캐릭터가 현재 특정 상태인지 확인합니다.

* 예: "캐릭터가 State.Stunned(기절) 태그를 가지고 있는가?" \-\> 그렇다면 입력을 무시함.

### **② UI 업데이트**

플레이어의 상태가 변할 때마다 태그 목록을 가져와 UI 디스플레이를 갱신합니다.

* 예: 독(Poison) 태그가 추가되면 화면에 독 아이콘을 표시.

### **③ 디버깅**

현재 캐릭터에게 어떤 복합적인 상태가 적용되어 있는지 한눈에 확인하고 싶을 때, 이 노드를 통해 모든 태그를 문자열로 출력해 볼 수 있습니다.

## **5\. 주의사항**

* **직접 비교 지양:** GetOwnedGameplayTags로 가져온 컨테이너를 단순히 "같음(==)"으로 비교하기보다는, Has Tag 같은 전용 함수를 사용하는 것이 성능과 정확도 면에서 좋습니다 (계층 구조 때문).  
* **정적 vs 동적 태그:** 액터에 기본으로 설정된 태그뿐만 아니라, 게임 플레이 중 동적으로 추가된(Effect 등에 의해) 태그까지 모두 포함하여 가져옵니다.

### **요약**

GetOwnedGameplayTags는 \*\*"오브젝트의 현재 상태(태그)를 스냅샷처럼 찍어서 리스트로 전달해 주는 우체부"\*\*와 같습니다. 이를 통해 복잡한 상태 이상이나 게임 로직을 태그 기반으로 깔끔하게 처리할 수 있습니다.
