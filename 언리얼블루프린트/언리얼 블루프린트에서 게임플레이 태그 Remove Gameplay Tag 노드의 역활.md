# **Unreal Engine: Remove Gameplay Tag 노드 상세 분석**

언리얼 엔진에서 \*\*Gameplay Tag(게임플레이 태그)\*\*는 액터나 오브젝트의 상태, 속성, 카테고리를 관리하는 매우 강력한 도구입니다. 이 중 Remove Gameplay Tag 노드는 특정 컨테이너나 인터페이스에서 태그를 삭제하는 역할을 합니다.

## **1\. 노드의 핵심 역할**

Remove Gameplay Tag 노드의 주요 목적은 \*\*"특정 데이터 구조(주로 Gameplay Tag Container)에서 지정된 태그를 찾아 제거하는 것"\*\*입니다.

* **상태 해제:** 캐릭터에게 부여된 '기절(Stun)', '화상(Burning)' 같은 상태 태그를 제거할 때 사용합니다.  
* **조건 변경:** 특정 행동이 끝났을 때(예: '공격 중') 해당 태그를 제거하여 다른 로직이 실행될 수 있도록 합니다.  
* **데이터 관리:** 게임 엔진 내에서 계층 구조를 가진 태그를 유연하게 관리합니다.

## **2\. 주요 입력 및 출력 (Inputs & Outputs)**

### **Inputs (입력)**

1. **Target (대상):** \- 보통 Gameplay Tag Container 변수나, IGameplayTagAssetInterface를 구현한 액터/컴포넌트가 들어갑니다.  
2. **Tag (태그):** \- 제거하고자 하는 구체적인 게임플레이 태그(예: State.Debuff.Poison)를 선택합니다.

### **Outputs (출력)**

1. **Return Value (Boolean):** \- 태그가 성공적으로 제거되었는지 여부를 반환합니다.  
   * 만약 컨테이너에 해당 태그가 원래 없었다면 False를 반환하고, 성공적으로 지워졌다면 True를 반환합니다.

## **3\. 작동 원리 및 특징**

### **계층적 제거**

게임플레이 태그는 A.B.C와 같은 계층 구조를 가집니다. 하지만 Remove Gameplay Tag는 **정확히 일치하는 태그만 제거**합니다.

* 예: State.Buff.SpeedUp을 제거한다고 해서 부모 태그인 State.Buff 자체가 사라지지는 않습니다.

### **Container 업데이트**

노드가 실행되면 연결된 Gameplay Tag Container 변수 내의 목록이 즉시 업데이트됩니다. 이를 통해 Has Tag 노드 등으로 상태를 체크하는 다른 로직들이 즉각적으로 반응하게 됩니다.

## **4\. 실전 활용 예시**

### **예시: 물약 효과 종료 (Poison 제거)**

1. 독에 걸렸을 때 Add Gameplay Tag로 State.Debuff.Poison 태그를 추가합니다.  
2. 일정 시간이 지나거나 해독제를 먹었을 때 Remove Gameplay Tag를 호출합니다.  
3. 캐릭터의 UI나 파티클 시스템은 Has Tag 노드로 이 태그의 존재 유무를 감시하다가, 태그가 사라지면 독 이펙트를 중단합니다.

### **예시: 무기 장착 시스템**

* 무기를 장착하면 Weapon.Equipped 태그를 추가합니다.  
* 무기를 해제하거나 파괴되면 Remove Gameplay Tag로 해당 태그를 제거하여 '맨손' 상태임을 로직상으로 알립니다.

## **5\. 주의사항**

* **일치 여부:** 태그 이름이 정확히 일치해야 합니다. (부분 일치로 제거되지 않음)  
* **참조 전달:** 블루프린트에서 태그 컨테이너를 함수 인자로 전달할 때 'Pass by Reference(참조로 전달)'를 체크하지 않으면, 복사본에서만 태그가 제거되고 원본 데이터는 변하지 않을 수 있으니 주의해야 합니다.

## **요약**

Remove Gameplay Tag 노드는 게임 내 **오브젝트의 특정 상태나 속성을 '꺼버리는(Off)' 스위치**와 같습니다. 이를 통해 하드코딩된 Boolean 변수(예: IsStunned, IsBurning)를 수십 개 만드는 대신, 태그 하나로 깔끔하게 상태를 관리할 수 있습니다.
