# **Switch on Gameplay Tag 노드 상세 가이드**

언리얼 엔진(Unreal Engine)의 블루프린트에서 **Switch on Gameplay Tag** 노드는 입력받은 Gameplay Tag 값에 따라 실행 흐름(Execution Flow)을 여러 갈래로 나누는 **흐름 제어(Flow Control)** 노드입니다.

표준 Switch on Int나 Switch on Enum과 유사하게 작동하지만, 단순한 숫자나 열거형 대신 **계층 구조를 가진 게임플레이 태그**를 기반으로 한다는 점이 특징입니다.

## **1\. 주요 역할 (Core Roles)**

### **① 로직 분기 (Branching Logic)**

특정 캐릭터의 상태, 아이템 유형, 또는 현재 발생한 이벤트의 태그를 확인하여 각각 다른 함수나 이벤트를 실행하게 합니다.

* 예: 캐릭터의 상태 태그가 State.Stunned이면 기절 로직을, State.Idle이면 대기 로직을 실행.

### **② 계층 구조 활용 (Hierarchy Support)**

게임플레이 태그는 Parent.Child 형태의 계층 구조를 가집니다. 이 노드는 설정을 통해 자식 태그가 들어왔을 때 부모 태그 핀으로 흐름을 보낼지 결정할 수 있습니다.

### **③ 가독성 및 유지보수 향상**

수많은 Branch (If-Else) 노드를 중첩해서 사용하는 대신, 하나의 노드에서 깔끔하게 여러 케이스를 관리할 수 있어 블루프린트 그래프가 단순해집니다.

## **2\. 주요 설정 및 옵션 (Pin & Details)**

### **입력 핀**

* **Selection**: 비교 대상이 될 Gameplay Tag 변수나 데이터를 입력받습니다.  
* **Exec (In)**: 노드를 실행시키는 실행 핀입니다.

### **출력 핀 (Pins)**

* **Selection Pins**: 사용자가 디테일 패널에서 추가한 태그들이 각각의 출력 실행 핀으로 나타납니다.  
* **Default**: 입력된 태그가 목록에 있는 어떤 태그와도 일치하지 않을 때 실행되는 기본 핀입니다.

### **디테일 패널 옵션**

* **Pin Tags**: 스위치 노드에 노출시킬 태그 목록을 추가합니다. 각 태그는 하나의 출력 실행 핀이 됩니다.  
* **Use Exact Tag Match**:  
  * **True (체크됨)**: 입력된 태그와 핀에 설정된 태그가 완전히 일치해야만 해당 경로로 이동합니다.  
  * **False (체크 해제)**: 입력된 태그가 핀에 설정된 태그의 자식인 경우에도 해당 경로로 이동할 수 있습니다. (예: Weapon.Melee.Sword 입력 시 Weapon.Melee 핀으로 실행 가능)

## **3\. 실제 활용 예시**

### **캐릭터 상태 머신 (State Handling)**

캐릭터가 공격을 받았을 때, 공격의 종류(태그)에 따라 다른 반응을 보이게 할 수 있습니다.

* Attack.Fire: 화염 이펙트 및 지속 데미지 로직 실행  
* Attack.Ice: 빙결 이펙트 및 속도 감소 로직 실행  
* Attack.Knockback: 밀쳐내기 물리 로직 실행

### **아이템 상호작용**

플레이어가 습득한 아이템의 태그를 확인합니다.

* Item.Consumable: 회복 아이템 사용 로직  
* Item.Equipment: 장비 장착 로직  
* Item.Quest: 퀘스트 아이템 팝업 표시

## **4\. 장점 요약**

1. **유연성**: 열거형(Enum)은 코드나 자산을 수정해야 항목을 늘릴 수 있지만, 태그는 데이터 기반이므로 훨씬 유연하게 관리할 수 있습니다.  
2. **검색 효율**: 프로젝트 전체에서 특정 태그가 어디서 사용되는지 찾기 쉽습니다.  
3. **가독성**: 복잡한 If문 중첩을 제거하여 스파게티 코드를 방지합니다.

## **주의사항**

* **성능**: 수백 번의 Branch 보다는 빠를 수 있지만, 매우 빈번하게 호출되는 Tick 이벤트 내에서 너무 복잡한 태그 비교는 신중히 설계해야 합니다.  
* **태그 관리**: 프로젝트가 커지면 태그가 너무 많아질 수 있으므로, 태그 사전(Gameplay Tag Table)을 통해 체계적으로 네이밍 룰을 정하는 것이 중요합니다.
