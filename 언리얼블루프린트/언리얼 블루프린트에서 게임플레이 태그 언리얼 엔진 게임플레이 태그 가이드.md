# **언리얼 엔진 게임플레이 태그 (Gameplay Tags) 가이드**

\*\*게임플레이 태그(Gameplay Tags)\*\*는 상태.이동.달리기와 같이 계층 구조를 가진 이름(Name) 데이터입니다. 불리언(Boolean) 변수나 열거형(Enum)의 한계를 극복하고, 프로젝트가 커져도 유연하게 상태를 관리할 수 있도록 설계되었습니다.

## **1\. 주요 역할 및 특징**

### **① 계층적 구조 (Hierarchical Structure)**

태그는 점(.)을 구분자로 사용하여 부모-자식 관계를 가집니다.

* 예: Weapon.Ranged.Rifle  
* 만약 어떤 대상이 Weapon.Ranged.Rifle 태그를 가지고 있다면, 시스템은 이 대상이 상위 태그인 Weapon과 Weapon.Ranged에도 해당한다고 판단할 수 있습니다.

### **② 성능 최적화 (Optimized for Performance)**

태그는 내부적으로 딕셔너리(Dictionary)나 문자열 비교가 아닌, **고유한 인덱스 번호**로 관리됩니다. 따라서 수천 개의 태그를 비교하더라도 매우 빠르며 메모리 효율적입니다.

### **③ 유연한 상태 관리 (Flexibility)**

* **부울(Bool)의 대체:** IsDead, IsStunned 같은 수많은 변수 대신 State.Dead, State.Stunned 태그 하나로 관리 가능합니다.  
* **열거형(Enum)의 확장:** 열거형은 코드 수정 없이 중간에 항목을 넣기 어렵지만, 태그는 에디터에서 자유롭게 추가/수정이 가능합니다.

## **2\. 블루프린트에서의 주요 활용법**

### **① 태그 컨테이너 (Gameplay Tag Container)**

단일 태그(Gameplay Tag)가 아닌 여러 개의 태그를 담는 \*\*컨테이너(Gameplay Tag Container)\*\*를 자주 사용합니다. 예를 들어, 캐릭터가 현재 '중독' 상태이면서 '슬로우' 상태라면 컨테이너에 두 태그를 모두 담을 수 있습니다.

### **② 주요 블루프린트 노드**

* **Has Tag:** 특정 태그가 있는지 확인합니다.  
* **Matches Tag:** 계층 구조를 고려하여 태그가 일치하는지 확인합니다. (예: Weapon.Ranged.Rifle은 Weapon에 매치됨)  
* **Has All Tags / Has Any Tags:** 여러 태그 중 전부 또는 일부가 포함되어 있는지 확인합니다.  
* **Switch on Gameplay Tag:** 태그 종류에 따라 로직을 분기합니다.

### **③ 인터페이스 활용 (IGameplayTagAssetInterface)**

액터나 오브젝트에 인터페이스를 구현하면, 다른 블루프린트에서 해당 액터가 어떤 태그를 가졌는지 직접 변수에 접근하지 않고도 Get Owned Gameplay Tags 노드를 통해 일관되게 가져올 수 있습니다.

## **3\. 실무 활용 예시**

| 상황 | 활용 방식 |
| :---- | :---- |
| **데미지 시스템** | 공격에 Damage.Fire 태그를 붙이고, 캐릭터가 Resistance.Fire 태그를 가졌다면 데미지를 경감시킵니다. |
| **AI 감지** | AI가 대상을 인식할 때 대상이 Team.Enemy 태그를 가졌는지 확인하여 공격 여부를 결정합니다. |
| **상태 이상** | 캐릭터가 기절했을 때 State.Stunned 태그를 추가하고, 이 태그가 있는 동안은 이동 입력을 무시하도록 로직을 짭니다. |
| **상호작용** | 문(Door) 액터에 Interaction.Locked 태그를 부여하여 플레이어가 상호작용할 때 잠김 메시지를 띄웁니다. |

## **4\. 설정 방법**

1. **Project Settings** (프로젝트 세팅) 접속  
2. **Gameplay Tags** 항목 선택  
3. **Tag List**에서 \+ 버튼을 눌러 태그 추가하거나, .ini 파일 또는 Data Table을 통해 대량으로 관리할 수 있습니다.

## **5\. 요약: 왜 사용해야 하나요?**

과거에는 "이 캐릭터가 죽었나?"를 체크하기 위해 IsDead 변수를 검사했다면, 이제는 \*\*"이 액터가 State.Dead 태그를 가지고 있는가?"\*\*를 묻는 방식으로 바뀝니다. 이는 코드의 결합도를 낮추고(Decoupling), 기획자가 에디터에서 데이터만으로 복잡한 게임 규칙을 만들 수 있게 도와줍니다.
