# **언리얼 엔진 게임플레이 태그(Gameplay Tags) 가이드**

게임플레이 태그는 상태.속성.세부사항과 같은 계층 구조(Hierarchical)를 가진 고유한 이름표입니다. (예: State.Debuff.Stun, Weapon.Ranged.Sniper)

## **1\. 주요 역할**

### **① 유연한 상태 관리 (State Management)**

과거에는 캐릭터의 상태를 IsStunned, IsBurning, IsDead와 같은 수많은 불리언(Boolean) 변수로 관리했습니다. 태그를 사용하면 하나의 **태그 컨테이너(Tag Container)** 안에 현재 활성화된 태그들을 넣고 빼는 방식으로 상태를 관리할 수 있습니다.

### **② 데이터 중심 설계 (Data-Driven Design)**

코드나 블루프린트 로직을 직접 수정하지 않고도, 에디터 상에서 태그를 추가하거나 변경하여 게임 동작을 제어할 수 있습니다. 예를 들어, 특정 아이템이 "독" 속성을 가졌는지 확인할 때 Item.Attribute.Poison 태그가 있는지 체크하는 식입니다.

### **③ 컴포넌트 간 결합도 감소 (Decoupling)**

특정 클래스 타입에 의존하지 않고 "태그"만으로 소통할 수 있습니다.

* **기존 방식:** A라는 캐릭터가 B라는 클래스인지 캐스팅(Cast to)해서 확인  
* **태그 방식:** 이 액터가 'Team.Enemy' 태그를 가지고 있는가? 라고 질문

### **④ Gameplay Ability System (GAS)의 핵심**

언리얼의 강력한 프레임워크인 GAS에서 태그는 필수입니다. 어빌리티의 실행 조건, 쿨타임, 효과 중첩 등을 모두 태그를 통해 판별합니다.

## **2\. 불리언(Boolean) vs 게임플레이 태그**

| 특징 | 불리언 (Bool) | 게임플레이 태그 (Tags) |
| :---- | :---- | :---- |
| **확장성** | 변수를 매번 새로 생성해야 함 | 계층 구조로 무한히 확장 가능 |
| **가독성** | 변수가 많아지면 관리가 힘듦 | State.Buff.\* 처럼 그룹화하여 관리 용이 |
| **복합 체크** | \`if (A && B |  |
| **성능** | 매우 빠름 | 내부적으로 정수(Hash)로 관리되어 매우 빠름 |

## **3\. 블루프린트 주요 노드**

블루프린트에서 태그를 다룰 때 주로 사용하는 함수들입니다.

* **Has Tag (Exact):** 특정 태그가 정확히 일치하는지 확인합니다.  
* **Has Any Tags:** 컨테이너가 주어진 태그 목록 중 **하나라도** 가지고 있는지 확인합니다. (OR 조건)  
* **Has All Tags:** 컨테이너가 주어진 태그 목록을 **모두** 가지고 있는지 확인합니다. (AND 조건)  
* **Matches Tag:** 계층 구조를 인식합니다. State.Debuff를 체크하면 State.Debuff.Stun도 참(True)으로 판정합니다.

## **4\. 실전 활용 예시: 기절(Stun) 시스템**

1. **태그 정의:** State.Debuff.Stun 태그를 생성합니다.  
2. **태그 추가:** 캐릭터가 기절 공격을 받으면 캐릭터의 GameplayTagContainer에 해당 태그를 추가합니다.  
3. **로직 제한:** 캐릭터의 '공격' 함수 시작 부분에서 Has Tag (State.Debuff.Stun)을 체크하여, 태그가 있다면 공격이 나가지 않도록 Branch 노드로 막습니다.  
4. **UI 연동:** UI는 캐릭터의 태그 변경을 감시하다가 Stun 태그가 추가되면 화면에 기절 아이콘을 띄웁니다.

## **5\. 요약 및 권장 사항**

* **계층 구조를 활용하세요:** Buff.Speed, Buff.Attack 처럼 카테고리를 나누면 나중에 Buff로 시작하는 모든 태그를 한 번에 찾기 쉽습니다.  
* **문자열 비교보다 안전합니다:** 단순 String 비교는 오타 위험이 크지만, 태그는 에디터에서 선택 방식이므로 안전합니다.  
* **단순한 게임이 아니라면:** 프로젝트 규모가 커질수록 불리언 변수를 줄이고 게임플레이 태그로 전환하는 것이 유지보수에 훨씬 유리합니다.
