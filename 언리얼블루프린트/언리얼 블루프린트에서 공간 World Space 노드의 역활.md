# **🌍 언리얼 블루프린트: 월드 스페이스(World Space) 노드 역할 정리**

언리얼 엔진에서 모든 사물은 좌표를 가집니다. **World Space** 노드는 '레벨 전체의 중심(0, 0, 0)'을 기준으로 하는 절대적인 데이터를 처리합니다.

## **1\. 월드 스페이스 vs 로컬 스페이스**

* **월드 스페이스 (World Space):** 레벨의 절대 원점(0,0,0)을 기준으로 한 위치입니다. 어떤 부모에게 속해 있든 상관없이 세상에서의 실제 위치를 나타냅니다.  
* **로컬 스페이스 (Local/Relative Space):** 자기 자신이나 부모 액터를 기준으로 한 위치입니다. 부모가 움직이면 로컬 좌표는 변하지 않아도 월드 좌표는 변하게 됩니다.

## **2\. 주요 월드 스페이스 노드와 그 역할**

### **① Get World Location / Rotation / Scale**

* **역할:** 액터나 컴포넌트의 현재 '진짜' 위치를 가져옵니다.  
* **사용 예시:** 플레이어가 현재 맵의 어디에 있는지 확인하거나, 두 물체 사이의 실제 거리를 계산할 때 필수적입니다.

### **② Set World Location / Rotation**

* **역할:** 액터를 월드의 특정 좌표로 즉시 이동시키거나 회전시킵니다.  
* **특징:** 부모가 어디에 있든 상관없이 입력한 좌표값으로 강제 고정됩니다. (텔레포트 등에 사용)

### **③ Add World Offset / Rotation**

* **역할:** 현재 위치에서 월드 축(X, Y, Z)을 기준으로 일정량만큼 더 움직입니다.  
* **차이점:** Add Local Offset은 캐릭터가 바라보는 방향(앞)으로 움직이지만, Add World Offset은 캐릭터가 어디를 보든 월드의 X축(동쪽 등) 방향으로 움직입니다.

## **3\. 좌표 변환 노드 (가장 중요한 역할)**

월드 스페이스 노드는 로컬 데이터를 월드로 바꾸거나 그 반대의 경우에 핵심적인 역할을 합니다.

| 노드 이름 | 역할 |
| :---- | :---- |
| **Transform Location** | 특정 액터의 로컬 좌표를 월드 좌표로 변환합니다. (예: "총구 앞 50cm"라는 로컬 위치를 월드 어디인지 계산) |
| **Inverse Transform Location** | 월드 좌표를 특정 액터 기준의 로컬 좌표로 변환합니다. (예: "적의 월드 위치"가 "내 캐릭터" 기준 어디인지 계산) |

## **4\. 실무 활용 사례**

1. **액터 스폰 (Spawn Actor):** 새로운 물체를 소환할 때는 항상 월드 좌표가 필요합니다. Get World Location으로 소환 위치를 정합니다.  
2. **라인 트레이스 (Line Trace):** 레이저를 쏠 때 시작점과 끝점은 반드시 월드 스페이스 좌표여야 합니다.  
3. **UI 월드 위젯:** 게임 속 캐릭터 머리 위에 이름을 띄울 때, 캐릭터의 World Location을 가져와 화면 좌표로 변환하여 배치합니다.

### **💡 팁: 월드 노드를 언제 쓸까요?**

* "세상 전체를 기준으로 정해진 위치에 배치하고 싶을 때" \-\> **World**  
* "부모를 따라다니거나 내 앞/뒤/좌/우를 계산하고 싶을 때" \-\> **Relative/Local**
