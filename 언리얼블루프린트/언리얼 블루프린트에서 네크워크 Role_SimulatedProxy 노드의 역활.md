# **언리얼 엔진 네트워킹: Role\_SimulatedProxy 이해하기**

멀티플레이어 게임에서 모든 클라이언트가 모든 액터를 직접 제어할 수는 없습니다. 언리얼 엔진은 **액터 역할(Actor Role)** 시스템을 통해 어떤 기기가 액터에 대해 어떤 권한을 가질지 결정합니다. 그중 Role\_SimulatedProxy는 타인의 캐릭터나 오브젝트를 내 화면에 보여줄 때 사용됩니다.

## **1\. SimulatedProxy란 무엇인가?**

\*\*SimulatedProxy (시뮬레이션된 프록시)\*\*는 해당 액터를 소유하지 않은 클라이언트에서 존재하는 액터의 상태입니다.

* **비유**: 멀티플레이어 게임에서 '나'는 AutonomousProxy이지만, 내 옆에서 뛰어다니는 '다른 플레이어'는 내 화면에서 SimulatedProxy로 보입니다.  
* **권한 없음**: 이 역할을 가진 기기(클라이언트)는 액터의 위치를 직접 바꾸거나 변수를 수정할 권한이 없습니다. 오직 서버로부터 받은 데이터를 바탕으로 "흉내(Simulate)"만 낼 뿐입니다.

## **2\. 주요 특징**

### **서버 권위주의 (Server Authority)**

SimulatedProxy는 서버가 보내주는 데이터를 100% 신뢰합니다. 서버에서 "이 캐릭터는 좌표 (100, 100)에 있다"라고 알려주면, 클라이언트의 프록시는 그 위치로 이동합니다.

### **보간 및 예측 (Interpolation)**

네트워크 지연(Lag) 때문에 서버 데이터는 실시간보다 조금 늦게 도착할 수 있습니다. SimulatedProxy는 캐릭터가 뚝뚝 끊겨 보이지 않도록 다음과 같은 처리를 합니다.

* **보간(Interpolation)**: 이전 위치와 새로 받은 위치 사이를 부드럽게 이어줍니다.  
* **시뮬레이션**: 물리 법칙이나 애니메이션 등을 로컬에서 실행하여 부드러운 움직임을 유지합니다.

## **3\. 다른 역할과의 비교**

| 역할 이름 | 설명 | 주로 사용되는 곳 |
| :---- | :---- | :---- |
| **Role\_Authority** | 액터의 모든 권한을 가짐. | 서버(Server) |
| **Role\_AutonomousProxy** | 내가 직접 조종하는 액터. 입력값을 서버로 보냄. | 소유한 클라이언트 (나) |
| **Role\_SimulatedProxy** | 타인이 조종하는 액터를 복제하여 보여줌. | 소유하지 않은 클라이언트 (타인) |

## **4\. 블루프린트에서의 활용 예시**

블루프린트에서 Get Local Role 노드를 사용하여 현재 액터가 어떤 상태인지 확인할 수 있습니다.

### **예시: 타격 이펙트 발생**

\[Event Any Damage\] \-\> \[Switch on ENetRole (Get Local Role)\]  
    \- Authority: 서버에서 실제 체력을 깎음.  
    \- SimulatedProxy: (수행하지 않음) \-\> 보통 시뮬레이션 프록시는 시각적 효과만 담당합니다.

### **왜 확인해야 하나요?**

만약 SimulatedProxy에서 중요한 로직(예: 아이템 획득, 점수 계산)을 실행하게 되면, 각 클라이언트마다 결과가 달라지는 **네트워크 비동기화** 현상이 발생합니다. 따라서 중요한 로직은 Authority에서 실행하고, 시각적인 효과나 사운드만 SimulatedProxy에서 실행되도록 필터링할 때 이 노드를 사용합니다.

## **요약**

Role\_SimulatedProxy는 \*\*"내 기기에서 돌아가고 있지만, 내가 주인은 아닌 액터"\*\*를 의미합니다. 서버의 지시를 받아 다른 플레이어나 발사체의 움직임을 부드럽게 재현하는 역할을 수행합니다.
