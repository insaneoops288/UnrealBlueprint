# **언리얼 블루프린트 노드(Node)의 역할과 구조**

언리얼 엔진의 **블루프린트(Blueprint)** 시스템에서 \*\*노드(Node)\*\*는 프로그래밍의 가장 작은 단위이자 핵심적인 빌딩 블록입니다. 텍스트 코딩에서의 '코드 한 줄' 혹은 '함수'와 같은 역할을 수행하며, 이들을 서로 연결하여 게임의 로직을 완성합니다.

## **1\. 노드의 주요 역할**

노드는 크게 네 가지 관점에서 게임 제작의 핵심 로직을 담당합니다.

### **① 이벤트 수신 (Triggering)**

게임 내에서 어떤 일이 "언제" 일어날지를 결정합니다.

* **예:** "게임이 시작될 때(BeginPlay)", "마우스 왼쪽 버튼을 눌렀을 때", "적과 충돌했을 때" 등.  
* 이러한 노드들은 보통 붉은색 제목 표시줄을 가집니다.

### **② 액션 수행 (Execution)**

특정 동작을 "어떻게" 수행할지 명령합니다.

* **예:** "캐릭터를 점프시켜라", "사운드를 재생해라", "폭발 이펙트를 생성해라".  
* 대부분의 함수 노드(파란색)가 이에 해당합니다.

### **③ 데이터 처리 및 계산 (Data Processing)**

숫자나 텍스트 같은 데이터를 계산하거나 가공합니다.

* **예:** "두 숫자를 더하기", "플레이어의 체력이 0인지 확인하기", "현재 위치 정보 가져오기".  
* 실행 핀 없이 데이터만 전달하는 '순수 함수' 노드(초록색)가 주로 이 역할을 합니다.

### **④ 흐름 제어 (Flow Control)**

로직이 진행되는 순서나 조건을 결정합니다.

* **예:** "조건이 참이면 A로, 거짓이면 B로 가라(Branch)", "이 작업을 10번 반복해라(For Loop)", "1초 기다렸다가 다음 작업을 해라(Delay)".

## **2\. 노드의 시각적 구조**

노드는 직관적인 이해를 돕기 위해 고유한 구성을 가지고 있습니다.

* **제목 표시줄 (Title Bar):** 노드의 이름과 아이콘이 표시됩니다. 색상에 따라 노드의 성격(이벤트, 함수, 변수 등)을 알 수 있습니다.  
* **실행 핀 (Execution Pins):** 상단 좌우에 있는 **하얀색 화살표** 모양입니다. 선을 연결하여 로직이 실행되는 순서를 정합니다.  
* **데이터 입력 핀 (Input Pins):** 노드 왼쪽에 위치한 동그란 핀입니다. 노드가 작업을 수행하는 데 필요한 값(데이터)을 받습니다.  
* **데이터 출력 핀 (Output Pins):** 노드 오른쪽에 위치한 동그란 핀입니다. 작업 결과로 나온 값을 다음 노드로 전달합니다.

## **3\. 핀 색상에 따른 데이터 타입**

노드의 데이터 핀 색상은 전달되는 정보의 종류를 나타냅니다.

| 색상 | 데이터 타입 | 설명 |
| :---- | :---- | :---- |
| **빨간색** | **Boolean** | 참(True) 또는 거짓(False) |
| **하늘색** | **Integer** | 정수 (소수점이 없는 숫자) |
| **진녹색** | **Float** | 실수 (소수점이 있는 숫자) |
| **분홍색** | **Text/String** | 문자열 (이름, 대사 등) |
| **노란색** | **Vector** | 3D 공간의 위치나 방향 (X, Y, Z) |
| **파란색** | **Object** | 액터, 캐릭터, 컴포넌트 등 객체 그 자체 |

## **4\. 요약**

블루프린트 노드는 \*\*"특정 상황(Event)이 발생했을 때, 필요한 데이터(Data)를 가지고, 정해진 순서(Execution)에 따라 특정 동작을 수행하는 부품"\*\*이라고 이해하면 쉽습니다. 개발자는 복잡한 C++ 코드를 직접 타이핑하는 대신, 이 노드들을 논리적으로 조립하여 게임을 만듭니다.
