# **언리얼 엔진 Apply Damage 함수의 역할과 활용**

언리얼 엔진의 대미지 시스템은 \*\*"대미지를 입히는 쪽(Apply Damage)"\*\*과 \*\*"대미지를 받는 쪽(Receive Damage)"\*\*이 명확히 구분되어 설계되어 있습니다. 그중 Apply Damage는 공격자가 호출하는 표준 인터페이스입니다.

## **1\. 주요 역할**

Apply Damage 함수는 다음과 같은 역할을 수행합니다.

* **표준화된 정보 전달**: 단순히 체력 수치를 깎는 것이 아니라, 누가(Instigator), 무엇으로(Causer), 어떤 종류의(Damage Type) 대미지를 입혔는지에 대한 상세 정보를 대상 액터에게 전달합니다.  
* **이벤트 트리거**: 대상 액터(Damaged Actor) 내부에 구현된 Event AnyDamage를 자동으로 실행시킵니다.  
* **결합도 감소(Decoupling)**: 공격자 액터는 대상 액터가 정확히 어떤 클래스인지(예: 몬스터인지, 나무인지) 알 필요가 없습니다. 그저 "대미지를 입힌다"라는 메시지만 던지면, 받는 쪽에서 알아서 처리하도록 설계되어 있습니다.

## **2\. 입력 파라미터 상세 분석**

Apply Damage 노드를 구성하는 주요 핀들의 의미는 다음과 같습니다.

| 파라미터 | 역할 |
| :---- | :---- |
| **Damaged Actor** | 대미지를 입을 대상 액터입니다. |
| **Base Damage** | 입힐 대미지의 기본 수치입니다 (float). |
| **Event Instigator** | 대미지를 발생시킨 \*\*책임자(Controller)\*\*입니다. 보통 대미지를 입힌 캐릭터의 컨트롤러를 연결합니다. (점수 계산이나 킬 로그 확인 시 중요) |
| **Damage Causer** | 대미지를 물리적으로 발생시킨 \*\*원인물(Actor)\*\*입니다. 총알, 칼, 폭탄 등이 해당됩니다. |
| **Damage Type Class** | 대미지의 속성입니다. (예: 불, 독, 낙하 대미지 등) 이를 통해 대상이 특정 속성에 저항을 갖게 하거나 특수한 이펙트를 재생하게 할 수 있습니다. |

## **3\. 대미지 전달 워크플로우**

1. **공격자**: Apply Damage 함수를 호출합니다 (예: 총알이 적에게 닿았을 때).  
2. **엔진**: 대상 액터가 유효한지 확인하고 대미지 데이터를 패키징합니다.  
3. **대상자**: 자신의 블루프린트에 배치된 **Event AnyDamage** 노드가 실행됩니다.  
4. **대상자**: 전달받은 Damage 수치를 현재 체력에서 차감하거나, Damage Type에 따른 후속 처리를 합니다.

## **4\. 파생 함수 (상황별 선택)**

단순한 Apply Damage 외에도 상황에 따라 다음 함수들을 사용합니다.

* **Apply Point Damage**: 특정 지점(Hit Location)에 대한 대미지입니다. 피격 부위별(헤드샷 등) 판정이 필요할 때 사용합니다.  
* **Apply Radial Damage**: 특정 범위를 가진 폭발 대미지입니다. 중심부에서 멀어질수록 대미지가 감소하는 감쇄(Falloff) 기능을 포함합니다.

## **5\. 요약 및 장점**

* **유지보수**: 모든 액터가 동일한 대미지 시스템을 공유하므로 새로운 적이나 오브젝트를 추가할 때 체력 감소 로직을 일관성 있게 유지할 수 있습니다.  
* **확장성**: Damage Type 클래스를 상속받아 커스텀 속성을 만들면 게임의 전투 시스템을 깊이 있게 확장할 수 있습니다.  
* **네트워킹**: 언리얼 엔진의 기본 대미지 시스템은 서버 측에서 검증하고 클라이언트에 전파하기 용이하도록 최적화되어 있습니다.
