# **언리얼 블루프린트에서  데미지 시스템의 역활?**

언리얼 엔진의 데미지 시스템은 "누가 데미지를 주는가(Apply Damage)"와 "누가 데미지를 받는가(Damage Event)"를 명확히 구분하여 액터 간의 의존성을 줄이고 유연한 설계를 가능하게 합니다.

## **1\. 데미지 시스템의 핵심 역할**

* **표준화:** 어떤 액터든 동일한 방식으로 데미지를 전달할 수 있습니다.  
* **디커플링(Decoupling):** 공격자가 피격자의 구체적인 클래스를 몰라도 데미지를 입힐 수 있습니다.  
* **정보 전달:** 데미지 양뿐만 아니라 공격자, 사용된 무기, 데미지 유형(용암, 총알 등) 등의 정보를 함께 전달합니다.

## **2\. 주요 노드 구성**

### **A. 데미지를 주는 쪽 (Apply Damage)**

공격자(무기, 캐릭터, 환경 등)가 호출하는 함수입니다.

* **Apply Damage:** 가장 기본적인 데미지 전달 방식입니다.  
* **Apply Point Damage:** 특정 지점에 타격이 발생했을 때 사용합니다 (예: 총기 사격). 히트 결과(Hit Result)를 포함합니다.  
* **Apply Radial Damage:** 특정 범위 내의 모든 액터에게 데미지를 줍니다 (예: 폭발). 거리별 감쇠 설정을 지원합니다.

### **B. 데미지를 받는 쪽 (Damage Events)**

피격자(플레이어, 몬스터, 파괴 가능한 물체 등)의 이벤트 그래프에서 호출되는 이벤트입니다.

* **Event AnyDamage:** 어떤 종류의 데미지든 들어오면 발동하는 범용 이벤트입니다.  
* **Event PointDamage:** 총알처럼 특정 지점에 맞았을 때 발동하며, 맞은 부위나 방향 정보를 포함합니다.  
* **Event RadialDamage:** 폭발 범위 내에 있을 때 발동하며, 폭발 중심지와의 관계를 계산할 수 있습니다.

## **3\. 주요 파라미터 설명**

| 파라미터 | 설명 |
| :---- | :---- |
| **Damage** | 전달된 데미지의 양 (Float 값). |
| **Damage Type** | 데미지의 성격 (예: 불, 독, 물리). 이를 통해 저항력이나 특수 효과를 구현합니다. |
| **Instigator** | 데미지를 일으킨 '원인'이 되는 컨트롤러 (보통 PlayerController). |
| **Damage Causer** | 데미지를 직접 가한 '도구' (예: 발사체 액터, 칼 액터). |

## **4\. 실제 구현 워크플로우 (예시)**

일반적으로 피격 액터 내부에서는 다음과 같은 로직을 구성합니다.

1. **Event AnyDamage** 노드를 배치합니다.  
2. 현재 **HP(체력) 변수**에서 들어온 **Damage** 값을 뺍니다.  
3. 결과값을 다시 HP 변수에 저장(Set)합니다.  
4. 만약 HP가 0보다 작거나 같으면 **'사망 로직'** 또는 \*\*'파괴 로직'\*\*을 실행합니다.

// 논리적 구조 예시  
Event AnyDamage (Damage) \-\>   
    Health \= Health \- Damage \-\>   
    If (Health \<= 0\) \-\>   
        Play Death Animation / Destroy Actor

## **5\. 데미지 시스템 사용의 장점**

1. **확장성:** 새로운 몬스터를 추가해도 Event AnyDamage만 넣어두면 기존의 모든 무기 공격이 즉시 적용됩니다.  
2. **유지보수:** 데미지 계산 방식을 변경하고 싶을 때, 각 무기를 수정할 필요 없이 피격자의 데미지 이벤트만 수정하면 됩니다.  
3. **네트워크 최적화:** 언리얼의 기본 데미지 시스템은 서버-클라이언트 간의 동기화를 고려하여 설계되어 있어 멀티플레이어 게임 구현에도 유리합니다.
