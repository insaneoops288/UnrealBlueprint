# **언리얼 엔진: Layered Animation Blueprint Linking의 역할과 활용**

언리얼 엔진 4.24 버전부터 도입된 **Layered Animation Blueprint Linking**은 애니메이션 블루프린트의 복잡성을 관리하고 코드의 재사용성을 극대화하기 위한 아키텍처입니다. 이 시스템의 주요 역할과 작동 원리를 분석합니다.

## **1\. 핵심 정의**

이 기능은 \*\*애니메이션 레이어 인터페이스(Anim Layer Interface)\*\*를 통해 정의된 "레이어(함수)"를 기반으로, 부모 애니메이션 블루프린트(Base ABP)와 자식 애니메이션 블루프린트(Linked ABP)를 동적으로 연결하는 기술입니다.

* **Base ABP (호스트):** 전체적인 애니메이션 흐름을 제어하며, 특정 부위에 구멍(레이어)을 뚫어놓은 상태입니다.  
* **Linked ABP (레이어):** 특정 상황(예: 특정 무기 장착)에 맞는 애니메이션 로직을 담고 있으며, Base ABP의 구멍에 끼워 맞워집니다.

## **2\. 주요 역할 및 이점**

### **① 모듈화 및 재사용성 (Modularization)**

과거에는 캐릭터가 사용하는 무기가 20개라면, 하나의 ABP 안에 20개 무기에 대한 스테이트 머신(State Machine)과 블렌딩 로직을 모두 넣어야 했습니다.

* **역할:** 이동(Locomotion)은 Base ABP에 두고, 무기별 동작(공격, 재장전)은 각각의 Linked ABP로 분리합니다.  
* **이점:** 다른 캐릭터(NPC 등)가 같은 이동 로직을 공유하면서 무기 레이어만 다르게 연결하여 재사용할 수 있습니다.

### **② 동적 런타임 교체 (Dynamic Swapping)**

게임 플레이 도중 캐릭터가 검을 버리고 활을 집어 들었을 때, 애니메이션 로직 자체를 즉시 교체할 수 있습니다.

* **역할:** Link Anim Class Layers 노드를 호출하여 현재 활성화된 레이어를 실시간으로 바꿉니다.  
* **이점:** 복잡한 Switch on Enum이나 방대한 Blend Poses by Int 노드를 줄여 그래프를 깔끔하게 유지합니다.

### **③ 협업 효율성 증대**

대규모 프로젝트에서 여러 애니메이터가 동시에 작업할 때 충돌을 방지합니다.

* **역할:** 한 명은 기본 이동 시스템을 수정하고, 다른 한 명은 특정 스킬의 레이어 ABP를 수정할 수 있습니다.  
* **이점:** 하나의 파일(ABP)을 동시에 수정할 필요가 없어 데이터 경합(Conflict)이 발생하지 않습니다.

### **④ 성능 최적화**

* **역할:** 필요한 로직만 메모리에 로드하고 실행합니다.  
* **이점:** 사용하지 않는 무기의 복잡한 스테이트 머신이나 변수들이 연산되지 않으므로 CPU 비용을 절감할 수 있습니다.

## **3\. 작동 프로세스**

1. **Anim Layer Interface 생성:** 어떤 레이어(예: ItemUpperBody, FullBodyAction)가 존재할지 정의합니다.  
2. **Base ABP 설정:** \- 인터페이스를 추가합니다.  
   * 애니메이션 그래프에서 Linked Anim Layer 노드를 배치하여 애니메이션이 흐를 '자리'를 만듭니다.  
3. **Linked ABP 생성:** \- 인터페이스를 구현(Implement)합니다.  
   * 해당 레이어 내부에서 실제 애니메이션 로직(시퀀스 재생, 블렌드 등)을 작성합니다.  
4. **런타임 연결:** \- 캐릭터 블루프린트에서 Link Anim Class Layers 노드를 사용하여 상황에 맞는 Linked ABP 클래스를 Base ABP에 주입합니다.

## **4\. Sub-Anim Instance와의 차이점**

| 구분 | Sub-Anim Instance | Layered ABP Linking |
| :---- | :---- | :---- |
| **연결 방식** | 정적 (컴파일 타임에 결정) | 동적 (런타임에 언제든 교체 가능) |
| **유연성** | 낮음 (특정 노드에 고정됨) | 높음 (인터페이스를 통한 추상화) |
| **주요 목적** | 그래프의 시각적 정리 | 기능의 분리 및 런타임 확장 |

## **5\. 실무 적용 예시 (Weapon System)**

1. **Base ABP:** 다리 동작(걷기, 달리기, 점프)을 처리하고, 상체는 WeaponOverlay라는 레이어 노드를 둡니다.  
2. **Sword ABP (Linked):** 검을 들고 있을 때의 유휴 자세와 공격 애니메이션을 가집니다.  
3. **Pistol ABP (Linked):** 총을 들고 있을 때의 조준 자세와 사격 로직을 가집니다.  
4. **교체:** 플레이어가 무기를 바꾸면, 캐릭터 블루프린트에서 Set Link Anim Class Layers를 호출해 Sword ABP에서 Pistol ABP로 즉시 교체합니다.
