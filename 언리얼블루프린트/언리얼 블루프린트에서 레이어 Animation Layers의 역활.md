# **언리얼 엔진 애니메이션 레이어 (Animation Layers)**

애니메이션 레이어 시스템은 애니메이션 블루프린트(AnimBP)의 로직을 독립적인 모듈로 분리하여 **재사용성**과 **가독성**을 극대화하는 도구입니다.

## **1\. 핵심 역할: 애니메이션의 모듈화**

전통적인 애니메이션 블루프린트는 하나의 거대한 그래프에 모든 무기 상태, 이동 로직 등을 넣어야 했습니다. 하지만 애니메이션 레이어를 사용하면 다음과 같은 작업이 가능합니다.

* **로직 분리:** 기본 이동(Locomotion)은 메인 AnimBP에 두고, 무기별 장전/사격 애니메이션은 별도의 '레이어'로 분리합니다.  
* **런타임 교체:** 캐릭터가 권총을 들었을 때와 소총을 들었을 때, 상체 애니메이션 로직만 담긴 레이어를 즉시 교체할 수 있습니다.  
* **협업 효율성:** 여러 명의 애니메이터가 서로 다른 레이어 파일을 작업하므로 머지(Merge) 충돌을 줄일 수 있습니다.

## **2\. 주요 구성 요소 및 노드**

### **① Animation Layer Interface (애니메이션 레이어 인터페이스)**

레이어의 '설계도'입니다. 어떤 입력(Pose)을 받고 어떤 출력을 내보낼지 함수처럼 정의합니다.

* 예: FullBodyOverride, UpperBodyLayer(InputPose) 등

### **② Linked Anim Layer 노드 (메인 그래프에서 사용)**

애니메이션 그래프 내에서 특정 레이어가 실행될 '지점'을 지정하는 노드입니다.

* **역할:** 인터페이스에서 정의한 레이어를 호출합니다.  
* **특징:** 실행 시점에 어떤 클래스가 링크되어 있느냐에 따라 실제로 구동되는 로직이 결정됩니다.

### **③ Link Anim Class Layers 노드 (이벤트 그래프에서 사용)**

실제로 어떤 레이어 로직(AnimBP)을 적용할지 결정하는 노드입니다.

* **역할:** 특정 Anim Instance 클래스를 현재 캐릭터의 애니메이션 레이어 슬롯에 바인딩합니다.  
* **활용:** 무기를 습득했을 때 Link Anim Class Layers를 호출하여 해당 무기 전용 애니메이션 블루프린트를 연결합니다.

## **3\. 작동 원리 (Workflow)**

1. **인터페이스 생성:** Animation Layer Interface 에셋을 만들고 레이어 함수를 정의합니다.  
2. **메인 AnimBP 설정:** 메인 AnimBP에서 해당 인터페이스를 추가하고, 그래프 안에 Linked Anim Layer 노드를 배치하여 포즈가 흐를 자리를 만듭니다.  
3. **레이어 구현:** 별도의 AnimBP를 생성하여 인터페이스를 구현(Implementation)하고, 실제 애니메이션 노드(Sequence, Blend Space 등)를 배치합니다.  
4. **연결:** 캐릭터 블루프린트에서 Link Anim Class Layers 노드를 사용해 메인 AnimBP와 구현된 레이어 AnimBP를 연결합니다.

## **4\. 장점 요약**

| 특징 | 설명 |
| :---- | :---- |
| **유연성** | 베이스 애니메이션을 건드리지 않고 특정 부위나 상태의 로직만 교체 가능 |
| **최적화** | 필요한 상황에만 특정 레이어 로직을 로드하고 실행하여 성능 관리 용이 |
| **가독성** | 스파게티처럼 꼬인 복잡한 스테이트 머신을 깔끔하게 정리 |

## **5\. 실무 예시: 무기 시스템**

* **Main AnimBP:** 다리의 이동(Walk/Run/Idle)만 담당. 상체는 WeaponLayer라는 노드에 연결됨.  
* **Pistol\_Layer\_BP:** WeaponLayer 위치에 권총 파지 및 반동 로직 제공.  
* **Rifle\_Layer\_BP:** WeaponLayer 위치에 소총 파지 및 조준 로직 제공.  
* **결과:** 무기를 바꿀 때마다 WeaponLayer에 연결되는 클래스만 바꿔주면 전체 애니메이션 시스템이 완성됨.
