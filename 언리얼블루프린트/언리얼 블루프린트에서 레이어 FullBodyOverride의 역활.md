# **언리얼 엔진: FullBodyOverride의 역할 및 활용 가이드**

언리얼 엔진의 애니메이션 블루프린트와 컨트롤 리그 시스템에서 **FullBodyOverride**는 캐릭터의 뼈대(Skeleton) 전체에 대해 기존 애니메이션 데이터를 무시하고, 새로운 데이터(주로 IK 또는 절차적 계산 결과)로 포즈를 완전히 덮어쓰는 기능을 의미합니다.

## **1\. 핵심 역할 (Core Roles)**

### **① 기존 애니메이션 데이터 무시 (Complete Replacement)**

일반적인 애니메이션 레이어링은 '팔만 흔들기'처럼 특정 부분만 수정(Partial)하거나 블렌딩하지만, FullBodyOverride는 루트(Root)부터 말단 노드까지의 모든 트랜스폼을 새로운 계산값으로 강제 대체합니다.

### **② FBIK(Full Body IK) 솔버와의 통합**

컨트롤 리그에서 **Full Body IK** 노드를 사용할 때 주로 발생합니다. 발의 위치를 고정(Planting)하거나 손이 특정 물체를 잡아야 할 때, 단순히 팔만 움직이는 것이 아니라 골반과 척추를 포함한 전신이 자연스럽게 반응하도록 포즈를 다시 계산하여 적용합니다.

### **③ 절차적 포즈 제어 (Procedural Animation)**

캐릭터가 가파른 지형을 걷거나, 벽을 붙잡는 동작(Climbing) 등을 수행할 때, 미리 제작된 FBX 애니메이션만으로는 한계가 있습니다. 이때 실시간 계산된 위치값을 전신에 적용하여 환경에 완벽히 적응하는 포즈를 만들어냅니다.

## **2\. 주요 사용 시나리오**

| 시나리오 | 설명 |
| :---- | :---- |
| **정밀한 상호작용** | 캐릭터가 아주 정밀하게 특정 버튼을 누르거나 복잡한 물체를 집어야 할 때, 전신의 무게중심을 맞추며 포즈를 덮어씁니다. |
| **지형 적응 (Foot Placement)** | 경사면에서 발의 높낮이에 따라 무릎, 골반, 허리의 각도가 모두 변해야 할 때 사용합니다. |
| **물리 기반 반응** | 외부 충격이나 특정 물리 연산 결과를 애니메이션에 강제로 반영해야 할 때 활용됩니다. |
| **시네마틱 컨트롤** | 시퀀서(Sequencer)에서 캐릭터의 포즈를 프레임 단위로 수동 제어할 때 컨트롤 리그를 통해 전신 오버라이드를 실행합니다. |

## **3\. 구현 방식 (Control Rig 예시)**

컨트롤 리그 내에서 Full Body IK 노드를 설정할 때 다음과 같은 흐름으로 작동합니다:

1. **Input Pose**: 기존 애니메이션 블루프린트에서 넘어온 포즈를 받습니다.  
2. **Solver Calculation**: 'Effectors'(손목, 발목 등)의 목표 위치를 기준으로 전체 뼈대의 각도를 계산합니다.  
3. **Override/Apply**: 계산된 결과값을 스켈레탈 메시에 직접 꽂아 넣습니다. 이때 Settings에서 강도(Alpha)를 조절하여 기존 애니메이션과 섞을 수도 있지만, 기본적으로 'Override'는 기존 값을 대체하는 데 목적이 있습니다.

## **4\. 주의사항**

* **성능 비용**: 전신의 모든 뼈를 실시간으로 재계산하므로, 단순한 Layered Bone Blend보다 연산 비용이 높습니다. 필요한 경우에만 활성화해야 합니다.  
* **포즈 깨짐 (Popping)**: 애니메이션과 오버라이드 포즈 간의 차이가 너무 크면 캐릭터가 갑자기 튀는 현상이 발생할 수 있습니다. Alpha값이나 Interpolation을 통해 부드럽게 전환하는 과정이 필요합니다.  
* **루트 모션과의 관계**: 루트 본(Root Bone)까지 오버라이드할 경우, 캐릭터의 실제 이동 캡슐과 메시의 위치가 어긋날 수 있으므로 주의 깊게 세팅해야 합니다.

### **요약**

FullBodyOverride는 \*\*"기존 애니메이션은 잠시 잊고, 내가 지금 계산한 이 완벽한 전신 포즈로 캐릭터를 그려라"\*\*라고 명령하는 엔진의 핵심 제어 장치입니다.
