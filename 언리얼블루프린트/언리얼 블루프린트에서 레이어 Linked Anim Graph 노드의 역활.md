# **언리얼 엔진: Linked Anim Layer 및 Graph 상세 가이드**

## **1\. 개요 (Overview)**

애니메이션 블루프린트가 커지면 관리하기가 매우 힘들어집니다. **Linked Anim Layer** 시스템은 프로그래밍의 '인터페이스(Interface)' 개념을 애니메이션 그래프에 도입한 것입니다. 이를 통해 베이스가 되는 애니메이션 블루프린트의 구조는 유지한 채, 세부적인 동작(걷기, 대기, 공격 등)만 외부 블록에서 정의하여 갈아끼울 수 있습니다.

## **2\. 주요 역할과 기능**

### **① 모듈화 (Modularity)**

과거에는 하나의 ABP 안에 모든 무기(검, 총, 활)의 애니메이션 로직을 넣어야 했습니다. 하지만 레이어를 사용하면 다음과 같이 분리할 수 있습니다.

* **Base ABP**: 캐릭터의 기본적인 이동 로직 정의  
* **Pistol Layer**: 권총 전용 조준 및 재장전 로직  
* **Sword Layer**: 칼 전용 휘두르기 로직

### **② 동적 교체 (Dynamic Swapping)**

런타임 중에 Set Layer Instance 함수를 사용하여 특정 레이어의 구현체를 즉시 바꿀 수 있습니다. 예를 들어, 캐릭터가 바닥에 놓인 '소총'을 집는 순간, 상체 애니메이션 레이어를 '소총 전용 레이어'로 교체하는 식입니다.

### **③ 협업 효율성**

애니메이터 A는 베이스 그래프를 수정하고, 애니메이터 B는 무기별 레이어만 작업할 수 있어 파일 충돌(Conflict)을 방지합니다.

## **3\. 작동 원리 (Workflow)**

레이어 시스템은 세 가지 단계로 구성됩니다.

### **1단계: Anim Layer Interface 생성**

* 어떤 "슬롯"이 필요한지 정의합니다.  
* 예: IdleAnim, MoveAnim, AttackAnim이라는 함수(레이어)를 가진 인터페이스 생성.

### **2단계: Base ABP에서 호출**

* 베이스 ABP의 AnimGraph 안에서 **Linked Anim Layer** 노드를 배치합니다.  
* 위에서 만든 인터페이스를 선택하고 특정 레이어(예: IdleAnim)를 연결합니다.  
* 이 노드는 "여기에 외부에서 정의된 애니메이션이 들어올 것"이라는 **플레이스홀더(Placeholder)** 역할을 합니다.

### **3단계: Implementation ABP 작성**

* 새로운 ABP를 만들고 해당 인터페이스를 구현(Implement)합니다.  
* 각 레이어 함수 내부에서 실제 애니메이션 노드들을 구성합니다.  
* 게임 실행 중 Link Anim Class Layers 노드를 통해 이 ABP를 베이스 ABP에 연결합니다.

## **4\. Linked Anim Graph vs Linked Anim Layer**

| 구분 | Linked Anim Graph | Linked Anim Layer |
| :---- | :---- | :---- |
| **성격** | 다른 ABP 전체를 서브 그래프로 호출 | 인터페이스 기반의 부분 교체 |
| **유연성** | 고정된 하위 구조 (정적) | 언제든 구현체를 교체 가능 (동적) |
| **주 용도** | 복잡한 로직을 별도 파일로 분리 | 무기, 도구, 상태별 로직 스위칭 |

## **5\. 실무 활용 팁**

* **Pose 입력**: 레이어 노드는 외부 포즈를 입력받을 수 있습니다. 예를 들어 베이스의 전신 포즈를 레이어로 넘겨서, 레이어 안에서 상체만 수정(Layered blend per bone)한 뒤 다시 반환하는 구조가 가능합니다.  
* **성능**: 수많은 State Machine을 하나에 몰아넣는 것보다, 필요한 레이어만 활성화하여 사용하는 것이 CPU 비용 최적화에 유리할 수 있습니다.
