# **언리얼 엔진: Linked Anim Layer (링크드 애니메이션 레이어) 가이드**

## **1\. 개요 (Definition)**

**Linked Anim Layer**는 애니메이션 블루프린트(AnimBP) 내에서 특정 로직을 별도의 자산으로 분리하여 관리할 수 있게 해주는 노드입니다. 이는 프로그래밍의 '인터페이스(Interface)' 개념과 유사하며, 메인 AnimBP의 복잡도를 낮추고 팀 협업을 용이하게 만듭니다.

## **2\. 주요 역할 (Key Roles)**

### **① 애니메이션 로직의 모듈화 (Modularity)**

과거에는 모든 애니메이션 로직(걷기, 뛰기, 공격, 무기별 포즈 등)을 하나의 거대한 AnimBP 안에 넣어야 했습니다. 하지만 링크드 레이어를 사용하면 다음과 같이 분리할 수 있습니다.

* **Main AnimBP**: 기본 이동(Locomotion) 및 상태 머신 관리.  
* **Weapon AnimBP**: 검, 총, 활 등 각 무기에 특화된 레이어 로직.

### **② 런타임 레이어 교체 (Dynamic Swapping)**

게임 플레이 중 캐릭터가 무기를 바꾸거나 상태가 변할 때, Set Layer Instance 함수를 사용하여 실행 중인 애니메이션 레이어를 즉시 교체할 수 있습니다. 예를 들어, 검을 들었을 때는 '검용 레이어'를 연결하고, 총을 들었을 때는 '총용 레이어'로 갈아 끼우는 방식입니다.

### **③ 애니메이션 인터페이스 활용**

Animation Layer Interface를 통해 입력(Input) 포즈와 출력(Output) 포즈를 정의합니다. 메인 AnimBP는 이 인터페이스가 어떻게 구현되었는지는 상관하지 않고, 오직 데이터가 들어오고 나가는 통로만 제공합니다.

## **3\. 작동 원리 (How it Works)**

1. **Animation Layer Interface 생성**: 사용할 레이어의 이름과 입/출력 구조를 정의합니다.  
2. **인터페이스 구현**: 별도의 애니메이션 블루프린트를 생성하여 해당 인터페이스를 추가하고, 실제 노드(애니메이션 시퀀스, 블렌드 스페이스 등)를 배치합니다.  
3. **Linked Anim Layer 노드 배치**: 메인 AnimBP의 애님 그래프(Anim Graph)에 인터페이스에 정의된 레이어 노드를 배치합니다.  
4. **레이어 연결**: 블루프린트 로직에서 Link Anim Class Layers 노드를 사용하여 실제 구현된 클래스를 할당합니다.

## **4\. 사용 시의 장점 (Benefits)**

| 장점 | 설명 |
| :---- | :---- |
| **가독성 향상** | 메인 그래프가 단순해져서 유지보수가 쉬워집니다. |
| **협업 효율** | 애니메이터 A는 이동 로직을, 애니메이터 B는 전투 로직을 서로 다른 파일에서 동시에 작업할 수 있습니다. |
| **메모리 최적화** | 필요한 상황에만 특정 레이어를 로드하여 메모리 사용량을 조절할 수 있습니다. |
| **재사용성** | 한 번 만든 '무기 애니메이션 레이어'를 여러 캐릭터 AnimBP에서 공통으로 사용할 수 있습니다. |

## **5\. 실무 적용 예시 (Use Case)**

**\[전투 시스템 예시\]**

* **입력**: 메인 이동 포즈 (Base Pose)  
* **레이어 노드**: WeaponOverrideLayer  
* **상황 A (맨손)**: 레이어에 아무것도 연결하지 않음 → Base Pose 그대로 출력.  
* **상황 B (칼 착용)**: SwordAnimLayer를 링크 → 메인 포즈 위에 칼을 든 상체 포즈와 애니메이션이 블렌딩되어 출력.  
* **상황 C (소총 착용)**: RifleAnimLayer를 링크 → 소총 조준 및 반동 로직이 적용된 포즈 출력.

## **6\. 결론**

**Linked Anim Layer** 노드는 대규모 프로젝트나 다양한 장비를 사용하는 캐릭터 시스템에서 **필수적인 요소**입니다. 애니메이션 블루프린트를 단순한 "데이터 저장소"가 아닌, 유연하고 확장 가능한 "시스템"으로 만들어 줍니다.
