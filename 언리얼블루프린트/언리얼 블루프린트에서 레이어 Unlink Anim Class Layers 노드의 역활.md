# **Unlink Anim Class Layers 노드 상세 가이드**

언리얼 엔진(Unreal Engine)의 **애니메이션 블루프린트(Animation Blueprint)** 시스템에는 애니메이션 로직을 모듈화하고 동적으로 교체할 수 있는 **애니메이션 레이어 인터페이스(Animation Layer Interface)** 기능이 있습니다. Unlink Anim Class Layers 노드는 이 시스템에서 '연결 해제'를 담당하는 핵심 노드입니다.

## **1\. 노드의 핵심 역할**

Unlink Anim Class Layers 노드는 특정 애니메이션 블루프린트 인스턴스에 적용되어 있던 **애니메이션 레이어 클래스(Anim Layer Class)의 연결을 끊는 기능**을 수행합니다.

주요 역할은 다음과 같습니다:

* **레이어 초기화:** Link Anim Class Layers 노드를 통해 강제로 덮어씌웠던 애니메이션 로직을 제거합니다.  
* **기본 로직 복구:** 레이어가 언링크되면, 해당 인터페이스 포인트는 메인 애니메이션 블루프린트 내에 정의된 **기본(Default) 그래프 로직**으로 돌아가게 됩니다.  
* **메모리 및 리소스 관리:** 더 이상 필요 없는 특정 무기나 상태 전용 애니메이션 클래스의 인스턴스를 정리합니다.

## **2\. 왜 사용하는가? (활용 시나리오)**

가장 대표적인 사용 사례는 **아이템 장착 및 해제** 시스템입니다.

### **예시: 무기 시스템**

1. **장착 (Linking):** 캐릭터가 '검'을 집으면, Link Anim Class Layers를 사용하여 '검 전용 애니메이션 클래스'를 캐릭터의 메인 AnimBP에 연결합니다. 이제 캐릭터는 검을 들고 걷거나 공격하는 애니메이션을 수행합니다.  
2. **해제 (Unlinking):** 캐릭터가 검을 버리거나 등에 메면, Unlink Anim Class Layers를 호출합니다.  
   * 결과적으로 '검 전용 애니메이션' 클래스가 제거됩니다.  
   * 캐릭터의 애니메이션은 다시 '기본 맨손(Default)' 상태의 포즈와 로직으로 자연스럽게 돌아갑니다.

## **3\. 노드 주요 핀(Pin) 설명**

| 핀 이름 | 설명 |
| :---- | :---- |
| **In Class** | 연결을 해제하고자 하는 특정 애니메이션 레이어 클래스를 지정합니다. |
| **Target** | 레이어를 제거할 대상 애니메이션 인스턴스(보통 캐릭터의 Mesh에서 얻은 Anim Instance)입니다. |

## **4\. 작동 원리 및 주의사항**

1. **인터페이스 기반:** 이 노드를 사용하려면 메인 애니메이션 블루프린트가 하나 이상의 \*\*애니메이션 레이어 인터페이스(Animation Layer Interface)\*\*를 구현하고 있어야 하며, 그래프 내에 'Layer' 노드가 배치되어 있어야 합니다.  
2. **동적 교체:** 런타임 중에 게임 플레이 로직(블루프린트)에서 호출됩니다. 애니메이션 그래프 내부가 아닌, \*\*이벤트 그래프(Event Graph)\*\*나 **캐릭터 블루프린트**에서 주로 실행됩니다.  
3. **상태 보존:** 레이어가 언링크될 때, 해당 레이어 내에서 관리되던 내부 변수나 상태는 소멸됩니다. 다시 링크할 때는 새로운 인스턴스가 생성되므로 초기화된 상태로 시작됩니다.

## **5\. 요약**

Unlink Anim Class Layers는 **"특정 상황(무기 해제, 탑승 종료 등)에서 추가되었던 애니메이션 오버라이드 로직을 제거하고, 캐릭터를 다시 기본 애니메이션 상태로 되돌릴 때"** 사용하는 필수적인 노드입니다. 이를 통해 복잡한 애니메이션 블루프린트 하나에 모든 데이터를 넣는 대신, 상황별로 필요한 애니메이션 셋을 가볍게 갈아끼울 수 있습니다.
