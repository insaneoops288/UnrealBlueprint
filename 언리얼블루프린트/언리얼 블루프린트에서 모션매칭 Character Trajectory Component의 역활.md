# **언리얼 엔진 모션 매칭: Character Trajectory Component의 역할**

모션 매칭(Motion Matching) 시스템은 현재 캐릭터의 포즈와 \*\*미래에 이동할 경로(Trajectory)\*\*를 분석하여 애니메이션 데이터베이스에서 가장 적합한 프레임을 찾아냅니다. 이때 **Character Trajectory Component**는 캐릭터의 과거 위치를 기록하고 미래의 이동 경로를 예측하는 핵심적인 역할을 수행합니다.

## **1\. 주요 역할 (Core Roles)**

### **① 과거 경로 추적 (History Tracking)**

캐릭터가 직전 몇 초 동안 어느 방향으로, 어떤 속도로 움직였는지에 대한 데이터를 유지합니다. 이는 애니메이션의 연속성을 보장하고, 급격한 방향 전환 시 관성을 표현하는 데 도움을 줍니다.

### **② 미래 경로 예측 (Future Prediction)**

플레이어의 입력(Input)이나 AI의 이동 명령을 바탕으로, 캐릭터가 향후 1\~2초 내에 도달할 예상 위치와 방향을 계산합니다.

* **입력 기반:** 조이스틱이나 키보드 입력값을 가속도로 변환하여 예측합니다.  
* **이동 속도:** 현재 Character Movement Component의 속도와 가속도 설정을 참조합니다.

### **③ 모션 매칭 노드에 데이터 전달**

예측된 궤적 데이터(Trajectory Sample)를 애니메이션 블루프린트(AnimBP)의 Motion Matching 노드에 전달합니다. 시스템은 이 예측된 선과 데이터베이스에 저장된 애니메이션들의 궤적을 비교하여 "가장 일치하는(Cost가 낮은)" 애니메이션을 선택합니다.

## **2\. 작동 원리 (How it Works)**

1. **데이터 샘플링:** 컴포넌트는 특정 시간 간격(예: 0.1초, 0.5초, 1.0초 뒤)으로 '샘플' 포인트를 생성합니다.  
2. **궤적 생성:** 현재 위치에서 미래의 예상 위치까지를 잇는 곡선(Trajectory)을 만듭니다.  
3. **포즈 매칭:** 모션 매칭 알고리즘은 다음과 같은 수식을 고려합니다:  
   * Total Cost \= Pose Cost (현재 자세 유사도) \+ Trajectory Cost (예측 경로 유사도)  
   * 여기서 Trajectory Component가 제공하는 데이터가 Trajectory Cost를 결정합니다.

## **3\. 주요 설정 및 파라미터**

| 파라미터 | 설명 |
| :---- | :---- |
| **History Settings** | 과거 데이터를 얼마나 오래, 얼마나 촘촘하게 저장할지 설정합니다. |
| **Prediction Settings** | 미래 예측 시간(Seconds)과 샘플 개수를 설정합니다. 보통 1초 정도의 미래를 예측합니다. |
| **Max Floating Distance** | 캐릭터 캡슐과 애니메이션 루트 모션 간의 허용 오차를 제어합니다. |
| **Blending/Smoothing** | 예측된 경로가 급격하게 변할 때 이를 얼마나 부드럽게 보정할지 결정합니다. |

## **4\. 왜 중요한가?**

기존의 **스테이트 머신(State Machine)** 방식은 "입력이 들어오면 Run 상태로 전이해라"라는 규칙 기반이었지만, **모션 매칭**은 "플레이어가 오른쪽 앞으로 가고 싶어 하니, 이 궤적과 가장 잘 맞는 애니메이션 프레임을 재생해라"라는 데이터 기반 방식입니다.

따라서 **Character Trajectory Component**가 정확하게 예측할수록 다음과 같은 효과를 얻을 수 있습니다:

* 발 미끄러짐(Foot Sliding) 현상 감소  
* 자연스러운 방향 전환(Leaning 및 Arc Turn)  
* 급정지 및 출발 시의 디테일한 반응성

## **5\. 블루프린트에서의 사용법**

1. **컴포넌트 추가:** 캐릭터 블루프린트에 Character Trajectory Component를 추가합니다.  
2. **데이터 전달:** 애니메이션 블루프린트의 Update Animation 이벤트에서 해당 컴포넌트의 궤적 데이터를 변수로 가져옵니다.  
3. **노드 연결:** Motion Matching 노드의 Trajectory 입력 핀에 이 데이터를 연결합니다.

*참고: 이 기능은 UE 5.4 이상에서 'Motion Matching' 플러그인이 활성화된 상태에서 최적으로 작동합니다.*
