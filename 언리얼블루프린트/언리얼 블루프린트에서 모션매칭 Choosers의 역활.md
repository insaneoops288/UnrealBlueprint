# **언리얼 엔진 모션 매칭: Choosers의 역할과 활용**

언리얼 엔진(UE 5.4 이상 핵심)의 모션 매칭 워크플로우에서 **Chooser**는 애니메이션 선택 로직을 데이터 중심(Data-driven)으로 관리할 수 있게 해주는 에셋입니다.

## **1\. Choosers의 핵심 정의**

Choosers는 \*\*"특정 조건(Context)에 따라 어떤 애니메이션 데이터베이스나 에셋을 사용할 것인가?"\*\*를 결정하는 테이블 기반의 선택 도구입니다.

기존에는 애니메이션 블루프린트(AnimBP)에서 수많은 Boolean 변수와 State Machine 노드를 연결해 상태를 전환했다면, Chooser는 이를 하나의 표(Table) 형태로 관리하여 가독성과 효율성을 극대화합니다.

## **2\. 주요 역할 (Key Roles)**

### **① 컨텍스트 기반의 데이터베이스 스위칭**

모션 매칭은 성능을 위해 하나의 거대한 데이터베이스보다 상황별(걷기, 뛰기, 전투, 수영 등)로 나누어진 여러 개의 **Pose Search Database**를 사용하는 것이 유리합니다.

* **Chooser의 역할:** 캐릭터의 현재 속도, 가속도, 태세(Stance), 장착 무기 등의 변수를 평가하여 현재 프레임에서 어떤 PSD(Pose Search Database)를 검색할지 실시간으로 결정합니다.

### **② 복잡한 스테이트 머신(State Machine) 대체**

전통적인 애니메이션 시스템은 상태가 늘어날수록 '스파게티 코드'처럼 노드가 복잡해지는 문제가 있습니다.

* **Chooser의 역할:** 복잡한 전이 조건(Transition Rules) 대신, 열(Column)과 행(Row)으로 구성된 규칙 테이블을 사용합니다. 예를 들어 "만약 IsCrouching이 True이고 Speed가 0보다 크면 Crouch\_Walk\_Database를 선택한다"는 로직을 직관적으로 작성할 수 있습니다.

### **③ 계층적 선택 구조 (Nested Choosers)**

Chooser는 다른 Chooser를 참조할 수 있습니다.

* **예시:** 상위 Chooser가 '이동(Locomotion)'과 '전투(Combat)' 중 하나를 고르면, 하위 Chooser가 '이동' 안에서 다시 '부상 여부'에 따른 데이터베이스를 고르는 방식입니다. 이를 통해 로직을 모듈화할 수 있습니다.

### **④ 포즈 검색(Pose Search) 최적화**

모든 애니메이션을 매 프레임 검색하는 것은 비용이 큽니다.

* **Chooser의 역할:** 현재 상황에 전혀 맞지 않는 데이터(예: 전투 중인데 비전투 포즈 검색)를 검색 대상에서 제외함으로써 성능을 최적화합니다.

## **3\. Chooser의 구조**

Chooser 에셋을 열면 다음과 같은 구성 요소를 볼 수 있습니다:

| 요소 | 설명 |
| :---- | :---- |
| **Columns (열)** | 판단 기준이 되는 변수나 조건 (예: 캐릭터 상태, 이동 속도, 게임플레이 태그). |
| **Rows (행)** | 각 조건의 조합. 상단에 위치한 행일수록 우선순위가 높습니다. |
| **Results (결과)** | 모든 조건이 충족되었을 때 반환할 값 (주로 Pose Search Database나 Anim Sequence). |

## **4\. 실전 활용 시나리오**

1. **애니메이션 블루프린트(AnimBP) 설정:**  
   * Motion Matching 노드의 입력 핀에 Chooser 에셋을 연결합니다.  
2. **데이터 입력:**  
   * 캐릭터의 Movement Component에서 전달되는 속도 값이나 Gameplay Tag를 Chooser의 입력 파라미터로 설정합니다.  
3. **결과 반환:**  
   * Chooser가 현재 태그가 State.Combat임을 확인하면 PSD\_Combat 데이터베이스를 모션 매칭 노드로 전달합니다.  
   * 모션 매칭 노드는 해당 데이터베이스 안에서 가장 캐릭터의 궤적(Trajectory)과 일치하는 포즈를 찾아 재생합니다.

## **5\. 요약**

**Choosers**는 모션 매칭 시스템의 \*\*'두뇌'\*\*와 같습니다.

어떤 애니메이션 데이터를 사용할지 결정하는 \*\*논리(Logic)\*\*를 애니메이션 블루프린트 노드에서 분리하여, 별도의 에셋으로 관리함으로써 대규모 프로젝트에서도 깔끔하고 확장 가능한 애니메이션 시스템을 구축할 수 있게 해줍니다.
