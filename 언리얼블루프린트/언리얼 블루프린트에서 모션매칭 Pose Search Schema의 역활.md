# **Unreal Engine 모션 매칭: Pose Search Schema 가이드**

모션 매칭 시스템에서 **Pose Search Schema**는 애니메이션 데이터베이스(Pose Search Database)가 생성되고 검색되는 방식을 결정하는 설계도 역할을 합니다.

## **1\. 주요 역할 (Core Roles)**

### **① 데이터 구조 및 특징 정의 (Feature Definition)**

스키마는 애니메이션의 수많은 데이터 중 \*\*'비교에 사용할 특징(Features)'\*\*이 무엇인지 정의합니다. 예를 들어, 캐릭터의 현재 위치, 속도, 발의 위치, 혹은 미래의 이동 경로(Trajectory) 중 무엇을 중요하게 볼 것인지 결정합니다.

### **② 특징 벡터 생성 (Feature Vector Generation)**

애니메이션 에셋을 데이터베이스화할 때, 스키마에 정의된 채널들을 바탕으로 각 프레임의 데이터를 수치화된 '특징 벡터'로 변환합니다. 시스템은 런타임에 이 벡터들 사이의 거리를 계산하여 가장 유사한 포즈를 찾습니다.

### **③ 가중치 부여 (Weighting)**

모든 데이터가 똑같이 중요하지는 않습니다. 스키마 내에서 각 채널(발 위치, 골반 속도 등)에 \*\*가중치(Weight)\*\*를 설정할 수 있습니다.

* *예: "발의 정확한 위치보다 캐릭터의 미래 경로가 더 중요하다"면 경로 채널에 높은 가중치를 부여합니다.*

### **④ 샘플링 및 최적화**

어떤 뼈(Bone)를 추적할지, 샘플링 빈도는 어떻게 할지 등을 설정하여 검색 성능과 정확도 사이의 균형을 맞춥니다.

## **2\. 주요 구성 채널 (Key Channels)**

스키마 내에는 다양한 '채널'을 추가하여 검색 기준을 세밀하게 조정할 수 있습니다.

| 채널 타입 | 설명 |
| :---- | :---- |
| **Trajectory (궤적)** | 캐릭터가 앞으로 이동할 예상 경로(위치 및 방향)를 비교합니다. 모션 매칭의 가장 중요한 요소입니다. |
| **Pose (포즈/뼈)** | 특정 뼈(예: 왼발, 오른발, 골반)의 위치나 속도를 비교합니다. 발이 땅에 닿는 타이밍 등을 맞출 때 필수적입니다. |
| **Velocity (속도)** | 캐릭터의 현재 이동 속도와 회전 속도를 검색 기준으로 삼습니다. |
| **Heading (방향)** | 캐릭터가 바라보는 방향과 이동 방향 사이의 관계를 정의합니다. |

## **3\. 작업 흐름에서의 위치**

1. **Schema 생성**: Pose Search Schema 에셋을 생성하고 필요한 채널(경로, 발 위치 등)을 추가합니다.  
2. **Database 연결**: 생성한 스키마를 Pose Search Database에 할당합니다.  
3. **데이터 분석**: 데이터베이스는 스키마의 설정에 따라 애니메이션 시퀀스를 분석하여 인덱스를 생성합니다.  
4. **런타임 검색**: 애니메이션 블루프린트의 Motion Matching 노드가 현재 캐릭터 상태를 스키마 규격에 맞춰 변환한 뒤, 데이터베이스에서 가장 적합한 포즈를 호출합니다.

## **4\. 요약**

**Pose Search Schema**가 없다면 모션 매칭 시스템은 애니메이션의 어떤 부분을 비교해야 할지 알 수 없습니다.

* **정확도 조절**: 특정 신체 부위의 움직임을 강조하고 싶을 때 스키마를 수정합니다.  
* **반응성 조절**: 미래 경로(Trajectory)의 가중치를 높이면 조작에 더 민감하게 반응하는 애니메이션을 얻을 수 있습니다.  
* **성능 최적화**: 불필요한 뼈 데이터를 제외하여 검색 속도를 높일 수 있습니다.
