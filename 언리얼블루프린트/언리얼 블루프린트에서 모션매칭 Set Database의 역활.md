# **언리얼 엔진 모션 매칭: Set Database의 역할과 활용**

모션 매칭(Motion Matching)은 캐릭터의 현재 상태(포즈, 속도 등)와 미래의 경로(Trajectory)를 계산하여 가장 적합한 애니메이션 프레임을 실시간으로 찾아내는 기술입니다. 이때 **Set Database**는 검색의 기반이 되는 데이터 셋을 결정하는 핵심적인 함수입니다.

## **1\. 핵심 역할: 검색 범위의 동적 변경**

모션 매칭 엔진은 매 프레임마다 "지금 내 캐릭터에게 가장 잘 어울리는 동작이 무엇인가?"를 데이터베이스에서 검색합니다. **Set Database**를 호출하면 이 검색의 대상이 되는 '라이브러리'가 통째로 바뀝니다.

* **상태 전환 (Stance Switching):** 서 있는 상태의 데이터베이스에서 앉아 있는(Crouch) 상태의 데이터베이스로 전환합니다.  
* **이동 스타일 변경 (Locomotion Styles):** 평상시 걷기/뛰기 데이터베이스에서 부상을 입었거나 긴박한 전투 상황(Alert)의 데이터베이스로 교체합니다.  
* **무기 체계 대응:** 검을 들고 있을 때와 총을 들고 있을 때의 포즈 및 이동 방식이 다르므로, 장착된 무기에 맞는 데이터베이스를 설정합니다.

## **2\. 왜 여러 개의 데이터베이스를 사용하나요?**

하나의 거대한 데이터베이스에 모든 애니메이션을 넣지 않고 Set Database로 나누어 관리하는 이유는 다음과 같습니다.

### **① 검색 효율성 및 성능 최적화**

모션 매칭은 연산량이 많은 기술입니다. 수천 개의 포즈를 매번 다 검색하는 것보다, 현재 상황(예: 수영 중)에 필요한 데이터만 모아둔 작은 데이터베이스를 검색하는 것이 CPU 성능 면에서 훨씬 유리합니다.

### **② 정확도 향상 (Context Precision)**

데이터베이스가 섞여 있으면, 캐릭터가 걷고 있어야 하는 상황에서 뜬금없이 전투 포즈가 선택될 가능성이 있습니다. 상황별로 데이터베이스를 분리하면 해당 상황에 불필요한 동작이 튀어나오는 '노이즈'를 방지할 수 있습니다.

## **3\. 블루프린트에서의 구현 흐름**

보통 **Pose Search Component**를 가진 액터나 애니메이션 블루프린트(AnimBP)에서 호출됩니다.

1. **컴포넌트 참조:** 캐릭터에 부착된 Pose Search Component를 가져옵니다.  
2. **노드 호출:** Set Database 함수를 실행합니다.  
3. **데이터 입력:** 미리 생성해 둔 Pose Search Database 에셋을 입력 핀에 연결합니다.  
4. **결과 반영:** 애니메이션 그래프(AnimGraph) 내의 Motion Matching 노드는 이제 새롭게 설정된 데이터베이스를 기준으로 포즈를 계산합니다.

## **4\. 실전 활용 예시**

| 상황 | 이전 데이터베이스 | Set Database 호출 후 |
| :---- | :---- | :---- |
| **비전투 → 전투** | DB\_Casual\_Walk | DB\_Warrior\_Stance |
| **맨손 → 방패 장착** | DB\_Unarmed | DB\_Shield\_Block |
| **지면 → 물속** | DB\_Land\_Movement | DB\_Swimming |

## **5\. 주의사항**

* **블렌딩(Blending):** 데이터베이스를 갑자기 바꾸면 포즈가 툭 튀는(Popping) 현상이 발생할 수 있습니다. 이를 방지하기 위해 애니메이션 그래프에서 모션 매칭 노드의 **Blending 설정**이나 **Inertialization(관성 블렌딩)** 노드를 함께 사용하여 부드럽게 연결해야 합니다.  
* **데이터 에셋 구성:** Set Database를 쓰기 전에, 각 상황에 맞는 애니메이션들이 Pose Search Database 에셋 안에 적절한 태그와 함께 잘 인덱싱되어 있어야 합니다.

### **요약**

**Set Database**는 캐릭터의 \*\*'행동 맥락(Context)'\*\*을 시스템에 알려주는 스위치입니다. 플레이어의 입력이나 게임 로직에 따라 적절한 애니메이션 데이터 셋을 연결해 줌으로써, 훨씬 자연스럽고 지능적인 캐릭터 움직임을 가능하게 합니다.
