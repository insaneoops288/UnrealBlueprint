# **언리얼 엔진 모션 매칭(Motion Matching) 시스템의 역할과 구조**

언리얼 엔진의 모션 매칭은 캐릭터의 현재 상태(포즈, 속도)와 미래의 예측 경로(트래젝토리)를 분석하여, 수많은 애니메이션 데이터 중 가장 적합한 프레임을 실시간으로 찾아 재생하는 **데이터 중심(Data-Driven) 애니메이션 시스템**입니다.

## **1\. 모션 매칭의 핵심 역할**

### **① 복잡한 스테이트 머신의 대체**

기존에는 '걷기 → 달리기', '달리기 → 정지' 등의 전환을 위해 수백 개의 노드와 조건(Transition Rule)을 연결해야 했습니다. 모션 매칭은 이러한 복잡한 로직 없이, 데이터베이스 내에서 가장 자연스러운 연결 지점을 스스로 찾아내어 애니메이터의 작업 시간을 획기적으로 줄여줍니다.

### **② 극도의 애니메이션 정밀도와 반응성**

사용자의 입력(조이스틱 이동 등)에 따라 캐릭터가 즉각적으로 반응하면서도, 발의 미끄러짐(Foot Sliding)이나 부자연스러운 끊김 없이 물리적으로 올바른 애니메이션을 출력합니다.

### **③ 트래젝토리(Trajectory) 기반 예측**

캐릭터가 1초 뒤에 어디에 있을지, 어떤 방향으로 몸을 틀지 예측하는 '경로선'을 기준으로 애니메이션을 선택하므로, 급격한 방향 전환이나 감속 시에도 매우 사실적인 움직임을 보장합니다.

## **2\. 블루프린트에서의 주요 구성 요소 및 상호작용**

모션 매칭을 구현하기 위해 블루프린트에서 제어하는 주요 요소는 다음과 같습니다.

### **1\) 모션 데이터베이스 (Motion Database)**

* **역할:** 사용 가능한 모든 애니메이션 시퀀스나 컴포지트를 저장하는 창고입니다.  
* **블루프린트 연동:** 개발자는 블루프린트에서 상황에 따라 다른 데이터베이스(예: 전투용, 탐험용, 부상 상태용)를 교체하거나 필터링할 수 있습니다.

### **2\) 캐릭터 트래젝토리 컴포넌트 (Character Trajectory Component)**

* **역할:** 액터의 이동 컴포넌트(Character Movement)로부터 속도와 방향 데이터를 받아 미래의 경로를 계산합니다.  
* **블루프린트 제어:** Get Character Trajectory 노드를 통해 현재 경로 데이터를 애니메이션 블루프린트로 전달합니다.

### **3\) 포즈 서치 노드 (Pose Search Node)**

* **역할:** 애니메이션 블루프린트(AnimBP)의 애님 그래프(AnimGraph) 내에서 작동하며, 실시간으로 데이터베이스를 쿼리(Query)합니다.  
* **작동 방식:** 현재 캐릭터의 포즈 정보와 트래젝토리 컴포넌트의 예측값을 비교하여 최적의 애니메이션 지점을 결정합니다.

## **3\. 기존 시스템(State Machine) vs 모션 매칭**

| 비교 항목 | 스테이트 머신 (State Machine) | 모션 매칭 (Motion Matching) |
| :---- | :---- | :---- |
| **제어 방식** | 로직 중심 (If/Else 조건문 기반) | 데이터 중심 (포즈 및 경로 비교) |
| **구현 난이도** | 애니메이션이 많아질수록 기하급수적으로 복잡해짐 | 데이터베이스 세팅이 중요하며 그래프는 단순함 |
| **비주얼 품질** | 블렌딩 지점이 정해져 있어 다소 경직됨 | 모든 프레임이 후보군이 되어 매우 부드러움 |
| **메모리 사용** | 낮음 | 상대적으로 높음 (포즈 인덱싱 데이터 필요) |

## **4\. 실무 블루프린트 워크플로우**

1. **데이터 준비:** 애니메이션 시퀀스를 Pose Search Database에 등록합니다.  
2. **컴포넌트 추가:** 캐릭터 블루프린트에 Character Trajectory Component를 추가합니다.  
3. **애님 블루프린트 설정:**  
   * Update Animation에서 트래젝토리 데이터를 변수로 저장합니다.  
   * AnimGraph에서 Motion Matching 또는 Pose Search 노드를 배치합니다.  
4. **튜닝:** 블루프린트에서 Pose Search Search Filter 등을 사용하여 특정 상황(예: 실내에서는 뛰지 않기)에서 검색 범위를 제한합니다.

## **5\. 결론: 언제 사용해야 하는가?**

모션 매칭은 **AAA급 퀄리티의 3인칭 액션 게임이나 오픈월드 게임**처럼 캐릭터의 움직임이 빈번하고 자연스러움이 생명인 프로젝트에 필수적입니다. 반면, 간단한 2D 게임이나 애니메이션 종류가 매우 적은 경우에는 기존의 스테이트 머신 방식이 더 효율적일 수 있습니다.
