# **Unreal Engine Blueprint: Last Index 노드 이해하기**

언리얼 블루프린트에서 **Last Index** 노드는 배열(Array) 작업을 할 때 가장 자주 사용되는 노드 중 하나입니다. 이 노드의 핵심 개념과 활용 사례를 정리했습니다.

## **1\. 주요 역할**

Last Index 노드는 대상 배열에서 **현재 마지막 아이템이 위치한 정수(Integer) 값**을 반환합니다.

### **핵심 공식**

배열의 인덱스는 항상 **0**부터 시작하기 때문에 다음과 같은 수학적 관계를 가집니다:

**Last Index \= 배열의 전체 길이(Length) \- 1**

예를 들어, 배열에 아이템이 5개 들어있다면:

* **Length**: 5  
* **Last Index**: 4 (0, 1, 2, 3, 4 순서이므로)

## **2\. 왜 사용하는가? (활용 사례)**

### **① 마지막에 추가된 요소 접근**

배열에 방금 Add 노드로 추가된 최신 항목을 즉시 가져와서 처리하고 싶을 때 사용합니다. Get 노드에 Last Index를 연결하면 항상 마지막 요소를 가리킵니다.

### **② For Loop (반복문) 제어**

특정 배열을 처음부터 끝까지 순회할 때 For Loop 노드의 **Last Index** 핀에 이 노드를 연결합니다.

* First Index: 0  
* Last Index: Last Index 노드 결과값

### **③ 배열 비우기 또는 마지막 요소 제거**

배열의 가장 뒤에 있는 요소를 지우고 싶을 때 Remove Index 노드와 결합하여 사용합니다.

### **④ 유효성 검사 (배열이 비었는지 확인)**

배열에 아무런 요소가 없을 때:

* Length는 **0**을 반환합니다.  
* Last Index는 **\-1**을 반환합니다.  
  이를 통해 배열이 비어있는지 로직상으로 체크할 수 있습니다.

## **3\. Length 노드와의 차이점 비교**

| 기능 | Length 노드 | Last Index 노드 |
| :---- | :---- | :---- |
| **반환 값** | 배열 내 실제 아이템의 개수 | 마지막 아이템의 주소(번호) |
| **시작 기준** | 1부터 카운트 (비어있으면 0\) | 0부터 카운트 (비어있으면 \-1) |
| **주 용도** | 개수 파악, 데이터 존재 유무 확인 | 특정 위치 접근, 루프 범위 지정 |

## **4\. 실전 팁**

* **Out of Bounds 에러 방지**: 배열에서 요소를 찾을 때 Length 값을 그대로 인덱스에 넣으면 "Index Out of Bounds" 에러가 발생합니다. (5개 배열에서 5번 인덱스를 찾으라고 하면 없기 때문). 이때 반드시 Last Index를 사용하거나 Length \- 1을 사용해야 합니다.  
* **역순 순회**: 배열의 끝에서부터 처음으로 거꾸로 검사해야 할 때(예: 요소를 지우면서 진행할 때), For Loop의 Start에 Last Index를 넣고 End에 0을 넣은 뒤 역순으로 돌릴 수 있습니다.
