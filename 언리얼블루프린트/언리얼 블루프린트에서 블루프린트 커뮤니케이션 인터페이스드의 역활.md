# **언리얼 엔진: 블루프린트 인터페이스(BPI)의 역할과 활용**

블루프린트 인터페이스는 클래스 간의 **유연한 통신**을 가능하게 하는 강력한 도구입니다. 특정 블루프린트가 어떤 클래스인지 정확히 알 필요 없이, "이 인터페이스를 가지고 있는가?"만 확인하고 명령을 내릴 수 있습니다.

## **1\. 핵심 역할**

### **① 디커플링 (Decoupling, 의존성 제거)**

가장 중요한 역할입니다. 일반적인 '형변환(Casting)' 방식을 사용하면 호출하는 블루프린트가 호출받는 블루프린트의 정보를 모두 알고 있어야 합니다(Hard Reference). 하지만 인터페이스를 쓰면 상대방이 누구인지 몰라도 함수를 던질 수 있어 프로젝트 구조가 가벼워집니다.

### **② 다형성 구현 (Polymorphism)**

하나의 인터페이스 함수를 여러 블루프린트에서 각기 다르게 구현할 수 있습니다.

* **예시:** Interact 인터페이스 함수  
  * **문(Door):** 문을 연다.  
  * **아이템(Item):** 인벤토리에 추가한다.  
  * **NPC:** 대화를 시작한다.  
  * **결과:** 플레이어는 대상이 무엇이든 상관없이 그냥 Interact(Message) 노드만 실행하면 됩니다.

### **③ 다중 상속의 대안**

블루프린트는 하나의 부모 클래스만 가질 수 있지만, 인터페이스는 한 블루프린트에 **여러 개를 추가**할 수 있습니다. 이를 통해 다양한 성격의 기능을 한 액터에게 부여할 수 있습니다.

## **2\. 왜 Casting(형변환) 대신 Interface를 쓰나요?**

| 특징 | Casting (형변환) | Blueprint Interface |
| :---- | :---- | :---- |
| **참조 방식** | 강한 참조 (Hard Reference) | 약한 참조 (Decoupling) |
| **메모리** | 참조된 모든 블루프린트를 메모리에 로드함 | 필요한 함수 규격만 확인하므로 가벼움 |
| **확장성** | 새로운 클래스가 추가될 때마다 캐스팅 노드 추가 필요 | 인터페이스만 추가하면 즉시 작동 |
| **안정성** | 대상이 해당 클래스가 아니면 Fail 발생 | 대상이 인터페이스를 구현하지 않았다면 그냥 무시됨 (Crash 방지) |

## **3\. 주요 사용 사례**

### **상호작용 시스템 (Interaction System)**

플레이어가 바라보는 물체가 무엇이든 '상호작용' 키를 눌렀을 때 반응하게 하고 싶을 때 사용합니다. BPI\_Interact라는 인터페이스를 만들고, 반응할 모든 액터의 Class Settings에서 이를 추가합니다.

### **데미지 시스템 (Damage System)**

엔진 기본 데미지 시스템 외에 커스텀 시스템을 만들 때 유용합니다. 공격자가 피격자가 정확히 누구인지(벽인지, 좀비인지, 플레이어인지) 몰라도 OnTakeDamage 메시지를 보낼 수 있습니다.

### **게임 상태 알림**

특정 이벤트(예: 게임 오버, 스테이지 클리어)가 발생했을 때, 해당 이벤트에 반응해야 하는 여러 액터들에게 동시에 신호를 보낼 때 유용합니다.

## **4\. 사용 방법 요약**

1. **인터페이스 생성:** 콘텐츠 브라우저에서 우클릭 \> 블루프린트 \> 블루프린트 인터페이스 선택.  
2. **함수 정의:** 인터페이스 내부에서 이름과 입력/출력 변수만 설정 (로직은 짜지 않음).  
3. **클래스 적용:** 대상 블루프린트의 Class Settings \-\> Interfaces 탭에서 생성한 인터페이스 추가.  
4. **로직 구현:** 해당 블루프린트의 Interfaces 섹션에 나타난 함수를 우클릭 \> 이벤트 구현하여 실제 동작 작성.  
5. **호출:** 다른 블루프린트에서 함수 이름 (Message) 노드를 사용하여 신호 전달.

## **요약**

블루프린트 인터페이스는 \*\*"서로 모르는 사이에도 대화할 수 있게 해주는 공용 통역기"\*\*입니다. 프로젝트가 커질수록 복잡한 캐스팅 구조를 피하고 유지보수를 쉽게 하기 위해 필수적으로 사용해야 하는 기능입니다.
