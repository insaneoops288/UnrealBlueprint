# **언리얼 엔진 블루프린트: Atan 및 Atan2 노드 가이드**

언리얼 엔진에서 Atan(Arctangent) 노드는 수학적인 탄젠트 값을 다시 \*\*각도(Angle)\*\*로 변환할 때 사용합니다. 주로 게임 개발에서는 두 지점 사이의 회전 값을 구하거나, 캐릭터가 특정 방향을 바라보게 할 때 필수적으로 사용됩니다.

## **1\. Atan vs Atan2 (가장 중요한 차이점)**

블루프린트에는 두 가지 주요 아크탄젠트 노드가 있습니다.

### **Atan (1개 입력)**

* **입력:** 단일 부동 소수점(Float) 값.  
* **출력:** \-90도 \~ 90도 (또는 \-π/2 \~ π/2 라디안) 사이의 값.  
* **특징:** 기울기 값만 알 때 사용하지만, 사분면(방향)을 구분하지 못해 게임 로직에서는 단독으로 쓰이는 경우가 드뭅니다.

### **Atan2 (2개 입력: Y, X) ⭐ 권장**

* **입력:** Y값과 X값 두 개를 따로 받습니다.  
* **출력:** \-180도 \~ 180도 (또는 \-π \~ π 라디안) 사이의 값.  
* **특징:** X와 Y의 부호를 모두 확인하므로 **360도 전 방향**을 정확히 계산할 수 있습니다. 벡터나 좌표를 각도로 변환할 때 표준으로 사용됩니다.

## **2\. 주요 활용 사례**

### **① 마우스 위치로 캐릭터 회전시키기 (Top-down 게임)**

탑다운 뷰에서 캐릭터가 마우스 커서 방향을 바라보게 하려면 Atan2가 필요합니다.

1. **목표 위치(마우스)** \- \*\*현재 위치(캐릭터)\*\*를 계산하여 상대적인 X, Y 거리를 구합니다.  
2. Atan2 (Degrees) 노드에 Y와 X를 입력합니다.  
3. 결과값(Degree)을 Set Actor Rotation의 **Z(Yaw)** 값에 연결합니다.

### **② 조이스틱 입력값을 각도로 변환**

게임패드의 아날로그 스틱 입력(Axis Config)은 X축과 Y축 값(-1.0 \~ 1.0)으로 들어옵니다.

* Atan2 노드에 스틱의 Y축, X축 값을 넣으면 플레이어가 스틱을 어느 방향(각도)으로 밀고 있는지 즉시 알 수 있습니다.

### **③ 두 벡터 사이의 각도 계산**

공중에 떠 있는 화살이 이동 방향(Velocity)에 맞춰 머리 방향을 바꾸어야 할 때, 해당 벡터의 성분을 분리하여 각도를 추출합니다.

## **3\. 블루프린트 사용 팁**

* **Degrees vs Radians:** 언리얼은 Atan2 (Degrees)와 Atan2 (Radians)를 별도로 제공합니다. Set Actor Rotation 같은 일반적인 회전 노드는 **Degree(도)** 단위를 사용하므로 Atan2 (Degrees)를 사용하는 것이 편리합니다.  
* **입력 순서 주의:** 수학적 정의에 따라 입력 핀이 **Y가 위, X가 아래**인 경우가 많습니다. 노드를 연결할 때 X와 Y가 바뀌지 않도록 주의하세요. (바뀌면 각도가 90도 돌아가게 됩니다.)

## **4\. 노드 요약**

| 노드 이름 | 입력 | 주요 용도 |
| :---- | :---- | :---- |
| **Atan** | Float (Value) | 단순 기울기 값의 역함수 계산 |
| **Atan2 (Degrees)** | Y, X (Float) | **좌표/벡터를 각도로 변환 (가장 많이 사용)** |
| **Atan2 (Radians)** | Y, X (Float) | 수학 공식이나 복잡한 계산식 내부용 |

**결론:** 두 지점 사이의 방향이나 캐릭터가 바라볼 각도를 계산하고 싶다면 고민하지 말고 **Atan2 (Degrees)** 노드를 사용하시면 됩니다.
