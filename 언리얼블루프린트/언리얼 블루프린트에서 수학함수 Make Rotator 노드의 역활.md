# **언리얼 블루프린트: Make Rotator 노드 완벽 이해하기**

언리얼 엔진에서 객체의 방향을 제어할 때 가장 기본이 되는 노드 중 하나가 바로 **Make Rotator**입니다. 이 노드는 세 가지 축의 회전 각도를 입력받아 하나의 회전 데이터(Rotator)로 묶어주는 역할을 합니다.

## **1\. 노드의 구성 요소**

Make Rotator 노드는 다음과 같은 3개의 입력 핀과 1개의 출력 핀으로 구성됩니다.

### **입력 (Inputs \- Float)**

* **X (Roll):** X축을 중심으로 하는 회전입니다. 비행기가 좌우로 기울어지는 동작을 상상하면 쉽습니다.  
* **Y (Pitch):** Y축을 중심으로 하는 회전입니다. 비행기가 기수를 위로 들거나 아래로 숙이는 동작입니다.  
* **Z (Yaw):** Z축을 중심으로 하는 회전입니다. 비행기가 왼쪽이나 오른쪽으로 방향을 트는(좌우 회전) 동작입니다.

### **출력 (Output \- Rotator)**

* **Return Value:** 입력된 X, Y, Z 값을 기반으로 생성된 **Rotator 구조체**입니다. 이 값은 Set Actor Rotation, Add Local Rotation 등 회전값을 필요로 하는 다른 노드에 연결하여 사용합니다.

## **2\. 주요 사용 사례**

### **① 특정 방향으로 액터 회전시키기**

가장 흔한 용도는 액터의 회전값을 수동으로 계산하여 설정할 때입니다. 예를 들어, 게임 중 특정 이벤트가 발생했을 때 캐릭터가 항상 정면(Yaw 0도)을 보게 하고 싶다면, Yaw 값에 0을 입력한 Make Rotator를 Set Actor Rotation에 연결합니다.

### **② 카메라 흔들림 또는 틸트 구현**

플레이어가 달릴 때 카메라가 좌우로 살짝 기우는 효과를 주고 싶다면, Timeline 노드에서 시간에 따라 변하는 값을 생성한 후, 이를 Make Rotator의 **Roll** 값에 연결하여 카메라의 회전값에 더해줄 수 있습니다.

### **③ 마우스 입력 처리**

1인칭 게임에서 마우스를 위아래로 움직일 때(Pitch)와 좌우로 움직일 때(Yaw)의 입력값을 받아 각각 해당 축에 매핑하여 카메라의 방향을 결정할 때 필수적으로 사용됩니다.

## **3\. 주의사항 및 팁**

1. **회전 순서:** 언리얼 엔진의 Rotator는 내부적으로 **Pitch \-\> Yaw \-\> Roll** 순서로 계산됩니다. 이를 'Gimbal Lock' 현상과 관련하여 이해해 두면 복잡한 3D 회전을 다룰 때 도움이 됩니다.  
2. **Break Rotator와 짝꿍:** 생성된 Rotator에서 특정 축의 값만 추출하고 싶을 때는 반대 역할을 하는 **Break Rotator** 노드를 사용합니다.  
3. **단위:** 입력되는 Float 값의 단위는 \*\*도(Degrees)\*\*입니다. (라디안이 아님에 주의하세요.)

## **요약**

**Make Rotator**는 "수치(Float)를 회전 데이터(Rotator)로 바꾸는 변환기"라고 생각하면 됩니다. 3D 공간에서 사물의 방향을 세밀하게 제어하기 위해 반드시 익혀야 하는 노드입니다.
