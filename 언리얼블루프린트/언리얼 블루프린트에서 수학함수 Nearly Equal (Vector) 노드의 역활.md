# **Nearly Equal (Vector) 노드 상세 가이드**

언리얼 엔진 블루프린트에서 Nearly Equal (Vector) 노드는 두 개의 벡터 값이 설정된 오차 범위 내에 있는지를 비교하여 Boolean (True/False) 값을 반환하는 함수입니다.

## **1\. 왜 사용하는가? (부동 소수점 오차)**

컴퓨터 과학에서 소수점(Floating Point) 계산은 완벽하게 정확하지 않을 때가 많습니다. 예를 들어, 수학적으로는 1.0이어야 할 값이 계산 과정에서 0.999999 또는 1.000001이 될 수 있습니다.

단순하게 Equal (==) 노드를 사용하면 이 미세한 차이 때문에 결과가 False로 나오게 되어, 게임 로직(예: 목적지 도착 판정)이 제대로 작동하지 않는 문제가 발생합니다. Nearly Equal은 이 문제를 해결하기 위해 \*\*허용 오차(Tolerance)\*\*를 제공합니다.

## **2\. 노드의 구성 요소**

| 입력/출력 | 이름 | 타입 | 설명 |
| :---- | :---- | :---- | :---- |
| **Input** | **A** | Vector | 비교할 첫 번째 벡터 값입니다. |
| **Input** | **B** | Vector | 비교할 두 번째 벡터 값입니다. |
| **Input** | **Error Tolerance** | Float | 허용할 오차의 범위입니다. (기본값은 매우 작음) |
| **Output** | **Return Value** | Boolean | 두 벡터 사이의 거리가 오차 범위 이내면 True, 아니면 False를 반환합니다. |

## **3\. 작동 원리**

이 노드는 내부적으로 두 벡터 간의 \*\*거리(Distance)\*\*를 계산합니다.

![][image1]그 후, 계산된 거리가 사용자가 설정한 Error Tolerance보다 작거나 같은지를 확인합니다.

## **![][image2]4\. 주요 활용 사례**

1. **AI 이동 목적지 확인**: AI가 특정 위치(벡터)에 도착했는지 확인할 때 사용합니다. 정확히 0, 0, 0 지점에 멈추는 것은 어렵기 때문에 약 5.0 정도의 오차 범위를 주어 도착 판정을 내립니다.  
2. **발사체 충돌 또는 근접 판정**: 투사체가 특정 지점에 충분히 근접했는지 체크할 때 유용합니다.  
3. **데이터 동기화**: 네트워크를 통해 받은 캐릭터의 위치 값이 이전 값과 "거의 변화가 없는지" 체크하여 불필요한 업데이트를 방지할 때 사용합니다.

## **5\. 팁: 적절한 Tolerance 값**

* **정밀한 계산**: 0.0001 (기본값에 가까운 값)  
* **일반적인 위치 판정**: 1.0 \~ 10.0 (캐릭터 크기나 속도에 따라 조절)  
* **회전/방향**: 벡터를 방향(Unit Vector)으로 사용할 때는 매우 작은 값을 사용해야 합니다.

## **6\. 요약**

Nearly Equal (Vector)는 \*\*"두 위치가 실질적으로 같은 위치라고 볼 수 있는가?"\*\*를 판단하는 노드입니다. 단순한 Equal 노드보다 게임 제작 환경에서 훨씬 더 안정적이고 자주 사용됩니다.
