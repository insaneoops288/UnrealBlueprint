# **Nearly Equal (Float) 노드 상세 가이드**

언리얼 엔진 블루프린트에서 Nearly Equal (Float) 노드는 두 Float(부동 소수점) 값을 비교하여, 두 값이 설정된 오차 범위(Tolerance) 내에 있는지 확인하고 그 결과를 Boolean(True/False)으로 반환합니다.

## **1\. 왜 이 노드가 필요한가요? (부동 소수점의 한계)**

컴퓨터 과학에서 실수(Float)는 이진수로 표현되므로 **정밀도 문제**가 발생합니다.

예를 들어, 수학적으로는 0.1 \+ 0.2가 0.3이지만, 컴퓨터 연산에서는 0.300000012...와 같은 미세한 차이가 발생할 수 있습니다.

따라서 일반적인 Equal (==) 노드를 사용하여 두 실수를 비교하면, 눈으로 보기엔 같아 보여도 결과가 False로 나올 가능성이 매우 높습니다.

## **2\. 노드 파라미터 구성**

| 입력 항목 | 데이터 타입 | 설명 |
| :---- | :---- | :---- |
| **A** | Float | 비교할 첫 번째 숫자입니다. |
| **B** | Float | 비교할 두 번째 숫자입니다. |
| **Error Tolerance** | Float | 허용할 오차 범위입니다. (두 값의 차이가 이 값보다 작거나 같으면 True) |

## **3\. 작동 원리 (수식)**

내부적으로는 다음과 같은 논리로 작동합니다.

abs(A \- B) \<= Error Tolerance

* 만약 A가 1.0이고 B가 1.001이며, Error Tolerance가 0.01이라면 결과는 **True**입니다.  
* 만약 동일한 상황에서 Error Tolerance가 0.0001이라면 결과는 **False**입니다.

## **4\. 주요 사용 사례**

### **① 캐릭터 위치/이동 확인**

캐릭터가 특정 좌표에 도착했는지 확인할 때 사용합니다. 캐릭터의 현재 위치 좌표가 정확히 0.0으로 떨어지는 경우는 거의 없으므로, 약 1.0 정도의 허용 오차를 주고 확인하는 것이 안전합니다.

### **② 물리 엔진 기반의 속도 체크**

물체가 멈췄는지 판단할 때 Velocity가 정확히 0인지를 체크하는 대신, Nearly Equal을 사용하여 0.1 이하의 속도면 멈춘 것으로 간주하게 처리합니다.

### **③ 타이머 및 게이지 비교**

진행률(0.0 \~ 1.0)이 끝에 도달했는지 확인할 때 유용합니다.

## **5\. 실무 팁**

* **기본값 활용**: 보통 아주 정밀한 계산이 아니라면 기본값인 0.0001 정도로도 충분합니다.  
* **벡터 비교**: Nearly Equal (Vector) 노드도 존재합니다. 이는 X, Y, Z 각각의 좌표를 오차 범위 내에서 한꺼번에 비교해 줍니다.  
* **\== 노드 지양**: 실수를 비교할 때는 습관적으로 Nearly Equal을 사용하는 것이 버그를 줄이는 지름길입니다.
