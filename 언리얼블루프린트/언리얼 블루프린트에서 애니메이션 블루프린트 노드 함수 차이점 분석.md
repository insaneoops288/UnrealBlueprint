# **애니메이션 블루프린트 노드 함수 차이점 분석**

애니메이션 블루프린트(AnimBP)의 각 노드(스테이트, 블렌드 노드 등)에서 설정할 수 있는 이 함수들은 \*\*연관성(Relevance)\*\*이라는 개념을 바탕으로 작동합니다. 연관성이란 해당 노드가 최종 포즈 출력에 기여하고 있는 상태를 의미합니다.

## **1\. On Initial Update (초기 업데이트 시)**

애니메이션 인스턴스가 생성된 후, 해당 노드가 **생애 처음으로** 연관성을 가질 때 딱 한 번 실행됩니다.

* **실행 횟수:** 1회 (게임 플레이 중 해당 노드가 처음 활성화될 때)  
* **실행 타이밍:** On Become Relevant 바로 직전에 실행됩니다.  
* **주요 용도:** \- 컴포넌트 레퍼런스 확보 (예: Get Pawn Owner 후 변수 저장)  
  * 스테틱한(변하지 않는) 값 초기화  
  * 비용이 높은 일회성 계산

## **2\. On Become Relevant (연관성이 생길 시)**

해당 노드가 활성화되어 포즈 계산에 참여하기 시작할 때마다 실행됩니다. 스테이트 머신에서 특정 스테이트로 **진입할 때마다** 호출된다고 이해하면 쉽습니다.

* **실행 횟수:** 해당 노드가 활성화될 때마다 (여러 번 호출 가능)  
* **실행 타이밍:** 노드가 활성화되는 순간, On Initial Update 이후에 실행됩니다.  
* **주요 용도:**  
  * 스테이트 진입 시 변수 리셋 (예: 경과 시간 0으로 초기화)  
  * 특정 스테이트 진입 시 사운드나 파티클 트리거  
  * 노드가 활성화될 때만 필요한 데이터 로드

## **3\. On Update (업데이트 시)**

해당 노드가 연관성을 유지하고 있는 동안(활성화된 상태 동안) **매 프레임** 실행됩니다.

* **실행 횟수:** 매 프레임 (노드가 활성화된 동안 지속적으로)  
* **실행 타이밍:** On Become Relevant가 호출된 이후부터 노드가 비활성화되기 전까지 매 업데이트 틱마다 실행됩니다.  
* **주요 용도:**  
  * 시간에 따라 변하는 파라미터 계산 (예: DeltaTime을 이용한 보간)  
  * 캐릭터의 현재 속도나 상태를 실시간으로 노드 변수에 반영  
  * 복잡한 수학 연산을 이벤트 그래프가 아닌 해당 노드 내에서 로컬하게 처리

## **💡 요약 비교표**

| 특성 | On Initial Update | On Become Relevant | On Update |
| :---- | :---- | :---- | :---- |
| **호출 빈도** | 전체 생애 중 **단 1회** | 노드 활성화 시마다 | 노드 활성 중 **매 프레임** |
| **주요 목적** | 초기화 및 레퍼런스 저장 | 상태 진입 로직 처리 | 실시간 파라미터 업데이트 |
| **비용(성능)** | 매우 낮음 | 낮음 | 상대적으로 높음 (매 프레임) |
| **비유** | "처음 만났을 때 인사" | "방에 들어올 때마다 인사" | "방 안에 있는 동안 대화" |

## **🛠️ 실전 활용 팁 (Optimization)**

1. **Thread Safe 권장:** 이 함수들은 기본적으로 \*\*쓰레드 안전(Thread Safe)\*\*하게 작성하는 것이 좋습니다. 이를 통해 애니메이션 업데이트를 워커 쓰레드(Worker Thread)에서 병렬로 처리하여 게임 성능(FPS)을 높일 수 있습니다.  
2. **이벤트 그래프 다이어트:** 과거에는 모든 로직을 Blueprint Update Animation 이벤트 그래프에 몰아넣었지만, 이제는 각 스테이트에 맞는 로직을 On Update 등으로 분산시켜 가독성과 관리 효율을 높이세요.  
3. **Property Access 활용:** 함수 내부에서 Get Pawn Owner 등을 직접 호출하기보다, 노드의 디테일 패널에서 **Property Access** 기능을 사용하여 캐릭터의 변수에 직접 바인딩하는 것이 성능상 가장 유리합니다.
