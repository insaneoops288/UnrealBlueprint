# **언리얼 엔진 블루프린트: 이벤트 디스패처(Event Dispatcher) 가이드**

이벤트 디스패처는 한 액터가 특정 사건(이벤트)이 발생했음을 알리면, 그 소식을 듣고자 대기(구독)하고 있던 다른 액터들이 즉각적으로 반응하게 만드는 통신 방식입니다. 이는 소프트웨어 디자인 패턴 중 **옵저버(Observer) 패턴**을 구현한 것입니다.

### **1\. 주요 역할: "느슨한 결합(Loose Coupling)"**

이벤트 디스패처의 가장 큰 역할은 클래스 간의 의존성을 줄이는 것입니다.

* **직접 참조 방식:** A가 B에게 명령을 내리려면 A는 B가 누구인지 정확히 알아야 합니다. (강한 결합)  
* **이벤트 디스패처 방식:** A는 "나 죽었어\!"라고 외치기만 합니다. 누가 그 소리를 듣고 무엇을 할지는 A가 신경 쓰지 않습니다. (느슨한 결합)

### **2\. 작동 프로세스**

이벤트 디스패처는 보통 다음과 같은 3단계로 작동합니다.

1. **디스패처 생성 (Create):** 특정 액터(예: 보스 몬스터) 안에 디스패처 변수를 만듭니다.  
2. **바인드 (Bind):** 이 소식을 듣고 싶어 하는 다른 액터(예: UI, 문, 사운드 매니저)들이 게임 시작 시 해당 디스패처에 자신들의 커스텀 이벤트를 연결합니다.  
3. **호출 (Call):** 사건이 발생한 순간(예: 보스 체력이 0이 됨), 보스 액터가 디스패처를 Call 합니다. 그러면 바인드되어 있던 모든 이벤트가 동시에 실행됩니다.

### **3\. 주요 기능(노드) 종류**

| 기능 | 설명 |
| :---- | :---- |
| **Call** | 등록된 모든 바인딩된 이벤트들에게 신호를 보냅니다. |
| **Bind** | 특정 이벤트를 디스패처에 등록합니다. (구독 시작) |
| **Unbind** | 등록된 이벤트를 해제합니다. (구독 해지) |
| **Unbind All** | 해당 디스패처에 연결된 모든 이벤트를 한꺼번에 제거합니다. |
| **Assign** | 바인드 노드와 커스텀 이벤트 생성 노드를 한 번에 만들어 줍니다. |

### **4\. 실전 활용 예시**

**상황: 플레이어가 레벨에 있는 '레버'를 당겼을 때**

* **레버 액터 (발신자):** OnLeverPulled라는 이름의 이벤트 디스패처를 가집니다. 플레이어가 클릭하면 이 디스패처를 Call 합니다.  
* **문 액터 (수신자 1):** BeginPlay에서 레버의 OnLeverPulled에 '문 열기' 이벤트를 바인드합니다.  
* **조명 액터 (수신자 2):** BeginPlay에서 레버의 OnLeverPulled에 '불 켜기' 이벤트를 바인드합니다.  
* **결과:** 레버 액터는 문이나 조명이 있는지 몰라도 됩니다. 그냥 신호만 보내면 세상의 모든 바인드된 객체들이 알아서 반응합니다.

### **5\. 왜 이벤트 디스패처를 써야 할까요?**

1. **성능 최적화:** Tick을 사용하여 매 프레임마다 "문이 열렸나?" 확인하는 것보다, 신호가 올 때만 실행되므로 훨씬 효율적입니다.  
2. **유지보수 용이:** 새로운 기능을 추가할 때 기존 코드를 수정할 필요 없이, 새 액터를 만들어 이벤트에 바인드만 하면 됩니다.  
3. **순환 참조 방지:** 두 액터가 서로를 참조하여 발생하는 메모리 문제나 복잡한 의존성 문제를 해결할 수 있습니다.

### **요약**

이벤트 디스패처는 \*\*"누가 나를 지켜보고 있는지는 모르겠지만, 방금 이 일이 일어났어\!"\*\*라고 공표하는 시스템입니다. 이를 통해 복잡한 프로젝트에서도 깔끔하고 확장성 있는 설계를 할 수 있습니다.
