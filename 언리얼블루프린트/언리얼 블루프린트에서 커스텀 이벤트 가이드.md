# **언리얼 엔진 블루프린트 커스텀 이벤트 가이드**

커스텀 이벤트(Custom Event)는 블루프린트 로직을 구조화하고, 특정 기능을 재사용하거나 다른 블루프린트에서 호출할 때 사용하는 핵심 기능입니다.

## **1\. 커스텀 이벤트 생성하기**

1. **그래프에서 우클릭**: 이벤트 그래프의 빈 공간에서 마우스 오른쪽 버튼을 클릭합니다.  
2. **검색**: Add Custom Event...를 입력하고 선택합니다.  
3. **이름 지정**: 이벤트의 이름을 입력합니다 (예: StartGame, DamagePlayer).  
4. **로직 연결**: 생성된 노드 뒤에 실행하고 싶은 로직(노드들)을 연결합니다.

## **2\. 동일한 블루프린트 내에서 호출하기**

자신이 만든 이벤트를 같은 블루프린트 안에서 호출하는 방법은 매우 간단합니다.

1. **노드 호출**: 그래프 빈 곳에 방금 만든 이벤트의 이름을 검색합니다.  
2. **함수 형태 선택**: Call \[이벤트 이름\]이라는 항목이 나타납니다. 이를 선택하면 실행 핀이 달린 호출 노드가 생성됩니다.  
3. **실행**: 특정 조건(예: BeginPlay, Tick, 버튼 클릭 등)이 만족될 때 해당 호출 노드를 연결하면 커스텀 이벤트가 실행됩니다.

## **3\. 다른 블루프린트에서 호출하기 (통신)**

가장 자주 쓰이는 패턴으로, A 블루프린트에서 B 블루프린트의 이벤트를 실행하는 방법입니다.

### **방법 A: 형변환 (Casting) 사용**

1. **참조 가져오기**: 대상 블루프린트(예: BP\_Player)의 객체 참조 변수를 가져옵니다.  
2. **형변환**: Cast to BP\_Player 노드를 사용합니다.  
3. **이벤트 호출**: 형변환 성공(As BP Player) 핀을 끌어당겨 Call \[이벤트 이름\]을 검색해 호출합니다.

### **방법 B: 블루프린트 인터페이스 (Interface)**

* 객체 간의 의존성을 줄이고 싶을 때 사용합니다. 인터페이스를 생성하고 함수를 정의한 뒤, 양쪽 블루프린트에서 구현 및 호출합니다.

## **4\. 커스텀 이벤트의 특징 및 팁**

### **입력 파라미터 (Inputs)**

* 커스텀 이벤트 노드를 선택한 후 **디테일(Details) 패널**에서 Inputs 항목의 \+ 버튼을 눌러 변수를 추가할 수 있습니다.  
* 이를 통해 이벤트 호출 시 데이터(데미지 수치, 이름 등)를 함께 전달할 수 있습니다.

### **리플리케이션 (Replication \- 멀티플레이어)**

* 멀티플레이어 게임 개발 시 매우 중요합니다.  
* **Not Replicated**: 로컬에서만 실행.  
* **Run on Server**: 클라이언트가 호출하면 서버에서 실행.  
* **Multicast**: 서버가 호출하면 접속한 모든 클라이언트에서 실행.  
* **Run on owning Client**: 서버가 호출하면 해당 객체를 소유한 클라이언트에서만 실행.

### **함수(Function)와의 차이점**

* **함수**: 실행 후 결과값을 즉시 반환(Return)할 수 있으며, 시간 지연(Delay) 노드를 사용할 수 없습니다.  
* **이벤트**: Delay, Timeline 등의 시간 흐름 관련 노드를 사용할 수 있지만, 직접적인 Return 값은 가질 수 없습니다 (Output 파라미터 미지원).

## **요약 예시**

\[이벤트 생성\]  
Custom Event (Name: HealPlayer) \-\> Print String ("Healed\!")

\[이벤트 호출\]  
Input Key F \-\> Call HealPlayer
