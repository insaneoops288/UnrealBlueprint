# **Unreal Engine: Box Trace For Objects 노드 상세 가이드**

Box Trace For Objects 노드는 보이지 않는 박스 형태의 충돌체를 특정 경로로 투사하여, 그 경로상에 있는 특정 \*\*오브젝트 타입(Object Type)\*\*의 물체들을 감지하는 역할을 합니다.

## **1\. 핵심 역할**

단순한 선(Line)이 아니라 **입체적인 박스(Box)** 공간을 사용하여 충돌을 계산합니다. 이는 다음과 같은 상황에서 유리합니다.

* 캐릭터의 전방에 넓은 범위의 공격 판정을 만들 때  
* 캐릭터가 좁은 통로를 지나갈 수 있는지 체격만큼의 부피로 검사할 때  
* 특정 구역 내에 있는 특정 물리 타입(예: WorldDynamic, PhysicsBody)의 오브젝트만 골라내고 싶을 때

## **2\. 주요 파라미터 (Input Pins)**

| 파라미터 | 설명 |
| :---- | :---- |
| **Start / End** | 박스 트레이스가 시작되고 끝나는 위치(Vector)입니다. |
| **Half Size** | 박스의 크기를 결정합니다. 중심에서 각 축 방향으로의 거리이므로, 실제 박스 크기는 이 값의 2배가 됩니다. |
| **Orientation** | 박스의 회전값(Rotator)입니다. 박스가 기울어진 채로 투사되게 할 수 있습니다. |
| **Object Types** | 감지할 오브젝트 채널의 배열입니다 (예: WorldStatic, Pawn, Vehicle 등). |
| **Trace Complex** | 참(True)이면 정밀한 폴리곤 단위 충돌을 계산하고, 거짓(False)이면 단순화된 캡슐/박스 콜리전으로 계산합니다. |
| **Actors to Ignore** | 트레이스에서 제외할 액터 리스트입니다. 보통 자기 자신(Self)을 넣어 본인이 감지되는 것을 방지합니다. |
| **Draw Debug Type** | 개발 중 트레이스 범위를 시각적으로 확인하기 위해 사용합니다 (None, For One Frame, For Duration 등). |

## **3\. 출력 데이터 (Output Pins)**

* **Return Value**: 무언가 감지되었다면 True, 아무것도 없다면 False를 반환합니다.  
* **Out Hit**: 충돌한 지점의 정보가 담긴 **Hit Result** 구조체입니다.  
  * **Hit Actor**: 감지된 액터가 무엇인지?  
  * **Location / Impact Point**: 정확히 어디서 부딪혔는지?  
  * **Impact Normal**: 부딪힌 표면의 방향은 어디인지?

## **4\. 실전 활용 사례**

### **A. 근접 공격 판정 (Melee Combat)**

검을 휘두를 때 검의 궤적을 따라 Box Trace를 실행하면, 얇은 선인 Line Trace보다 훨씬 관대한 판정을 줄 수 있어 플레이어 경험이 좋아집니다.

### **B. 상호작용 시스템**

플레이어 앞에 있는 아이템을 집을 때, 정확히 조준하지 않아도 캐릭터 정면의 일정 부피 안에 있는 아이템을 쉽게 찾을 수 있습니다.

### **C. 이동 가능 여부 체크**

캐릭터가 순간이동(Blink)하거나 특정 위치로 이동할 때, 도착 지점에 캐릭터의 크기(Half Size)만큼의 공간이 비어있는지 미리 확인할 수 있습니다.

## **5\. Line Trace vs Sphere Trace vs Box Trace**

* **Line Trace**: 가장 가볍고 빠름. 정확한 조준(총기 발사 등)에 적합.  
* **Sphere Trace**: 구체 형태. 모든 방향으로 둥글게 감지할 때 적합.  
* **Box Trace**: 각진 부피. 캐릭터의 체격이나 직사각형 형태의 공격 범위, 문 형태의 통과 여부 확인에 최적.

**Tip**: 성능 최적화를 위해 너무 큰 Half Size를 사용하거나 매 프레임 너무 많은 Box Trace를 호출하는 것은 피하는 것이 좋습니다.
