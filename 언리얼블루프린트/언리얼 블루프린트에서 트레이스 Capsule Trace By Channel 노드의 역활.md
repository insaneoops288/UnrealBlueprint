# **Capsule Trace By Channel 노드 완벽 가이드**

언리얼 엔진에서 **Capsule Trace By Channel**은 점과 점 사이를 잇는 단순한 선(Line Trace) 대신, 두께와 높이가 있는 **캡슐(Capsule)** 형태의 범위를 밀어서(Sweep) 경로 상에 있는 물체를 찾아내는 노드입니다.

## **1\. 주요 역할 (Core Function)**

이 노드는 \*\*시작점(Start)\*\*에서 \*\*끝점(End)\*\*까지 지정된 반지름(Radius)과 높이(Half Height)를 가진 캡슐을 이동시킵니다. 이 이동 경로 중에 설정된 \*\*콜리전 채널(Collision Channel)\*\*과 반응하는 물체가 있다면 그 정보를 반환합니다.

## **2\. 주요 입력 파라미터 (Inputs)**

| 파라미터 | 설명 |
| :---- | :---- |
| **Start / End** | 트레이스가 시작되고 끝나는 3D 공간의 좌표입니다. |
| **Radius** | 캡슐의 반지름입니다. 체크하고자 하는 범위의 너비를 결정합니다. |
| **Half Height** | 캡슐의 절반 높이입니다. (전체 높이의 1/2) |
| **Trace Channel** | 어떤 콜리전 채널을 대상으로 검사할지 결정합니다. (예: Visibility, Camera, Pawn 등) |
| **Trace Complex** | 체크 시 복잡한 콜리전(메쉬 디테일)을 계산할지, 단순한 콜리전(캡슐/박스 등)을 계산할지 정합니다. |
| **Actors to Ignore** | 트레이스 결과에서 제외할 액터 리스트입니다. (주로 자기 자신을 포함시킵니다.) |
| **Draw Debug Type** | 개발 중 트레이스 범위를 시각적으로 확인하기 위해 사용합니다. (None, One Frame, For Duration 등) |

## **3\. 주요 출력 파라미터 (Outputs)**

* **Return Value (Boolean):** 무언가에 부딪혔다면 True, 아무것도 없다면 False를 반환합니다.  
* **Out Hit (Hit Result):** 부딪힌 대상에 대한 상세 정보가 담긴 구조체입니다.  
  * **Hit Actor:** 부딪힌 액터가 무엇인가?  
  * **Location:** 부딪힌 정확한 위치.  
  * **Normal:** 부딪힌 표면의 방향(법선 벡터).  
  * **Distance:** 시작점으로부터의 거리.

## **4\. 왜 Capsule Trace를 사용하나요? (활용 사례)**

1. **캐릭터 이동 및 충돌 예방:**  
   * 캐릭터가 특정 방향으로 이동할 때, 캐릭터의 실제 크기(캡슐 컴포넌트)만큼 공간이 확보되었는지 미리 확인할 때 사용합니다.  
2. **벽 타기 / 난간 잡기 (Ledge Detection):**  
   * 캐릭터 전방에 잡을 수 있는 난간이 있는지 확인할 때, 라인 트레이스보다 넓은 범위를 커버하므로 더 정확한 판정이 가능합니다.  
3. **근접 공격 판정:**  
   * 칼 휘두르기 같은 근접 공격 시, 궤적을 따라 캡슐을 생성하여 적중 여부를 판정하면 얇은 선보다 훨씬 부드럽고 관대한 판정을 줄 수 있습니다.  
4. **아이템 상호작용 범위:**  
   * 플레이어 주변에 상호작용 가능한 아이템이 있는지 원통형/캡슐형 범위로 검사할 때 유용합니다.

## **5\. Line Trace vs Capsule Trace**

* **Line Trace:** 아주 얇은 선입니다. 총알 궤적이나 정확한 조준점 확인에 적합하지만, 물체 사이의 좁은 틈을 지나칠 수 있습니다.  
* **Capsule Trace:** 볼륨을 가집니다. 캐릭터와 비슷한 부피를 가진 물체가 통과할 수 있는지 확인하거나, 판정 범위를 넉넉하게 잡아야 할 때 필수적입니다.

## **6\. 사용 팁**

* **Self Ignore:** 트레이스를 시작할 때 자기 자신의 캡슐 컴포넌트에 부딪혀서 True가 나오는 경우가 많습니다. 반드시 Actors to Ignore에 Self를 추가하세요.  
* **Draw Debug:** 트레이스가 생각한 대로 작동하지 않는다면 Draw Debug Type을 For Duration으로 설정하여 붉은색/녹색 캡슐이 제대로 그려지는지 확인하는 것이 디버깅의 핵심입니다.
