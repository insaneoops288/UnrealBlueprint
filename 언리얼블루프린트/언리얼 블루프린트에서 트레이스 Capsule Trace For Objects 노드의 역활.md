# **Capsule Trace For Objects 노드 상세 분석**

언리얼 엔진에서 **Capsule Trace For Objects**는 보이지 않는 '캡슐' 형태의 광선을 쏘아 특정 오브젝트들과의 충돌 여부를 판별하는 노드입니다. 주로 캐릭터의 이동 경로 확인이나 복잡한 충돌 감지에 사용됩니다.

## **1\. 노드의 핵심 역할**

이 노드는 단순히 선(Line)을 긋는 것이 아니라, **두께와 높이가 있는 캡슐 공간**을 일정 거리만큼 밀어내어(Sweep) 그 경로에 걸리는 물체를 찾습니다.

* **모양(Shape):** 실린더 양 끝에 구체가 붙은 캡슐 형태입니다.  
* **필터링(Filtering):** 'Trace Channel'(Visibility 등)이 아닌 **'Object Type'**(WorldStatic, WorldDynamic, Pawn 등)을 기준으로 충돌을 검사합니다.

## **2\. 주요 입력 파라미터 (Inputs)**

| 파라미터 | 설명 |
| :---- | :---- |
| **Start** | 캡슐 추적을 시작할 세계 좌표(Vector)입니다. |
| **End** | 캡슐 추적이 끝날 세계 좌표(Vector)입니다. |
| **Radius** | 캡슐의 반지름입니다. 두께를 결정합니다. |
| **Half Height** | 캡슐의 절반 높이입니다. 캐릭터의 키와 맞출 때 유용합니다. |
| **Object Types** | 감지하려는 오브젝트 타입들의 배열입니다. (예: Pawn, PhysicsBody 등) |
| **Trace Complex** | 체크 시 복잡한 콜리전(메쉬 정밀도)을 사용할지 여부입니다. (성능을 위해 보통 false 권장) |
| **Actors to Ignore** | 추적에서 제외할 액터 목록입니다. (자기 자신을 제외할 때 자주 사용) |
| **Draw Debug Type** | 개발 중 추적 범위를 시각적으로 확인하기 위한 설정입니다. (None, For One Frame, Persistent 등) |

## **3\. 주요 출력 결과 (Outputs)**

* **Return Value (Boolean):** 무언가에 부딪혔다면 True, 아니면 False를 반환합니다.  
* **Out Hit (Hit Result):** 부딪힌 대상에 대한 상세 정보를 담고 있는 구조체입니다.  
  * **Hit Actor:** 부딪힌 액터가 무엇인지 알려줍니다.  
  * **Impact Point:** 실제로 충돌이 일어난 정확한 지점입니다.  
  * **Impact Normal:** 충돌한 표면의 방향(법선 벡터)입니다. 캐릭터가 벽을 타고 흐르게 할 때 사용합니다.

## **4\. 왜 사용하는가? (활용 사례)**

1. **캐릭터 이동성 체크 (Ledge Detection):**  
   캐릭터가 앞으로 이동할 때, 캐릭터의 실제 몸체 크기(캡슐 콜리전)와 동일한 크기로 Trace를 쏘아 앞에 장애물이 있는지, 혹은 올라갈 수 있는 턱이 있는지 확인합니다.  
2. **공격 판정:**  
   검을 휘두를 때 칼날의 궤적을 캡슐 형태로 추적하여 적(Pawn 타입)이 범위 내에 있는지 정밀하게 판별합니다.  
3. **착지 지점 계산:**  
   점프 후 땅에 닿기 전, 발 밑의 지형이 평평한지 혹은 특정 오브젝트(예: 움직이는 플랫폼)인지 확인할 때 유용합니다.

## **5\. Line Trace vs Capsule Trace**

* **Line Trace:** 아주 얇은 선입니다. 작은 틈새 사이로 빠져나갈 수 있어 정밀한 사격 등에 적합합니다.  
* **Capsule Trace:** 부피를 가집니다. 캐릭터가 통과할 수 있는 통로인지 확인하거나, 실제 물리적 부피를 가진 물체의 이동 경로를 시뮬레이션할 때 필수적입니다.

### **💡 팁**

캐릭터 블루프린트에서 사용할 때는 Get Actor Location을 **Start**로, Get Actor Forward Vector에 일정 수치를 곱한 값을 더해 **End**로 설정하면 캐릭터 정면의 장애물을 쉽게 감지할 수 있습니다.
