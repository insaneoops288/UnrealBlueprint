# **언리얼 엔진 블루프린트: 트레이스 콜리전 설정 (Trace Collision Settings)**

언리얼 엔진에서 \*\*트레이스(Trace)\*\*는 특정 지점에서 다른 지점으로 보이지 않는 선(Line)이나 도형(Box, Sphere, Capsule)을 투사하여 물리적인 충돌을 체크하는 기술입니다. 이때 **Collision Setting**은 트레이스가 "무엇과 부딪힐 것인가?"를 결정하는 가장 중요한 기준입니다.

## **1\. 트레이스의 두 가지 주요 방식**

트레이스 노드는 크게 두 가지 방식으로 나뉩니다. 각 방식에 따라 콜리전 설정의 접근법이 다릅니다.

### **① LineTraceByChannel (채널 기반 트레이스)**

특정 \*\*콜리전 채널(Collision Channel)\*\*을 기준으로 트레이스를 수행합니다.

* **주요 설정**: Trace Channel (기본값: Visibility, Camera)  
* **작동 원리**: 물체의 콜리전 프리셋 설정 중 해당 채널이 Block으로 설정된 물체만 감지합니다.  
* **용도**: 가시성 체크, 카메라 가림 확인, 일반적인 투사체 판정 등.

### **② LineTraceForObjects (오브젝트 기반 트레이스)**

물체의 \*\*오브젝트 타입(Object Type)\*\*을 기준으로 트레이스를 수행합니다.

* **주요 설정**: Object Types (배열 형태)  
* **작동 원리**: 물체의 'Object Type'이 배열에 포함된 유형(예: WorldStatic, Pawn, PhysicsBody 등)과 일치하면 무조건 감지합니다.  
* **용도**: 특정 그룹의 물체(예: '적군'들만, 또는 '아이템'들만)를 찾을 때 유리합니다.

## **2\. 콜리전 응답(Collision Response)의 역할**

트레이스 채널 방식(ByChannel)을 사용할 때, 월드에 배치된 액터의 콜리전 설정에는 세 가지 응답 상태가 존재합니다.

| 응답 (Response) | 설명 |
| :---- | :---- |
| **Ignore (무시)** | 트레이스가 이 물체를 통과하며, 아무런 충돌 정보를 반환하지 않습니다. |
| **Overlap (겹침)** | 트레이스가 통과하지만, 겹쳤다는 신호를 보낼 수 있습니다. (주로 일반 트레이스보다는 Overlap 전용 노드에서 사용) |
| **Block (차단)** | 트레이스가 이 물체에 부딪혀 멈춥니다. Hit Result를 통해 충돌 지점, 노멀값 등을 반환합니다. |

## **3\. 커스텀 트레이스 채널 설정 방법**

기본 제공되는 Visibility나 Camera 채널 외에 게임 로직에 맞는 전용 채널(예: 'Bullet', 'Interaction')이 필요할 때 설정하는 방법입니다.

1. **Project Settings(프로젝트 세팅)** 이동.  
2. **Engine \> Collision** 섹션 선택.  
3. **Trace Channels** 항목에서 New Trace Channel 클릭.  
4. 이름 설정 후 Default Response(기본 응답)를 설정 (보통 Ignore로 두고 필요한 물체만 Block으로 바꿈).  
5. 블루프린트 노드의 Trace Channel 드롭다운에서 생성한 채널을 선택하여 사용.

## **4\. 주요 노드 옵션 설명**

트레이스 노드 내부에는 콜리전 판정에 영향을 주는 추가 설정들이 있습니다.

* **Actors to Ignore (무시할 액터)**: 트레이스를 쏘는 본인(Self)이나 특정 아군 등 충돌 계산에서 제외할 액터 리스트를 배열로 넘겨줍니다.  
* **Trace Complex (복합 트레이스)**:  
  * True: 메시의 정밀한 폴리곤 단위(Complex Collision)로 충돌을 체크합니다. (사격 게임의 정밀 헤드샷 등에 사용)  
  * False: 단순화된 캡슐이나 박스 형태(Simple Collision)로 체크합니다. (성능 최적화에 유리)  
* **Draw Debug Type**: 트레이스가 실제로 어디로 날아가는지 화면에 선으로 그려주는 시각화 옵션입니다.

## **5\. 실전 활용 팁**

1. **최적화**: 모든 물체를 Visibility 채널에 Block으로 두면 트레이스 비용이 커질 수 있습니다. 필요한 물체만 특정 커스텀 채널에 반응하도록 설계하세요.  
2. **레이어 구분**: 바닥은 감지하지만 풀(Grass)은 무시하고 싶다면, 풀의 콜리전 설정에서 사용 중인 트레이스 채널을 Ignore로 변경하면 됩니다.  
3. **결과 분석**: 충돌이 발생하면 Break Hit Result 노드를 사용해 부딪힌 액터(Hit Actor), 컴포넌트(Hit Component), 정확한 위치(Location) 등을 가져와 후속 로직을 처리합니다.
