# **Line Trace By Channel 노드 상세 가이드**

## **1\. 개요 (Overview)**

**Line Trace By Channel**은 특정 시작점에서 끝점까지 직선을 그어, 지정된 \*\*충돌 채널(Collision Channel)\*\*에 반응하는 물체가 있는지 탐색하는 노드입니다. 흔히 '레이캐스팅(Raycasting)'이라고도 불립니다.

## **2\. 주요 입력 파라미터 (Inputs)**

| 항목 | 설명 |
| :---- | :---- |
| **Start** | 라인 트레이스가 시작되는 3차원 좌표(Vector)입니다. |
| **End** | 라인 트레이스가 끝나는 3차원 좌표(Vector)입니다. |
| **Trace Channel** | 어떤 충돌 채널을 검사할지 선택합니다. (기본값: Visibility, Camera 등) |
| **Trace Complex** | 복잡한 충돌(메시 자체의 폴리곤)을 검사할지 결정합니다. 체크하면 정교하지만 성능 비용이 높습니다. |
| **Actors to Ignore** | 검사에서 제외할 액터 리스트입니다. (보통 자기 자신을 제외할 때 사용) |
| **Draw Debug Type** | 개발 중에 라인이 눈에 보이도록 설정합니다. (None, One Frame, For Duration 등) |

## **3\. 주요 출력 파라미터 (Outputs)**

| 항목 | 설명 |
| :---- | :---- |
| **Return Value** | 무언가에 부딪혔다면 True, 아무것도 맞지 않았다면 False를 반환합니다. |
| **Out Hit** | 부딪힌 대상에 대한 상세 정보가 담긴 **Hit Result 구조체**입니다. |

### **💡 Out Hit(Hit Result)에서 얻을 수 있는 정보**

노드에서 Break Hit Result를 사용하면 다음과 같은 세부 정보를 볼 수 있습니다.

* **Hit Actor**: 부딪힌 액터가 누구인지.  
* **Hit Component**: 액터의 어떤 컴포넌트에 맞았는지.  
* **Location**: 월드 공간에서 실제로 부딪힌 위치.  
* **Normal**: 부딪힌 표면의 방향(법선 벡터). 도탄(리플렉션) 계산 시 유용합니다.  
* **Distance**: 시작점에서 부딪힌 지점까지의 거리.

## **4\. "By Channel"의 의미**

이 노드가 By Channel인 이유는 \*\*충돌 설정(Collision Settings)\*\*에 기반하기 때문입니다.

* 예를 들어, 채널을 Visibility로 설정하면 '보이는 물체'에는 부딪히지만, '투명한 트리거 박스'는 그냥 통과할 수 있습니다.  
* 프로젝트 세팅에서 사용자 정의 채널(예: Bullet, Interaction)을 만들어 특정 물체들만 골라낼 수도 있습니다.

## **5\. 주요 활용 사례 (Use Cases)**

1. **슈팅 게임 (Hitscan)**: 총을 쐈을 때 즉시 탄환이 맞았는지 판정합니다.  
2. **상호작용 시스템**: 화면 중앙(에임)에서 앞쪽으로 라인을 쏴서 앞에 문이나 아이템이 있는지 확인합니다.  
3. **바닥 감지**: 캐릭터의 발밑으로 짧은 라인을 쏴서 현재 경사도나 바닥의 재질(Physical Material)을 파악합니다.  
4. **AI 시야**: AI 캐릭터가 플레이어를 직선상에서 가로막는 장애물 없이 볼 수 있는지 체크합니다.

## **6\. 사용 시 주의사항**

* **자기 자신 제외**: 라인 트레이스 시작점이 캐릭터 내부라면, 캐릭터 본인의 캡슐 컴포넌트에 먼저 부딪힐 수 있습니다. Actors to Ignore에 자신을 추가하는 것이 필수입니다.  
* **성능**: 매 프레임 수백 개의 라인 트레이스를 실행하는 것은 성능에 부담을 줄 수 있습니다. 필요한 경우에만 실행하도록 최적화가 필요합니다.
