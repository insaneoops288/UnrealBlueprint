# **LineTraceForObjects 노드 완벽 이해하기**

언리얼 엔진의 **LineTraceForObjects**는 시작점과 끝점 사이에 가상의 선(Ray)을 긋고, 그 선에 부딪히는 물체 중 \*\*사용자가 지정한 오브젝트 타입(Object Type)\*\*에 해당하는 것이 있는지 검사하는 노드입니다.

## **1\. 주요 역할 (Role)**

이 노드의 핵심은 \*\*"내가 관심 있는 종류의 물체만 골라서 찾겠다"\*\*는 것입니다. 예를 들어, 세상에 수많은 장애물이 있어도 오직 'Pawn(캐릭터)'이나 'PhysicsBody(물리 시뮬레이션 물체)'만 감지하고 싶을 때 사용합니다.

## **2\. 주요 입력 파라미터 (Inputs)**

| 항목 | 설명 |
| :---- | :---- |
| **Start** | 선이 시작되는 월드 좌표 (벡터) |
| **End** | 선이 끝나는 월드 좌표 (벡터) |
| **Object Types** | 감지하고자 하는 오브젝트 타입들의 배열 (예: WorldStatic, Pawn, Vehicle 등) |
| **Trace Complex** | 체크 시 복잡한 콜리전(메시 형태 그대로)을 계산합니다. 해제 시 단순 콜리전(캡슐, 박스 등)을 사용합니다. |
| **Actors to Ignore** | 트레이스에서 제외할 액터들의 목록 (자기 자신을 제외할 때 자주 사용) |
| **Draw Debug Type** | 개발 중 선이 실제로 어떻게 나가는지 시각적으로 보여줍니다 (None, One Frame, For Duration 등) |

## **3\. 출력 데이터 (Outputs)**

* **Return Value**: 무언가(지정한 타입)에 부딪혔다면 True, 아니면 False를 반환합니다.  
* **Out Hit**: 부딪힌 지점의 상세 정보가 담긴 **Hit Result** 구조체를 반환합니다.  
  * Hit Actor: 부딪힌 대상 액터  
  * Hit Location: 부딪힌 정확한 좌표  
  * Normal: 부딪힌 표면의 방향(법선 벡터)  
  * Distance: 시작점부터 부딪힌 곳까지의 거리

## **4\. LineTraceByChannel vs LineTraceForObjects**

가장 많이 비교되는 두 노드의 차이점은 \*\*"필터링 방식"\*\*입니다.

1. **LineTraceByChannel**:  
   * **채널 기준**: Visibility(눈에 보이는가?) 또는 Camera 채널 등을 사용합니다.  
   * 용도: 총을 쏘거나, 마우스 클릭으로 물체를 선택할 때처럼 "보이는 모든 것"을 대상으로 할 때 유리합니다.  
2. **LineTraceForObjects**:  
   * **오브젝트 성격 기준**: Pawn, WorldDynamic, Projectile 등 물리 설정에 정의된 오브젝트 타입을 사용합니다.  
   * 용도: "내 주변에 있는 적(Pawn)만 찾고 싶다"거나 "물리 효과가 적용되는 물체만 밀어내고 싶다"처럼 특정 부류를 찾을 때 유리합니다.

## **5\. 실전 활용 사례**

1. **아이템 상호작용**: 'InteractiveObject'라는 커스텀 오브젝트 타입을 만들고, 캐릭터 앞에 이 타입의 물체가 있는지 확인할 때.  
2. **AI 시야 체크**: AI가 'Pawn' 타입만 감지하도록 설정하여 벽(WorldStatic)은 무시하고 캐릭터만 조준하게 할 때.  
3. **범위 공격 보정**: 폭발 중심에서 주변으로 트레이스를 쏴서 특정 타입(예: Destructible)의 물체에만 데미지를 입힐 때.

## **6\. 주의사항**

* **Object Types 입력**: 이 노드는 배열을 입력받으므로, Make Array 노드를 연결하여 감지할 타입들을 하나 이상 지정해줘야 작동합니다. 빈 배열이면 아무것도 감지하지 못합니다.  
* **콜리전 설정**: 대상 액터의 **Collision Preset** 설정에서 Object Type이 트레이스에서 찾는 타입과 일치해야 합니다.
