# **Multi Box Trace By Channel 노드 상세 가이드**

언리얼 엔진의 Multi Box Trace By Channel 노드는 가상의 **박스(Box)** 모양을 특정 경로로 이동시키면서, 그 경로상에 걸리는 **모든 물체**를 감지하는 역할을 합니다.

## **1\. 핵심 역할**

* **다중 충돌 감지**: 단일 충돌(Single)과 달리, 경로 상에 있는 첫 번째 물체에서 멈추지 않고 지정된 채널에 해당하는 **모든 물체**를 배열(Array) 형태로 반환합니다.  
* **박스 형태의 검사**: 선(Line)이나 구체(Sphere)가 아닌, 가로/세로/높이가 있는 박스 형태로 검사하므로 문(Door), 복도, 특정 구역 등을 검사할 때 유리합니다.

## **2\. 주요 입력 파라미터 (Inputs)**

| 파라미터 | 설명 |
| :---- | :---- |
| **Start** | 트레이스(검사)가 시작되는 세계 공간의 좌표입니다. |
| **End** | 트레이스가 끝나는 세계 공간의 좌표입니다. |
| **Half Size** | 박스의 크기를 결정합니다. (중심에서 각 면까지의 거리, 즉 반지름 개념) |
| **Orientation** | 박스의 회전값입니다. 박스가 기울어진 채로 이동해야 할 때 설정합니다. |
| **Trace Channel** | 검사할 충돌 채널을 선택합니다. (예: Visibility, Camera, Pawn 등) |
| **Actors to Ignore** | 검사에서 제외할 액터 목록입니다. (자기 자신을 제외할 때 자주 사용) |
| **Draw Debug Type** | 화면에 검사 과정을 시각적으로 표시합니다. (None, For One Frame, For Duration, Persistent) |

## **3\. 출력 결과 (Outputs)**

* **Return Value**: 물체가 하나라도 감지되면 True, 없으면 False를 반환합니다.  
* **Out Hits**: 감지된 모든 물체의 정보를 담은 **Hit Result 구조체 배열**입니다. 각 요소에는 충돌 지점, 충돌한 액터, 노멀 값 등이 포함됩니다.

## **4\. Single vs Multi 차이점**

1. **Box Trace By Channel (Single)**:  
   * 경로 상에서 **가장 처음 만난** 물체 하나만 반환하고 종료합니다.  
   * 예: 총알이 벽에 맞았는지 확인할 때.  
2. **Multi Box Trace By Channel**:  
   * 경로 상의 **모든** 유효한 타겟을 뚫고 지나가며 수집합니다.  
   * 예: 관통하는 거대 미사일, 범위 내의 모든 적에게 대미지를 주는 스킬.

## **5\. 실무 활용 사례**

### **① 광역 공격 (AOE Damage)**

캐릭터가 앞을 향해 휘두르는 커다란 대검 스킬을 구현할 때, 칼의 궤적만큼 박스 트레이스를 쏘아 그 안에 있는 모든 적 액터를 찾아내어 대미지를 입힐 수 있습니다.

### **② 센서 및 구역 검사**

지정된 통로(Box 형태)를 지나가는 모든 유닛을 카운트하거나, 특정 영역 안에 어떤 아이템들이 놓여 있는지 한 번에 스캔할 때 사용합니다.

### **③ 관통 투사체**

장애물을 무시하고 경로에 있는 모든 적을 타격해야 하는 레이저나 관통 화살의 로직을 짤 때 필수적입니다.

## **6\. 주의사항**

* **성능**: Multi 트레이스는 Single보다 더 많은 계산을 필요로 합니다. 특히 결과값 배열이 클 경우 매 프레임(Tick) 호출 시 주의가 필요합니다.  
* **채널 설정**: 충돌 설정(Collision Presets)에서 해당 액터가 선택한 Trace Channel에 대해 'Block' 혹은 'Overlap'으로 설정되어 있어야 감지됩니다.
