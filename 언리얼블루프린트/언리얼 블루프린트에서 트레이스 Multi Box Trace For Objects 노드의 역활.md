# **Multi Box Trace For Objects 노드 분석**

언리얼 엔진에서 물리적 충돌이나 감지를 구현할 때 자주 사용되는 이 노드는 레이캐스트(Line Trace)보다 넓은 영역을 검사할 수 있으며, 여러 대상을 동시에 찾는 데 특화되어 있습니다.

## **1\. 노드의 이름이 의미하는 것**

* **Multi**: 충돌한 첫 번째 대상에서 멈추지 않고, 경로 상에 있는 **모든** 유효한 대상을 배열(Array) 형태로 반환합니다.  
* **Box**: 감지 영역의 형태가 **상자(Box)** 모양입니다.  
* **Trace**: 시작점(Start)에서 끝점(End)까지 해당 모양을 스윕(Sweep)하며 지나갑니다.  
* **For Objects**: 특정 콜리전 채널(Visibility, Camera 등)이 아니라, \*\*오브젝트 타입(WorldStatic, WorldDynamic, Pawn, PhysicsBody 등)\*\*을 기준으로 필터링합니다.

## **2\. 주요 입력 핀 (Inputs)**

| 핀 이름 | 설명 |
| :---- | :---- |
| **Start / End** | 박스 트레이스가 시작되고 끝나는 3D 위치(Vector)입니다. |
| **Half Size** | 박스의 크기를 결정합니다. 중심에서 각 면까지의 거리(Extents)이므로, 실제 박스 크기의 절반 값을 입력합니다. |
| **Orientation** | 박스의 회전값(Rotator)입니다. 박스가 기울어진 채로 이동해야 할 때 사용합니다. |
| **Object Types** | 감지하고자 하는 오브젝트 타입의 배열입니다. (예: Pawn만 감지하고 싶다면 Pawn 타입을 추가) |
| **Trace Complex** | 체크 시 복잡한 콜리전(메시 디테일)을 검사합니다. 비활성 시 단순 캡슐/박스 콜리전을 검사하여 성능을 아낍니다. |
| **Actors to Ignore** | 트레이스 결과에서 제외할 액터들의 배열입니다. (보통 자기 자신 'Self'를 추가합니다) |
| **Draw Debug Type** | 화면에 트레이스 영역을 선으로 그려줍니다. 개발 단계에서 시각적으로 확인할 때 필수적입니다. |

## **3\. 주요 출력 핀 (Outputs)**

| 핀 이름 | 설명 |
| :---- | :---- |
| **Return Value** | 하나라도 감지된 대상이 있으면 True, 없으면 False를 반환합니다. |
| **Out Hits** | 충돌한 모든 대상의 정보를 담은 **Hit Result 구조체 배열**입니다. For Each Loop를 통해 각 충돌체의 위치, 액터 이름, 노멀 값 등을 추출할 수 있습니다. |

## **4\. 언제 사용하나요? (Use Cases)**

1. **광역 데미지(AOE) 판정**: 캐릭터 앞에서 박스 모양의 공격을 휘둘렀을 때, 그 범위 안에 있는 모든 적을 한꺼번에 찾아내어 데미지를 입힐 때 사용합니다.  
2. **아이템 수집 범위**: 캐릭터 주변의 일정 사각형 영역 내에 있는 'Item' 타입의 오브젝트들을 모두 찾아 인벤토리에 넣을 때 유용합니다.  
3. **통로 검사**: 특정 복도나 구역에 장애물이나 플레이어가 있는지 한꺼번에 체크할 때 사용합니다.

## **5\. Line Trace vs Box Trace 차이점**

* **Line Trace**: 아주 가는 선으로 검사합니다. 정밀하지만 문틈 사이로 지나가면 충돌을 놓칠 수 있습니다.  
* **Box Trace**: 일정한 부피(부피)를 가지고 검사합니다. "실제로 부딪힐 만한 공간"을 검사하므로 게임적인 판정(공격 범위 등)에서 훨씬 안정적입니다.

## **💡 팁**

* **Multi** 노드는 성능 비용이 단일(Single) 트레이스보다 높으므로, 매 프레임(Tick)마다 너무 많은 개수를 실행하는 것은 피하는 것이 좋습니다.  
* 충돌 결과가 너무 많을 경우, Out Hits 배열을 루프 돌리기 전에 Length를 체크하여 필요한 로직만 수행하세요.
