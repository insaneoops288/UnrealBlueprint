# **Multi Capsule Trace By Channel 노드 가이드**

언리얼 엔진의 Multi Capsule Trace By Channel 노드는 물리 엔진의 '스윕(Sweep)' 기능을 활용하여 특정 공간 내의 여러 대상을 동시에 감지할 때 필수적인 도구입니다.

## **1\. 핵심 역할**

이 노드의 주요 목적은 \*\*시작점(Start)\*\*과 **종료점(End)** 사이의 직선 경로를 따라 **캡슐(Capsule)** 모양의 충돌체를 이동시키며, 설정된 \*\*콜리전 채널(Collision Channel)\*\*에 반응하는 모든 물체를 찾아내는 것입니다.

## **2\. 주요 특징**

* **Multi (다중 감지):** 'Single' 트레이스와 달리 첫 번째 충돌 지점에서 멈추지 않습니다. 경로상에 있는 모든 유효한 대상을 배열(Array) 형태로 반환합니다.  
* **Capsule Shape:** 구(Sphere)와 원통(Cylinder)이 합쳐진 캡슐 형태를 사용하므로, 캐릭터(Pawn)의 충돌 범위와 유사한 영역을 검사할 때 매우 효율적입니다.  
* **By Channel:** 특정 콜리전 채널(예: Visibility, Camera, Pawn 등)을 기준으로 필터링하여, 원하는 유형의 물체만 감지할 수 있습니다.

## **3\. 주요 입력 핀 (Inputs)**

| 핀 이름 | 설명 |
| :---- | :---- |
| **Start / End** | 트레이스가 시작되고 끝나는 세계 공간(World Space) 좌표입니다. |
| **Radius / Half Height** | 캡슐의 반지름과 반높이(중심에서 끝까지)를 설정하여 크기를 정의합니다. |
| **Trace Channel** | 어떤 채널과 충돌할지 결정합니다. (예: Visibility로 설정 시 투명하지 않은 물체 감지) |
| **Trace Complex** | True일 경우 복잡한 메시 표면(충돌 프리셋 기준)까지 정밀하게 검사합니다. |
| **Actors to Ignore** | 검사에서 제외할 액터 리스트를 지정합니다. (보통 자기 자신을 넣습니다) |
| **Draw Debug Type** | 개발 중 트레이스 경로를 시각적으로 확인할 수 있게 해줍니다. |

## **4\. 주요 출력 핀 (Outputs)**

* **Return Value:** 하나라도 감지된 물체가 있다면 True, 없으면 False를 반환합니다.  
* **Out Hits:** 감지된 모든 물체에 대한 상세 정보가 담긴 **Hit Result 구조체 배열**입니다. 충돌 지점, 충돌한 액터, 노멀 값 등이 포함됩니다.

## **5\. 활용 예시**

1. **범위 공격 (AOE Skill):** 휘두르는 칼의 궤적이나 캐릭터 앞 점프 공격 시 착지 지점의 여러 적을 동시에 타격할 때.  
2. **관통 탄환:** 총알이 첫 번째 벽이나 적을 뚫고 지나가며 뒤에 있는 대상들까지 맞추는 로직 구현.  
3. **경로 장애물 검사:** 로봇이나 AI가 이동할 경로(캡슐 크기)에 걸리는 물체들이 무엇인지 미리 파악할 때.

## **6\. Single Trace와의 차이점 요약**

* **Single:** 경로상 **가장 먼저** 부딪힌 물체 하나만 반환하고 종료합니다.  
* **Multi:** 경로상 부딪힌 **모든** 물체를 순서대로 수집하여 배열로 넘겨줍니다.
