# **Multi Capsule Trace For Objects 노드 완벽 이해하기**

언리얼 엔진의 **Multi Capsule Trace For Objects**는 캡슐 형태의 가상의 충돌체를 특정 경로로 쏘아보내(Sweep), 해당 경로에 겹치는 모든 지정된 타입의 오브젝트 정보를 배열 형태로 반환하는 노드입니다.

## **1\. 노드의 핵심 역할**

이 노드의 이름은 세 가지 핵심 키워드로 나뉩니다.

* **Multi**: 첫 번째 충돌에서 멈추지 않고, 경로상에 걸리는 **모든** 대상(Hit)을 다 찾아냅니다. (결과가 배열로 반환됨)  
* **Capsule**: 탐지 범위의 모양이 **캡슐(원통 \+ 양끝 구체)** 형태입니다. 캐릭터의 충돌체와 유사한 모양이라 캐릭터 크기의 장애물 검사에 유용합니다.  
* **For Objects**: 'Visibility'나 'Camera' 같은 채널 방식이 아니라, \*\*Object Type(WorldStatic, WorldDynamic, Pawn, PhysicsBody 등)\*\*을 기준으로 필터링하여 탐지합니다.

## **2\. 주요 입력 핀 (Inputs)**

| 핀 이름 | 설명 |
| :---- | :---- |
| **Start / End** | 트레이스가 시작되는 지점과 끝나는 지점의 세계 좌표(Vector)입니다. |
| **Radius** | 캡슐의 반지름입니다. |
| **Half Height** | 캡슐의 절반 높이입니다. (전체 높이의 1/2) |
| **Object Types** | 탐지할 오브젝트 타입들의 배열입니다. (예: Pawn, WorldDynamic만 선택 가능) |
| **Actors to Ignore** | 탐지에서 제외할 액터들의 배열입니다. 보통 'Self'를 넣어 자신은 안 걸리게 합니다. |
| **Draw Debug Type** | 개발 중 트레이스 범위를 눈으로 확인할 수 있게 선을 그려줍니다. (None, For One Frame, For Duration 등) |

## **3\. 주요 출력 핀 (Outputs)**

| 핀 이름 | 설명 |
| :---- | :---- |
| **Return Value** | 하나라도 걸린 대상이 있다면 True, 없으면 False를 반환합니다. |
| **Out Hits** | 충돌한 모든 대상의 정보가 담긴 **Hit Result 구조체 배열**입니다. |

## **4\. 작동 방식 (내부 로직)**

1. **필터링**: 먼저 지정된 Object Types에 해당하는 오브젝트들만 후보군에 올립니다.  
2. **스캔**: Start 지점에서 End 지점까지 캡슐 모양을 이동시킵니다.  
3. **수집**: 이동 경로 중에 후보 오브젝트와 겹치거나 부딪히면 해당 정보를 Out Hits 배열에 차곡차곡 쌓습니다.  
4. **결과**: End 지점에 도달하면 모든 수집된 정보를 반환합니다.

## **5\. 실전 활용 사례**

### **A. 광역 공격 (AOE) 스킬 구현**

* 칼을 휘두르는 궤적을 따라 캡슐 트레이스를 실행하여, 범위 내에 있는 모든 적(Enemy Object Type)을 찾아 한꺼번에 대미지를 줄 때 사용합니다.

### **B. 캐릭터 전방 장애물 체크**

* 캐릭터가 대시(Dash)를 할 때, 이동 경로상에 벽이나 다른 캐릭터가 있는지 미리 확인합니다. 캐릭터와 비슷한 캡슐 모양을 사용하므로 정확도가 높습니다.

### **C. 아이템 파밍 범위 확인**

* 플레이어 주변에 있는 모든 'PickUp' 타입의 아이템들을 한 번에 스캔하여 리스트를 만들 때 유용합니다.

## **6\. 주의사항 및 팁**

* **성능 고려**: Multi 트레이스는 Single보다 비용이 큽니다. 너무 긴 거리나 너무 많은 오브젝트 타입을 한꺼번에 검사하면 성능에 영향을 줄 수 있습니다.  
* **결과 처리**: 반환된 Out Hits 배열은 **For Each Loop** 노드를 사용해 하나씩 꺼내어 대미지를 주거나 정보를 읽어야 합니다.  
* **Trace Complex**: 복잡한 폴리곤 메시 자체를 검사할지 결정합니다. 정밀함이 필요하면 켜고, 성능이 중요하면 끕니다.
