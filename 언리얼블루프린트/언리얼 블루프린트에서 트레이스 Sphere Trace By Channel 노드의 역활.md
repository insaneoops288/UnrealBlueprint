# **Sphere Trace By Channel 노드 상세 가이드**

Sphere Trace By Channel은 언리얼 엔진 블루프린트에서 공간 상의 충돌을 감지하기 위해 사용하는 핵심 노드입니다. 단순히 점과 점을 잇는 '라인 트레이스(Line Trace)'와 달리, \*\*두께(반지름)\*\*를 가진 구체를 발사하여 더 넓은 범위를 체크합니다.

## **1\. 주요 역할 및 작동 방식**

* **볼륨 쿼리 (Volume Query):** 시작점(Start)에서 끝점(End)까지 지정된 \*\*반지름(Radius)\*\*의 구체를 이동시키며(Sweep), 이 경로에 걸리는 물체가 있는지 확인합니다.  
* **채널 필터링 (Trace Channel):** 모든 물체가 아닌, 특정 콜리전 채널(예: Visibility, Camera, Pawn 등)에 반응하도록 설정된 물체만 감지합니다.  
* **정밀한 결과 반환:** 단순히 충돌 여부만 알려주는 것이 아니라, 충돌한 위치, 표면의 법선(Normal), 충돌한 액터(Actor), 컴포넌트 등의 상세 정보를 제공합니다.

## **2\. 주요 입력 파라미터 (Inputs)**

| 파라미터 | 설명 |
| :---- | :---- |
| **Start** | 구체 트레이스가 시작되는 3D 좌표 (Vector)입니다. |
| **End** | 구체 트레이스가 끝나는 3D 좌표 (Vector)입니다. |
| **Radius** | 트레이스에 사용할 구체의 반지름입니다. 값이 클수록 더 넓은 범위를 감지합니다. |
| **Trace Channel** | 감지할 대상의 콜리전 채널을 선택합니다. (보통 Visibility가 기본) |
| **Trace Complex** | 체크 시 복잡한 콜리전(메시 자체)을 계산합니다. 해제 시 단순 콜리전(캡슐, 박스 등)을 사용합니다. |
| **Actors to Ignore** | 트레이스에서 제외할 액터 배열입니다. (예: 자기 자신) |
| **Draw Debug Type** | 화면에 트레이스 경로를 시각적으로 표시합니다. (None, For One Frame, For Duration, Persistent) |

## **3\. 주요 출력 결과 (Outputs)**

* **Return Value (Boolean):** 무언가에 부딪혔다면 True, 아무것도 감지되지 않았다면 False를 반환합니다.  
* **Out Hit (Hit Result Struct):** 충돌 시 발생하는 상세 데이터 묶음입니다. Break Hit Result 노드를 연결하여 다음 정보를 추출할 수 있습니다.  
  * **Blocking Hit:** 유효한 충돌이었는지 확인.  
  * **Impact Point:** 실제 구체가 표면과 닿은 지점.  
  * **Hit Actor:** 부딪힌 대상 액터가 누구인지 확인.  
  * **Normal:** 충돌한 표면이 바라보는 방향 (튕겨나가는 효과 등에 사용).

## **4\. Line Trace vs Sphere Trace: 언제 쓸까?**

| 비교 항목 | Line Trace | Sphere Trace |
| :---- | :---- | :---- |
| **정밀도** | 매우 정밀 (바늘 끝 같음) | 부피가 있어 관대함 |
| **난이도** | 조준이 정확해야 함 | 대략적인 방향만 맞아도 감지됨 |
| **주요 용도** | 레이저, 히트스캔 총기, 정밀 상호작용 | 근접 공격 판정, 아이템 줍기 범위, 캐릭터 주변 위험 감지 |

## **5\. 실전 활용 예시**

1. **근접 공격 판정 (Melee Combat):** 칼을 휘두를 때 칼날의 궤적을 따라 Sphere Trace를 실행하면, 얇은 선보다 훨씬 안정적으로 적을 타격할 수 있습니다.  
2. **아이템 상호작용 (Interaction):** 플레이어가 바라보는 방향으로 약간의 두께를 가진 트레이스를 쏘면, 아이템을 아주 정확히 조준하지 않아도 쉽게 상호작용 UI를 띄울 수 있습니다.  
3. **바닥 감지 (Ground Check):** 캐릭터가 공중에 떠 있는지 확인할 때, 캐릭터 발밑에 얇은 구체를 쏘아 계단이나 경사로에서도 안정적으로 접지력을 계산할 수 있습니다.

## **💡 팁 (Tips)**

* **Draw Debug Type**을 For Duration으로 설정하고 실행해 보세요. 구체가 어떻게 이동하고 어디서 막히는지 빨간색/초록색 선과 원으로 직접 확인할 수 있어 디버깅에 매우 유리합니다.  
* 구체의 시작점과 끝점이 같으면 이동하지 않고 해당 위치의 충돌만 검사하는 **Overlap** 체크처럼 동작하게 됩니다.
