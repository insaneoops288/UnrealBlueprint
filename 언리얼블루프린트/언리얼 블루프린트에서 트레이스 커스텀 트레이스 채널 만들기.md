# **언리얼 엔진 커스텀 트레이스 채널 생성 및 활용**

언리얼 엔진의 기본 채널(Visibility, Camera) 외에 게임 로직에 특화된(예: AI 탐지 전용, 아이템 상호작용 전용) 채널을 만드는 방법입니다.

## **1\. 커스텀 트레이스 채널 생성하기**

먼저 프로젝트 설정에서 새로운 채널을 등록해야 합니다.

1. **Project Settings 오픈**: 상단 메뉴의 Edit \-\> Project Settings를 선택합니다.  
2. **Collision 설정 이동**: 왼쪽 리스트에서 Engine 섹션 아래의 Collision을 클릭합니다.  
3. **New Trace Channel 클릭**: Trace Channels 항목 옆의 New Trace Channel... 버튼을 누릅니다.  
4. **세부 설정 입력**:  
   * **Name**: 채널 이름 (예: Interactable, BulletTrace).  
   * **Default Response**: 기본 반응을 설정합니다.  
     * Ignore: 기본적으로 모든 물체가 이 트레이스를 무시합니다. (특정 물체만 체크하고 싶을 때 유리)  
     * Block: 기본적으로 모든 물체가 이 트레이스를 막습니다.  
5. **Accept**: 설정을 완료하면 리스트에 추가됩니다.

## **2\. 오브젝트에 채널 할당하기**

트레이스에 감지되길 원하는 액터(Actor)나 컴포넌트에서 해당 채널에 대한 반응을 설정해야 합니다.

1. **액터 선택**: 월드에 배치된 액터나 블루프린트 에셋을 엽니다.  
2. **Collision 컴포넌트 선택**: 스태틱 메시나 콜리전 박스를 선택합니다.  
3. **Details 패널 설정**:  
   * Collision Presets를 Custom으로 변경합니다.  
   * 방금 만든 커스텀 채널(예: Interactable)을 찾습니다.  
   * 이 물체가 트레이스에 걸리게 하려면 해당 채널을 **Block**으로 체크합니다.

## **3\. 블루프린트에서 트레이스 실행하기**

이제 블루프린트 노드를 통해 해당 채널을 타겟팅합니다.

### **사용 노드: Line Trace By Channel**

1. **Trace Channel 선택**: 노드의 입력 핀 중 Trace Channel 드롭다운 메뉴를 클릭하여 생성한 커스텀 채널을 선택합니다.  
2. **로직 연결**:  
   * **Start**: 트레이스가 시작될 위치 (예: 카메라 위치).  
   * **End**: 트레이스가 끝날 위치 (예: 카메라 전방 500단위).  
3. **결과 처리**: Out Hit 핀을 Break하여 부딪힌 액터가 무엇인지 확인합니다.

### **장점**

* **성능 최적화**: 불필요한 오브젝트(바닥, 벽 등)를 무시하고 상호작용이 필요한 대상만 정확히 필터링할 수 있습니다.  
* **코드 가독성**: 채널 이름을 통해 해당 트레이스가 어떤 용도인지 명확히 알 수 있습니다.

## **4\. 주의사항**

* **채널 제한**: 언리얼 엔진은 최대 18개의 커스텀 트레이스/오브젝트 채널을 지원합니다. 채널을 너무 남발하지 않도록 주의하세요.  
* **Default Response**: Ignore로 설정했다면, 감지되길 원하는 모든 물체의 콜리전 설정을 일일이 Block으로 바꿔줘야 합니다. 반대로 Block으로 설정했다면 감지되지 않아야 할 물체들을 Ignore로 바꿔야 합니다.  
* **C++ 연동**: C++에서 사용할 때는 Config/DefaultEngine.ini 파일을 확인하여 해당 채널이 ECC\_GameTraceChannel1 등 어떤 열거형 값에 할당되었는지 확인해야 합니다.

// 예시: C++에서 커스텀 채널 호출 시  
// DefaultEngine.ini에서 설정된 번호를 확인하세요.  
ECollisionChannel TraceChannel \= ECC\_GameTraceChannel1;
