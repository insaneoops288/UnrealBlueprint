# **언리얼 엔진 블루프린트 노드 색상 의미 정리**

블루프린트 에디터에서 노드(함수) 상단의 색상은 해당 노드가 어떻게 동작하는지, 어떤 성격을 가지고 있는지 나타냅니다.

## **1\. 파란색 (일반 함수 / Impure Functions)**

가장 흔히 볼 수 있는 색상입니다.

* **의미**: 실행 핀(하얀색 화살표)이 있는 일반적인 함수입니다.  
* **특징**:  
  * 호출될 때 데이터나 월드의 상태를 변경할 수 있습니다(예: 변수 값 설정, 액터 파괴 등).  
  * 반드시 실행 흐름(Execution Line)에 연결되어야 작동합니다.  
  * 보통 '객체의 상태를 변화시키는 액션'과 관련이 많습니다.

## **2\. 녹색 (순수 함수 / Pure Functions)**

* **의미**: 실행 핀이 없는 함수입니다.  
* **특징**:  
  * 월드의 상태를 변경하지 않고 오직 **데이터 값만 반환**합니다(예: 수학 계산, 거리 측정, 변수 값 읽기).  
  * 실행 흐름에 연결할 필요가 없으며, 출력 핀(데이터)을 다른 노드에 연결할 때만 실행됩니다.  
  * 호출될 때마다 다시 계산되므로 최적화 시 주의가 필요합니다.

## **3\. 빨간색 (이벤트 / Events)**

* **의미**: 블루프린트 로직의 **시작점**입니다.  
* **특징**:  
  * 게임 엔진이나 시스템에서 특정 상황(예: 게임 시작, 충돌, 키 입력)이 발생했을 때 호출됩니다.  
  * 실행 흐름의 가장 왼쪽에 위치하며, 오른쪽으로 실행 신호를 보냅니다.  
  * 사용자 정의 이벤트(Custom Event)도 빨간색으로 표시됩니다.

## **4\. 노란색/금색 (매크로 / Macros)**

* **의미**: 자주 사용되는 노드 조합을 하나로 묶어놓은 매크로입니다.  
* **특징**:  
  * For Loop, Gate, Sequence와 같은 흐름 제어 매크로가 대표적입니다.  
  * 함수와 달리 실행 핀과 데이터 핀을 자유롭게 구성할 수 있습니다.

## **5\. 시안색 (타임라인 / Timelines)**

* **의미**: 시간에 따른 값의 변화를 제어하는 특수 노드입니다.  
* **특징**:  
  * 문이나 엘리베이터 이동, 부드러운 색상 변경 등 애니메이션 효과를 줄 때 주로 사용됩니다.  
  * 노드 내부에 그래프 에디터가 포함되어 있습니다.

## **6\. 회색 (흐름 제어 / Flow Control)**

* **의미**: 로직의 방향을 결정하는 노드들입니다.  
* **특징**:  
  * Branch(If문), Switch, Delay 등이 포함됩니다.  
  * 데이터 처리가 아닌, '실행 순서'를 관리하는 역할을 합니다.

## **7\. 보라색 (건축 스크립트 / Construction Script 관련)**

* **의미**: 주로 Construction Script 내에서 사용되거나, 클래스의 구조와 관련된 함수들입니다.  
* **특징**:  
  * 에디터 상에서 액터를 배치하거나 수정할 때 즉각적으로 실행되는 로직에 자주 쓰입니다.

### **요약 팁**

* **실행 핀이 있는가?** \- YES: 파란색(함수) 또는 노란색(매크로)  
  * NO: 녹색(순수 함수)  
* **로직의 시작인가?**  
  * YES: 빨간색(이벤트)
