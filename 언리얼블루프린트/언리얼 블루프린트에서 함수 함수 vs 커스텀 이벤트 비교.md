# **언리얼 엔진 블루프린트: 함수(Function) vs 커스텀 이벤트(Custom Event)**

블루프린트에서 로직을 구조화할 때 가장 먼저 고민하게 되는 것이 함수와 커스텀 이벤트의 선택입니다. 이 문서에서는 두 방식의 기술적 차이와 용도별 권장 사항을 다룹니다.

## **1\. 핵심 차이점 요약**

| 특징 | 함수 (Function) | 커스텀 이벤트 (Custom Event) |
| :---- | :---- | :---- |
| **반환값 (Output)** | 가능 (여러 개의 출력 핀 가능) | **불가능** |
| **지연 노드 (Latent)** | 사용 불가 (Delay, Timeline 등) | **사용 가능** |
| **네트워크 복제 (RPC)** | 지원 안 함 | **지원함 (Server, Client, Multicast)** |
| **로컬 변수** | 지원함 (함수 내 전용 변수) | 지원 안 함 (이벤트 그래프 변수 공유) |
| **실행 방식** | 즉시 실행 (Synchronous) | 호출 또는 예약 실행 (Asynchronous 가능) |
| **상속 및 오버라이드** | 자식에서 오버라이드 가능 | 오버라이드 불가 (단, 부모 이벤트 호출 가능) |

## **2\. 상세 분석**

### **🟢 함수 (Functions)**

함수는 특정 입력에 대해 결과를 즉시 반환하거나, 상태를 변경하는 **순수 로직**을 수행할 때 적합합니다.

* **반환값의 존재:** 함수는 로직이 끝난 후 결과를 Return Node를 통해 전달할 수 있습니다. 계산식이나 데이터 조회에 매우 유리합니다.  
* **Pure(순수) 함수:** 실행 핀(Exec)이 없는 'Pure' 함수로 설정하면 값이 필요할 때만 계산되도록 만들 수 있습니다.  
* **캡슐화와 로컬 변수:** 함수 내부에서만 사용하는 'Local Variable'을 생성할 수 있어, 이벤트 그래프가 지저분해지는 것을 방지합니다.  
* **제약:** Delay, Timeline, Retriggerable Delay 같은 \*\*시간 관련 노드(Latent Action)\*\*를 사용할 수 없습니다. 함수는 호출된 프레임 내에서 즉시 완료되어야 하기 때문입니다.

### **🔴 커스텀 이벤트 (Custom Events)**

커스텀 이벤트는 로직의 **시작점(Entry Point)** 역할을 하며, 복잡한 흐름 제어나 네트워크 통신에 필수적입니다.

* **지연 및 시간 기반 로직:** Delay나 Timeline을 사용하여 시간에 따라 변하는 로직을 짤 수 있습니다.  
* **네트워크 복제(Replication):** 언리얼의 멀티플레이어 환경에서 서버에서 실행되거나 클라이언트로 전파되는 RPC(Remote Procedure Call)는 오직 커스텀 이벤트를 통해서만 구현됩니다.  
* **단방향 흐름:** 출력 핀(Return)을 가질 수 없으므로, 로직을 실행하고 "잊어버리는(Fire and Forget)" 방식에 가깝습니다. 만약 결과가 필요하다면 다른 이벤트를 호출하거나 변수를 업데이트해야 합니다.

## **3\. 언제 무엇을 사용해야 할까?**

### **함수를 사용해야 할 때**

1. 계산 결과값이 필요할 때 (예: 데미지 계산, 거리 측정).  
2. 부모 클래스의 로직을 자식에서 다르게 구현(Override)하고 싶을 때.  
3. 로컬 변수를 사용하여 메모리를 효율적으로 관리하고 싶을 때.  
4. 실행 핀 없이 값만 가져오는 순수 노드가 필요할 때.

### **커스텀 이벤트를 사용해야 할 때**

1. Delay, Timeline 등 시간이 걸리는 로직이 포함될 때.  
2. 멀티플레이어 게임에서 서버/클라이언트 간 통신이 필요할 때.  
3. 다른 블루프린트나 시스템에서 이벤트를 '바인딩(Bind to Event)'하여 구독 모델을 만들 때.  
4. 입력(키보드, 마우스 등)에 반응하는 로직의 시작점을 만들 때.

## **4\. 실전 팁**

* **함수 내 이벤트 호출:** 함수 내부에서 커스텀 이벤트를 호출할 수는 있지만, 그 반대로 이벤트 그래프에서 함수를 사용하는 것이 일반적인 구조화 방법입니다.  
* **가독성:** 로직이 너무 길어지면 Collapse to Function 또는 Collapse to Macro 기능을 적절히 사용하여 시각적으로 정리하는 습관이 중요합니다.
