# **Link Anim Class Layers 노드 상세 설명**

언리얼 엔진의 **Link Anim Class Layers** 노드는 애니메이션 블루프린트(AnimBP)의 특정 부분을 실행 중에(Runtime) 동적으로 교체할 수 있게 해주는 기능을 수행합니다. 이는 주로 \*\*애니메이션 레이어 인터페이스(Animation Layer Interface)\*\*와 함께 사용됩니다.

## **1\. 핵심 역할: "동적 모듈 교체"**

가장 큰 역할은 메인 애니메이션 블루프린트의 복잡도를 낮추고, 상황에 따라 필요한 애니메이션 로직만 '끼워 넣는' 것입니다.

* **인터페이스 기반 연결:** 미리 정의된 '애니메이션 레이어 인터페이스'를 통해 메인 AnimBP와 서브 AnimBP를 연결합니다.  
* **런타임 스왑:** 게임 플레이 도중 캐릭터가 무기를 바꾸거나 상태가 변했을 때(예: 부상 상태), 그 상태에 맞는 애니메이션 레이어 클래스를 즉시 적용합니다.

## **2\. 주요 구성 요소**

이 기능을 제대로 활용하기 위해서는 세 가지 요소가 필요합니다.

1. **애니메이션 레이어 인터페이스 (Anim Layer Interface):** 어떤 "구멍(Layer)"을 만들지 정의하는 설계도입니다. (예: WeaponUpperBody, IdleLayer)  
2. **메인 애니메이션 블루프린트:** 인터페이스를 추가하고, 그래프 내에 Linked Anim Layer 노드를 배치하여 '애니메이션이 들어갈 자리'를 확보합니다.  
3. **레이어 애니메이션 블루프린트:** 인터페이스를 구현(Implement)하여 실제로 어떤 애니메이션이 재생될지 정의하는 별도의 클래스입니다.

## **3\. Link Anim Class Layers 노드의 입력 핀**

* **Target:** 레이어를 교체할 대상인 애니메이션 인스턴스(보통 메인 AnimBP 인스턴스)입니다.  
* **In Class:** 적용하고자 하는 '레이어 애니메이션 블루프린트' 클래스입니다.

## **4\. 왜 사용하는가? (장점)**

### **① 그래프의 단순화 (Anti-Spaghetti)**

하나의 AnimBP 안에 모든 무기(권총, 소총, 칼 등)의 애니메이션 로직을 다 넣으면 그래프가 매우 복잡해집니다. Link Anim Class Layers를 사용하면 메인 그래프는 깨끗하게 유지하고, 무기별 로직은 각각의 파일로 분리할 수 있습니다.

### **② 메모리 효율 및 최적화**

모든 애니메이션 데이터를 한꺼번에 로드하지 않고, 현재 링크된 레이어에 필요한 데이터만 관리할 수 있어 대규모 프로젝트에서 성능 이점이 있습니다.

### **③ 협업 효율성**

애니메이터 A는 메인 캐릭터 이동을 작업하고, 애니메이터 B는 특정 무기의 동작만 별도 파일에서 작업하여 합칠 수 있으므로 작업 충돌이 줄어듭니다.

## **5\. 실제 사용 예시 (워크플로)**

1. **무기 장착:** 캐릭터가 땅에 떨어진 '소총'을 줍습니다.  
2. **이벤트 발생:** 캐릭터 블루프린트에서 Link Anim Class Layers 노드를 호출합니다.  
3. **클래스 할당:** In Class에 ABP\_Rifle\_Layers 클래스를 전달합니다.  
4. **결과:** 메인 AnimBP의 Linked Anim Layer 노드 위치에서 이제 소총 전용 애니메이션 로직이 실행됩니다.

### **관련 노드: Unlink Anim Class Layers**

링크했던 레이어를 제거하고 다시 기본 상태(또는 빈 상태)로 되돌리고 싶을 때는 Unlink Anim Class Layers 노드를 사용하면 됩니다.
