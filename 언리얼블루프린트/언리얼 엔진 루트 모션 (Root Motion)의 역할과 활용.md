# **언리얼 엔진 루트 모션 (Root Motion)의 역할과 활용**

언리얼 엔진에서 \*\*루트 모션(Root Motion)\*\*이란, 애니메이션의 '루트 본(Root Bone)'의 움직임을 분석하여 이를 캐릭터 캡슐 컴포넌트의 실제 월드 좌표 이동으로 변환하는 기능을 말합니다.

## **1\. 루트 모션의 핵심 역할**

### **① 애니메이션과 이동의 완벽한 일치 (Precision)**

일반적인 'In-place' 애니메이션은 캐릭터가 제자리에서 걷고, 코드로 속도를 주어 캡슐을 이동시킵니다. 이 경우 발이 지면에서 미끄러지는 '피트 슬라이딩(Foot Sliding)' 현상이 발생하기 쉽습니다. 루트 모션은 애니메이션의 발걸음 속도와 캡슐의 이동 속도를 완전히 일치시켜 시각적 몰입감을 극대화합니다.

### **② 복잡한 기동 구현 (Complex Maneuvers)**

파쿠르(담 넘기), 사다리 타기, 특정 방향으로 회전하며 공격하는 동작 등 **비정형적인 움직임**을 코드로 구현하려면 매우 복잡한 물리 계산이 필요합니다. 루트 모션을 사용하면 애니메이터가 제작한 정교한 움직임을 그대로 게임 내 물리 이동으로 사용할 수 있습니다.

### **③ 공격 및 피격 판정의 정확성**

액션 게임에서 캐릭터가 휘두르는 무기의 거리나 회전 반경이 애니메이션마다 다를 때, 루트 모션을 사용하면 캐릭터가 공격 애니메이션에 맞춰 자연스럽게 전진하거나 회전하므로 판정 로직을 짜기가 훨씬 수월해집니다.

## **2\. 루트 모션 작동 원리**

1. **애니메이션 에셋:** 루트 본(보통 0,0,0 좌표에 위치)이 애니메이션 도중 이동하거나 회전합니다.  
2. **엔진 분석:** 엔진이 매 프레임 루트 본의 변화량(델타)을 측정합니다.  
3. **캡슐 전송:** 측정된 변화량을 캐릭터의 **Capsule Component**에 적용하여 실제 물리 위치를 옮깁니다.  
4. **루트 리셋:** 비주얼 메시의 루트 본은 다시 원점으로 돌아와 캡슐 안에 머물게 됩니다.

## **3\. 블루프린트 및 에셋 설정 방법**

루트 모션이 정상 작동하려면 세 가지 단계의 설정이 필요합니다.

### **Step 1: 애니메이션 시퀀스 설정**

* 애니메이션 에셋을 열고 **Asset Details** 패널로 이동합니다.  
* Enable Root Motion 체크박스를 선택합니다.  
* Force Root Lock을 설정하여 애니메이션 재생 시 메시가 캡슐 밖으로 나가지 않게 고정합니다.

### **Step 2: 애니메이션 블루프린트(AnimBP) 설정**

* AnimBP의 **Anim Preview Editor** 또는 **Class Defaults**를 확인합니다.  
* **Root Motion Mode** 옵션을 설정합니다:  
  * **From Everything:** 모든 상황(상태 머신, 몽타주 등)에서 루트 모션 적용.  
  * **Root Motion from Montages Only:** 애니메이션 몽타주 재생 시에만 루트 모션 적용 (가장 흔히 사용됨).

### **Step 3: 캐릭터 무브먼트 컴포넌트**

* Character Movement Component는 기본적으로 루트 모션을 받아들일 준비가 되어 있습니다.  
* 네트워크 환경(멀티플레이어)의 경우, 루트 모션 복제가 복잡할 수 있으므로 주의가 필요합니다.

## **4\. 장단점 요약**

| 장점 | 단점 |
| :---- | :---- |
| 발 미끄러짐 없는 극강의 사실감 | 애니메이션 데이터에 전적으로 의존 (수정 시 리소스 필요) |
| 복잡한 이동 경로를 쉽게 구현 가능 | 멀티플레이어 환경에서 동기화 구현이 까다로움 |
| 애니메이터의 의도를 100% 반영 가능 | 캡슐 충돌 처리가 애니메이션 속도에 따라 급격히 변할 수 있음 |

## **5\. 실무 팁: Root Motion Montage**

일반적인 이동(걷기, 뛰기)은 제어의 용이성을 위해 **In-place** 방식을 사용하고, 특수 기술(대시, 콤보 공격, 벽 타기)은 **Root Motion Montage**를 사용하여 필요한 순간에만 루트 모션을 활성화하는 방식이 가장 효율적입니다.
